## 规则

1. 后续提交或阶段自检优先运行：根工程 build、集成版 build、默认烟测、`--write`。
2. `.E5S` 只能作为“E5S 存档/旧探针兼容”检查对象，不能重新作为 R/S 剧本主格式。
3. 写入烟测必须只写测试副本，不能修改当前游戏本体。
4. 若某次旧全量探针失败，不得因此回退 R/S eex 主线；应拆分失败项并明确归入 E5S 兼容或高级调试。

## 2026-06-01 补充：SmokeTests 输出目录被占用的 build 失败规避

在 Windows 上，如果某个旧的 SmokeTests/调试进程仍占用 `bin\\Debug\\...` 下的 `CCZModStudio.dll`，`dotnet build` 可能出现：

- `MSB3027/MSB3021 无法复制 ... because it is being used by another process`

这不是代码错误。建议改用独立输出目录构建 SmokeTests，再用该目录的 DLL 运行烟测：

```powershell
dotnet build 工具整合包\CCZModStudio.SmokeTests\CCZModStudio.SmokeTests.csproj -c Debug -p:OutDir=..\..\CCZModStudio_TestBins\SmokeTests\
dotnet CCZModStudio_TestBins\SmokeTests\CCZModStudio.SmokeTests.dll --rs-smoke
dotnet CCZModStudio_TestBins\SmokeTests\CCZModStudio.SmokeTests.dll --write
```
