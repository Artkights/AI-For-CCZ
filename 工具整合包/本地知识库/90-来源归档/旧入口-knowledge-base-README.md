#### 核心概念

##### 双渠道判定

曹操传引擎通过同时检查**装备渠道**和**个人渠道**来判断角色是否拥有某项特技：

```
装备特效号 ← 检查角色携带的装备
    ↓
  任一通过 → 判定为"拥有该特技"
    ↑
个人特效号 ← 检查角色自身属性
```

##### 效果值标志

| 标志值 | 含义 | 返回值类型 |
|--------|------|------------|
| `1` | 不需要效果值 | `1`（有）或 `0`（无） |
| `0` | 需要效果值 | 具体数值或 `0` |

##### 状态数组 0x508B00

用于跟踪每个单位的二次行动使用状态，确保每回合只触发一次。

##### 两种编号体系 ⭐

游戏中存在两套平行的编号系统：

| 编号 | 全称 | 范围 | 生命周期 | 转换函数 |
|------|------|------|----------|----------|
| **Data号** | 角色数据号 | 0~1023 | 永久固定 | → `0x484002` → 角色数据指针 |
| **战场编号** | 战场编号 | 0~249 | 单场战斗 | → `0x4061F9` → 战术单位指针 |

两个编号通过 [战斗数据结构](../02-数据结构/战斗数据结构.md) 实现互相转换。详见 [两种编号体系详解](../02-数据结构/单位结构体.md#两种编号体系)。
