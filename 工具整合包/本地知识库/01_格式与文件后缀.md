# 01 格式与文件后缀

## 已确认

| 后缀/文件 | 当前确认 | 依据 | 实现影响 |
| --- | --- | --- | --- |
| `.E5S` | 存档信息 | 用户 2026-05-30 明确校正；`B形象指定器\System.ini` 中也有 `CountSV=900` 注释为“存档数量” | 不能继续作为 R/S 剧本主格式；旧 `SV/E5S 剧本` 口径已降级为存档信息/旧兼容探针 |
| `.eex` | R/S 剧本/场景/战场相关格式 | 用户 2026-05-30 明确校正；项目 RS 目录中存在 R/S eex 文件；轩辕春秋 5.6 教程也把 `R_XX.eex/S_XX.eex` 称为 R/S 剧本；2026-06-03 再次复核后不再把它当人物图像封包 | 剧本制作、战场制作与 EEX 探针扫描 `RS\R_*.eex/S_*.eex`；人物 R/S 图像预览不得从这些文件取帧 |
| `RS` 目录 | R/S 剧本 EEX 目录 | `B形象指定器\形象指定器6.5\System.ini` 的 `UserPath2` 指向 `...\RS\`；`a新剧本编辑器` 读取 R/S eex | 剧本资源解析扫描此目录；人物 R/S 编号只把它作为剧本/战场语境，不作为图像本体来源 |
| `Ekd5.exe` | 多个核心表和形象指定数据位于 exe | `B形象指定器\System.ini` 注释：`FileHead` 是 exe 中 S 形象数据位置，`RFileHead` 是 exe 中 R 形象数据位置 | 人物 R/S 指定表必须按形象指定器设置校验偏移 |

## 2026-05-31 eex 与 E5S 实现校正

- 剧本制作读取列表阶段只允许扫描 `RS\R_*.eex` / `RS\S_*.eex`。
- 列表阶段只能读取文件名、长度和前 256 字节 EEX 头部，禁止一次性全文扫描所有 eex，避免 UI 卡死。
- 选择具体剧本后，才允许后台解析命令候选和 GBK 文本线索。
- `.E5S` 后续如保留，必须放入“存档信息/旧探针/待校正”语境，不得在核心剧本制作中称为 R/S 剧本。
- 文本写回需区分两条路径：兼容短写回仅限已识别 GBK 文本的原偏移、原容量内等长/缩短写回；旧版完整树写回可通过 `LegacyScenarioWriter` 重建 Scene/Section/子块长度和 `0x76` 跳转，允许文本参数随结构扩容，但仍不得跳过复读校验或修改未知命令结构。

## 2026-05-31 验证入口校正

- `CCZModStudio.SmokeTests` 默认不再运行旧 `SV004.E5S` 全量断言，而是运行 R/S eex 主线只读烟测。
- `--write` 当前指向 R/S eex 核心写入烟测，只写测试副本，验证 `RS\R_00.eex` 文本和 `Ekd5.exe` 人物 R/S 指定表；这是烟测隔离策略，不代表正式功能必须以测试副本为前置条件。
- 旧 `.E5S` 兼容检查入口为 `--legacy-e5s-smoke`，输出中明确标注 `compat_only_not_rs_script` / `old_probe_only`。
- 历史全量探针入口保留为 `--legacy-all-smoke`，不作为当前核心创作工具的成功标准。
- 主界面高级页名称为 `R/S eex高级探针`，读取按钮为 `读取 RS/*.eex`；旧 `SV剧本/关卡探针` 不再作为核心或高级 R/S 入口名称。

## 2026-05-31 08:05 补充：R/S eex 与 E5S 命名边界

- 新建/显示给创作者的剧本命令、文本、模板、核对清单统一使用 `R/S eex` 语义。
- `.E5S` 只保留为 `旧 E5S 兼容探针/存档信息`，不能再作为“读取剧本”的主入口文案。
- 旧备注范围 `SV命令`、`SV文本`、`SV命令模板` 只作兼容别名；新备注应使用 `R/S命令`、`R/S文本`、`R/S命令模板`。
