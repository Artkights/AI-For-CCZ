# 联网深度专题

## 结论速览

- 本页汇总联网资料的深度专题结论，只记录可复核信息、版本边界、迁移价值和待验证项。
- 外部资料不能替代本地样本、旧工具源码、复读、烟测或实机验证。
- 6.5 未加密基底是当前项目默认版本；6.6/6.6x 只作为差异和未来适配。

## 适用版本

- 默认吸收目标：曹操传加强版 6.5。
- 参考版本：5.6、6.3、6.4、6.6、6.6x。
- 写回规则只开放给已完成本地验证的 6.5 能力。

## 已确认事实

- 6.5 发布资料可作为当前项目的外部主线入口，但具体写回仍以本地 6.5 样本、HexTable、旧工具源码和烟测为准。
- 6.6/6.6x 在 6.5 基础上继续扩展脚本、资源和动态图能力，不能直接混入 6.5。
- 工具集、教程导航和 Meff/E5 资料适合补足制作流程、资源范围和术语边界，不适合不加验证地生成写入规则。

## 实现/使用方法

- 查版本能力时，先读 `6X版本差异与适配.md`，再读本页来源摘要。
- 查外部工具和教程时，把资料当作“线索”，再回到 `00-总览与规范/工具来源映射.md` 和本地旧工具目录验证。
- 将联网资料落地为代码或写回规则前，必须补到对应主题页的 `证据来源` 与 `待验证项`。

## 风险与边界

- 论坛资料可能跨版本、跨工具、跨作者习惯混杂，不能自动合并。
- 发布帖描述的是能力范围，不等于二进制结构、参数槽位和资源表已经可写。
- 教程中的旧称呼、旧工具按钮名和历史项目路径需要映射到当前 6.5 基底。

## 专题结论

### 6.5 发布与版本能力

- 6.5 是当前项目的主目标版本，外部资料中出现的 R 插图、`Tr.e5`、脚本整数变量输入、负数参数等能力只作为迁移线索。
- 当前可落地的写回能力必须回到本地样本、旧剧本编辑器源码、`LegacyScenarioReader/Writer`、E5 索引读取和烟测结果。
- `信息传送 28`、R 插图槽位、透明/遮挡、取消规则和坐标越界仍需实机验证后才能扩展写回能力。
- `4017/4018`、`4020/4021`、`4023` 等变量体现 6.5 引擎有战斗集气、自由 R 鼠标行为和负面状态恢复模式等全局脚本开关；这些目前只能作为待验证索引，不直接生成 UI 保存项。

### 6.6/6.6x 差异边界

- 6.6/6.6x 出现动态图、R 文字、`72 信息传送` 新子功能和资源文件变化。
- 这些能力说明资源边界和脚本语义已经变化，不能直接套用到 6.5。
- 若未来支持 6.6，必须单列版本检测、HexTable、资源文件、脚本命令模板和写入护栏。
- `4031` 这类解释开关会影响多条脚本命令的角色编号语义，`4038` 则由 6.6 动态图控制写入；未来脚本解析需要记录命令上下文状态，不能只按单条命令做静态参数解释。

### 6.x 整型变量线索

- 公开资料中“整形变量/整型变量”是同一类脚本变量口径；知识库统一写作“整型变量”，引用旧帖时保留原始含义。
- 已归档变量包括：`4017` 初始气、`4018` 集气模式、`4020/4021` 自由 R 鼠标行为、`4023` 负面状态恢复模式、`4031` Data 号/战场编号解释、`4038` 动态图显示次数、`4080` 旧物品扩充兼容线索。
- 这些变量的证据等级均为外部佐证或旧口径。进入工具实现前，需要最少补一个本地脚本样本和一个运行时验证记录。

### 工具集与补档

- 工具集资料用于确认社区长期使用的工具入口和命名习惯。
- 当前项目优先参考本地已有旧工具：B 形象指定器、CczRSX 6.5、普罗补丁、a 新剧本编辑器和相关源码。
- 外部工具名不能替代本地路径、样本、偏移和读写结果。

### MOD 制作教程导航

- 教程导航适合建立学习路线和术语映射。
- R/S 剧本、人物形象、宝物、兵种、地图、资源等主题必须回到对应专题文档。
- 教程里的历史习惯若与本地 6.5 样本冲突，以本地证据为准，并把旧口径归档。

### 旧形象指定器全局设置

- 联网资料确认旧“新形象指定器”在打开 `Ekd5.exe` 后存在全局设置入口，功能线索包括能力显示、能力条件、转职等级、等级上限、升级经验、装备经验、兵种/职业名称和游戏标题等。
- 该资料是 5.6 教程口径，不能直接套用到当前 6.5 未加密基底的偏移或写回规则。
- 当前 6.5 本地已验证并迁入工具的全局设定写回项只有：`6.5-3 兵种系`、`6.5-4 详细兵种` 和 `Ekd5.exe@0x8D3C4` 游戏标题。
- 2026-06-08 已撤回对联网地址线索和本地机器码上下文的数字项解释：等级阈值、升级经验、装备经验、功勋、装备等级上限等旧窗口左侧全局数值项一律回调为不可读取、不可写入。
- 旧窗口左侧全局数值项仍需旧工具逆向、字节差异样本或实机验证后才能开放保存。

### E5/Meff/图片资源

- Meff/E5 资料可帮助识别资源类别、编辑器用途和图片资源工作流。
- 当前项目已经确认部分 E5 `0x110` 图片索引和 DLL `RT_BITMAP` 路径；通用 Ls/E5 重封包、整体重排、批量动作帧替换仍未完全闭环。
- DLL 图标资料必须区分 PE `RT_BITMAP`、`RT_ICON`、`RT_GROUP_ICON`，不能把标准 `.ico` 字节直接写入 DLL。

### 2026-06-08 扩大检索专题

- Hexzmap 相关旧帖可以佐证社区长期把 `Hexzmap.e5` 与战场地形格、地形编辑器和地形导入流程关联；但旧帖多为猜测、工具说明或跨版本工具链，不提供足够的 6.5 二进制目录表定义。当前 `0x110` 目录表、12 字节大端目录项和 `fileOffset + 2` 数据起点仍以本地样本探针为准。
- `Unit_spc.e5 / Unit_mov.e5 / Unit_atk.e5` 在 6.1 发布资料中被明确归入 S 形象资源，外部问答也能佐证旧工具导入依赖支持文件；这些只能确认资源职责，不能确认 6.5 动作帧完整顺序、朝向表或三文件同步替换规则。
- `MEff 文件初步研究` 明确带有估测和未实测口径，适合补充 Meff/Mcall/LS 研究入口；当前工具只能继续保守预览 `Meff.e5` 的标准图、RAW 和未知字节图候选，不能把推测当作动画重封包规则。
- `S 剧本完全教程基础篇` 能补充战前设置、战中事件、战后处置的传统制作结构习惯；实际参数槽、子事件长度、跳转和写回仍以本地 `全剧本指令详解（2023）.docx`、旧剧本编辑器源码和 `LegacyScenarioWriter` 复读为准。
- `如何做MOD` 和 97 经验帖进一步补充 R/S 跳转、子事件承接命令、即时判定、无条件跳转和修改后复读的排错口径；这些与旧源码规则互相印证的部分可转成“保护规则”和“核对清单”，不能转成自动修复。
- 6.5p/6.5pl 与旧 EXE 变量资料显示 Star/Puluo 分支会继续扩展装备、道具、特技书、策略书和单挑变量；它们是分支适配线索，不属于当前 6.5 未加密基底默认写入能力。

## 待验证项

- R 插图实机行为：图号 0 取消、透明度、遮挡层级、坐标越界、多插图共存和缺 `Tr.e5` 时的发布边界。
- 6.5 脚本整数变量和负数参数：命令模板、参数槽位、变量范围和写回规则。
- 6.x 整型变量开关：`4017/4018/4020/4021/4023/4031/4038/4080` 的默认值、作用域、重置时机和跨命令影响。
- Hexzmap 权威格式：块顺序、尺寸表、尾部字节和当前样本大小差异。
- `Unit_atk.e5`、`Unit_mov.e5`、`Unit_spc.e5` 动作帧序列和批量替换规则。
- Meff/E5 通用重封包能力是否能安全迁移到当前工具链。
- 6.5p/6.5pl 分支资料与 6.5 未加密默认基底的核心文件、变量和脚本命令差异矩阵。
- R/S 教程口径与本地样本的逐关对照：三段式 S Scene、R 入 S/S 回 R 跳转、即时判定、`0x76` 跳转、子事件承接命令和修改后复读流程。
