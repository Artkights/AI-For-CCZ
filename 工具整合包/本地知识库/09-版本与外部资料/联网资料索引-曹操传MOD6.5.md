#### 2026-06-07 图片/DLL 图标资料复核

已复核来源：

- Microsoft Learn：`BeginUpdateResourceW` / `UpdateResourceW` / `EndUpdateResourceW`，用于更新 PE 文件资源。
  https://learn.microsoft.com/en-us/windows/win32/api/winbase/nf-winbase-beginupdateresourcew
  https://learn.microsoft.com/en-us/windows/win32/api/winbase/nf-winbase-updateresourcew
  https://learn.microsoft.com/en-us/windows/win32/api/winbase/nf-winbase-endupdateresourcew
- Microsoft Learn：资源类型常量，包括 `RT_BITMAP`、`RT_ICON`、`RT_GROUP_ICON`。
  https://learn.microsoft.com/en-us/windows/win32/menurc/resource-types
- Raymond Chen / The Old New Thing：Windows 图标资源由 `RT_GROUP_ICON` 和 `RT_ICON` 组成，`.ico` 文件目录项与 PE 资源组目录项格式不同。
  https://devblogs.microsoft.com/oldnewthing/20120720-00/?p=7083
- 轩辕春秋 6.6 发布资料：新版资料可见图标资源迁移线索，但属于 6.6 边界；6.5 当前样本仍以 DLL `RT_BITMAP` 为主。
  https://xycq.org.cn/forum/viewthread.php?authoruid=0&extra=&page=1&tid=310256

迁移结论：

- 6.5 当前样本的 `Itemicon.dll`、`Mgcicon.dll`、`Cmdicon.dll` 不能按标准 `ExtractIconEx` 图标 DLL 处理；应解析 PE `RT_BITMAP`。
- 标准 `.ico` 替换不能直接把 `.ico` 文件字节写入 DLL；若后续遇到标准图标 DLL，需要分别写 `RT_GROUP_ICON` 与 `RT_ICON`。
- 当前代码实现只开放样本已确认的 `RT_BITMAP` 路径：按字段编号定位成对位图资源、写前备份、调用 Win32 UpdateResource 系列 API 写 DIB、写后生成报告。
- 6.6 资料里出现的 `Item.e5` / `Mtem.e5` 等图标资源迁移线索不能直接套入 6.5 DLL；如果项目切换到 6.6，需要重新走版本护栏、样本探针和烟测。
