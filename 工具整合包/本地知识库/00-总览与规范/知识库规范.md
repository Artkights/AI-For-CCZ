# 知识库规范

## 结论速览

- 本页定义知识库的写作、证据、清洗和迁移规则。
- 根入口固定为 `README.md`，旧平铺文件和嵌套 `knowledge-base` 不再作为主入口。
- 文档目标是有序、严密、丰富、实用，且不引入脏数据。

## 适用版本

- 默认版本：曹操传加强版 6.5（未加密）。
- 6.6/6.6x、5.6、6.3、6.4 等资料只能作为版本差异或外部佐证。
- 非 6.5 资料进入实现前，必须有独立版本护栏和本地样本验证。

## 文档模板

每篇主题文档使用以下结构：

```markdown
# 标题

## 结论速览

## 适用版本

## 已确认事实

## 实现/使用方法

## 风险与边界

## 证据来源

## 待验证项
```

可以在模板后追加 `## 详细记录`、`## 示例`、`## 调用链` 等章节，但前七个章节必须保留。

## 证据等级

| 等级 | 定义 | 处理方式 |
|------|------|----------|
| 已验证 | 本地样本、旧工具源码、烟测、复读或实机确认 | 可进入实现规则 |
| 外部佐证 | 发布帖、教程、论坛资料可复核，但未完成本地验证 | 可进入背景和边界，不单独写回 |
| 待证 | 样本不足、只有推断或跨版本线索 | 只放在 `待验证项` |
| 废弃/旧口径 | 已被本地证据修正或不适合当前版本 | 归档并说明替代结论 |

## 清洗规则

- 不保留旧项目绝对路径、Windows 旧盘符链接和无法在当前仓库解析的导航。
- 不复制论坛长帖全文，只保留结论、链接、版本、用途和验证状态。
- 不把外部经验直接写成 6.5 实现规则。
- 不让同一主题在多个文件中并行维护；重复内容合并后，旧入口写入 `90-来源归档/迁移索引.md`。
- 不把历史未确认内容写进正文结论；全部集中到 `待验证项`。

## 编码规则

- Markdown 统一为 UTF-8 with BOM。
- 使用 PowerShell 读取中文时，优先显式设置 UTF-8，避免把无 BOM UTF-8 误判为 GBK。
- 文档中可保留中文目录和中文文件名，链接路径必须使用当前知识库相对路径。

## 联网资料规则

- 优先整理可复核的轩辕春秋发布帖、工具帖、教程帖和本地已有 docx。
- 每条联网结论必须包含来源链接、版本范围、是否已被本地证据确认。
- 6.6/6.6x 资料必须写清“不能直接套用 6.5”的边界。
- 外部资料进入写回能力前，必须补本地样本、旧工具源码、烟测或实机验证。

## 维护流程

1. 先确认主题是否已有主文档。
2. 读取本地样本、旧工具源码、烟测或现有专题。
3. 若需要联网，先记录来源，再抽取可复核结论。
4. 按模板写入主题文档。
5. 更新根 README 或迁移索引。
6. 运行结构、链接、编码和脏数据扫描。
