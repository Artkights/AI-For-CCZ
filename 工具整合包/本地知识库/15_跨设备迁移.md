# 跨设备迁移与路径解析

## 根因

- 旧逻辑把 `WorkspaceRoot`、游戏根目录和 `HexTable.xml` 绑定在同一个固定目录结构下。
- 旧逻辑下 `HexTable.xml` 只查找：
  - `老版游戏制作工具\CczRSX 6.5\ConfigTable\HexTable.xml`
  - `CczRSX 6.5\ConfigTable\HexTable.xml`
- 跨设备复制、只复制游戏目录、从 `bin` 输出目录启动、或测试副本位于独立目录时，固定相对路径会失效，导致后续数据表、剧本和资源模块无法读取表结构。

## 当前规则

- 项目检测入口统一在 `CCZModStudio.Core.ProjectDetector`。
- `HexTable.xml` 只在当前项目范围内查找；失败后使用工具内置备份，不再从当前工作目录或程序目录向外递归捞取其他项目的旧配置。
- 旧工具依赖资源统一按“项目内优先，工具内置备份兜底”解析。工具包独立运行时不应依赖外部 `老版游戏制作工具` 目录。
- 当前项目范围包括：
  - 用户选择的游戏目录
  - 由游戏目录推导出的工作区目录
- `HexTable.xml` 可迁移到：
  - `<工作区>\老版游戏制作工具\CczRSX 6.X\ConfigTable\HexTable.xml`（例如 `CczRSX 6.6`、`CczRSX 6.5`、`CczRSX 6.4`）
  - `<工作区>\CczRSX 6.X\ConfigTable\HexTable.xml`
  - `<游戏目录或工作区>\ConfigTable\HexTable.xml`
  - `<游戏目录或工作区>\HexTable.xml`
- 6.6/其它 6.X 项目应优先提供对应版本 `HexTable.xml`；当前临时允许缺表时回退内置 6.5 表只读打开，后续版本计划取消该兜底。
- `HexTable.xml` 内置备份位于工具输出目录 `ConfigTable\HexTable.xml`，源码中对应 `CCZModStudio\Assets\ConfigTable\HexTable.xml`。
- 旧工具依赖内置备份位于工具输出目录 `LegacyResources\`，源码中对应 `CCZModStudio\Assets\LegacyResources\`：
  - `a新剧本编辑器v0.23\CczString.ini`
  - `a新剧本编辑器v0.23\CczCustom.ini`
  - `a新剧本编辑器v0.23\CczSceneEditor2.ini`
  - `B形象指定器\形象指定器6.5\System.ini`
  - `普罗-综合工具v0.3\素材库\**`
  - `普罗-搬运 注入\*.txt`
- `CczString.ini`、素材库、B形象指定器、普罗搬运配置都走 `ProjectDetector` 的 portable search API，不再各自硬编码两三个固定路径；找不到项目内资源时回退 `LegacyResources`。
- `CczProject` 必须保存项目级已解析路径：`SceneDictionaryPath`、`SceneEditorDirectory`、`ImageAssignerDirectory`、`ImageAssignerSystemIniPath`、`MaterialLibraryRoot`、`PatchConfigRoot`。
- 主界面加载项目后应主动读取 `CczString.ini` 与素材库索引，而不是等到各模块首次使用时再各自猜测路径；手动按钮只作为刷新入口。
- 各模块读取旧工具资源时优先使用 `CczProject` 的已解析路径，并先校验 `File.Exists`/`Directory.Exists`；路径失效后再回退 portable search，避免旧设备绝对路径短路。
- 素材库索引、人物 R/S 形象预览、宝物大类边界、普罗补丁/搬运配置都应接收或引用 `CczProject`，不要只传 `WorkspaceRoot`。

## 受影响模块

- 数据表：`HexTableParser` 只解析传入 XML，不做路径猜测；路径统一由 `ProjectDetector` 解析。
- 剧本：`CczString.ini` 使用项目级搜索，缺失时使用内置字典备份。
- 人物 R/S 形象预览：新剧本编辑器配置和 B形象指定器配置使用项目级搜索，缺失时使用内置配置备份。
- 宝物/物品大类：`System.ini` 从项目内 B形象指定器目录解析，缺失时使用内置配置备份。
- 素材库、普罗补丁/搬运配置：使用 portable directory search，缺失时使用内置素材库/文本配置备份。
- 回归测试：`--migration-smoke` 应输出 `sceneEditor`、`imageAssigner`、`materialRoot`、`patchRoot` 与素材数量，并校验非空解析路径仍存在。
