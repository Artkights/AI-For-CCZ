#### 核心概念

##### 特技判定机制

引擎的特技判定系统有三个关键组件：

| 组件 | 地址 | 作用 |
|------|------|
| **核心引擎** | `4101D9` | 双渠道判定（装备+个人），返回效果值 |
| **桩函数** | `421080-421180` | 标准桩函数区域（原生用） |
| **调度函数** | `600000+` | 设置 EDX 偏移 + 9BBFE4 查表 |

##### 两种特技注册方式

###### 方式一：原生的标准路径（复杂，不推荐用于新特技）

1. 在 `421080-421180` 区域找一个空位置创建**桩函数
2. 在 `600000` 区域找一个空位置创建**调度函数**，设置 EDX 偏移
3. 在特技数据表中新增一行（`9BBFE4` 偏移表）
4. 在游戏逻辑中找到合适的调用点调用

###### 方式二：内联直接调用路径（推荐 ⭐）

1. 在游戏逻辑中找到合适的注入点（已有 NOP 区域或函数尾部）
2. 写入 `jmp <代码洞地址>` 覆盖 5 字节
3. 在代码洞中编写特技逻辑：
   ```asm
   ; 内联桩核心：直接调用核心引擎
   push <效果值标志>       ; 00=需要效果值, 01=不需要
   push <叠加标志>         ; 00=可叠加, 01=不可叠加
   push <装备特效号>       ; 用户可编辑
   push <个人特效号>       ; 用户可编辑
   call 4101D9           ; → 直接判定
   test eax, eax
   jz  <跳过>
   ; [特技效果代码]
   ```

**优势**：跳过调度函数和查表，无需创建调度函数或 EDX 偏移，更简洁灵活。

##### 代码空洞

引擎代码段中存在多个未使用空间（全部为 NOP 填充），可用于放置新代码：

| 空洞 | 地址 | 大小 |
|------|------|------|
| A | `407675-4076FF` | 138B |
| B | `407308-407357` | 79B |
| D ⭐ | `4529A7-452F05` | **1374B** |
| E ⭐⭐⭐ | `601A9A-602500+` | **2150B+** |

> 详情见 [代码空洞与安全补丁方案](../01-核心引擎/代码空洞与安全补丁.md)

##### 两种编号体系 ⭐

游戏中每个角色同时拥有两套不同的编号：

###### Data号（角色数据号）— 角色"身份证"

| 属性 | 值 |
|------|-----|
| 范围 | 0 ~ 1023（共 1024 个角色） |
| 生命周期 | 永久不变，编译期固定 |
| 转指针 | `4061E4` → `RuntimeCharacter*`（`[4CEA00] + id × 48H`） |

Data号是角色的唯一永久标识，保存在数据库（如设计工具）中。无论在哪个关卡、哪场战斗中出现，同一个角色的Data号始终不变。

###### 战场编号（战场编号）— 出战"座位号"

| 属性 | 值 |
|------|-----|
| 范围 | 0 ~ 249（最多 250 个单位同时在场） |
| 生命周期 | 仅单场战斗有效，每场重新分配 |
| 转指针 | `4061F9` → `TacticalUnit*`（`4A7B20 + id × 30`） |

战场编号是临时编号，角色进入战场时被动态分配一个"座位号"。同一角色在不同战斗中可能获得不同的战场编号。

###### 两者关系

```
战场编号 ──4061F9──→ TacticalUnit* ──WORD[+00]──→ Data号 ──4061E4──→ RuntimeCharacter*
```

战术单位的 `[+00..+01]` 字段存储对应 Data 号，`[+04]` 是出场/显示编号载体。战斗数据结构 `4927F0` 同时存储攻击方战场编号（`+00`）、Data 号（`+04`）、人物运行时指针（`+08`）和战术单位指针（`+0C`），是两种编号体系的汇聚枢纽。

> 详见 [两种编号体系详解](../02-数据结构/单位结构体.md#两种编号体系)

##### 关键工具函数

| 地址 | 功能 | 调用方式 |
|------|------|----------|
| `4101D9` | 核心引擎：edx=单位, 装备号, 个人号→效果值 | 标准调用 |
| `4061E4` | Data号 → 人物运行时记录（48H 字节/角色，动态基址） | push Data号 + call |
| `4061F9` | 战场编号 → 战术单位指针（30字节/单位） | movzx + call |
| `484002` | 策略号 → 策略记录指针（61H 字节/策略） | mov/movzx + call |
| `42518F` | 包装判定：预检后调 4101D9，失败 回退 | 标准调用 |
| `425AD9` | 百分比计算：`x * y / 100` | call 425AD9 |
| `41A534` | 7档属性计算 | 直接调用 |

   ```asm
   push <个人特效号>    ; 用户可编辑

   ```

#### 常用调试技巧

##### 设置断点

在关键函数设置断点，观察参数和返回值：

```
bp 4101D9      ; 核心引擎
bp 4210D0      ; 引导攻击桩函数
bp 421175      ; 二次行动桩函数
```

##### 观察内存

使用内存窗口查看数据结构：

```
Memory[4A7B20] ; 单位数组基址
Memory[508B00] ; 二次行动状态数组
```

##### 追踪调用

使用调用栈窗口观察函数调用链：

```
Call Stack:
  4212F1  chase_attack_wrapper
  417C67  target_list_builder
  43AAF7  main_dispatcher
  43B2E0  battle_processor
```

#### 常见问题

##### Q: 如何添加新特技？

1. 在数据文件中添加特技配置（特效号、效果值）
2. 创建对应的桩函数（或复用现有桩函数）
3. 在特技触发点调用桩函数

##### Q: 如何修改现有特技的效果？

1. 定位特技的调用点
2. 修改调用逻辑或返回值处理

##### Q: 如何理解复杂的调用链？

使用 x64dbg 的追踪功能，逐步执行并记录调用：

```
TraceRecord.Start()
... 执行操作 ...
TraceRecord.Stop()
```
