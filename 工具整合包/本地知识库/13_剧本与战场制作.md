# 13 剧本与战场制作

## 已确认主线格式

- `.eex` 才是 R/S 剧本/资源相关主线格式。
- 剧本制作扫描 `RS\R_*.eex` / `RS\S_*.eex`。
- `.E5S` 是存档信息/旧兼容对象，不能重新作为 R/S 剧本主格式。
- 联网复核的 5.6 制作教程也把 `R_XX.eex` / `S_XX.eex` 解释为支配过场剧情和战斗关卡的 R/S 剧本；该外部口径只用于确认命名边界，实际读写仍以本地旧工具源码、样本复读和烟测为准。

## 当前剧本制作实现

- 2026-06-02 起，剧本制作主线优先使用旧版完整树：
  - `LegacyScenarioReader` 读取真实 `EEX\0 + Scene offset table + Scene/Section/CommandBlock`。
  - 内存模型为 `LegacyScenarioDocument -> LegacyScenarioScene -> LegacyScenarioSection -> LegacyScenarioCommandNode -> LegacyScenarioCommandBlock`。
  - 左侧树直接来自真实子块关系，不再把 `Section` 结构只当作 UI 启发式。
  - 文本优先绑定到命令内嵌文本参数，不再作为 Section 附近偏移线索猜测。
- 剧本制作页新增旧版完整结构保存：
  - 入口：剧本制作页 `完整保存剧本`，以及旧版文本参数编辑后的保存。
  - 写入方式：`LegacyScenarioWriter.Save`。
  - 写入内容：重建 10 字节 EEX 头、Scene 偏移表、Scene 内 Section 数、Section 长度、子块长度、命令参数和 `0x76` 跳转相对位移。
  - 正式项目也允许完整保存；保存前自动备份，先写临时文件，替换前按旧版规则重读校验。
  - 未修改完整写回烟测已验证 `R_00.eex changedBytes=0`，说明当前样本可字节级重建。
- 2026-06-06 起，单条指令正式修改入口对齐旧版 ItemData 链路：
  - TreeView 命令节点 `Tag` 绑定 `LegacyScenarioItemData`，不再把通用参数表作为正式修改主体。
  - 双击命令或点击修改时走 `LegacyCommandEditDispatcher -> LegacyMfcItemDataEditDialog`，按旧 `OnEditModify()` 分派到 71 个旧式 Dialog 行为。
  - Dialog 控件文案、下拉项、勾选框、列表和按钮来自旧 `cczEditor2.rc`/`DLGINIT`；OK 写回规则来自旧 `OnInitDialog/OnBnClickedOk`，只写 `IntData` / `LongCharData`。
  - 打开时使用快照工作对象，OK 才写回，Cancel 不修改原命令；文本显示和保存按旧版 `\n <-> 多行文本` 规则处理。
  - 保存仍由 `LegacyScenarioWriter` 按 `code_instruct` 整体重新编码，`0x05/0x35/0x04/16位参数/0x76` 跳转回填均走旧版完整树保存链路。
- 2026-06-06 继续统一旧版树 UI：
  - 剧本制作、战场制作 S 剧本、R 场景制作 R 剧本共用旧式可视树生成规则，读取成功时只显示 `文件路径 -> Scene -> Section -> 真实指令节点`。
  - `测试条件`、`正文根`、`事件正文`、`子事件主体/载体`、独立 `文本参数` 等调试包装节点不再出现在旧版完整树 UI；真实父子块仍保留在 `LegacyScenarioDocument/LegacyScenarioCommandBlock` 中用于保存和跳转回填。
  - 剧本制作右侧改为上方大图片预览、中部内嵌旧式 Dialog 参数区、下方 `备注`；备注写入 `CCZModStudio_Notes`，不写入 `.eex`，也不触发剧本未保存状态。
  - 顶部和右键菜单暂时隐藏新增、删除、复制、移动、粘贴预览等结构编辑入口，本轮只保留单条指令 Dialog 修改和完整保存。
  - 弹窗版旧式 Dialog 与剧本页右侧内嵌参数区共用同一套 `LegacyMfcDialogHostControl` 初始化、校验和提交逻辑。
- 原 `ScenarioStructureProbeReader` / `ScenarioTextWriter.SaveInPlace` 保留为兼容和诊断路径：
  - 旧版完整树读取失败时，剧本页回退到现有探针。
  - 原地短写仍只允许 GBK 字节数不超过原容量。
  - 高级探针页继续作为诊断/导出入口，不再作为剧本制作主模型。
- 剧本制作页按 CczSceneEditor2 习惯组织 Scene/Section/Command 树，并把这棵树作为主导航面；中间恢复当前上下文命令列表、文本线索和搜索结果，右侧显示当前树节点的属性、参数说明和文本参数编辑区。辅助表只跟随树节点刷新，不再作为反向 `SelectionChanged` 导航源。
- 2026-06-03 起，旧版完整树模式下开放保守命令节点编辑：
  - 新增命令来自 `CczString.ini` 命令字典和旧版 `code_instruct` 参数布局表；默认参数按布局生成，数值为 0，文本为空，可变数组为空。
  - `添加到正文末尾` 优先插入正文根的事件正文末尾，并避开尾部 `0x0C / 0x0D / 子事件结束 0x00` 分界命令。
  - `前插` / `后插` 只允许作用于普通命令；正文根、子事件设定、子事件载体、结束标记和 `0x0C / 0x0D` 分界命令旁不直接插入。
  - `追加到子块` 只在选中命令已有子块时启用，同样插在子块尾部分界命令之前。
  - `删除命令` 只允许删除无子块、未被 `0x76` 引用、且非结构分界的普通命令。
  - 选中参数槽后可在右侧输入十进制或 `0x` 十六进制并应用普通数值参数；当前只开放非结构/非跳转命令的 `Word16` 与非 `0x76` 的 `Dword32`，文本参数仍走文本节点编辑，可变数组保持只读保护。
  - 增删和参数修改只修改内存中的 `LegacyScenarioDocument`，需要点击 `完整保存剧本` 才写入文件；写入仍走备份、临时文件、旧版 Reader 复读校验和结构化报告。
- 剧本读取后选择具体 R/S eex 的 UI 性能边界：
  - 2026-06-02 复核 `R_00.eex`：旧版完整树可读到 `Scene=5`、`Section=7`、`Command=8528`、`Text=712`。
  - 最终定位：旧版不会卡死，是因为 `CTreeCtrl` 是唯一导航源，选中节点只更新当前项/状态栏；新版卡死来自树、命令表、文本表、搜索表之间的双向 `SelectionChanged` 级联，而不是 R/S eex 数据本身。
  - 当前规则：普通剧本完整生成旧版命令树，单个 Section 只有超过 `20000` 条命令时才触发异常折叠保护；命令表、文本表、搜索结果表作为树节点上下文视图保留，但不再订阅 `SelectionChanged` 作为导航入口。
  - 旧版文本参数使用 `LegacyScenarioCommandNode -> ScenarioStructureRow -> ScenarioTextEntry` 的精确映射，不再为旧版完整树执行偏移评分扫描。
  - 搜索仅作为显式定位：点击搜索后定位第一条命中并在右侧显示摘要，清空搜索不再重置当前树节点。
  - 2026-06-02 再次复核 `System.StackOverflowException`：全部 119 个 R/S eex 旧版完整读取最大嵌套深度为 13，不足以造成数据递归栈溢出；实际高风险链路是 WinForms `AfterSelect/SelectionChanged` 在树、命令表、文本表、搜索表之间互相触发。
  - 当前规则：用户点击只允许 `_scriptTree.AfterSelect` 触发一次展示；内部显式定位继续通过 `_bindingScriptDocument` 压制事件，并使用 `_selectedScriptCommandRow` / `_selectedScriptTextEntry` 保存当前对象，防止隐藏辅助表留下旧选择。
  - 旧版命令模型枚举已改为迭代式；Reader/Writer 对异常深嵌套加显式上限，UI 树超过 64 层时折叠为 `嵌套层级已折叠`，避免不可恢复的 StackOverflow。
- 左侧树与旧剧本编辑器的阅读顺序进一步对齐：
  - `Scene -> Section -> 测试条件 -> 正文根 -> 事件正文 / 子事件`
  - 文本线索优先归属到 `Section文本线索`，只把无法稳定归到某个 Section 的剩余文本放到 `Scene补充文本线索` 或 `未归属文本线索`。
  - 选中 `Scene` / `Section` 节点时，右侧只显示该结构节点详情，不再让命令/文本表反向选中其它对象。
  - 选中文本节点时，仅设置当前文本参数并刷新右侧编辑框；要复制/备注命令必须选中命令树节点本身。
- 文本线索支持直接写入当前项目 `RS\*.eex`：
  - 入口：剧本制作页 `保存选中文本`。
  - 写入方式：`ScenarioTextWriter.SaveInPlace`。
  - 写入限制：只支持 GBK 字节数不超过原容量的原偏移短写回。
  - 保存前自动备份目标 eex。
  - 保存后重新读取同偏移校验。
  - 结构化报告类型：`R/S eex 剧本文本写回`。
- 旧高级文本探针页的批量保存按当前项目可写口径执行，但仍受原 GBK 容量、目标路径、备份、复读和报告护栏限制；测试副本用于隔离验证和发布前对比，不再作为所有写入的前置条件。
  - 提示：剧本制作页的“剧本列表”在 UI 上按传统流程排序为 `R_00..R_nn` 在前、`S_00..S_nn` 在后，同类内按编号排序，避免按字典序出现 `R_1`/`R_10`/`R_2` 的错觉。

## MCP 只读剧本结构入口

- 2026-06-04 起，MCP 新增 `list_scenario_files`、`read_scenario_commands`、`search_scenario_scripts`。
- `list_scenario_files` 只列出项目 `RS` 目录下匹配 `R_*.eex / S_*.eex` 的剧本文件，可按 R/S 和关键字筛选。
- `read_scenario_commands` 复用 `LegacyScenarioReader`，可读取单个 R/S eex 的 Scene、Section、Command、参数、文本参数数量、跳转目标候选和结构标记。
- `search_scenario_scripts` 可在单个或多个 R/S eex 中搜索命令名、命令号、参数预览和 GBK 文本线索，用于无头定位剧情、胜败条件、变量、出场和战场事件。
- 这些 MCP 工具只读，不开放命令结构写回；需要修改文本仍优先走 `write_scenario_texts` 的容量内写回或 UI 的完整树 Writer，涉及结构/跳转/可变数组时必须继续保守处理。

## 当前战场制作实现

- 战场制作页读取 R/S eex 关卡后，显示标题、胜败条件文本、地图底图、地形层和战场命令候选。
- 战场关卡列表只取 `RS\S_XX.eex`；当前下拉框选中的 `S_XX.eex` 是标题/胜败条件、出场/坐标候选、战场命令定位三者共同的数据源。
- 战场命令候选必须优先来自 `LegacyScenarioReader` 的旧版完整树：
  - 读取真实 `Scene / Section / CommandIndex / Offset / Id`。
  - `TargetKey` 格式为 `Scene=...;Section=...;Command=...;Offset=...;Id=...`，必须能回查同一个旧版命令树。
  - `ScenarioStructureProbeReader(maxCommandRows:...)` 只作为旧版读取失败时的兜底，不能作为主导航键来源。
- 2026-06-06 起，战场制作左侧 S 剧本树也绑定 `LegacyScenarioItemData`：
  - 双击 S 剧本命令或点击 `旧版修改...` 进入同一个 `LegacyCommandEditDispatcher`。
  - OK 后复制 `ItemData` 回 `LegacyScenarioCommandNode`，刷新当前树节点，启用 `完整保存S剧本`。
  - 完整保存继续走 `LegacyScenarioWriter.Save`，保留备份、临时替换、旧版 Reader 复读校验和保存报告。
- 切换 S 关卡时，如果上一关仍在异步加载，必须记录待重载并在当前加载结束后补读当前选择，不能直接丢掉 `SelectedIndexChanged`。
- 标题/胜败条件文本支持原容量短写回，保存前备份，保存后复读。
- 出场单位、坐标、阵营、AI 等目前是命令候选整理和项目侧核对备注；未知 R/S eex 命令结构不强写。

- 战场制作页已改为左右分栏：

  - 左侧文本参数可编辑并保存；旧版完整树文本走 `LegacyScenarioWriter` 完整保存，兼容探针文本走 `ScenarioTextWriter.SaveInPlace` 原容量短写回。
  - 左侧 `标题/胜败` 页签保留原标题、胜败条件编辑入口。
  - 右侧从 `Map\Mxxx.jpg` 与 `Hexzmap.e5` 联动读取地图，按真实 `48x48 像素 = 1 格` 绘制网格。
- 右侧角色列表来自 `6.5-0 人物`、`6.5-0-4 R形象`、`6.5-0-5 S形象` 与 `6.5-4 详细兵种`：

  - 选中角色后用现有人物 R/S 预览服务显示 S 形象；S=0 时按职业和当前阵营槽计算 Unit 图号。
  - 可把角色从左侧列表拖到地图格，生成我军/友军/敌军、隐藏、等级修正、等级档、AI 方针、转向等项目侧布阵记录。
- 布阵记录保存到 `CCZModStudio_Notes/*_BattlefieldUnitReviews.json`：
  - `BattlefieldUnitReview.IsPlacement=true` 表示地图摆放项。
  - 坐标、阵营、S/R 形象、职业、等级、AI、隐藏、转向都保存在 JSON 中。
  - 该 JSON 是创作核对/计划文件，不写入 `S_XX.eex`，不参与发布封包。
- 安全边界：
  - 当前允许像旧工具一样拖放和可视化布阵，但出场命令参数字段尚未完全确认前，不自动强写 S 剧本中的坐标/阵营/AI/等级。
  - 真正修改出场命令参数仍需先明确命令 ID、参数槽语义、长度、跳转/Section 结构影响，再走备份、复读校验和实机验证。

## 2026-06-05 战场出场指令解析修正

- 问题原因：战场页能定位 `46/47/4B` 命令，但早期只按“每条命令一行”和短参数预览处理；兼容探针也没有同步旧源码中 `46`、`47` 的变长循环规则，导致友军/敌军出场大块被截断，后续命令可能错位，地图预加载只能看到少量或错误坐标。
- 已确认来源：
  - `全剧本指令详解（2023）.docx`：`44/45/46/47/4B` 是 S 战场部署核心命令，当前仍只做候选核对，不批量强写。
  - `Star175引擎发布文稿5.6-6.6.docx`：6.5 支持剧本数字输入框用负数表示整型变量，最常见用途是坐标；解析时不能把负数坐标当作非法值丢弃。

## 2026-06-01 剧本制作页 Scene 树 UI：按“事件/子事件结构”优化（启发式）

背景知识（来自旧剧本编辑器语法说明）：

- 一个剧本可由若干部分组成，每一部分内部包含多个事件；每个事件分为“测试条件（可多个）”与“事件体（可多个，可包含子事件）”，并由事件结束标志分隔。
- 子事件以“子事件开始标志”开头，测试条件只能一个，子事件内部也可继续嵌套子事件，最后以事件结束标志结束。

当前 CCZModStudio 的实现策略：

- `ScenarioStructureProbeReader` 仍是只读探针：只可靠识别 `0x0D` 结束 Scene、`0x0C` 结束 Section，并把 `0x00` 标为“事件结束/分隔标记候选”。参数长度未完全确认，不作为写回依据。
- 剧本制作页左侧树在 UI 层新增 “事件块” 分组：
  - `Scene -> Section -> 事件 -> (测试条件 / 事件 / 分隔标记候选) -> Command`
  - 事件分割规则（启发式）：在同一个 Section 下，以 `0x0000` 命令作为事件结束/分隔候选，把命令切成多个“事件”。
  - “测试条件”块规则（启发式）：事件起点到第一个 `0x0000` 之前的命令视为测试条件；第一个 `0x0000` 之后到下一个 `0x0000` 之前视为事件体。
  - 若某事件直到 Section 结束都没有遇到 `0x0000`，则整段视为“事件体”。
- 子事件 UI 嵌套（启发式）：基于旧工具 `CczString.ini` 的先验知识：
  - `0x0001` = 子事件设定（子事件开始标志候选）
  - `0x0000` = 事件结束（用于关闭最近的子事件块；如果没有子事件在栈中，则作为事件体结束边界候选）
  - 目前的树展示为：`事件 -> 事件(体) -> 子事件 -> ... -> Command`
  - 注意：这仍是 UI 层启发式，参数长度与真实跳转语义未完全确认；遇到不成对标记时，UI 会尽量“就近闭合/终止”避免无限嵌套。

相关代码位置：

- 树构建：`CCZModStudio/MainForm.cs` -> `BuildScriptTree(...)`
- Section/Scene 文本归属与联动：`CCZModStudio/MainForm.cs` -> `BuildScriptSectionTextAssignments(...)`、`BuildScriptSceneFallbackTextAssignments(...)`、`GetScriptTextsForNodeContext(...)`
- 结构探针：`CCZModStudio/Formats/ScenarioStructureProbeReader.cs`

- 根因：战场制作页旧版完整树初建时已走 `BuildLegacyEditorScriptTree()` 和 `LegacyScenarioCommandDisplayFormatter`，但 `ApplyBattlefieldScriptPreviewToNode()` 会为了显示地图预览覆盖，重新用通用 `BuildScriptCommandSummary(row, ...)` 写回节点文本，导致左侧 S 剧本树又显示裸参数/通用摘要。
- 修正：新增战场节点文本 helper，若节点 `Tag` 是 `LegacyScenarioItemData` 且存在 `Command`，预览刷新也使用 `BuildLegacyScriptCommandSummary(row, command, ...)`；只有兼容探针节点才回退通用摘要。
- 同步修正 `RefreshBattlefieldScriptPreviewTree()`：旧版完整树节点的 `Tag` 不是 `ScenarioStructureRow`，而是 `LegacyScenarioItemData`，刷新时必须同时识别两种 Tag，避免后续预览刷新漏掉旧版节点。
- 结果：战场制作页左侧 S 剧本树与剧本制作页一样，稳定显示旧版 `UpdateShow()` 中文映射；地图预览追加文字只在中文映射后面附加 `[地图预览已调整: ...]`。
