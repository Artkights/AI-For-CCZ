#### 2026-06-10 GameDebug MCP 阶段函数目录

`CCZModStudio.GameDebugMcpServer` 新增两个内部调试入口：

- `game_process_start`：直接启动/检查 `Ekd5.exe`，启动前校验 SHA256；真实启动需 `allow_launch=true`，不依赖鼠标、截屏或 x32dbg。
- `debug_function_catalog`：按阶段导出函数目录，离线复读 `Ekd5.exe` 的文件偏移和入口原字节，输出 `function-catalog.json/md`。
- `debug_static_xref_scan`：离线扫描 `.text` 中指向阶段函数的 `call/jmp/jcc rel32`，输出 `static-xref-scan.json/md` 和调用者断点候选。
- `debug_phase_probe_plan`：按阶段生成 `internal-probe-plan.json`，供 `debug_internal_probe_run` 设置断点并采集寄存器、反汇编、调用栈、只读内存和战场状态。
- `debug_autonomy_plan` / `debug_autonomy_run`：把启动/状态确认、阶段函数目录、阶段断点计划、可选探针运行和知识草稿串成一个无鼠标、无截屏的可审计流程。
- `debug_address_probe_run`：一键生成跨阶段地址报告和探针计划，并在显式 `run_probes=true` 时调用 x32dbg bridge 运行探针；默认安全模式不启动游戏、不启动调试器、不下断点。
- `debug_battle_profile_probe_run`：先执行只读战场 profile 门禁，再生成跨阶段地址探针计划；动态 probe 默认必须等 profile 匹配或攻击后 HP 差分出现。
- `debug_live_probe_readiness`：实机动态 probe 前置检查，确认目标进程/窗口、x32dbg bridge、运行态、战场 profile 和探针计划是否齐备。
- `debug_live_probe_auto_run`：一步式实机入口；可选启动游戏/调试器，执行 readiness，并在显式 `run_probes=true` 且 gate 通过后进入动态探针。
- `debug_capture_evidence`：默认只采集 x32dbg 状态、寄存器、断点、反汇编、栈、目标内存和只读战场状态；截图改为显式 `include_screenshot=true`。
- `game_runtime_state_classify`：运行态只读分类入口，用进程状态、x32dbg 状态和战场单位数组判断 `not_running`、`process_no_battle_signal`、`partial_battle_signal` 或 `battle_loaded`，为后续阶段选择提供条件。
- `game_runtime_wait_for_state`：轮询 `game_runtime_state_classify`，直到命中目标分类或超时，输出 `runtime-wait-summary.json/md`；不发送输入、不截屏、不写进程内存。
- `game_battle_state_match`：只读匹配已知战场 profile。v1 支持 `yingchuan_cao_zhangliang`，检查曹操 `unit[0] (10,6)`、张梁 `unit[61] (10,5)`，并用张梁 HP `< 168` 标记 `attack_after_observed`。
