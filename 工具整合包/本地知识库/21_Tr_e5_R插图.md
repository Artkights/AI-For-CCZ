## 信息传送 28：R 插图参数

旧剧本编辑器源码中 `信息传送28` 的示例说明可归纳为：

```text
第一行：R插图编号，取值 0-7，最多同时使用 8 个 R 插图。
第二行：X,Y，R 场景直角坐标。
第三行：光标变手 & 绘图层次, 透明程度。
第四行：图片在 Tr.e5 里的序号，从 0 开始计数；但 0 表示取消。
取消：除第一行插图编号外，其余数据全部填 0。
```

第三行第一项是类似“4按钮”的位标志：

- `0`：在 R 人物形象下方绘图，不变手。
- `1`：在 R 人物形象上方绘图。
- `3`：二进制 `11`，表示光标变手且在人物形象上方绘图。

第三行第二项是透明程度：

- 范围 `0..100`。
- 数字越大越清楚。

第四行图片编号注意事项：

- 源码说明“从 0 开始计数，但 0 又表示取消”。
- 实际制作时 `Tr.e5` 第一个图通常不能作为有效显示图，建议放占位图凑数。
- 当前工具侧若将来做 UI，应把“文件索引 #1”与“脚本可用编号 1?” 分开显示，避免 0/1 基准混淆。

## Tr.e5 样本结构

本地样本 `Tr.e5` 已按现有 E5 图片索引规则验证：

- 索引表起点：`0x110`。
- 索引项大小：`12` 字节。
- 字段：`sizeA`、`sizeB`、`dataOffset`，均为大端 32 位；`sizeA == sizeB`。
- 样本大小：`1163157` 字节。
- 样本条目数：`24`。
- 载荷类型：混合 `JPEG`、`BMP`、`PNG`。

## 与现有 MCP/E5 工具的关系

- `Tr.e5` 的结构与当前 `list_e5_image_entries` 支持的 E5 图片索引一致。
- 未来可以让 MCP/形象页读取 `E5\Tr.e5` 的条目列表和预览，但写入前还需要明确：
  - 当前 6.5 基底是否自带 `Tr.e5`，没有时是否允许新增文件。
  - 剧本中 `信息传送28` 的图号与 E5 条目是否严格按旧源码说明映射。
  - 图片尺寸是否有运行时上限、透明处理差异和坐标越界行为。
  - 取消 R 插图时 0 编号和其它行全 0 的实机行为。

## 当前安全边界

- 不自动生成 `信息传送28` 命令。
- 不自动新增或替换 `Tr.e5`。
- 不把样本 `Tr.e5` 加入 GitHub。
- 可做只读定位、知识检索、旧工具对照、实机验证记录。
