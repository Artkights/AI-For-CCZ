﻿# 旧扳手调试器源码对照

## 结论速览

- `老版游戏制作工具\扳手` 是 Python + PyQt5 写成的运行时内存调试器，不是离线资源编辑器；它通过拖拽准星附加游戏进程，再直接读写 `Ekd5.exe` 进程内存。
- 源码覆盖控制、人物、战场、仓库、天赋/专属/套装、变量、自定义地址和必杀配置等页面，可作为旧 6.x 运行时结构、界面字段命名和调试入口的源码级证据。
- 它的地址体系主要服务 6.1、6.2、6.3、6.3mp+、6.4 和 6.4 状态修正版/圣三分支；当前项目默认是 6.5 未加密，不能把本页旧地址直接当作 6.5 写回规则。
- 当前 6.5 已由实机调试确认战场单位数组 `004A7B20`、stride `30`、当前 HP `unit+10`；若与扳手旧版本映射冲突，以当前 6.5 本地样本和实机证据为准。

## 适用版本

- 扳手源码自身版本标识：`version = 1`，注释说明 `-1` 为 6.4 状态修正版，`0` 为 6.4，`1` 为 6.3mp+，`2` 为 6.3，`3` 为 6.2，`4` 为 6.1。
- 本页适用范围：整理旧工具源码事实、旧版运行时地址线索、页面行为和风险边界。
- 当前 6.5 使用范围：只能作为源码级交叉参照；进入 CCZModStudio 写回能力前，必须重新用 6.5 样本、x32dbg、只读探针、复读或烟测闭环确认。

## 已确认事实

### 源码文件角色

| 文件 | 角色 | 关键事实 |
|------|------|----------|
| `main.py` | 程序入口 | 创建 `QApplication`，实例化 `MyDesiger(QMainWindow, Ui_MainWindow)`，调用 `setupUi()` 并显示主窗体。 |
| `mywindow.py` | 主窗口和全部业务逻辑 | 前约 3000 行为 PyQt UI 生成代码，后续为进程附加、版本识别、页面切换、读写内存、后台锁定线程和 `DIY.csv` 解析。 |
| `hook.py` | 远程线程注入辅助 | 使用 `VirtualAllocEx`、`WriteProcessMemory`、`CreateRemoteThread` 把小段 x86 shellcode 和参数写入目标进程，调用游戏内部函数。 |
| `untitled.ui` | Qt Designer UI | 与 `mywindow.py` 生成界面对应。 |
| `README.md` | 发布说明 | 描述为通用曹操传调试/修改器，功能含逻辑控制、人物属性、战场信息、仓库、天赋、必杀、变量和自定义修改项；发布帖为 `http://www.xycq.online/forum/viewthread.php?tid=310288&extra=page%3D1`。 |
| `准星.cur` | 附加进程交互资源 | 拖动准星到游戏窗口后识别目标 HWND 和进程。 |

### 进程附加和版本识别

- 工具通过鼠标拖动准星获取窗口句柄，再由 `win32process.GetWindowThreadProcessId()` 找到进程，使用 `OpenProcess(PROCESS_ALL_ACCESS, ...)` 获取句柄。
- 附加前读取 `40100E`，若不是期望值 `774`，还会检查窗口类名是否为 `豪华曹操传`。
- 自动版本识别读取 exe 文件版本低字：`4 -> 6.4`，`3 -> 6.3`，`2 -> 6.2`，其它走 `6.1`；也支持 UI 手动覆盖版本。
- 6.3 会额外读取 `41A8A0`，若首字节为 `55`，改判为 `6.3mp+`。
- 读取 `401007` 若为 `F4E9`，切到 `version = -1`，并禁用部分页面，作为 6.4 状态修正版/圣三分支处理。

## 实现/使用方法

### 人物数据线索

- 人物数据基址来自指针 `4CEA00`，每人记录长度 `48`。
- 姓名读取：`base + id * 48 + 8`，最多 8 字节 GBK。
- R 形象编号：`50F800 + id * 2`；S 形象编号：`501000 + id * 2`；头像编号：`50F000 + id * 2`。
- 五维偏移：攻击 `+11`、防御 `+13`、精神 `+15`、爆发 `+17`、士气 `+19`，均 2 字节。
- HP 上限：`version <= 1` 为 `+1B` 4 字节，否则为 `+1C` 4 字节。
- MP 上限：`version <= 1` 为 `+1F` 2 字节，否则为 `+20` 1 字节。
- 我军标记：`+2A`，工具把 0 视为我军勾选，非我军写 255。
- 等级 `+2C`，经验 `+2D`。
- 五项成长/档位：`+21` 到 `+25`。
- 出阵 `+27`，幸存 `+28`，撤退 `+29`。
- 兵种 `+2B`；武器 `+2E/+2F/+30`，防具 `+31/+32/+33`，辅助 `+34`，其中 255 表示空。
- 上述人物装备槽是旧扳手读取运行时人物数据的偏移线索，不能与当前离线 S 剧本的 `48` 装备命令混用。当前 CCZModStudio 写回 `48` 时使用“人物、武器编码、武器等级、防具编码、防具等级、辅助编码”六参数，并保持 `0=默认、1=卸去、2+=分类内相对编号+2` 的脚本编码；武器/防具/辅助边界来自当前项目 `System.ini` 的 `DefID/AssID`。
- 击杀数 `+35`；`+06`、`+44`、`+46` 作为原始 2 字节字段显示。
- 五维功勋表：6.4 使用 `508000`，其它旧版使用 `508400`；只处理 `id <= 101`，每人 10 字节，五个 2 字节字段。
- 重算能力调用 `hook.recal()`，传参为 `base + 48 * id`，远程调用旧版内部函数 `4075DF`。

### 战场单位线索

- 战场记录首 2 字节为 Data 编号；`FFFF` 表示空槽。
- 旧工具把我军、友军、敌军按固定容量连续排列，并用 `cnt_wo/cnt_you/cnt_di` 决定遍历范围。
- 常用偏移：

| 偏移 | 含义 |
|------|------|
| `+00` | Data 编号，2 字节。 |
| `+04` | 出场/显示编号。 |
| `+05` | 阵营/队伍标记。 |
| `+06/+07` | 当前 X/Y 坐标。 |
| `+08` | AI 目标武将编号；255 表示使用坐标目标。 |
| `+09/+0A` | 目标坐标 X1/Y1。 |
| `+0C` | 显示/死亡状态；工具在 HP 为 0 且该字段为 3 时启用复活。 |
| `+0D` | 行动状态位；保存时按“已行动/未行动”写 `| 6` 或 `& 248`。 |
| `+0E` | AI/控制策略：3 被动、1 主动、2 坚守、0 攻击武将/到点、4 跟随/逃跑、7 友军可控。 |
| `+0F` | 朝向。 |
| `+10` | 当前 HP，4 字节。 |
| `+14` | 当前 MP，4 字节。 |
| `+18..+1D` | 攻击、防御、精神、爆发、士气、移动临时升降档；工具用 1/3/6 表示减少/正常/增加。 |
| `+1E` | 异常状态。普通模式按 bit 表示麻痹、禁咒、混乱、中毒；状态修正版按 4 个 2-bit 值拆解。 |
| `+20/+21` | 第二目标坐标 X2/Y2。 |
| `+22/+23` | 原始扩展字段，界面显示为 `22号` 和 `戮`。 |

- SP 不在单位记录内，而是按 Data 编号读取 `501C00 + data_code`。
- `version == 0` 额外显示一组扩展战场字段，按 `addr_war + war_code * 36 + 24/26/28/2A/2C/2E` 读取。
- 战场全局：
  - 回合上限：`4B3D0B`。
  - 当前回合：`4B3D0A`，界面显示时加 1。
  - 下一关：`4B0776`，战斗中保存为 `stage * 2 - 1`。
  - 天气：读取 `4B3D0C` 和 `4B3D19`；写入晴/阴/小雨/大雨/雪时分别写 `0/3/4/5/16` 到 `4B3D0C`，并把 `4B3D19` 清零。
- `onKill()` 按界面矩形坐标批量把敌方单位 HP 写 0。
- `onLife()` 读取方向、坐标、Data 编号，调用 `hook.life()` 在游戏进程内创建/复活单位。
- `LockThread()` 每秒检查进程和锁定项：可自动复活我军，也可锁定选中单位 HP/MP。

### 仓库和物品线索

- 装备仓库固定 200 个槽，每槽 3 字节：`4B0783 + slot * 3` 为物品 id，`+1` 等级，`+2` 经验；物品 id 为 255 表示空。
- 消耗品数量：
  - `version >= 1`：前 17 项位于 `4B09DB + i`，剩余位于 `510C80 + i - 17`，总 43 项。
  - `version < 1`：全部位于 `510C80 + i`，总 105 项。
- 金钱：`4B077C` 4 字节；忠奸度：`4B0782` 1 字节；功勋池：`505F40` 4 字节。
- 装备名表：`4A1140 + id * 25`，读取前 12 字节 GBK；物品类型位在 `+14`。
- 消耗品名表：`version >= 1` 使用 `4A19BF + id * 25`，6.4 使用 `4A1FE6 + id * 25`。
- “全宝物”会找空槽写入可用宝物，6.3/6.2/6.1 等旧版会跳过 `87..129` 区间，等级最高 9；经验为 255 且等级不是 9 时写 250。
- “全道具”把消耗品数量写 255；“清空仓库”把装备 id 写 255、消耗品数量写 0。

### 天赋、专属、套装和必杀线索

- 天赋表 `addr_tianfu + row * 8`：
  - `+0/+2/+4` 为三个 2 字节人物 id。
  - `+6` 为兵种 id，255 表示空。
  - `+7` 为效果值。
- 专属/套装表 `addr_zhuanshu + row * 16`：
  - 专属 1：人物 `+0`，物品 `+2`，效果 `+3`。
  - 专属 2：人物 `+4`，物品 `+6`，效果 `+7`。
  - 套装 1：物品 `+8/+9/+10`，效果 `+11`。
  - 套装 2：物品 `+12/+13/+14`，效果 `+15`。
  - 保存时若效果值为 0，工具会把对应人物或物品字段清零。
- 必杀表 `addr_bisha + row * 16`：
  - 五个角色 id 分别位于 `+0/+3/+6/+9/+12`，均为 2 字节。
  - 五个等级分别位于 `+2/+5/+8/+11/+14`，均为 1 字节。
  - 效果值位于 `+15`。
- 名称表按版本读取：
  - 6.4：天赋名 `4FF560` 共 180 项，必杀名 `4A7510` 共 80 项。
  - 6.3/6.3mp+：天赋名 `4FF7A0` 共 144 项，必杀名 `4FF551` 共 36 项。
  - 6.2：尝试在 `4F0000` 与 `4FF960` 之间判断天赋名表，必杀名 `4FF710` 共 36 项。
  - 6.1：天赋名 `4FFA80` 共 88 项，必杀名 `4FF710` 共 36 项。

### 变量和自定义地址线索

- 布尔变量区：`492FC8 + index * 4`，共 4096 项，界面显示 True/False。
- 整型变量区：`502000 + index * 4`，共 4096 项。
- 指针变量区：`506000 + index * 4`，共 4096 项。
- 保存变量时，工具把表格选中行的值按 4 字节写回对应地址。
- `DIY.csv` 位于工具运行目录，不在扳手源码目录内。CSV 格式由代码推断如下：

| CSV 列 | 含义 |
|--------|------|
| `row[0]` | 变量名；当 `row[1..5]` 为空时表示分组标题。 |
| `row[1]` | 值解释类型，已见 `战场编号`、`Data编号`、`宝物`。 |
| `row[2]` | 起始地址，16 进制文本。 |
| `row[3]` | 字节数或循环步长，16 进制文本。 |
| `row[4]` | 循环次数，10 进制文本。 |
| `row[5]` | 注释。 |

- DIY 页面按 `地址 = 起始地址 + i * 步长` 展开多行，读取指定字节数并解释为整数；保存时只把界面中的十进制数值写回选中地址。

### 远程调用线索

`hook.py` 的五个函数都遵循同一模式：在目标进程分配可执行内存，写入 shellcode 和参数区，然后 `CreateRemoteThread()` 执行。

| 函数 | 目标用途 | 关键内部地址 |
|------|----------|--------------|
| `life(h_process, dir, x, y, code, add)` | 创建/复活战场单位 | `add` 来自主程序版本表，即旧版复活函数指针。 |
| `recal(h_process, code)` | 重算人物能力 | 调用 `4075DF`，参数为人物记录指针。 |
| `changeDir(h_process, dir, code)` | 调整朝向/刷新显示 | 调用 `4B5DF0` 和 `457428`。 |
| `changePos(h_process, dir, x, y, code)` | 调整坐标/刷新显示 | 调用 `4B5DF0` 和 `4594DD`。 |
| `changeWeather(h_process, weather)` | 改天气并触发游戏内部刷新 | 设置 `ECX = 4B3D08`，调用 `41D9D1`。 |

### 控制页补丁线索

- 控制页不是单纯改数据，它会直接 patch 多处代码，例如敌友军可控、禁止 AI、穿越移动、待命后可移动、自动回归等。
- 部分控制功能会在 `485E00` 一带写入一段新逻辑，并把 `len_war`、`addr_war` 等运行时参数嵌入该逻辑；这是旧工具内存热补丁方式，不等同于当前项目可持久注入方案。

## 风险与边界

- 扳手直接写运行中进程，默认没有项目文件备份、结构化报告或复读校验；这不符合当前 CCZModStudio 写回护栏。
- 本页旧地址大多来自 6.1 到 6.4 工具适配。当前 6.5 的 `Ekd5.exe` 地址、函数、结构和补丁空洞必须重新确认，不能直接搬用。
- 当前 6.5 已确认的战场单位数组 `004A7B20`、stride `30`、HP `+10` 与扳手 `version == 0` 几何一致，但这只能说明线索相近，不能证明其它字段、函数和全局地址也一致。
- `mywindow.py` 混合了生成 UI 与业务逻辑，函数耦合高，很多字段语义只能从界面命名和读写位置反推。
- `version == 3` 的天赋名表判断分支使用了未在该分支中定义的 `i/j` 循环变量，存在源码缺陷。
- `saveWar()` 中读取下一关阶段的分支位于 `war_code` 赋值之前；如果该分支命中，可能触发未定义变量问题。
- `DIY.csv` 未随源码目录出现，本页格式是从读取逻辑推断，仍需找到样本 CSV 才能定稿。
- 控制页 patch 和 `hook.py` 远程线程调用属于动态调试能力，迁移到当前工具链时应转换为可审计计划、断点命中证据或持久补丁包，不能默默写进程内存。
