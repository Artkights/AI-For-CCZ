# 人工智能垂域S形象模型-延期计划

## 结论速览

- 本计划记录“文 + 角色参考图 + 曹操传 S 格式参考图 + 图生图”的垂域 S 形象模型方向。
- 当前状态：延期/资源不足，不进入现阶段实现，不训练模型，不接入 MCP 生产流程。
- 已选择的默认方向：S 形象专用、云端 80GB 显卡、仅使用授权/自有素材。
- 当前可执行工作只限于提示词、参考图规则、后处理规则和质检标准沉淀。

## 适用版本

- 默认服务于曹操传加强版 6.5 未加密当前基底。
- 后续若扩展到 6.6/通用 6.x，必须重新验证 `Unit_mov.e5`、`Unit_atk.e5`、`Unit_spc.e5` 的图号、条带尺寸、动作帧序和导入效果。

## 已确认事实

- 现有本地可见素材不足以支撑稳定训练一个可发布的 S 形象垂域模型。
- 通用绘图模型可以辅助产出草稿，但很难稳定满足曹操传 S 形象的低像素比例、洋红色键、条带格式、动作差异和武器一致性。
- `D:\Downloads\蛋卷AI形象包0608` 的经验表明：更可靠的工作流是“角色图 + 4 倍硬边 S/R 格式参考图 + 人工智能 草稿 + 缩回目标尺寸 + 手修”。
- 未公开 R/S 形象只能作为私有参考或学习观察；除非后续确认授权，不能进入训练集、模型发布或公开资料包。

## 实现/使用方法

- 以后重启本计划前，先补齐授权数据集：
  - 角色参考图、S 格式参考图、最终 `mov/atk/spc` 条带必须能一一配对。
  - 每套素材要记录授权状态、来源、角色类型、武器类型、动作类别和是否允许训练/发布。
  - 优先覆盖刀、枪、剑、弓、扇/杖、双持、文官、怪物、大体型单位等类别。
- 默认模型路线：
  - 不使用 GAN 作为主路线。
  - 不直接让扩散模型生成 `48x48/64x64` 小图。
  - 优先考虑低像素 token / masked transformer 路线，或先做“人工智能 草稿到曹操传条带”的修复模型。
- 默认输入协议：
  - 第一张图：角色身份、服装、发型、主色、武器和关键装饰。
  - 第二张图：曹操传 S 格式、4x6 布局、动作、朝向、人物比例、占格比例、洋红色键、轮廓粗细和特效尺度。
  - 文本：只做修正和约束，例如武器类型、职业、动作强调、配色保留、禁止项。
- 默认输出协议：
  - 中间输出为 4x6/24 格动作表。
  - 后处理导出 `Unit_mov.e5` 预览条带 `48x528`、`Unit_atk.e5` 预览条带 `64x768`、`Unit_spc.e5` 预览条带 `48x240`。
  - 生成结果只做替换预览，不直接写入游戏资源。

## 风险与边界

- 资源不足时训练会过拟合，容易只记住参考图角色或动作模板。
- 授权不清的素材不能用于可发布模型；私用实验也必须和公开发布数据分开管理。
- 即使模型训练完成，S 形象仍需要自动质检和必要手修，不能承诺一次生成即可入库。
- 当前 MCP 只优化提示词和后处理，不承担训练、模型管理、数据标注和版权审计。
