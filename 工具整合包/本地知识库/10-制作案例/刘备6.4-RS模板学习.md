# 刘备 6.4 R/S 模板学习

## 结论速览

- `基底/刘备6.4模组 - 副本/RS/R_01.eex` 与 `S_01.eex` 可作为成熟关卡 R/S 结构样本，用来学习过场、战前准备、战场初始化和战中事件的组织方式。
- 该样本只能学习结构、节奏和命令组合，不能照抄剧情、变量号、地图坐标、人物 ID、`0x72` 参数或作者个人风格。
- 独立首关仍以《三英战龙帝》已实机确认的 `R_00/S_00` 双 Scene 出战路线为基线；刘备 `R_01/S_01` 更适合作为已有流程中的后续关模板。
- 文本导入模板只处理对白、旁白、出场、移动、转向和动作等宏；背景、绘图、音轨、出战测试、部署、胜败、子事件和跳转必须走旧版完整树蓝图与写入器。

## 样本复读摘要

| 文件 | 结构 | 用途 |
| --- | --- | --- |
| `R_01.eex` | `10 Scene / 13 Section / 约 501 Commands` | 成熟 R 关卡：大地图引子、剧情 Scene、入队装备、奖励、战前出战、玩家切换和点击反应 |
| `S_01.eex` | `3 Scene / 14 Section / 约 366 Commands` | 成熟 S 关卡：战场初始化、战中事件、战后收束 |
| 三英 `R_00.eex` | `2 Scene / 2 Section / 约 129 Commands` | 独立首关：剧情 Scene + 玩家切换 Scene，已实机确认可进入战前准备 |
| 三英 `S_00.eex` | `3 Scene / 28 Section / 约 332 Commands` | 独立首关战场：初始化链、三阶段事件、四坛、轻量单挑、胜败 |

## R 剧本结构经验

刘备 `R_01` 的成熟结构可归纳为三层：

| 层级 | 样本用途 | 泛化规则 |
| --- | --- | --- |
| 引子 / 过场 Scene | 大地图、旁白、背景切换 | 可用多个 Scene，但必须确认当前基底入口会按预期进入；独立首关不要随意拆真实 Section |
| 正式剧情 Scene | `2F/24/27/30/1C/18/17/09/14/32/33/34/1E` | 初始人物先 `30`，再 `1C` 绘图；中途新人物 `30` 后尽快 `1E` 重绘 |
| 战前准备 Scene | 剧情末尾写 `4B`，玩家切换 Scene 写 `07/08/06/1D/24/0D` | 出战入口必须参考本地样本；不要用裸 `0x11` 或未验证 `0x72` 猜测链 |

常见 R 命令顺序：

```text
2F/24/27 -> 初始 30 人物出现 -> 1C 绘图 -> 18 事件名 -> 17 场所名
-> 09 停顿 -> 14/16 对白信息 -> 32/33/34 演出
-> 中途 30 出现 + 1E 重绘 -> 31 消失 + 1E 重绘
-> 3B 入队 / 3E 装备 / 3D 奖励
-> 4B 出战位 -> 08/06 出场限制 -> 00
```

玩家切换 Scene 的样本骨架：

```text
07 出战测试 -> 00
08 菜单处理 false -> 14 出发对白
06 我军出场限制或清理 -> 1D 调色板设定 -> 24 255
0D 结束 Scene -> 00
```

`0x72 信息传送` 在刘备样本中可见，但不是通用默认值。只有当目标基底已有同类成功样本、参数已复读、实机已验证时，才允许进入自动生成蓝图。

## S 剧本结构经验

刘备 `S_01` 可稳定拆成三段：

| Scene | 样本职责 | 泛化规则 |
| --- | --- | --- |
| Scene 1 | 战场初始化 | `44/45/46/47/48/4B/27/1C/24/19/1A/5A/08` 是检查重点；缺 `27/1C/24` 可能导致入场闪退或显示异常 |
| Scene 2 | 战中事件 | 每个事件 Section 把触发测试放在正文根之前，不能写进正文块 |
| Scene 3 | 战后收束 | 通常处理音乐、菜单、调色板和结束命令；具体跳转需按样本复读 |

S 战中事件标准结构：

```text
02 内部信息
-> 触发测试命令，如 25/2E/36/3F/41
-> 05 变量门或防重复
-> 00 正文根
-> 正文动作，如 14/16/23/3D/4E/4F/50/53/55/59/49/0E
-> 0B 变量赋值
-> 00 事件结束
```

不要生成：

```text
02 -> 00 正文根 -> 触发测试命令 -> 05 -> 正文动作
```

这种结构会让战场循环只看到一个立即结束的事件段，后续四坛、单挑、回合提示、击破检测都可能不触发。

## 可泛化事件蓝图

| 设计语义 | 触发命令 | 正文命令 | 备注 |
| --- | --- | --- | --- |
| 坐标奖励 | `25 进入指定地点测试` + `05` | `3D 获得物品` + `0B` | 物品 ID 和坐标必须复读 |
| 邻接对话 / 轻量单挑 | `2E 武将相邻测试` + `05` | `4F/50/14/53/38/0B` | 首版优先于完整单挑 |
| 回合提示 | `3F 回合测试` + `40 行动方测试` + `05` | `16/23/0B` | 防重复变量必填 |
| AI 变化 | `3F/36/25` + `05` | `4E AI 方针` + `16/0B` | 必须绑定真实战场记录 |
| 击破 / 复活 | `36 武将状态测试` + `05` | `14/55/0B` | Boss 护阵可用此类事件实现 |
| 失败 | `36/43` + `05` | `53/16/49/0E/0D/0B` | 撤退失败必须有明确收尾 |
| 胜利 | `41/42` + `05` | `59/49/0D/0B` | 显示目标 `19/1A` 与真实判定分开 |
| 完整单挑 | `2E` 或剧情触发 | `68/60/61/64/65/66/63/5F` | 样本可见，但自动化默认列为高风险 |

## 与三英战龙帝的边界

- 三英当前 `R_00/S_00` 是独立首关实测基线：`R_00` 用剧情 Scene + 玩家切换 Scene，不硬写 `0x11`，不使用 `0x72`。
- 刘备 `R_01/S_01` 是成熟后续关样本：可以学习多 Scene 组织、点击反应 Section、完整单挑和战后收束，但不能直接替代三英首关入口。
- 生成新关时先判断项目类型：独立首关用三英路线，既有模组后续关优先找同模组前后关样本，再参考刘备模板。
- 所有结构命令必须走旧版完整树写入、备份、复读和实机验证；文本导入宏只服务已定稿的可见文本与演出小段。

## 生成前检查清单

- 是否明确目标是独立首关、后续关，还是只替换文本。
- 是否复读目标 R/S 的 Scene、Section、命令数量和入口路线。
- 是否把 R 里的文学分幕与真实 Section 区分开。
- 是否把 S 事件触发条件放在正文根 `00` 之前。
- 是否检查 S 初始化链包含背景、绘图和音乐。
- 是否标注 `0x72`、`0x76`、完整单挑、跨 Section 复制为高风险。
- 是否把文本导入宏与完整树蓝图区分输出。
