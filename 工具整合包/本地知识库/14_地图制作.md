## 当前实现

- `地图制作` 页读取 `Map` 目录图片列表。
- 支持缩放、显示网格、叠加变量尺寸地形层。
- 支持从地图制作页直接修改当前地图对应的 Hexzmap 地形层。
- 网格尺寸由当前底图分辨率决定：`gridWidth = 图片宽度 / 48`，`gridHeight = 图片高度 / 48`。
- 网格绘制、鼠标坐标换算、地形叠加、地形画笔写回都必须共用同一套 `gridWidth/gridHeight`，不能再使用固定格数。
- `替换底图` 调用 `MapImageReplaceService.ReplaceMapImage`：
  - 目标必须在当前项目 `Map` 目录内。
  - 目标和替换来源必须是 JPG/JPEG。
  - 保存前自动备份目标底图。
  - 写入后用 SHA256 和图片尺寸复读校验。
  - 生成结构化报告，报告类型为 `地图底图替换`。

## 边界

- 当前不重封包未知地图格式，只处理 `Map\Mxxx.jpg` / `.jpeg` 整文件替换。
- 替换底图不会同步修改 `Hexzmap.e5`；底图变化后必须叠加地形格核对坐标。
- 若替换图尺寸变化，服务会允许写入但给出风险提示；尺寸变化会改变地图格数，必须检查对应 `Hexzmap.e5` 段是否同尺寸。
- 图片宽高不能被 48 整除时，不能安全启用地形层叠加与地形写回。

## 2026-05-31 最高优先级校正：地图制作必须驱动地形网格

- 地图底图不是单纯图片替换对象，它决定地形编辑网格。
- 规则：图片宽高必须按 48 像素折算为格子宽高。
- 示例：`20x20` 地图应为 `960x960`，`37x30` 地图应为 `1776x1440`。
- 地图替换后必须复读图片尺寸，并提示对应 Hexzmap 地形块是否需要同步调整。

## 2026-05-31 已落地：地图与地形同尺寸模型

- 地图制作页、地形编辑页、战场地图预览、剧本地图预览已统一使用地图/地形块真实尺寸。
- `M000 960x960 -> 20x20`、`M056 1776x1440 -> 37x30` 已在本轮手工校验中确认。
- 对应 `Hexzmap.e5` 目录表解析见 `12_地形编辑.md`；地图制作侧只应消费解析后的 `Width/Height/DataOffset`，不自行猜测固定块。

## 2026-05-31 规则固化：格子尺寸=48x48 像素

- 统一规则：`48x48 像素 = 1 个地图格`。
- 因此地图网格数计算为：
  - `gridWidth = imageWidth / 48`
  - `gridHeight = imageHeight / 48`
- 例子：
  - `960x960 -> 20x20`
  - `1776x1440 -> 37x30`
- 地图制作与地形编辑必须共用同一套 `gridWidth/gridHeight`，并以地图底图分辨率为唯一来源；任何写死 `30x30` 的假设都会导致坐标错位和错误写回风险。

## 2026-06-01 校正结论：地图制作与地形编辑必须共享同一“格数推导函数”

- 为避免两边逻辑漂移，地图制作页与地形编辑页不应该各自“再算一次格数”，而是统一消费同一套来源数据：
  - 地图底图分辨率（Width/Height）
  - 统一规则 `MapTilePixelSize=48`
  - 推导出 `GridWidth/GridHeight`
- 代码侧约束：当底图宽高不能被 48 整除时，必须禁用叠加与写回，并在 UI 明确提示原因（防止产生“看起来能画但写回错位”的风险）。
