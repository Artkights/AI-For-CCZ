# 曹操传 6.5 独立单关 MOD 一键制作提示词模板

本模板用于让 AI 直接制作“独立单关游戏包”：一次输入主题后，生成可测试的 `R + S` 小型游戏体验，而不是只改人物名、兵种名、物品名或追加几句对白。

默认路线已经改为：

`StandaloneScenarioDesign -> force_open ModPackage -> preview -> test_copy apply -> auto_validate_mod -> export_mod_report`

`force_open` 只移除阻碍完整关卡生成的无用限制：白名单命令阻断、模板块限制、复杂跳转降级成说明文本、资源缺失直接停止。备份、写后复读、结构报告和测试副本仍必须保留。

## 一键版

把下面整段复制给 AI，并替换尖括号内容：

```text
我要制作一个曹操传加强版 6.5 未加密基底的独立单关 MOD：<一句话主题，例如“水战奇袭龙帝军”>。

目标是“一口气生成可玩的独立单关游戏包”，不是改名包。信息缺失时你必须自动补全，不要反复问我。

必须使用 standalone_single_stage / force_open 路线：
1. detect_project
2. auto_make_mod(prompt=<完整主题>, automation_mode=force_open, max_repair_attempts=3)
3. 如果 auto_make_mod 没有完整输出设计和包，则改走手动流水线：
   - analyze_standalone_scenario_request(prompt=<完整主题>)
   - compile_standalone_scenario_package(design=<上一步 Design>, slot_limit=64)
   - preview_mod_package(package=<Package>, automation_mode=force_open)
   - 只修复具体问题：GBK 超长、槽位冲突、路径不存在、资源格式错误、命令参数结构错误；不要退回旧白名单模板。
   - apply_mod_package(package=<修复后的 Package>, write_mode=test_copy, automation_mode=force_open)
4. auto_validate_mod(package=<Package>, run_smokes=true)
5. export_mod_report(package=<Package>, report_kind=standalone_force_open)

必须生成并写入测试副本的内容：
- StandaloneScenarioDesign：标题、主题、剧情梗概、R 编号、S 编号、角色、战斗部署、事件图、奖励、资源需求。
- R 剧本：开场背景、人物登场、移动、对白、选择或目标说明、跳转到 S。
- S 剧本：地图初始化、我军/友军/敌军部署、AI、胜败条件、回合事件、击破事件、敌将台词、胜利分支、失败分支、战后收束。
- 表数据：人物、职业、物品、等级、能力、装备绑定、必要特效。
- 资源计划：头像、R 形象、S 形象、地图、图标；没有素材时写入可执行的 resourceUpdates generate_task 或占位资源计划。
- 验证计划：R 能打开、R 能跳 S、S 能载入地图、部署数量正确、胜败条件存在、战中事件存在、S 能回 R 或结束、复读后命令树不损坏。

默认内容规模：
- 4 名我方角色。
- 1 名友军角色。
- 8-12 名敌军。
- 1 名关底敌将。
- 至少 2 个战中事件。
- 至少 1 个敌将台词事件。
- 必须有胜利剧情、失败分支、奖励结算和战后收束。

禁止退回旧行为：
- 不要把 analyze_mod_request / compile_mod_package 当作独立单关主流程。
- 不要用 compile_scenario_patch 生成主 R/S 结构。
- 不要声明“只使用白名单命令”。
- 不要把复杂跳转、部署、胜败测试、子事件降级成说明文本。
- 不要只输出 ModDesign 或文字方案，必须输出并应用 ModPackage 到测试副本。

最终回复必须列出：
- StandaloneScenarioDesign 摘要。
- 事件图节点清单。
- 战场部署清单：阵营、坐标、AI、等级、装备。
- R/S 剧本补丁范围和关键命令类型。
- 表数据、特效、资源生成任务的写入范围。
- metadata.tags.constraint_mode=force_open 是否存在。
- 测试副本路径、apply 报告路径、验证报告路径。
- 可玩性验收结果；如果失败，列出失败项和已强制放行的风险。
```

## 完整工程版

适合已经想清楚人物、剧情、玩法和资源风格时使用：

```text
目标：为曹操传加强版 6.5 未加密基底制作一个可运行的独立单关游戏包。只做单关闭环，不做多关战役。

MOD 设定：
- 标题：<MOD 标题>
- 主题：<魏军外传/蜀汉外传/水战/火攻/妖术/架空/历史改写等>
- 剧情梗概：<开端、冲突、反转、结局>
- 主角：<姓名、性格、战斗定位>
- 我方角色：<默认 4 人；可写姓名/兵种/定位>
- 友军角色：<默认 1 人；可写姓名/兵种/定位>
- 敌方角色：<敌将、精英兵、普通兵、特殊机制>
- 战斗地图：<城池/山地/渡口/野外/营寨/自定地图编号>
- 胜利条件：<击破敌将/占领地点/坚持回合/护送>
- 失败条件：<主角撤退/友军撤退/超过回合/敌军进入区域>
- 战斗节奏：<标准/困难/高压/解谜/剧情演出优先>
- 奖励：<宝物/装备/金钱/隐藏奖励>
- 资源风格：<原版像素风/写实三国风/暗黑异闻风/水军风/其他>
- 额外要求：<商店、隐藏事件、地形变化、伏兵、援军、特殊 AI 等>

强制执行路线：
1. detect_project。
2. analyze_standalone_scenario_request，生成完整 StandaloneScenarioDesign。
3. 检查 Design 是否包含：
   - Roles：我方、友军、敌军、关底敌将。
   - BattleDeploymentPlan：地图、目标、回合限制、胜败条件、单位坐标、AI、等级。
   - ScenarioEventGraph：intro、deploy、turn_start、unit_defeated、victory、defeat、epilogue。
   - ScenarioResourcePlan：头像、R/S 形象、地图、图标或 generate_task。
4. compile_standalone_scenario_package，生成带 force_open 标记的 ModPackage。
5. preview_mod_package(package, automation_mode=force_open)。
6. 如果 preview 失败，只允许围绕具体错误修复：GBK 长度、ID 冲突、路径、资源格式、命令参数、表字段范围。不得降级到旧模板或白名单。
7. apply_mod_package(package, write_mode=test_copy, automation_mode=force_open)。
8. auto_validate_mod(package, run_smokes=true)。如果 smoke 运行环境不可用，必须仍输出验证计划和未运行原因。
9. export_mod_report(package, report_kind=standalone_force_open)。

必须产出的 ModPackage 要求：
- metadata.tags.constraint_mode 必须为 force_open。
- scenarioPatches 必须同时包含 R 和 S。
- R 必须包含开场、登场、对白、目标说明、跳转到 S。
- S 必须包含部署、AI、胜败条件、至少 2 个战中事件、敌将台词、胜利、失败、战后收束。
- tableUpdates 必须覆盖人物、职业、物品或装备绑定中的实际可玩数据，不能只改显示名。
- effectPackages 必须覆盖至少 1 个可感知的装备、兵种、个人或套装特效。
- resourceUpdates 缺素材时必须使用 generate_task 或占位资源计划，不得只写“以后生成”。
- validationPlan 必须包含可玩性检查，不得默认 skipped。

验收标准：
- 不是改名包：不能只有人物名、兵种名、物品名变化。
- R 能作为开场剧情打开。
- R 存在跳转到 S 的路径。
- S 能载入地图并部署单位。
- 我方、友军、敌军、敌将数量符合设计。
- 胜利条件和失败条件均存在。
- 至少 2 个战中触发和 1 个敌将台词事件存在。
- 胜利后有奖励结算或 epilogue，失败后有失败分支。
- 测试副本写入、复读、报告均成功；验证失败时报告必须列出可玩性风险。
```

## AI 执行硬性规则

- 用户只给一句主题时，自动设计人物、剧情、兵种、装备、特效、地图、资源风格和事件结构。
- 独立单关默认走 `auto_make_mod(automation_mode=force_open)`；需要手动拆解时才走 `analyze_standalone_scenario_request -> compile_standalone_scenario_package`。
- `analyze_mod_request -> compile_mod_package` 只用于普通表/资源/特效包，不作为独立单关主路径。
- R/S 主结构必须由 standalone 编译器生成，不让自然语言模型直接手填二进制偏移。
- 允许编译器生成结构插入、跨段跳转、部署命令、胜败测试、子事件和 `76` 跳转。
- preview 失败优先自动修复并重试，不直接结束。
- 测试副本通过后才允许建议正式推广；正式基底只通过 `promote_test_copy_mod(confirm_promote=true)` 推进。
- 任何强制放行都必须写进报告，不能在最终回复里隐瞒风险。
