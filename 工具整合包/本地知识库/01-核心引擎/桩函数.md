## 详细记录

#### 概述

桩函数是特技判定系统的入口点，每个特技对应一个固定格式的桩函数。它们位于内存的 `0x421080-0x42115A` 区域，每个占 17 字节。

#### 函数格式

##### 汇编结构

```asm
; 每个特技桩函数的固定格式
push <效果值标志>      ; 参数1
push <叠加标志>        ; 参数2
push <装备特效号>      ; 参数3
push <个人特效号>      ; 参数4
call <调度函数>        ; 调度函数地址
```

##### 参数说明

| 参数 | 名称 | 取值范围 | 说明 |
|------|------|----------|------|
| 参数1 | 效果值标志 | `0` 或 `1` | 控制是否需要读取效果值 |
| 参数2 | 叠加标志 | `0` 或 `1` | 控制效果值能否叠加 |
| 参数3 | 装备特效号 | 0x0000-0xFFFF | 装备提供的特技编号 |
| 参数4 | 个人特效号 | 0x0000-0xFFFF | 角色自身的特技编号 |

#### 参数详解

##### 效果值标志

控制是否需要读取效果值（用于控制特技强度的数值）：

| 值 | 含义 | 典型用途 |
|----|------|----------|
| `1` | 不需要效果值 | 纯开关型特技（如引导攻击、二次行动） |
| `0` | 需要效果值 | 有强度等级的特技（如MP恢复、伤害加成） |

##### 叠加标志

控制个人和装备两个渠道的效果值能否叠加：

| 值 | 含义 | 说明 |
|----|------|------|
| `1` | 不可叠加 | 只取其中一个渠道的效果值 |
| `0` | 可叠加 | 两个渠道的效果值累加 |

##### 特效号

特技编号是 16 位数值，用于在游戏数据中定位具体的特技配置。

#### 代码示例

##### 引导攻击 (0x4210D0)

```asm
; 引导攻击桩函数
; 效果值标志=1（不需要效果值）
; 叠加标志=1（不可叠加）
; 装备特效号=0x36(54)
; 个人特效号=0x18(24)
004210D0: push 0x01    ; 不需要效果值
004210D2: push 0x01    ; 不可叠加
004210D4: push 0x36    ; 装备特效号 = 54
004210D6: push 0x18    ; 个人特效号 = 24
004210D8: call 0x006002C6
```

##### 二次行动 (0x421175)

```asm
; 二次行动桩函数
; 效果值标志=1（不需要效果值）
; 叠加标志=1（不可叠加）
; 装备特效号=0x5A(90)
; 个人特效号=0x38(56)
00421175: push ebp
00421176: mov ebp, esp
00421178: push 0x01        ; 效果值标志 = 1
0042117A: push 0x01        ; 叠加标志 = 1
0042117C: push 0x5A        ; 装备特效号 = 90
0042117E: push 0x38        ; 个人特效号 = 56
00421180: call 0x00600819  ; 调度函数
00421185: pop ebp
00421186: ret
```

##### MP恢复 (0x421106)

; 叠加标志=0（可叠加）
; 装备特效号=0x1B(27)

00421106: push ebp
00421107: mov ebp, esp

0042110B: push 0x00        ; 叠加标志 = 0（可叠加）
0042110D: push 0x1B        ; 装备特效号 = 27
0042110F: push 0x01        ; 个人特效号 = 1
00421111: call 0x0060002A  ; 调度函数
00421116: pop ebp
00421117: ret

#### 内联桩函数

##### 概述

并非所有特技都必须使用 `0x421080-0x421180` 区域的标准桩函数。在实际模组开发中，**内联桩**是更常用且更灵活的模式：

**内联桩**：在游戏逻辑的任意合适位置直接内联桩函数的核心逻辑，即 `push 4个参数 → call 0x4101D9`，跳过调度函数和查表环节。

##### 内联桩模板

```asm
; 内联桩函数标准模板
; 使用场景：任意游戏逻辑函数内部
mov ecx, <单位指针>           ; 设置 ECX = 单位指针
push <效果值标志>             ; 0x00=需要效果值, 0x01=不需要
push <叠加标志>               ; 0x00=可叠加, 0x01=不可叠加
push <装备特效号>             ; 用户可编辑
push <个人特效号>             ; 用户可编辑
call 0x4101D9                 ; → 直接调用核心引擎
test eax, eax                 ; 检测结果
jz  <skip_label>              ; 无特技 → 跳过
; [特技效果代码]
skip_label:
```

##### 内联桩对比标准桩

| 特性 | 标准桩 (0x421xxx) | 内联桩 |
|------|-------------------|--------|
| 位置 | 固定区域 | 任意位置 |
| 调度函数 | 必须经过 0x600xxx | 跳过（直接调核心引擎） |
| 查表 (0x9BBFE4) | 必须 | 跳过 |
| 创建难度 | 需在 0x421xxx 找空位 | 只需在目标函数附近找5字节空间 |
| 代码洞需求 | 通常需要 | 通常需要 |
| 适用场景 | 引擎原生特技 | **模组新增特技（推荐）** |
