#### 二、二次行动特技 - 完整解析 (412D52)

> ⚠️ 本小节所有函数**仅服务于二次行动这一个特技**。没有二次行动的单位在攻击/施法/移动后不会自动恢复行动力。

##### 2.1 函数完整流程反汇编

```asm
; 函数: try_second_action (412D52)
; 参数: [ebp+08] = 单位索引 (1字节)
; 返回: EAX (0 = 失败, 1 = 成功)
412D52:   push    ebp
412D53:   mov     ebp, esp
412D55:   mov     cl, [ebp+08]  ; cl = 单位索引

; =====================
; 检查 1: 单位有效性
; =====================
412D58:   call    4061F9    ; 获取单位指针 (ecx -> ecx)
412D5D:   push    ecx
412D5E:   cmp     word [ecx], FFFF  ; 检查 unit[00]
412D63:   jz      412DB0    ; 无效单位 -> 返回

; =====================
; 检查 2: 单位是否存活
; =====================
412D65:   xor     eax, eax
412D67:   cmp     [ecx+10], eax  ; 检查 HP (unit[10])
412D6A:   jz      412DB0    ; HP = 0 -> 死亡，返回

; =====================
; 检查 3: 状态标志
; =====================
412D6C:   movzx   edx, byte [ebp+08]
412D70:   add     edx, 508B00    ; edx = &status_flags[unit_index]
412D76:   push    edx
412D77:   cmp     byte [edx], al   ; al = 0
412D79:   jnz     412DB0    ; != 0 -> 已使用/不可用，返回

; =====================
; 检查 4: 特技判定
; =====================
412D7B:   nop                ; 5个NOP (可能是预留空间)
412D7C:   nop
412D7D:   nop
412D7E:   nop
412D7F:   nop

412D80:   call    421175    ; 调用二次行动桩函数
412D85:   test    eax, eax
412D87:   jz      412DB0    ; 判定失败 -> 返回

; =====================
; 成功分支: 恢复行动
; =====================
412D89:   mov     ecx, [ebp-04]  ; ecx = 单位指针
412D8C:   push    06        ; 参数 = 06 (二次行动 bit)
412D8E:   call    406690    ; 恢复行动能力函数

412D93:   push    88        ; 参数 3
412D98:   push    06        ; 参数 2
412D9A:   mov     ecx, [ebp-04]
412D9D:   push    [ecx+04]  ; 参数 1: 单位 ID
412DA0:   call    43E6AA    ; 更新界面显示

; =====================
; 标记已使用
; =====================
412DA5:   mov     eax, 01
412DAA:   mov     ecx, [ebp-08]
412DAD:   mov     byte [ecx], FF  ; status_flags[index] = FF

; =====================
; 函数返回
; =====================
412DB0:   mov     esp, ebp
412DB2:   pop     ebp
412DB3:   ret     04
```

##### 2.2 行动恢复函数 406690

```asm
; 参数: [ebp+08] = 行动类型 (bit位置)
; ECX = 单位指针
406690:   push    ebp
406691:   mov     ebp, esp
406693:   movzx   eax, byte [ebp+08]  ; eax = 行动类型
406697:   not     eax                   ; 取反 (mask)
406699:   and     byte [ecx+0D], al  ; 清除对应 bit
40669C:   mov     esp, ebp
40669E:   pop     ebp
40669F:   ret     04
```

**算法：** `unit[0D] &= ~(1 << action_type)`

##### 2.3 状态数组 508B00 的值含义

| 值 | 含义 | 说明 |
|----|------|------|
| 00 | 待激活 | 回合开始被重置为此状态 |
| 01 | 不可用 | 单位不拥有该特技 |
| FF | 已使用 | 本回合已触发过二次行动 |

##### 3.2 参数详解

###### 参数 1 - 效果值标志

| 值 | 含义 | EAX 返回值含义 | 典型场景 |

| 值 | 含义 | 说明 |
|----|------|------|
| 01 | 不可叠加 | 只取个人或装备渠道的单一结果 |
| 00 | 可叠加 | 个人和装备渠道的效果值可以累加 |

##### 8.2 6.3/6.4 到 6.6 的跨版本变量线索

- 6.4 系统笔记说明 `4031` 用于在部分剧本指令中切换“Data 号 / 战场编号”的解释方式，典型用于复制人或同 Data 多单位的场景。
- 6.4 物品扩充资料提到旧流程曾用 `4080` 处理新道具读取问题；在当前 6.5 工具链中，这只能作为旧编辑器兼容线索，不能替代 `HexTable.xml` 和本地表结构。
- 6.6 发布资料说明动态图控制会把停止时的当前显示次数保存到 `4038`，且资源边界迁移到 `DT.e5`。这属于 6.6 差异，不进入 6.5 默认写回路径。

##### 8.3 处理原则

- 论坛发布帖可以确认“存在某能力”和“作者公开描述的用途”，但不能确认当前基底的二进制偏移、参数槽长度和所有边界行为。
- 任何整型变量映射进入可写功能前，必须补齐三类证据：本地样本命令、旧工具源码或 docx 说明、实机/烟测复读。
- 6.6 的 `DT.e5`、R 文字和 R 插图微调只进入版本差异；若未来支持 6.6，需要独立 `HexTable.xml`、资源探针和写入护栏。
