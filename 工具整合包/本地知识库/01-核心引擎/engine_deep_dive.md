#### 二、二次行动特技 - 完整解析 (0x412D52)

> ⚠️ 本小节所有函数**仅服务于二次行动这一个特技**。没有二次行动的单位在攻击/施法/移动后不会自动恢复行动力。

##### 2.1 函数完整流程反汇编

```asm
; 函数: try_second_action (0x412D52)
; 参数: [ebp+0x08] = 单位索引 (1字节)
; 返回: EAX (0 = 失败, 1 = 成功)
0x412D52:   push    ebp
0x412D53:   mov     ebp, esp
0x412D55:   mov     cl, [ebp+0x08]  ; cl = 单位索引

; =====================
; 检查 1: 单位有效性
; =====================
0x412D58:   call    0x4061F9    ; 获取单位指针 (ecx -> ecx)
0x412D5D:   push    ecx
0x412D5E:   cmp     word [ecx], 0xFFFF  ; 检查 unit[0x00]
0x412D63:   jz      0x412DB0    ; 无效单位 -> 返回

; =====================
; 检查 2: 单位是否存活
; =====================
0x412D65:   xor     eax, eax
0x412D67:   cmp     [ecx+0x10], eax  ; 检查 HP (unit[0x10])
0x412D6A:   jz      0x412DB0    ; HP = 0 -> 死亡，返回

; =====================
; 检查 3: 状态标志
; =====================
0x412D6C:   movzx   edx, byte [ebp+0x08]
0x412D70:   add     edx, 0x508B00    ; edx = &status_flags[unit_index]
0x412D76:   push    edx
0x412D77:   cmp     byte [edx], al   ; al = 0
0x412D79:   jnz     0x412DB0    ; != 0 -> 已使用/不可用，返回

; =====================
; 检查 4: 特技判定
; =====================
0x412D7B:   nop                ; 5个NOP (可能是预留空间)
0x412D7C:   nop
0x412D7D:   nop
0x412D7E:   nop
0x412D7F:   nop

0x412D80:   call    0x421175    ; 调用二次行动桩函数
0x412D85:   test    eax, eax
0x412D87:   jz      0x412DB0    ; 判定失败 -> 返回

; =====================
; 成功分支: 恢复行动
; =====================
0x412D89:   mov     ecx, [ebp-0x04]  ; ecx = 单位指针
0x412D8C:   push    0x06        ; 参数 = 0x06 (二次行动 bit)
0x412D8E:   call    0x406690    ; 恢复行动能力函数

0x412D93:   push    0x88        ; 参数 3
0x412D98:   push    0x06        ; 参数 2
0x412D9A:   mov     ecx, [ebp-0x04]
0x412D9D:   push    [ecx+0x04]  ; 参数 1: 单位 ID
0x412DA0:   call    0x43E6AA    ; 更新 UI 显示

; =====================
; 标记已使用
; =====================
0x412DA5:   mov     eax, 0x01
0x412DAA:   mov     ecx, [ebp-0x08]
0x412DAD:   mov     byte [ecx], 0xFF  ; status_flags[index] = 0xFF

; =====================
; 函数返回
; =====================
0x412DB0:   mov     esp, ebp
0x412DB2:   pop     ebp
0x412DB3:   ret     0x04
```

##### 2.2 行动恢复函数 0x406690

```asm
; 参数: [ebp+0x08] = 行动类型 (bit位置)
; ECX = 单位指针
0x406690:   push    ebp
0x406691:   mov     ebp, esp
0x406693:   movzx   eax, byte [ebp+0x08]  ; eax = 行动类型
0x406697:   not     eax                   ; 取反 (mask)
0x406699:   and     byte [ecx+0x0D], al  ; 清除对应 bit
0x40669C:   mov     esp, ebp
0x40669E:   pop     ebp
0x40669F:   ret     0x04
```

**算法：** `unit[0x0D] &= ~(1 << action_type)`

##### 2.3 状态数组 0x508B00 的值含义

| 值 | 含义 | 说明 |
|----|------|------|
| 0x00 | 待激活 | 回合开始被重置为此状态 |
| 0x01 | 不可用 | 单位不拥有该特技 |
| 0xFF | 已使用 | 本回合已触发过二次行动 |

#### 三、特技判定的参数详解 (桩函数传递的 4 个参数)

##### 3.1 参数结构

```
[栈顶]
+-----------------------+
| 参数 4: 个人特效号    | [ebp+0x08]
+-----------------------+
| 参数 3: 装备特效号    | [ebp+0x0C]
+-----------------------+

| 参数 1: 效果值标志    | [ebp+0x14]
+-----------------------+
```

##### 3.2 参数详解

###### 参数 1 - 效果值标志

| 值 | 含义 | EAX 返回值含义 | 典型场景 |
|----|------|----------------|----------|
| 0x01 | 不需要效果值 | 1 = 拥有, 0 = 没有 | 引导攻击、二次行动 |
| 0x00 | 需要效果值 | 效果值 (0 = 失败) | MP恢复、HP恢复等 |

###### 参数 2 - 叠加标志

| 值 | 含义 | 说明 |
|----|------|------|
| 0x01 | 不可叠加 | 只取个人或装备渠道的单一结果 |
| 0x00 | 可叠加 | 个人和装备渠道的效果值可以累加 |

#### 四、单位结构体 (Unit Structure)

##### 4.1 完整结构体布局 (已知字段)

```
偏移  大小  含义                    说明
---------------------------------------------------------
0x00  2     单位有效性标记          0xFFFF = 无效
0x04  1     单位ID                  用于索引 0x508B00
0x0D  1     行动状态位掩码          每个 bit 控制一种行动类型
0x10  4     当前HP                  归零 = 死亡
0x14  4     最大HP
0x1E  1     异常状态标志            中毒、混乱等
0x23  1     回合状态标志            新回合清零
```

##### 4.2 行动状态位掩码 (unit[0x0D])

| Bit | 对应行动 | 对应清除参数 |
|-----|----------|--------------|
| 0 | 移动 | 0x00 |
| 1 | 攻击 | 0x01 |
| 6 | 二次行动 | 0x06 |

##### 8.2 6.3/6.4 到 6.6 的跨版本变量线索

- 6.4 系统笔记说明 `4031` 用于在部分剧本指令中切换“Data 号 / 战场编号”的解释方式，典型用于复制人或同 Data 多单位的场景。
- 6.4 物品扩充资料提到旧流程曾用 `4080` 处理新道具读取问题；在当前 6.5 工具链中，这只能作为旧编辑器兼容线索，不能替代 `HexTable.xml` 和本地表结构。
- 6.6 发布资料说明动态图控制会把停止时的当前显示次数保存到 `4038`，且资源边界迁移到 `DT.e5`。这属于 6.6 差异，不进入 6.5 默认写回路径。

##### 8.3 处理原则

- 论坛发布帖可以确认“存在某能力”和“作者公开描述的用途”，但不能确认当前基底的二进制偏移、参数槽长度和所有边界行为。
- 任何整型变量映射进入可写功能前，必须补齐三类证据：本地样本命令、旧工具源码或 docx 说明、实机/烟测复读。
- 6.6 的 `DT.e5`、R 文字和 R 插图微调只进入版本差异；若未来支持 6.6，需要独立 `HexTable.xml`、资源探针和写入护栏。
