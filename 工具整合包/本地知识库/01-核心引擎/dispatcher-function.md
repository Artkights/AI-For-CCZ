## 详细记录

#### 概述

调度函数位于内存 `0x600000-0x600E00` 区域，是一个**大规模的特技数据表查表引擎**。该区域包含数百个结构几乎完全相同的函数，唯一的差异是 `mov edx, 0xXXXX` 中的立即数值。

**核心原理**：EDX 是一个**特技数据表偏移量**（数据库键），`0x9BBFE4` 是通用的查表函数（`bool HasAbility(unit*, table_offset)`），`0x4101D9` 是通用的特技核心引擎。

#### 统一模板

##### 模板 A：单特技调度（一个 EDX → 一个能力）

```
                每个调度函数占用 0x2A 字节
┌──────────────────────────────────────────────┐
│ push ebp                                     │
│ mov ebp, esp                                 │
│ push ecx                  ; 保护ecx           │
│ mov ecx, [ecx]            ; ★ 解引用单位指针   │
│ mov edx, 0xXXXX           ; ★ 唯一差异点！     │
│ call 0x9BBFE4             ; ★ 查表验证        │
│ test eax, eax                                │
│ jnz skip                  ; 表未命中 → 跳过   │
│ pop ecx                   ; 恢复ecx           │
│ push [ebp+0x14]           ; 效果值标志        │
│ push [ebp+0x10]           ; 叠加标志          │
│ push [ebp+0x0C]           ; 装备特效号        │
│ push [ebp+0x08]           ; 个人特效号        │
│ call 0x4101D9             ; ★ 核心引擎        │
│ skip:                                        │
│ leave                                        │
│ ret 0x10                                     │
└──────────────────────────────────────────────┘
```

##### 模板 B：多特技调度（根据 `al` 分支到不同 EDX）

```
┌──────────────────────────────────────────────┐
│ push ebp / mov ebp, esp / push ecx           │
│ mov ecx, [ecx]                               │
│                                              │
│ cmp al, 0xNN      ; 参数al选择子类型          │
│ jz  set_edx_NN                               │
│ cmp al, 0xMM                                 │
│ jz  set_edx_MM                               │
│ ...                                          │
│                                              │
│ set_edx_NN: mov edx, 0x34XX  ; jmp merge     │
│ set_edx_MM: mov edx, 0x34YY  ; jmp merge     │
│                                              │
│ merge:                                       │
│ call 0x9BBFE4             ; 统一查表          │
│ test eax, eax                                │
│ jnz skip                                     │
│ pop ecx / push... / call 0x4101D9            │
│ skip: leave / ret 0x10                       │
└──────────────────────────────────────────────┘
```

#### EDX 值的含义

##### EDX = 特技数据表中的行偏移

```
全局特技数据表 (基址动态分配或位于 0x9Bxxxx 区域):

  偏移    内容大小              说明
  ────────────────────────────────────────────
  0x3400   16 bytes   特技行 #1 (未使用/保留)
  0x3410   16 bytes   特技行 #2
  0x3420   16 bytes   特技行 #3
  0x3430   16 bytes   特技行 #4 (MP恢复)
  ...
  0x34E0   16 bytes   特技行 #14 (属性下降)
  ...
  0x35C0   16 bytes   特技行 #28 (必定暴击)
  ...
  0x37A0   16 bytes   特技行 #58 (二次行动)
  ...

  每行 = 16 字节 (0x10)，间隔均匀
  EDX值之间的步长 = 0x10
```

##### 查表函数 `0x9BBFE4` 的语义

```c
// 伪代码
bool HasAbility(CharacterData* unit, int table_offset) {
    // unit 来自 [ecx]，是角色数据指针（非战术单位）
    // table_offset 对应 EDX 值
    
    byte* row = &global_ability_table[table_offset];
    
    // 在角色数据中查找是否拥有该表行定义的特技
    // 返回 0 = 有该特技 (匹配成功)
    // 返回 非0 = 无该特技 (未匹配)
    return (unit->has_ability(row) ? 0 : 1);
}
```

#### 完整调用链

```
┌──────────────────────────────────────────────────────┐
│                   桩函数 (Stub)                        │
│  地址: 0x421080-0x421180 区域 或 内联 (0x410F0A等)    │
│  职责: push 效果值标志, push 叠加标志,                 │
│        push 装备特效号, push 个人特效号,               │
│        设置 al=子类型选择, call 调度函数               │
└────────────────────────┬─────────────────────────────┘
                         │
                         ▼
┌──────────────────────────────────────────────────────┐
│               调度函数 (Dispatcher)                    │
│  地址: 0x600000-0x600E00 区域                         │
│  职责:                                                │
│    1. mov ecx, [ecx]  → 解引用单位指针                  │
│    2. mov edx, 0xXXXX → 设定特技数据表偏移              │
│    3. call 0x9BBFE4   → 查表: 该单位有此特技吗?        │
│    4. call 0x4101D9   → 是 → 执行核心引擎              │
└────────────────────────┬─────────────────────────────┘
                         │
                         ▼
┌──────────────────────────────────────────────────────┐
│             查表函数 (Lookup) — 0x9BBFE4               │
│  bool CheckAbility(character_data, table_offset)       │
│    - 在全局特技数据表中                              │
│      查找 table_offset 行的定义                      │
│    - 比对角色数据中是否有该特技                       │
│    - 返回0=有, 非0=无                                 │
└────────────────────────┬─────────────────────────────┘
                         │ (返回0 → 继续)
                         ▼
┌──────────────────────────────────────────────────────┐
│           核心引擎 (Core Engine) — 0x4101D9            │
│  职责: 双渠道判定 (个人特技 + 装备特技)                │
│  参数: [ebp+8]=个人特效号, [ebp+C]=装备特效号,        │
│        [ebp+10]=叠加标志, [ebp+14]=效果值标志          │
│  返回: eax = 特技效果值 (0=未激活)                     │
└──────────────────────────────────────────────────────┘
```

#### 对 Mod 开发者的意义

##### 编写新特技需要做的事

**方法一：标准路径**（需要创建调度函数）
1. **定义特技效果** → 在特技数据表中新增一行（分配一个新 EDX 偏移）
2. **创建桩函数** → 在 0x421080 区域或内联，push 4 个参数，call 新调度函数
3. **创建调度函数** → 复制任意现有调度函数（0x2A 字节），修改 `mov edx` 为新偏移
4. **Hook 调用点** → 在游戏逻辑中找到合适的调用时机

**方法二：内联直接调用（⭐ 推荐）**
1. **定义特技效果** → 在代码洞中编写效果逻辑
2. **内联桩** → 在效果代码中直接 `push 4参数 → call 0x4101D9`
3. **Hook 调用点** → 在游戏逻辑中找到合适的调用时机，jmp 到代码洞

> 方法二绕过了调度函数和 0x9BBFE4 查表，无需创建新的调度函数或 EDX 偏移。所有现有注入补丁均采用此方法。

##### EDX 偏移的计算

```
EDX = 0x3400 + slot_index × 0x10

例如: slot #14 → EDX = 0x3400 + 14×0x10 = 0x3400 + 0xE0 = 0x34E0
     slot #28 → EDX = 0x3400 + 28×0x10 = 0x3400 + 0x1C0 = 0x35C0
     slot #58 → EDX = 0x3400 + 58×0x10 = 0x3400 + 0x3A0 = 0x37A0
```
