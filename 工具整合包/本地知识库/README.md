# CCZModStudio 本地知识库

## 结论速览

- 本目录是当前项目唯一主知识库入口，默认服务于 `曹操传加强版6.5（未加密）` 的模组制作、工具开发、资源替换和剧本编辑。
- 知识库已经从旧平铺编号文件和嵌套 `knowledge-base` 合并为分层框架；旧入口只保存在 `90-来源归档`。
- 所有可写规则必须优先依赖本地样本、旧工具源码、复读、烟测或实机验证；联网资料只作为版本边界、制作习惯和外部佐证。
- 6.6/6.6x 资料只进入版本差异和未来适配，不混入 6.5 默认写入路径。

## 适用版本

- 默认适用：曹操传加强版 6.5（未加密）当前项目基底。
- 6.6/6.6x 只进入版本差异和未来适配，不混入 6.5 默认写入路径。

## 已确认事实

- 根目录只保留本入口 Markdown；主题文档统一迁入分层目录。
- 旧平铺编号文件和嵌套 `knowledge-base` 已迁移或归档。
- 所有 Markdown 统一为 带 BOM 的 UTF-8。

## 实现/使用方法

- 按“快速导航”进入对应主题，再按主题页的证据等级和风险边界执行。
- 修改游戏文件前必须确认目标版本、目标路径、写前备份、写后复读和结构化报告。
- 联网资料先进入 `09-版本与外部资料`，再由本地证据决定能否迁移到实现规则。

## 风险与边界

- 本知识库不能把外部资料直接当成 6.5 写入规则。
- 历史旧口径和未确认线索只允许出现在归档、证据说明或待验证项中。
- 跨版本适配必须单列版本护栏。

## 证据来源

- 本地旧平铺知识库、嵌套 `knowledge-base`、本地 docx、旧工具源码、样本复读、烟测记录和联网复核资料。
- 迁移关系见 `90-来源归档/迁移索引.md`。

## 待验证项

- 继续补齐 R 插图实机行为、Hexzmap 权威格式、动作帧序列和部分脚本命令参数槽位。
- 继续验证 6.x 整型变量开关、负数参数、Data 号/战场编号切换和 6.6 动态图/R 文字边界。
- 若工具或脚本仍引用旧平铺路径，需要改为新分层路径。
## 使用规则

1. 开发或制作前先看本页，再看 `00-总览与规范/开发执行规则.md` 和目标专题。
2. 遇到格式、偏移、命令参数、资源编号或跨版本能力，先确认证据等级，再决定是否能写回。
3. 标为 `待验证项` 的内容不能作为正式写入依据，必须补本地样本、旧工具源码、复读结果、烟测或实机验证。
4. 外部链接只沉淀可复核结论和版本边界，不复制论坛长帖或未经确认的口口相传数据。
5. 后续新增文档必须使用 `00-总览与规范/知识库规范.md` 的模板。

## 证据等级

| 等级 | 含义 | 是否可作为写入依据 |
|------|------|--------------------|
| 已验证 | 本地样本、旧工具源码、烟测、复读或实机确认 | 可以，但仍需备份和复读 |
| 外部佐证 | 发布帖、教程、论坛资料可复核，但未完成本地验证 | 不能单独作为写入依据 |
| 待证 | 只有线索、假设或样本不足 | 不能作为实现规则 |
| 废弃/旧口径 | 已被本地证据修正或只适用于旧版本 | 不能进入主流程 |

## 三步学习路线

1. 新手先读 `05-教程指南/入门指南.md`，理解特技判定、代码空洞、两套编号体系和基本工具链。
2. 做工具或写回功能前读 `00-总览与规范/开发执行规则.md`、`06-项目与工具链/验证与烟测.md`。
3. 做引擎相关工作先看 `01-核心引擎/引擎工作索引.md`，再进入具体机制页。
4. 做具体内容时进入对应专题：角色、兵种、宝物统一按“游戏设定”口径查 `07-数据表与资源`，地图、战场、剧本统一按“游戏制作”口径查 `07-数据表与资源` 和 `08-剧本与战场`，引擎特技看 `01-核心引擎` 与 `03-机制详解`。

## 分层目录

- `00-总览与规范`：项目约束、执行规则、证据等级、待验证清单、工具索引、更新记录。
- `01-核心引擎`：特技判定、桩函数、调度函数、代码空洞、安全补丁。
- `02-数据结构`：单位结构、全局变量、战斗数据结构、Data 号与战场编号体系。
- `03-机制详解`：物理攻击、暴击、连击、追击、二次行动、状态、属性下降、战斗公式、功勋模式、6.5 注入特效索引。
- `04-函数速查`：函数地址、调用链、用途和验证状态。
- `05-教程指南`：入门路线、工具准备、战场操作自动化和实操流程。
- `06-项目与工具链`：CCZModStudio、MCP、烟测、跨设备迁移、旧扳手运行时调试器源码对照。
- `07-数据表与资源`：游戏设定、地图、地形、E5 图片、动态链接库图标、坐标规律。
- `08-剧本与战场`：游戏制作、R/S eex、旧剧本编辑器源码、v0.24 静态对照、剧本指令、R 插图。
- `09-版本与外部资料`：6.5/6.6 差异、联网专题、外部资料迁移结论。
- `10-制作案例`：刘备传 6.5 方案、实施记录和可复用案例。
- `90-来源归档`：旧入口、迁移索引、原始 docx 和不可直接采用资料。

## 维护规则

- Markdown 统一保存为 带 BOM 的 UTF-8，避免 Windows PowerShell 按 GBK 误读中文。
- 每个主题只保留一个主文档；旧口径、重复入口和来源全文进入 `90-来源归档`。
- 新增联网资料必须写明链接、版本、用途、证据等级和本地验证状态。
- 更新代码、格式判断、烟测结论或写回护栏时，同步更新对应专题和 `00-总览与规范/更新记录.md`。
