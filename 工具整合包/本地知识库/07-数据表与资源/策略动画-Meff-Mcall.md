# 策略动画 / Meff 与 Mcall

## 结论速览

- `6.5-5-2 策略动画1` 是大动画字段，写 `Ekd5.exe` 的 `MgMcall=668144`；值 `<100` 或 `255` 表示无 Mcall，大于等于 `100` 时映射 `Mcall{值-100}.e5`。
- `6.5-5-3 策略动画2` 是小动画字段，写 `Ekd5.exe` 的 `MgMeff=668000`；值 `N` 映射 `E5\Meff.e5` 图号 `N+1`，`255` 表示无动画或保留。
- 当前工具只做只读预览：按结构化头部、20 字节记录和 8bpp 帧平面渲染动画，不做导入、重封包或实机写回。

## 已确认结构

- `Meff.e5` 是标准 E5 `110` 索引资源；条目可能为 LS12 压缩，需先按 E5 字典解码。
- `Mcall*.e5` 不是 E5 `110` 索引资源，文件本体就是动画数据。
- Meff/Mcall 解码后的动画数据结构：
  - byte0：记录数减一，实际 `recordCount = byte0 + 1`。
  - byte1：声明帧平面数量。
  - byte2/byte3：动画标记，含义仍待验证。
  - `int32LE@4`：帧间隔，样本常见 50、100、150。
  - `int32LE@8` / `int32LE@12`：帧宽、帧高。
  - `int32LE@16`：音效或效果参数候选。
  - 记录区长度：`recordCount * 20`，后续载荷按 `width * height` 切连续 8bpp 帧平面。
- 预览调色板优先级：当前项目 `Spalet.e5`，其次工具内置 `Assets\Palettes\tsb`，缺失时灰度降级。
- 透明规则：索引 0 透明；洋红色也按透明键处理。

## 本地样本校验

- 样本：`基底\加强版6.5未加密版\E5\Meff.e5`。
- 60 个 Meff 条目中 56 个严格满足 `recordCount*20 + declaredPlaneCount*width*height == decodedLength`。
- Meff #12、#13、#26、#33 的载荷包含额外完整帧平面：例如 #12 声明 20 帧、实际可切 23 帧。工具按“全部平面”预览，并在信息中标明声明帧数与实际帧数。
- 样本：`基底\加强版6.5未加密版\E5\Mcall00.e5` 至 `Mcall08.e5`。
- Mcall 样本可按同类结构解析，常见尺寸包括 `240x240`、`300x300`、`320x240`，帧间隔常见 100 或 150。

## 风险与边界

- 当前结构足以做正确的只读帧预览，但还不能证明所有 byte2/byte3、记录参数、音效参数和实机合成顺序。
- 6.6/6.6x 可能存在真彩化或扩展载荷；遇到尾部不整除、尺寸异常或帧数异常时，工具应降级为状态说明，不生成猜测帧。
- 导入、压缩、重封包、Mcall/Meff 新增条目和实机调用参数仍是待验证项。
