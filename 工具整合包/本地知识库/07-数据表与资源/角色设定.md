#### 当前实现

- `角色设定` 创作页已能读取人物基础字段、头像、职业、等级、能力和 R/S 形象编号。
- `职业` 自动引用 `6.5-4 详细兵种` 名称，方便创作者确认人物职业。
- 人物 R/S 形象链路以 `B形象指定器\形象指定器6.5\System.ini` 为准：`RFileHead=E1000`、`FileHead=D2800`。
- 头像预览来自 `E5\Face.e5` 的 PNG 候选，不把 `.E5S` 或关卡 R/S 误当人物形象。
- `R=0` / `S=0` 是合法值：表示使用普通形象（与兵种/初始设定相关），不是读取错误；当前 S=0 预览可按人物 `职业` 和预览阵营计算 Unit 图号，R=0 默认兵种形象后续再单独纠偏。
- 保存策略：保存前自动备份，保存后复读校验。

#### 2026-06-01 暴击/撤退台词读写修正（6.5）

基于本仓库的 6.5 基底文件（`Imsg.e5` 尺寸 `450000`）与 `HexTable.xml`：

- `6.5-0-2 暴击台词`：`Imsg.e5 @ 140000`，`99 x 200B`。
- `6.5-0-3 撤退台词`：`Imsg.e5 @ 440200`，`49 x 200B`，位于 `Imsg` 末尾追加区（原 6.4 的 `Imsg` 尺寸为 `440200`，6.5 在尾部追加 `9800` 字节正好容纳 49 条撤退台词）。
- `6.5-0 人物` 的字段 `撤退台词/暴击台词` 都是“台词编号”，用于定位对应台词表的 `ID` 行：
  - 撤退台词：人物 `撤退台词` 在 6.5 基底前 49 人为 `0..48` 的排列（不等同于人物 ID 顺序），用于指定每个角色引用哪条撤退台词文本。
  - 但在部分 模组 中，人物 `撤退台词` 可能出现 `>48` 的取值；此时在工具侧应视为“该人物没有可编辑的撤退台词行或由剧本/其他机制控制”，不要因为撤退台词行找不到而连带禁止保存人物列传/暴击台词。

对应代码修正点（CCZModStudio）：

- `MainForm.ShowRoleTextDetails(...)`：
  - 改为允许“列传/暴击/撤退”三者任意一项可编辑就启用保存按钮。
  - 当撤退台词 `ID` 超出 `0..48` 时，右侧文本区显示“未在撤退台词表中找到”的提示，不阻断其它文本编辑。
- `MainForm.SaveSelectedRoleTextDetails(...)`：
  - 撤退台词行用 `TryFindRowById`，找不到则跳过撤退台词写回，仅保存列传/暴击台词改动。

#### 2026-06-04 暴击/撤退台词真实映射纠偏

联网复核来源：

- 轩辕春秋 `用star175 5.6版exe做自己的曹操传MOD`：角色文本章节说明撤退台词按 Data 人物顺序前 49 人设置；致命一击台词前 21 条为“特定人物台词”，后续每 3 条一组，对应 Data 中的“致命一击台词类型”。
  https://www.xycq.org.cn/forum/viewthread.php?action=printable&tid=201504
- 轩辕春秋 `請問...爆擊，撤退台詞分別對應哪些人物？`：补充说明撤退台词仅在我军身份撤退时显示；老帖口径写作 0-47，和 6.5 本地表 `49 x 200B` 存在版本/口径差异，本项目以 6.5 本地 `HexTable.xml` 与样本文件为准。
  https://xycq.org.cn/forum/thread-230322-1-1.html

本地 6.5 复核结论：

- `6.5-0-3 撤退台词` 在本项目仍为 `49 x 200B`，实际显示规则按人物行 ID 对齐撤退台词表同 ID 行，而不是按人物表字段 `撤退台词` 的值跳转。
  - 例：曹操人物 ID=0，字段 `撤退台词=1`，旧逻辑会错显示夏侯惇行；真实应读撤退台词 #0。
  - 人物 ID 超过 48 时没有 49 行表内可编辑撤退台词；可能无通用撤退台词，或由 S 剧本事件处理。
- `6.5-0-2 暴击台词` 不是“人物字段值直接指向文本行”：
  - 前 21 行（#0..#20）是特殊人物台词。
  - 特殊人物归属在 `Ekd5.exe @ 0x89C30`，按 21 个 Int32 人物 ID 对应暴击台词 #0..#20。
  - 当前 6.5 基底读得特殊人物 ID 序列：`0,1,2,3,6,7,9,13,15,16,17,33,34,36,57,84,95,108,109,119,32`。
  - 未命中特殊人物表时，人物字段 `暴击台词=t` 是普通台词类型，实际候选行为 `21 + t * 3` 起连续 3 行，战斗中随机显示其一。
  - 例：曹仁 ID=5 不在特殊表，字段 `暴击台词=1`，实际候选为 #24/#25/#26；刘备 ID=32 在特殊表，实际使用 #20，字段值不再决定台词行。

代码落地：

- 新增 `RoleQuoteMappingService` 统一解析真实显示行。
- `角色设定` 右侧文本区：
  - 撤退台词按人物 ID 定位。
  - 暴击台词若为特殊人物显示单行；若为普通类型显示 3 条候选，空行分隔，保存时拆回各自 200B 行。
- `TableReferenceLookupService` 不再把人物表 `暴击台词/撤退台词` 字段生成为直接跨表行跳转，只输出规则说明。
- `TableReferenceDiagnosticService` 删除这两个字段的直接单表引用诊断，避免误报/误导。
