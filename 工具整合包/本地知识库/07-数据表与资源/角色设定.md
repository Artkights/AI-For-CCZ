#### 2026-06-30 人物默认装备尾字节确认（6.5/6.3）

本地 6.5 未加密基底与 `HexTable.xml` 复读确认：`6.5-0 人物` 每行 `32B` 中原先未命名的最后 3 个字节不是保留区，而是人物默认装备槽。

| 字段 | 行内偏移 | 文件偏移公式 | 存储口径 | 空值 |
| --- | ---: | --- | --- | --- |
| `武器` | `+0x1D` / `+29` | `0x18C + 人物ID * 0x20 + 0x1D` | 物品绝对 ID | `255` |
| `防具` | `+0x1E` / `+30` | `0x18C + 人物ID * 0x20 + 0x1E` | 物品绝对 ID | `255` |
| `辅助` | `+0x1F` / `+31` | `0x18C + 人物ID * 0x20 + 0x1F` | 物品绝对 ID | `255` |

确认规则：

- 三槽使用物品绝对 ID，不是 R/S 剧本 `0x3E` 战场装备命令的相对编码。
- `255` 表示空/未指定；`0` 是合法物品 ID，例如短剑，不能当空值处理。
- `6.5-0 人物` 的 `DataPos=396`、`RowSize=32`、`RowCount=1024` 不变；只是补齐尾部 3 个现存字节的列名和读写语义。
- `6.3-0 人物` 同结构也应同步补齐 `武器/防具/辅助`，避免跨版本兜底读取时丢失尾部装备字段。
- 物品分段按 `ItemCategoryBoundaryService.Resolve(project)` 读取项目 `System.ini`；找不到或无效时默认 `DefID=70`、`AssID=109`。

值域护栏：

- `武器` 允许 `255` 或 `WeaponStartId..DefenseStartId-1`。
- `防具` 允许 `255` 或 `DefenseStartId..AccessoryStartId-1`。
- `辅助` 允许 `255` 或 `AccessoryStartId..255`；若物品分类表完整，应优先只允许 `ItemClassificationService` 判定为 `AccessoryEquipment` 的辅助装备，排除道具/消耗品和预留空位。
- 不允许把防具 ID 填入武器列，也不允许把道具/消耗品填入辅助列。

本地样本证据：

- `基底\加强版6.5未加密版\Data.e5` 中人物 ID `162` 的尾部三字节复读为 `武器=46、防具=90、辅助=143`。
- 写回烟测在测试副本中把人物 ID `162` 临时改为 `武器=0、防具=70、辅助=109`，直接校验 `Data.e5 @ 0x18C + 162 * 0x20 + 0x1D..0x1F` 三字节成功；随后恢复原值并复读成功。

#### 2026-06-01 暴击/撤退台词读写修正（6.5）

基于本仓库的 6.5 基底文件（`Imsg.e5` 尺寸 `450000`）与 `HexTable.xml`：

- `6.5-0-2 暴击台词`：`Imsg.e5 @ 140000`，`99 x 200B`。
- `6.5-0-3 撤退台词`：`Imsg.e5 @ 440200`，`49 x 200B`，位于 `Imsg` 末尾追加区（原 6.4 的 `Imsg` 尺寸为 `440200`，6.5 在尾部追加 `9800` 字节正好容纳 49 条撤退台词）。
- `6.5-0 人物` 的字段 `撤退台词/暴击台词` 都是“台词编号”，用于定位对应台词表的 `ID` 行：
  - 撤退台词：人物 `撤退台词` 在 6.5 基底前 49 人为 `0..48` 的排列（不等同于人物 ID 顺序），用于指定每个角色引用哪条撤退台词文本。
  - 但在部分 模组 中，人物 `撤退台词` 可能出现 `>48` 的取值；此时在工具侧应视为“该人物没有可编辑的撤退台词行或由剧本/其他机制控制”，不要因为撤退台词行找不到而连带禁止保存人物列传/暴击台词。

对应代码修正点（CCZModStudio）：

- `MainForm.ShowRoleTextDetails(...)`：
  - 改为允许“列传/暴击/撤退”三者任意一项可编辑就启用保存按钮。
  - 当撤退台词 `ID` 超出 `0..48` 时，右侧文本区显示“未在撤退台词表中找到”的提示，不阻断其它文本编辑。
- `MainForm.SaveSelectedRoleTextDetails(...)`：
  - 撤退台词行用 `TryFindRowById`，找不到则跳过撤退台词写回，仅保存列传/暴击台词改动。

#### 2026-06-04 暴击/撤退台词真实映射纠偏

联网复核来源：

- 轩辕春秋 `用star175 5.6版exe做自己的曹操传MOD`：角色文本章节说明撤退台词按 Data 人物顺序前 49 人设置；致命一击台词前 21 条为“特定人物台词”，后续每 3 条一组，对应 Data 中的“致命一击台词类型”。
  https://www.xycq.org.cn/forum/viewthread.php?action=printable&tid=201504
- 轩辕春秋 `請問...爆擊，撤退台詞分別對應哪些人物？`：补充说明撤退台词仅在我军身份撤退时显示；老帖口径写作 0-47，和 6.5 本地表 `49 x 200B` 存在版本/口径差异，本项目以 6.5 本地 `HexTable.xml` 与样本文件为准。
  https://xycq.org.cn/forum/thread-230322-1-1.html

本地 6.5 复核结论：

- `6.5-0-3 撤退台词` 在本项目仍为 `49 x 200B`，实际显示规则按人物行 ID 对齐撤退台词表同 ID 行，而不是按人物表字段 `撤退台词` 的值跳转。
  - 例：曹操人物 ID=0，字段 `撤退台词=1`，旧逻辑会错显示夏侯惇行；真实应读撤退台词 #0。
  - 人物 ID 超过 48 时没有 49 行表内可编辑撤退台词；可能无通用撤退台词，或由 S 剧本事件处理。
- `6.5-0-2 暴击台词` 不是“人物字段值直接指向文本行”：
  - 前 21 行（#0..#20）是特殊人物台词。
  - 特殊人物归属在 `Ekd5.exe @ 89C30`，按 21 个 Int32 人物 ID 对应暴击台词 #0..#20。
  - 当前 6.5 基底读得特殊人物 ID 序列：`0,1,2,3,6,7,9,13,15,16,17,33,34,36,57,84,95,108,109,119,32`。
  - 未命中特殊人物表时，人物字段 `暴击台词=t` 是普通台词类型，实际候选行为 `21 + t * 3` 起连续 3 行，战斗中随机显示其一。
  - 例：曹仁 ID=5 不在特殊表，字段 `暴击台词=1`，实际候选为 #24/#25/#26；刘备 ID=32 在特殊表，实际使用 #20，字段值不再决定台词行。

代码落地：

- 新增 `RoleQuoteMappingService` 统一解析真实显示行。
- `角色设定` 右侧文本区：
  - 撤退台词按人物 ID 定位。
  - 暴击台词若为特殊人物显示单行；若为普通类型显示 3 条候选，空行分隔，保存时拆回各自 200B 行。
- `TableReferenceLookupService` 不再把人物表 `暴击台词/撤退台词` 字段生成为直接跨表行跳转，只输出规则说明。
- `TableReferenceDiagnosticService` 删除这两个字段的直接单表引用诊断，避免误报/误导。

#### 2026-06-22 《安西军传》人物 Data 落表经验

本次以 `基底\安西军传` 为正式 MOD 根目录，先用 ModPackage 写入测试副本，复读核心表后再推广正式基底。落表包为 `anxi_data_modpackage.json`，核心内容只覆盖人物、列传、暴击台词、撤退台词；不写 R/S 形象、S 形象、头像资源和个人特技。

- 人物 ID 分段固定为：`0-35` 我军/核心与长期可控人物，`36-78` 友军/中立与预留，`79-139` 敌军命名人物与预留，`140-199` 我军/友军群体单位，`200-259` 敌军群体单位。
- 撤退台词只写 `6.5-0-3` 的 `0..48` 共 49 行。人物 ID `49+` 不强占撤退台词表，敌军死亡/撤退句优先交给 S 剧本事件。
- 暴击台词仍按 6.5 规则处理：前 21 行为特殊人物定制句，普通类型按 3 行一组；人物字段不是简单跨表行号。
- 列传入表必须只讲人物本身，不写“教学关”“送经验”“后续关卡”“预留位”“战场单位”等制作语汇。用于占位的人物行可保留在 Data 中，但 `6.5-0-1 人物列传` 宜写成“某某，名籍未书。”这类中性文本。
- `data信息设计.md` 可保留“用途、出场范围、预留”等项目管理信息；导入 `Imsg.e5` 的列传文本必须另行扫描，避免把设计说明带进游戏内文本。
- 正式推广后若发现列传文案问题，可用 `write_table_rows` 对 `6.5-0-1 人物列传` 做小范围修正；该写入会自动生成 `Imsg.e5` 备份和 `WriteOperationReport.json`。
- R/S 形象和头像不能因人物表已落地而顺手写默认替换。图片资源槽是“可替换槽”而非空槽，必须先预览 `Face.e5 / Pmapobj.e5 / Unit_mov.e5 / Unit_atk.e5 / Unit_spc.e5` 再写编号。
- 生成落表清单时用 UTF-8；PowerShell 终端可能把中文显示成乱码，但可用 Python 或 `[Text.Encoding]::UTF8.GetString(...)` 复核文件本体是否正常。

#### 2026-06-22 《安西军传》暴击/撤退台词重设计经验

本次发现原始台词近似的根因是：撤退台词仅少数人物显式写入，其余 `0..48` 走默认模板；普通暴击按人物 ID 粗分三行组；且特殊暴击前 21 行若不同步修改 `Ekd5.exe @ 0x89C30`，会被旧 6.5 特殊人物表错误引用。

- 撤退台词必须按人物 ID `0..48` 全量定制，且脚本遇到缺句应直接失败，不能回退为“某某今日暂退，来日再战”之类模板。
- 特殊暴击写入要拆成两步同步执行：`6.5-0-2` 的 `#0..#20` 文本行，以及 `Ekd5.exe @ 0x89C30` 的 21 个 Int32 人物 ID。只改文本不改 EXE 映射，等于让旧人物念新台词。
- 普通暴击类型应按职业/战场职责分组，而非按人物 ID 分桶。推荐先分为守城步军、远射弓弩、游骑突击、军议谋略、护送医护、马匪劫掠、吐蕃攻坚、暗线离间八类。
- 台词静态检查至少包括：撤退 49 行全覆盖、特殊暴击 21 行、普通暴击 8 组三句、无重复、GBK 不超过 200B、无“游戏/关卡/Boss/预留/特技/数值/落表”等制作语汇。
- 高位敌军名将不强占原生撤退台词表；其败亡、投降、撤走语更适合放入对应 S 战场事件，便于结合关卡局势和人物关系。
- 对 EXE 偏移写入，优先使用 `domain=patch` 的 EffectPackage，并设置 `expectedOldBytesHex`；若工具链不可用，才用受控脚本写测试副本，写前备份，写后复读。

#### 2026-06-22 《安西军传》人物五维重调经验

本次以 `anxi_stat_rebalance_modpackage.json` 做小型 Data 包，只写 `6.5-0 人物` 的五维字段；先写测试副本 `CCZModStudio_TestCopies\20260622_181112_安西军传`，复读与 `MOD_PACKAGE_SMOKE_OK` 通过后推广正式基底。

- 文档五维名称按项目原表头保留，不因工具字段名而批量改文案。落表时按工具字段写入：`武力、智力、统帅、敏捷、士气`。
- 需要区分“文档展示顺序”和“Data 字段名”：文档表头仍可写 `统率/运气`，但 ModPackage 字段必须用 `统帅/士气`，否则 `read_table/write_table_rows` 会找不到列。
- 已命名人物和已命名群体单位可统一抬到 `60-100`；预留槽如 `友预xx、敌预xx、友兵xx、敌兵xx` 不应纳入重调包，避免污染后续空位。
- 敏捷不宜只作为少数斥候属性。骑射、斥候、沙陀、回鹘、细作、信使、护商等单位应整体提高敏捷，让战场移动、追击和侧翼价值更鲜明。
- 百姓、文书童、僧侣、保护目标不必用 20/30 段低五维表现脆弱；可用等级、职业、HP/MP、装备、AI、胜败条件和事件脚本控制风险，五维仍保持基本体面。
- 五维小包不要顺手重算 `初始HP/初始MP`，也不要改职业、台词、头像、R/S 形象、兵种或特效。单字段小包的优势是复读容易、回滚范围小。
- 验证口径：包内更新行数、唯一 ID 数一致；字段集合只包含五维；数值最小/最大在目标区间；预留段 ID 不出现；正式推广后抽查 HP/MP、职业、台词、Army 未变化。
