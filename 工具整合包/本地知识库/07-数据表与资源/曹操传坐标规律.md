## 详细记录

#### 基本信息

地图原图尺寸：

```text
640 x 400 像素
```

地图格子是斜菱形。

一个完整格子：

```text
宽 16 像素
高 8 像素
面积 64 像素
```

半格：

```text
横向 8 像素
纵向 4 像素
```

#### 坐标方向

```text
右上 1 格：像素 (+8, -4)，游戏坐标 (X, Y-1)
右下 1 格：像素 (+8, +4)，游戏坐标 (X+1, Y)
左下 1 格：像素 (-8, +4)，游戏坐标 (X, Y+1)
左上 1 格：像素 (-8, -4)，游戏坐标 (X-1, Y)
```

简单记法：

```text
X 轴沿右下方向增加
Y 轴沿左下方向增加
往右上走，Y 减少
往右下走，X 增加
```

#### 像素转游戏坐标

设图片内像素坐标为：

```text
px = 图片内横向像素，左边为 0，右边为 640
py = 图片内纵向像素，上边为 0，下边为 400
```

转换公式：

```text
X = floor(px / 16 + py / 8 + 4)
Y = floor(py / 8 - px / 16 + 46)
```

#### 游戏坐标转像素

设游戏坐标为：

```text
X, Y
```

转换公式：

```text
px = 8 * (X - Y + 42)
py = 4 * (X + Y - 50)
```

#### 四角验证

```text
左上角 (0, 0)       => (4, 46)
右上角 (640, 0)     => (44, 6)
左下角 (0, 400)     => (54, 96)
右下角 (640, 400)   => (94, 56)
```

#### 格子数量

整张图像素面积：

```text
640 * 400 = 256000 像素
```

单个菱形格子面积：

```text
16 * 8 / 2 = 64 像素
```

整张图等价完整格子数量：

```text
256000 / 64 = 4000 个格子
```

结论：

```text
一张 640x400 地图，等价包含 4000 个完整格子的面积。
边缘存在半格或截断格，但合计面积等于 4000 个完整格子。
```

#### CCZModStudio 应用边界

- `R场景制作` 页的 `Mmap.e5` 背景按本页公式绘制坐标：画布固定为 `640x400`，格子为 `16x8` 斜菱形。
- 鼠标拖放/点击时先把画布像素换算为游戏坐标，再保存到 R 场景草稿的 `GridX/GridY`。
- 角色绘制时把游戏坐标换算为像素锚点，R 形象帧水平居中、脚底贴近该菱形坐标点。
- 不要把 R 场景背景当作战场地图的 `48x48` 正方格，也不要使用可调“格宽”来解释 R 背景坐标。
