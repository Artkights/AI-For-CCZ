## 详细记录

本页记录 `图片处理` 模块当前已确认的资源清单、别名、读写边界和烟测结果。战场地图底图仍属于 `地图制作` 模块，不能混入本模块。

#### 模块边界

- 目标：统一读取、预览、替换非战场地图的图片资源，包括人物头像、R/S 形象、范围图、策略动画、R 背景、R 插图、封面/背景、界面图等。
- 写入方式：
  - 对可读取 `0x110` 图片索引表的 `.e5` 文件开放单条替换、批量导入和批量清空；写前备份，写后逐条复读索引和条目字节，生成结构化报告。
  - 对 `Itemicon.dll` / `Mgcicon.dll` / `Cmdicon.dll` 开放按字段编号替换/清空 `RT_BITMAP` 位图资源；写前备份 动态链接库，写后生成报告。
- 不做的事：
  - 不替换 `Map\M*.jpg` 战场地图底图；该操作继续走地图制作模块。
  - 不把 `Data.e5` / `Imsg.e5` / `Star.e5` / `Hexzmap.e5` 当图片资源替换。
  - 不重排 E5 条目编号；“删除/清空”语义是写入 1x1 透明 PNG 预留，不从索引表移除条目。
  - 不重排 动态链接库资源 ID；“删除/清空”语义是写入透明 RT_BITMAP，占用原资源 ID。
  - 不对未识别为 `0x110` 索引封包的文件盲替换，例如当前样本中的 `Mark.e5`。

#### 2026-06-04 自检：为什么图片处理曾读不到图标预览

- 原因：图片处理页最初只把 `Itemicon.dll / Mgcicon.dll / Cmdicon.dll` 当作文件级资源列出，`ReadEntries` 只读取 E5 `0x110` 索引条目；动态链接库图标不是 E5 索引封包，因此没有生成可选择的编号条目。
- 宝物设定页能预览，是因为它直接按物品表 `图标` 字段调用 `ItemIconPreviewService`，该服务会解析 动态链接库 的 `RT_BITMAP` 资源。
- 对齐：图片处理页现在对 动态链接库图标资源也生成编号条目，并复用 `ItemIconPreviewService.BuildPreview(...)` 渲染；2026-06-07 起新增 `IconResourceReplaceService`，按字段编号写回 `RT_BITMAP` 成对资源。

#### 2026-06-07 补齐：预览、动态链接库图标、批量操作

- 新增 `E5ImageRenderService`：
  - 标准 BMP/JPG/PNG 直接解码。
  - `Pmapobj.e5`、`Unit_atk.e5`、`Unit_mov.e5`、`Unit_spc.e5` 的 RAW 帧条按固定宽高和 `tsb` 调色板渲染代表帧。
  - 对 `Meff.e5`、`Weather.e5` 等 LS 解码后仍不是标准图片的条目，生成“未知字节图候选”预览；这是制作参考，不等同于已确认的真实帧宽高。
- 新增 `E5ImageReplaceService.ReplaceBatch`：
  - 支持一次写入多条 E5 图片条目。
  - 批量导入匹配规则：选中行数量等于文件数量时按选中行顺序匹配；否则从文件名最后一段数字提取目标图号，例如 `12.png` -> 图号 `12`。
  - 批量删除/清空写入 1x1 透明 PNG，占用原图号，不删除索引项。
- 新增 `IconResourceReplaceService`：
  - 解析 PE 资源目录中的 `RT_BITMAP`，按字段编号定位成对资源 ID。
  - 替换时把 BMP/JPG/PNG 来源缩放/居中到目标 RT_BITMAP 尺寸，再用 Win32 `BeginUpdateResourceW` / `UpdateResourceW` / `EndUpdateResourceW` 写回 DIB 数据。
  - 清空时写入透明 RT_BITMAP，不移除资源 ID。
  - 仍建议重要图标替换后用旧工具或实机复查透明、尺寸和调色板表现。

#### 2026-06-07 人工智能 绘图素材接入

- 新增专题：`人工智能绘图-图像工作室-模型上下文协议.md`，记录自然语言生成 R 背景、动态链接库图标、头像、R 形象、S 形象的 预设、提示词模板、尺寸、编号映射和来源。
- MCP 工具 `prepare_ccz_generated_image` / `draw_ccz_image_asset` 会把生成图后处理到当前 E5/动态链接库资源可预览的尺寸，并写入 `CCZModStudio_Exports\AiImageAssets`。
- 该流程只调用 `preview_e5_image_replace`、`preview_e5_image_batch_replace` 或 `preview_dll_icon_replace` 做替换预览；不直接写入 E5 或 动态链接库。
- 已验证默认尺寸：`Mmap.e5 640x400 jpg`、`Face.e5 120x120 png`、`Pmapobj.e5 48x1280 png`、`Unit_mov/atk/spc 48x528 / 64x768 / 48x240 png`。

#### 代码落地

- 新增 `ImageResourceCatalogService`：汇总已知图片资源和项目内可发现的其它 E5 图片索引资源。
- `图片处理` 主标签包含：
  - `图片资源`：读取文件清单、条目列表、预览、定位、替换/还原单条 E5 图片。
  - `人物R/S指定`：原人物 R/S 编号指定表，继续写 `Ekd5.exe` 的 R/S 编号表。
- 旧入口 `形象设定` 兼容跳转到 `图片处理`。
