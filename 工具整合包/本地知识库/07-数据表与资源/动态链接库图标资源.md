## 详细记录

本页记录物品/命令/策略等图标资源的已确认信息。当前重点用于 `宝物设定` 页右侧图标预览。

#### 6.6 修正版边界

- 6.1-6.5：物品/宝物图标继续走 `Itemicon.dll` 的 `RT_BITMAP` 成对资源，字段 `N` 优先预览 32x32 位图 `minId + N*2 + 1`。
- 6.6 修正版：物品/宝物图标不走 `Itemicon.dll`，改读 `E5\Item.e5`。字段值 `N` 成对映射到小图 `#2N+1`、大图 `#2N+2`，宝物设定预览默认显示大图。例如字段 `1` 应预览 `Item.e5 #4`，不是空白 `#2`。
- 6.6 修正版策略图标改读 `E5\Mtem.e5`，本轮暂保持 `字段+1` 映射；舌战/命令图标仍可从 `Cmdicon.dll` 读取。
- 这一边界只修 6.6 `Item.e5` 适配，不改变 6.1-6.5 的 `Itemicon.dll/Mgcicon.dll/Cmdicon.dll` 预览和写回逻辑。

#### 已确认：Itemicon.dll

文件：`加强版6.5未加密版\Itemicon.dll`

当前解析结论：

- `Itemicon.dll` 不是普通 `ExtractIconEx` 可枚举的标准图标 动态链接库；Windows 标准图标枚举返回 0。
- 它包含 PE 资源目录中的 `RT_BITMAP` 资源。
- 已解析到 `RT_BITMAP` 资源 ID `100-439`，共 340 个 位图 资源。
- 这些资源按 16x16 / 32x32 成对出现，因此当前程序按如下候选规则预览：

```text
物品“图标”字段 N
-> 优先显示 RT_BITMAP 资源 ID = 100 + N * 2 + 1（32x32）
-> 若不存在，再退回 ID = 100 + N * 2（16x16）
```

示例：

- 物品 ID `0`，名称 `短剑`，图标字段 `0`
- 预览资源：`Itemicon.dll` 的 `RT_BITMAP ID=101`
- 烟测可导出短剑图标 PNG。

#### 实现边界

- 当前已经能从 `Itemicon.dll` 读取并显示候选图标。
- `图片处理` 模块会把 `Itemicon.dll`、`Mgcicon.dll`、`Cmdicon.dll` 纳入图标资源定位/预览范围，并复用同一个图标预览服务按编号生成条目。
- 2026-06-07 起，图片处理页支持替换/清空 动态链接库图标：
  - 当前样本走 `RT_BITMAP` 写回，而不是标准 `RT_GROUP_ICON`/`RT_ICON` 图标组写回。
  - 替换时按编号定位 `minId + N*2` / `minId + N*2 + 1` 两个资源 ID，把来源图片缩放/居中为对应尺寸 DIB。
  - 清空时写入透明 DIB，占用原资源 ID，不删除资源项。
  - 写入前自动备份 动态链接库，写后生成文本和 JSON 结构化报告。
- 2026-06-04 自检对齐后，样本可预览候选数为：`Itemicon.dll=170`、`Mgcicon.dll=49`、`Cmdicon.dll=41`。
- 当前映射规则来自资源排列和字段值的对应推断，已能用于创作者快速确认，但仍建议结合旧工具或实机画面最终校验。

#### Windows PE 图标/位图资源写回资料结论

- 标准 Windows `.ico` 资源在 PE 中通常拆成 `RT_GROUP_ICON` 和多个 `RT_ICON`；`.ico` 文件头中的目录项需要转换成资源组目录项，图像数据分别写入 `RT_ICON`。
- 本项目样本的 `Itemicon.dll`、`Mgcicon.dll`、`Cmdicon.dll` 不是 `ExtractIconEx` 可枚举的标准图标 动态链接库，主要图标载体是 `RT_BITMAP`。因此当前实现只写 `RT_BITMAP`，不写 `RT_GROUP_ICON`/`RT_ICON`。
- Win32 写回 接口 采用 `BeginUpdateResourceW` -> `UpdateResourceW` -> `EndUpdateResourceW`。失败时必须丢弃更新；成功后再计算新 SHA256 和报告。
- 写回仍需实机复查：GDI+ 转 DIB 后的透明度、位深和旧引擎读取方式可能影响显示效果。

#### 2026-06-07 人工智能 图标草稿

- `人工智能绘图-图像工作室-模型上下文协议.md` 已新增 `dll_icon` 预设：自然语言生成方形图标，默认后处理为 `32x32 png`。
- 前景提示词要求单体、居中、无文字、无水印、清晰轮廓、纯洋红或纯色背景；后处理会把洋红/纯色背景尽量透明化。
- 人工智能 图标流程只生成导出图并调用 `preview_dll_icon_replace`；正式写回仍必须单独调用 动态链接库图标替换工具，并在旧工具或实机中复查透明和缩放表现。
- 2026-06-16 追加：6.6 项目中的 `dll_icon` 预设默认切到 `E5\Item.e5`，字段 `N` 会分别生成 16x16 小图和 32x32 大图，对应预览/替换目标 `#2N+1/#2N+2`；6.1-6.5 仍调用 DLL 图标预览/替换工具。
