#### 当前实现

- `兵种设定` 页面包含三个创作向子页：
  1. `详细兵种`：合并名称、说明、成长、穿透。
  2. `兵种系/地形`：合并兵种系名称、地形发挥、移动消耗。
  3. `相克矩阵`：读取/保存 `6.5-3-3 兵种相克` 与 `6.5-3-4 兵种属性`；列头按 `两位兵种系ID:兵种系名` 显示。
  4. `兵种策略`：合并策略基础属性、EKD5 策略附表、80 个详细兵种学会等级；学会等级 0 表示无法学习。
  5. `兵种特效`：合并特效说明、分配，并只读显示人物名/兵种名。
- `详细兵种` 右侧已新增范围预览与可装备类别预览：单击兵种行显示该详细兵种可装备类别；选中 `攻击范围` 单元格时按字段值 + 1 读取 `E5\Hitarea.e5` 图号；选中 `穿透` 单元格时按字段值 + 1 读取 `E5\Effarea.e5` 图号。预览只读，不改 E5 资源。双击兵种行可用复选框编辑可装备类别。
- `6.5-4-2 兵种成长` 每行 35 字节中，前 9 个命名字段之后的 26 个单字节对应旧工具“部队属性编辑”里的装备类别：普通/特殊剑、枪、弓、刀、炮车、锤、斧、扇、宝剑、将剑、铠甲、衣服、袍服。`0` 表示不能装备，非 `0` 表示允许装备；本工具按复选框写 `0/1`。
- 80 个详细兵种与 40 个兵种系的显示归属固定为：详细兵种 ID `0..59` 中每 3 个归入一个兵种系，公式 `seriesId = jobId / 3`；详细兵种 ID `60..79` 中每 1 个归入一个兵种系，公式 `seriesId = 20 + (jobId - 60)`。`宝物设定 / 装备类型 / 可装备部队` 的兵种系列使用该映射。
- 保存策略：保存前自动备份，保存后复读校验。
- `6.5-7 兵种特效` 名称表在 `HexTable.xml` 中 `Bytes=0`，当前不开放改名；只读读取名称，避免破坏未拆清结构。
- 2026-06-13 起，兵种/人物/套装特效读取通过 `CczEngineProfileService.TableHints` 定位：`JobEffectNameTable`、`JobEffectDescriptionTable`、`JobEffectAssignmentTable` 和 `PersonalEffectTable` 均走 `HexTableNameResolver.ResolveForProject`，服务层不再固定 `6.5-7-*` 字面表名。
- EffectPackage 的 `job` 域写 `兵种特效说明` 与 `兵种特效分配`，删除模式按固定空值清理：`1/2/3号武将=1024`、`兵种=255`、`特效值=0`，并清空说明文本。
- EffectPackage 的 `personal` 域写 `人物专属、套装专属`。非删除模式只修改包中明确指定的槽位，例如 `person_item_1` 只写 `武将1/装备1/特效值1`，不会清空 `武将2/装备2/特效值2` 或套装槽；删除模式才清空 14 个固定列。
- 专属/套装槽的 `特效值=0` 视为清空该槽：写入时同步把该槽人物/物品字段置 `0`。这条规则来自旧扳手“天赋/专属”源码口径，但当前写入仍以 6.5 HexTable、备份、复读和烟测为准。

#### 形象指定器与兵种属性口径（2026-07-06）

- `B形象指定器\形象指定器6.5\System.ini` 配置 `UserXK=A3280`，注释为“exe 中兵种相克数据的位置”。当前 6.5 未加密基底以 `Ekd5.exe` 文件偏移为准；运行时 VA 换算为 `VA = fileOffset + 0x42E800`。
- `UserXK=A3280` 块已拆成三个边界：
  - `0xA3280..0xA38BF`：`6.5-3-3 兵种相克`，`40*40=1600` 字节，运行时 VA 起点 `0x004D1A80`，公式 `offset = 0xA3280 + 攻方兵种系ID*40 + 守方兵种系ID`。
  - `0xA38C0..0xA39FF`：`6.5-3-4 兵种属性`，`8*40=320` 字节，运行时 VA 起点 `0x004D20C0`，公式 `offset = 0xA38C0 + 属性序号*40 + 兵种系ID`。
  - `0xA3A00..0xA3A27`：图一 `1960` 尾部 40 字节候选，运行时 VA 起点 `0x004D2200`。该段只纳入证据报告和待验证清单，不开放写入。
- `6.5-3-4 兵种属性` 按 `6.5-3 兵种系` 的 40 个大类逐列对齐，不按 `6.5-4` 的 80 个详细职业对齐；详细职业的可装备类别仍属于 `6.5-4-2 兵种成长`。
- `形象指定器65.exe` VB6 `Option1` 控件数组顺序确认了 8 行官方字段名。旧知识库/MCP 把 `row0` 标成“兵种大类”的口径已废弃；`row0` 官方字段是“移动声音”。6.6 骑马/步兵类补丁读取 `byte [兵种系ID + 0x004D20C0]` 并把 `0/2` 当骑/步代理，只能作为旧补丁实现说明，不能再作为字段名。

- 当前工具整合包 GUI 已在 `兵种设定 / 相克矩阵` 下开放 `兵种属性` 子页，读取/保存 `6.5-3-4 兵种属性`。底层表仍是 `8 行属性 x 40 列兵种系`，但 GUI 转置显示为 `40 行兵种系 x 8 列属性`：每行显示 `ID` 和兵种系名（如 `00:君主`、`01:步兵`），列头显示 `移动声音/移动速度/攻击声音/远程兵种/攻击延迟/兵种类型/策略伤害/参与围攻`；除 `策略伤害` 列使用 `0..255` 数值输入外，其余 7 列使用 `中文名称：数字` 下拉，底层仍按原始单字节矩阵保存。

- 保存预览和 MCP preview/write 必须列出精确文件偏移；GUI 下拉行可显示中文值，例如 `兵种属性 移动声音[01:步兵] 0xA38C1 脚步声：2 -> 马蹄声：0`，`策略伤害` 行保持纯数字，例如 `兵种属性 策略伤害[00:君主] 0xA39B0 100 -> 110`。保存后沿用 `HexTableWriter` 自动备份、结构化报告和复读校验。
- MCP `read_job_settings(include_matrices=true)` 的 `Matrices.Attributes` 和 `AttributeRows` 返回官方字段名、行起点偏移、运行时 VA 和值口径；`preview_job_settings/write_job_settings` 的 `attribute_matrix` 使用 `{ row_id, column_id, value }`，校验 `row_id=0..7`、`column_id=0..39`、`value=0..255`。

#### 兵种系与详细职业的关系（2026-06-20）

- `6.5-3 兵种系` 与 `6.5-4 详细兵种/详细职业` 不是同一张表，也不是简单一一对应。
- 人物 Data 表里的“职业”字段引用 `6.5-4 详细兵种` 编号；S 形象 `S=0` 默认兵种图也按人物职业与阵营计算。
- 6.5 常见显示/进阶口径按“三个详细职业行归属一个兵种系”处理：`详细职业ID / 3` 向下取整得到兵种系。例如详细职业 `0/1/2` 都属于兵种系 `0`，所以把岳夜/襄慎/刘生分别写成职业 `0/1/2` 会在实机显示成同一个兵种系。
- 独立基底需要九个不同兵种系时，首阶详细职业应写为 `系ID * 3`：`0/3/6/9/12/15/18/21/24`，而不是连续 `0-8`。`+1/+2` 可作为同系二三阶或预留行。
- 《三英战龙帝》当前复读案例：`6.5-3:0-8` 为夜剑、锁脉、裂枪、义勇、血帝、血卫、血祭、鳞卫、影侍；人物 Data `0-8` 的职业为 `0/3/6/9/12/15/18/21/24`；`6.5-4:1/2、4/5、7/8...` 是同系预留，不给首关人物直接引用。

#### 兵种方案设计规范（2026-06-22）

本节来自《安西军传》36 关中篇 MOD 的兵种规划整理，可复用于后续从纯净 6.5 基底开新项目。

- 先分清 `兵种系` 和 `职业/详细兵种`：`6.5-3` 是 40 个大类，用于地形、相克和三阶归属；`6.5-4` 是 80 个实际职业，人物和战场单位引用它。普通三阶按 `0-59`、每 3 行一系处理；特殊职业按 `60-79`，对应系 `20-39`。
- 新项目若要“清空原职业/兵种”，设计层面应覆盖 `6.5-3`、`6.5-4`、`6.5-4-1`、`6.5-4-2`、`6.5-4-3` 和 `6.5-5` 的 80 个职业学习等级列；不要只改详细职业名，否则原版职业、说明、成长、策略会残留。
- 兵种系名和职业名建议 2-4 字。短名更适合 6.5 表格宽度、战场显示和后续文案引用；避免把完整官号、族属、人物名和功能全堆进一个职业名。
- 普通三阶特技采用继承式设计：一转、二转各 1 个特技，且必须是同一个基础特技；三转保留基础特技，再新增 1 个进阶特技。这样玩家能读出同系成长，而不是每阶像换了一个职业。
- 特殊职业固定 2 个特技，但不是普通三转的第四阶。它们服务名将、敌帅、剧情单位或终局单位，可大胆使用更鲜明的效果组合，但要避免所有名将共用同一套“减伤+全防”。
- 主力人物分配应尽量“一名命名人物对应一条普通职业线”。阶段可控友军可用来补足地域兵种覆盖；敌军专属兵种不要为了凑覆盖强塞给我军。群体单位可以复用兵种系，但命名可培养人物应避免同一时期重复。
- 个人特技应与兵种特技分工：兵种特技写职业共性，个人特技补人物短板或强化其战场职责。不要只因效果酷就给角色；例如“深入敌后”只适合潜入、刺杀、细作，不适合正面作战主将。
- 个人特技表必须带“协同方向”列，写清它是补命中、补生存、补策略环境、补远程压制、补反击、补护送还是补攻坚；同时校验个人特技 ID 不与当前兵种特技 ID 重复。
- 策略学习要按功能递进：自身强化早于单体/范围强化，如 `气合` 早于 `奋起`；小补给早于大补给、输送、援队；单体妨碍早于强妨碍或范围妨碍；基础火水风地策略早于高级术法。物理兵种只给少量气合、奋起、坚固、疾行等，文吏、谋士、医僧再开放完整策略链。
- `6.5-4-1 兵种说明` 应单独写 80 行，与详细职业 ID 一一对应。说明采用游戏内口吻，不写“游戏设定”“数值”“转职”等元话语；半文言即可，重点写气势、职责、地域感和战场用法。
- 兵种特效风险分层：稳定层可用常见攻击、防御、命中、恢复、移动、远程/山水战辅助；大胆层如策略追击、疾风攻击、侧翼突袭、固定减伤、恢复周围等，需要后续实机验证手感；暂避层包括标注“剧本特技，扳手无效”的 `211`，以及 `220+` 空白、死亡台词或未明效果。
- 特效值先写为设计建议，不等于最终写入值。正式写入 `6.5-7-2` 前必须备份、复读，并用至少一场测试战确认触发、叠加、显示和 AI 行为。
