# AI绘图 / Image Studio MCP

## 结论速览

- CCZModStudio MCP 已接入无头 AI 绘图素材流程：先生成提示词和目标映射，再调用 Image Studio 兼容上游，最后后处理为 6.5 图片资源可用尺寸。
- 所有产物只写入 `CCZModStudio_Exports\AiImageAssets`，并做替换预览；不直接写入 `E5`、`Pmapobj.e5`、`Unit_*.e5` 或 DLL 资源。
- S 形象提示词已从“单帧小人”升级为“4x6、24 格动作表”。后处理按格切帧，导出 `Unit_mov.e5`、`Unit_atk.e5`、`Unit_spc.e5` 三套条带。
- 本地 6.5 样本尺寸和编号映射为“已验证”；OpenAI 文档、Sprite sheet/像素动画资料为“外部佐证”。

## 适用版本

- 默认适用：曹操传加强版 6.5 未加密当前基底。
- 不自动适配 6.6/通用 6.x；跨版本需要重新验证资源文件、编号范围、条带尺寸和写回护栏。

## 绘图工具选择

- R/S 形象默认上游已切换为 RetroDiffusion，原因是它面向像素画和游戏素材，公开资料显示支持像素角色、sprite sheet、原生小尺寸和参考图/编辑类工作流，比通用 Image2 更适合低分辨率点阵约束。
- `r_actor` / `s_unit` 默认使用像素专用上游：`CCZ_PIXEL_IMAGE_PROVIDER` 未配置时等同于 `retrodiffusion`。
- `r_background` / `face` / `dll_icon` 仍使用 Image Studio 兼容上游；这些素材不是当前低分辨率 S/R 点阵痛点。
- 若 RetroDiffusion 暂不可用，可临时设置 `CCZ_PIXEL_IMAGE_PROVIDER=image_studio` 回退原 Image Studio 请求形态，但 R/S 质量预期应降级为草稿辅助。

RetroDiffusion 环境变量：

```text
RETRO_DIFFUSION_BASE_URL=https://api.retrodiffusion.ai
RETRO_DIFFUSION_API_KEY=<本机私有密钥，不写入仓库>
RETRO_DIFFUSION_MODEL=RD_FLUX
CCZ_PIXEL_IMAGE_PROVIDER=retrodiffusion
```

Image Studio 环境变量仍用于非 R/S 素材和回退：

```text
IMAGE_STUDIO_BASE_URL=<兼容 OpenAI Images/Responses 的上游>
IMAGE_STUDIO_API_KEY=<本机私有密钥，不写入仓库>
IMAGE_STUDIO_TEXT_MODEL=<文本/Responses 模型>
IMAGE_STUDIO_IMAGE_MODEL=<图像模型>
IMAGE_STUDIO_API_MODE=responses|images
```

## S 形象提示词工程

### 蛋卷 AI 形象包本地经验

- `D:\Downloads\蛋卷AI形象包0608\01提示词`：记录了 S/R 形象的两种提示词思路。`S提示词一` 是自由动作型，适合没有足够格式参考图时让模型按 4x6 描述自行组织动作；`S提示词二` 是替换型，要求用第一张角色图替换第二张格式图中的角色，并保留第二张图的布局、动作和朝向。当前默认优先采用替换型。
- `D:\Downloads\蛋卷AI形象包0608\02参考图`：包含 R 参考图、S 4x4/4x6/5x5/8x8 参考图和 SR 参考图。其中 `s-4-6-*.png` 为 `1024x1536`，本质是把低分辨率曹操传 S 小人按 4 倍硬边放大，便于通用绘图模型理解比例、动作和像素边缘。
- `D:\Downloads\蛋卷AI形象包0608\03成品（修完）` / `04未修待处理的跑图`：显示 AI 输出通常只是草稿，需要缩回目标尺寸后再检查、修图和整理成 `mov.bmp`、`atk.bmp`、`spc.bmp`。不要把通用模型输出直接当作可入库成品。
- `01提示词\03说明.txt` 明确提醒“现阶段只有辅助作用”，并提示不要随意使用未公开 R/S 形象。未公开素材只能作为私有参考或学习观察；除非后续确认授权，不能默认进入训练集、公开模型或公开资料包。

### 输入图角色分工

- 第一张参考图：角色身份、服装、发型、配色、武器、关键装饰；可以是全身像、立绘、补全后的全身图或三视图。
- 第二张参考图：曹操传 MOD S 格式图，负责动作、朝向、人物比例、占格比例、像素画法、布局和风格。优先使用 4 倍硬边放大的 `1024x1536` 4x6 格式图。
- 后续真实 `mov.bmp / atk.bmp / spc.bmp` 或 Unit 条带参考图：用于补充原生像素画法、洋红色键背景、轮廓粗细、动作幅度和特效尺度。
- 不要让模型复制参考图背景、UI、文字、水印；只抽取角色设计和 sprite 风格。
- 武器不能写死为剑；必须从参考图或描述中抽取“当前武器”，例如枪、戟、刀、剑、斧、锤、弓、弩、扇、杖、法器或徒手格斗武器。
- 有第二张格式参考图时，提示词优先要求“把第一张角色替换到第二张格式图的所有动作格”，而不是让模型重新设计整套动作。

### 曹操传 S 风格约束

- 已验证帧规格：`Unit_mov.e5` 单帧 `48x48`，`Unit_atk.e5` 单帧 `64x64`，`Unit_spc.e5` 单帧 `48x48`。
- 已验证真实样例：`D:\Downloads\mov.bmp` 为 `48x528`、11 帧；`atk.bmp` 为 `64x768`、12 帧；`spc.bmp` 为 `48x240`、5 帧；背景为接近 `RGB(247,0,255)` 的纯洋红色键。
- 真实移动帧几乎撑满 `48x48`，样例平均可见包围盒约 `46.8x47.8`；攻击帧利用 `64x64` 大画布，样例平均包围盒约 `60.8x57.5`；特技帧可按动作需要略小，但仍有明确状态/施法/倒地姿态。
- 角色在格内应是低分辨率战棋小人，而不是高清像素插画：粗轮廓、硬边、大块色、少量明暗层次、有限调色盘。
- 比例应接近曹操传 S 小人：头盔/头部偏大但不是巨头，躯干紧凑，腿短，肩背宽，整体约 `2 到 3 头身`。
- 细节要概括：服装、甲片、披风、发色、阵营色只保留缩小到 `48x48/64x64` 后仍可读的关键色块。
- 每格角色、当前武器、裙摆和特效应接近撑满单帧，允许贴近边界但不能被裁断；不要画成小图标悬在大空白里。
- 负面方向：不要精细立绘、写实长身比例、柔边抗锯齿、复杂光影、现代高清像素画、白底/透明底、大面积留白，也不要把非剑武器改成剑。

### 失败样例复盘

- 仅写“像素风、居中、不出界、留边距”会诱导模型生成现代像素贴纸：角色小、白底、细节多、攻击只是站立角色加小特效。
- 真实曹操传 S 的关键不是“干净居中”，而是“目标帧内原生绘制、洋红色键背景、几乎撑满格、低色块、粗轮廓、强动作剪影”。
- 4x6 中间表可继续用于工程化切帧，但提示词必须说明最终条带语义：move 11 帧、attack 12 帧、special 5 帧，攻击帧要使用 `64x64` 的全画布。
- 替换型提示词比自由动作型更稳定：自由动作型容易改布局、漏格、动作重复、比例漂移；替换型能把布局和动作压力交给第二张格式图。
- 即使用替换型提示词，通用绘图模型仍可能产生白底、柔边、脏色、格内比例漂移或局部出界，必须后处理和手修检查。

## API 请求形态

RetroDiffusion R/S 默认请求形态：

```json
{
  "prompt": "<替换型或自由动作型 S/R 提示词>",
  "negative_prompt": "<负面约束>",
  "model": "${RETRO_DIFFUSION_MODEL}",
  "width": 1024,
  "height": 1536,
  "num_images": 1,
  "image_format": "png",
  "pixel_art": true,
  "style": "low-resolution pixel art sprite sheet",
  "palette": "limited",
  "transparent_background": false,
  "background_color": "#F700FF",
  "remove_background": false
}
```

说明：当前 MCP v1 只传文本约束和像素参数；如果后续上游账号支持参考图上传/编辑端点，再把“第一张角色图 + 第二张格式图”的二图输入接入同一后处理链路。

Responses API 可用于文字生图，也可在 content 中加入 `input_image` 作为参考图。Images API 可用于生成和编辑，编辑端点适合“角色参考图 + 像素风格参考图”的工作流。常用参数包括 `size`、`quality`、`output_format`，响应可返回 base64 图片。

Responses API 文字生图示例：

```json
{
  "model": "${IMAGE_STUDIO_TEXT_MODEL}",
  "input": [{"role": "user", "content": [{"type": "input_text", "text": "<prompt>"}]}],
  "tools": [{
    "type": "image_generation",
    "model": "${IMAGE_STUDIO_IMAGE_MODEL}",
    "size": "1024x1536",
    "quality": "high",
    "output_format": "png"
  }],
  "tool_choice": {"type": "image_generation"},
  "store": false,
  "stream": true
}
```

Images API 编辑/参考图示例：

```text
POST /v1/images/edits
multipart/form-data:
- model = gpt-image-1.5 或兼容图像编辑模型
- prompt = <4x6 S形象动作表提示词>
- image = <角色参考图>
- image = <Unit/S风格参考图>
- size = 1024x1536
- quality = high
- output_format = png
```

## 后处理规则

- 背景：按目标比例居中裁切，缩放到 `640x400`，保存 JPG。
- 头像：居中裁切为正方形，缩放到目标尺寸，保存 PNG；纯洋红或纯色边角背景会转透明。
- DLL 图标：居中缩放到方形画布，输出 PNG，再由 DLL 预览服务缩放为目标 RT_BITMAP 尺寸。
- R 形象：仍为静态条带草稿，将同一立绘缩放成 `48x64` 帧，纵向复制 20 帧，输出 `48x1280 png`。
- S 形象：优先要求上游生成 `4x6/24格` 动作表；后处理按格切帧并生成 move/attack/special 三条带，不再复制单帧。

## 风险与边界

- S 形象 v2 已避免重复单帧，但 `Unit_mov/atk/spc` 的精确方向、动作语义和实机表现仍需逐条验证。
- 生成后只做替换预览，不直接写 E5 或 DLL；正式替换必须显式调用现有 `replace_*` 工具，并走备份、复读和报告。
- 不依赖模型直接输出透明背景；提示词要求纯洋红或纯色背景，后处理负责透明化。
- RetroDiffusion 是当前 R/S 默认工具，但仍不能保证一次生成可入库；仍按草稿、预览、手修流程执行。
- 通用绘图模型输出只视为草稿，不能直接作为入库成品；必须缩回目标尺寸、检查条带、必要时手修。
- 未公开 R/S 形象只能作为私有参考或学习观察；除非确认授权，不能默认进入训练集、公开模型或公开资料包。
- 垂域 S 形象模型计划已延期归档到 `10-制作案例/AI垂域S形象模型-延期计划.md`，当前不训练模型、不接入生产流程。
- API Key 不得写入仓库、知识库或 generated config；只允许本机环境变量或本机私有配置。
