#### B形象指定器校正

来源：`B形象指定器\形象指定器6.5\System.ini`。

- `DefID=70`：防具起始编号，取值范围 0~255。
- `AssID=109`：辅助装备段起始编号，取值范围 0~255。

当前工具保留 `大类` 作为边界分段，同时新增内容感知 `物品大类`：

- ID `< 70`：武器。
- ID `70-108`：防具。
- ID `>= 109`：辅助段编码范围。段内继续按物品行内容区分：`装备特效号=2` 为辅助装备，`装备特效号=3` 为道具/消耗品；空名、0/102 等保守标为预留/未知。
- `DefID` 和 `AssID` 是下一段的起始 ID，不是上一段末尾的占位开关；因此默认配置下 ID `69` 仍属于武器，ID `108` 仍属于防具。

#### 当前实现

- `宝物设定` 创作页合并基础字段和说明文本，界面按旧版编辑器习惯显示关键列。
- 可写字段：基础字段与 `介绍`。
- 可见表头顺序：`ID`、`名称`、`图标`、`物品大类`、`类型码`、`初始能力`、`能力成长`、`价格（/100）`、`特效号`、`特效名`、`特效值`、`图鉴`、`介绍`。
- 可见界面隐藏：`分段`、`大类`、`类型说明`、`价格显示`、`实际效果号`、`实际效果说明`、`特效提示`、`来源文件`。
- `类型` 原始字段在界面标题为 `类型码`，用 `ItemTypeCatalogService.BuildShortName(...)` 显示短名，例如 `普通剑系`、`普通枪系`；它不是兵种设定里的可装备类别，底层仍保存原始单字节类型码。下拉覆盖 0-255，未知码显示 `类型N` 或按大类显示 `武器类型N` 等。
- `宝物图鉴` 是物品 25 字节行最后 1 字节，界面显示为 `图鉴` 并允许按 0..255 原值编辑；它只作为图鉴/宝物标记字节处理，不作为辅助装备可装备部队列表。
- 界面只保留原始可编辑的 `价格（/100）`，不再额外显示第二个 `价格显示` 列。
- 工具栏新增 `宝物特效`：打开项目侧宝物特效目录编辑器，表头为 `特效号`、`特效名`、`特效说明`。
- `宝物特效` 存储路径为 `CCZModStudio_Notes\{项目名}_ItemEffectCatalog.json`，使用 UTF-8 JSON 保存；允许重复 `特效号`，`特效名` 与 `特效说明` 支持变长中文。
- `装备特效名` 按旧版习惯显示中文特效名称；普通/无特效显示 `无`。读取逻辑为：优先读取项目侧 UTF-8 宝物特效目录，未命中时再回退到 `6.5-1-2/6.5-1-3 装备特效名称` 基础表；同一特效号在项目侧目录中有多条记录时，宝物页按 `名称1 / 名称2` 合并显示。
- `宝物特效` 不写回 `Ekd5.exe` 固定宽度特效名称区。原因是旧 EXE 表是固定宽度 GBK 字段，无法稳定支持重复编号、变长中文名和无编码异常写回。
- 2026-06-13 起，装备特效名称基础表读取通过 `CczEngineProfileService.TableHints.ItemEffectNameLowTable/HighTable` 和 `HexTableNameResolver.ResolveForProject` 定位，不再在服务层固定 `6.5-1-2/6.5-1-3` 字面表名。当前 6.5 写回规则不变。
- EffectPackage 的 `item` 域用于维护宝物特效：项目侧 UTF-8 JSON 目录负责中文名/说明，物品低段/高段表负责 `装备特效号` 与 `装备特效号-效果值` 绑定。删除模式会自动扫描当前引擎的低/高物品表，清理所有引用该特效号的物品绑定。
- 辅助段中 `装备特效号=2/3` 的类别标记会先按内部有效效果号纠偏，再查询中文名；`2` 归类为辅助装备，`3` 归类为道具/消耗品。
- 宝物设定不提供辅助装备“可装备部队”选择；辅助能否装备由兵种设定里的可装备类别和物品自身 `类型码` 匹配决定，不能复用 `宝物图鉴` 字节或辅助段其它字段当作部队列表。
- 右侧图标预览按 `图标` 字段从 `Itemicon.dll` 的 `RT_BITMAP` 资源中显示候选图标，便于创作者确认外观。
- 保存路径：
  - ID 0-103 写回 `Data.e5` 与 `Imsg.e5`。
  - ID 104+ 写回 `Star.e5` 与 `Imsg.e5`。
- 保存策略：写入前自动备份，写入后重新读取校验。
- 烟测：`--write` 会在测试副本中修改 0 段与 104+ 段物品名称/说明，输出 `ITEM_WRITE_SMOKE_OK`。
- 烟测：`--rs-smoke` 会导出短剑候选图标 PNG，输出 `ITEM_ICON_PREVIEW`。

#### 2026-07-06 清儿吕布传 6.6 物品行布局证据

本节只适用于 `基底\清儿吕布传` 这一 6.6 本地样本。该项目 `Ekd5.exe` 大小为 `1413120`，版本资源低字为 `6`，应按旧扳手低字映射识别为 `6.6`。不能因为 EXE 大小不同于 `新改曹操傳6.6修正版` 就降级为 `6.1` 或套用 6.5 物品字段解释。

清儿 6.6 物品基础行仍为 `0x19`/25 字节，但行内字段与当前 6.5 HexTable 的解释不同。已复核低段基址为 `Data.e5 @ 0x818C`，高段基址为 `Star.e5 @ 0`：

| 偏移 | 清儿 6.6 解释 | 证据/处理 |
| ---: | --- | --- |
| `row+0x00..0x0E` | 可见 GBK 名称 | 读取到首个 `00` 截断；后续尾标/保留字节不得混入显示名。 |
| `row+0x0F` | 正确的宝物/物品图标字段 | 低段 `Data.e5 @ 0x819B + ID*0x19`；高段 `Star.e5 @ (ID-104)*0x19 + 0x0F`。 |
| `row+0x10` | 保留/终止候选 | 当前样本多为 `0`，写入时应保留原值。 |
| `row+0x11` | 类型字段；辅助/道具段常作为实际特效候选 | 当 `row+0x12` 为 `2/3` 类别标记时，实际效果优先从此字节解释。 |
| `row+0x12` | 原始装备特效字节；辅助/道具段可为类别标记 | 武器/防具可作为特效号；辅助常见 `2`，消耗/道具常见 `3`，不得直接当普通特效解释。 |
| `row+0x13` | 价格 | 单字节，沿用原始值解释。 |
| `row+0x14` | 旧 6.5 图标位置；清儿 6.6 下不是图标字段 | 当前工具若从这里读图标，会出现图标号跳脱和预览与游戏不一致。 |
| `row+0x15` | 初始能力 | 单字节。 |
| `row+0x16` | 特效值 | 单字节。 |
| `row+0x17` | 成长 | 单字节。 |
| `row+0x18` | 图鉴/宝物标记 | 单字节。 |

图标错位根因：当前旧 6.5 布局把 `row+0x14` 当 `图标`，但清儿 6.6 正确图标字段在 `row+0x0F`。已验证样例：

- `ID=0 短剑`：`Data.e5 @ 0x819B = 1`，旧 `row+0x14 = 1` 只是碰巧接近。
- `ID=1 大剑`：`row+0x0F = 2`，旧 `row+0x14 = 1`，已经错位。
- `ID=2 钢剑`：`row+0x0F = 3`，旧 `row+0x14 = 2`。
- `ID=30 仁义之剑`：`row+0x0F = 31`，旧 `row+0x14 = 21`。
- `ID=104 镶玉甲`：`Star.e5 @ 0x0F = 105`。

图标资源换算仍采用 6.6 `E5\Item.e5` 成对规则：表内 `图标=N -> 小图 #2N+1 / 大图 #2N+2`，宝物预览默认取大图。也就是说 `图标=1 -> #3/#4`。注意“图标字段号”与“E5 条目号”不能混用；直接浏览 `Item.e5` 时才使用 `#3/#4` 这类条目号。

清儿样本的图标字段大体按物品 ID 顺序递增，但不能强制重排。已发现存在人工指定的特殊高值，例如 `ID=36 方天画戟` 图标 `248`、`ID=51 芭蕉扇` 图标 `232`、`ID=148 墨麒麟` 图标 `231`。因此正确规则是“从 `row+0x0F` 读原值并按 E5 成对映射预览”，不是按 ID 自动生成连续编号。

辅助/道具特效读取根因：清儿 6.6 的辅助装备和消耗品中，`row+0x12=2/3` 多为类别标记，不是实际特效号。实际效果候选常在 `row+0x11`。已验证样例：

- `ID=109 铜盾`：`row+0x11=0x3D`，`row+0x12=2`，应解释为 `盾反 / 辅助攻击防御`，而不是特效 `2`。
- `ID=111 风神盾`：`row+0x11=0x3F`，`row+0x12=2`，应解释为 `辅助全防御`。
- `ID=112 遁甲天书`：`row+0x11=0x53`，`row+0x12=2`，应解释为 `策略绝对命中 / 策略无视地形 / 策略无视天气`。
- `ID=140 玉玺`：`row+0x11=0x5C`，`row+0x12=2`，应解释为 `神魔附体`。
- `ID=160 印绶` 及消耗品段常见 `row+0x12=3`，也应优先把 `3` 视作道具/消耗品类别标记。

“未定义特效”根因：只使用形象指定器清单会漏掉清儿物品表实际引用的部分编号。清儿 `Ekd5.exe` 中存在 6.6 特效名称槽池：`0x9E800 + id * 16`，16 字节 GBK 定宽槽。形象指定器清单是高可信绑定表；若物品引用编号未在清单中出现，但名称槽有有效文本，可作为 `6.6 EXE 名称槽候选` 显示，不能直接判为“未定义”。已复核槽位示例：

#### 2026-06-21 辅助装备与道具/消耗品区分

- 扳手源码中装备仓库和消耗品数量是两套运行时结构：装备仓库槽写装备 ID、等级、经验三字节；消耗品另有独立数量表。
- `AssID..255` 只是辅助段编码范围，不代表全都可装备。工具按物品行内容新增 `物品大类`：武器、防具、辅助装备、道具/消耗品、预留/空位、未知。
- 战场 `0x48` 辅助槽继续保留 `AssID` 相对编码，不因过滤消耗品而重排；若候选是道具/消耗品，会显示“不可装备”提示。
- 三英样本当前低位规划：`109-117` 为辅助装备，`118-122` 为道具/消耗品；其 `宝物图鉴` 全 0 解释为图鉴未启用，不是辅助适性表为空。

#### 2026-06-03 宝物特效目录

- 新增项目侧 `宝物特效` 编辑器，入口位于 `宝物设定` 页面工具栏，紧邻 `读取宝物/物品`、`保存宝物/物品`。
- 目录文件位于 `CCZModStudio_Notes\{项目名}_ItemEffectCatalog.json`，保存前会自动给旧 JSON 生成同目录时间戳 `.bak.json` 备份。
- 目录字段：
  - `EffectId`：特效号，可重复。
  - `Name`：特效名，变长中文。
  - `Description`：特效说明，变长中文。
- 宝物页、字段说明面板和跨表引用解释统一采用同一口径：
  - `0` 仍解释为 `无`。
  - `255` 仍解释为 `普通装备/无扩展特效`。
  - 其余编号优先查项目侧目录；未命中时回退基础装备特效名称表。
- 该方案用于解决三类旧表限制：
  - 固定宽度 GBK 字段导致的中文截断/编码异常。
  - 同一特效号无法稳定维护多条中文名。
  - 跨设备后不同系统区域设置下的编码不稳定。
