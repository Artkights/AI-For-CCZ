#### B形象指定器校正

来源：`B形象指定器\形象指定器6.5\System.ini`。

- `DefID=70`：道具/防具起始编号，取值范围 0~255。
- `AssID=109`：道具/辅助起始编号，取值范围 0~255。

当前工具按此推断 `大类`：

- ID `< 70`：武器。
- ID `70-108`：防具。
- ID `>= 109`：辅助/道具。
- ID `69` / `108`：结束或预留行，需要谨慎处理。

#### 当前实现

- `宝物设定` 创作页合并基础字段和说明文本，界面按旧版编辑器习惯显示关键列。
- 可写字段：基础字段与 `介绍`。
- 可见表头顺序：`ID`、`名称`、`图标`、`类别`、`初始能力`、`能力成长`、`价格（/100）`、`特效号`、`特效名`、`特效值`、`介绍`。
- 可见 UI 隐藏：`分段`、`大类`、`宝物图鉴`、`类型说明`、`价格显示`、`实际效果号`、`实际效果说明`、`特效提示`、`来源文件`。
- `类型` 原始字段在界面标题为 `类别`，用 `ItemTypeCatalogService.BuildShortName(...)` 显示短名，例如 `普通剑系`、`普通枪系`；底层仍保存原始类型码。下拉覆盖 0-255，未知码显示 `类型N` 或按大类显示 `武器类型N` 等。
- 界面只保留原始可编辑的 `价格（/100）`，不再额外显示第二个 `价格显示` 列。
- 工具栏新增 `宝物特效`：打开项目侧宝物特效目录编辑器，表头为 `特效号`、`特效名`、`特效说明`。
- `宝物特效` 存储路径为 `CCZModStudio_Notes\{项目名}_ItemEffectCatalog.json`，使用 UTF-8 JSON 保存；允许重复 `特效号`，`特效名` 与 `特效说明` 支持变长中文。
- `装备特效名` 按旧版习惯显示中文特效名称；普通/无特效显示 `无`。读取逻辑为：优先读取项目侧 UTF-8 宝物特效目录，未命中时再回退到 `6.5-1-2/6.5-1-3 装备特效名称` 基础表；同一特效号在项目侧目录中有多条记录时，宝物页按 `名称1 / 名称2` 合并显示。
- `宝物特效` 不写回 `Ekd5.exe` 固定宽度特效名称区。原因是旧 EXE 表是固定宽度 GBK 字段，无法稳定支持重复编号、变长中文名和无编码异常写回。
- 辅助/道具中 `装备特效号=2/3` 的类别标记会先按内部有效效果号纠偏，再查询中文名。
- 右侧图标预览按 `图标` 字段从 `Itemicon.dll` 的 `RT_BITMAP` 资源中显示候选图标，便于创作者确认外观。
- 保存路径：
  - ID 0-103 写回 `Data.e5` 与 `Imsg.e5`。
  - ID 104+ 写回 `Star.e5` 与 `Imsg.e5`。
- 保存策略：写入前自动备份，写入后重新读取校验。
- 烟测：`--write` 会在测试副本中修改 0 段与 104+ 段物品名称/说明，输出 `ITEM_WRITE_SMOKE_OK`。
- 烟测：`--rs-smoke` 会导出短剑候选图标 PNG，输出 `ITEM_ICON_PREVIEW`。

#### 2026-05-31 最高优先级校正：辅助装备特效号=2 的含义

- `B形象指定器 System.ini`：`DefID=70` 为防具起始，`AssID=109` 为辅助起始。
- 武器/防具中 `装备特效号` 基本可作为实际装备特效解释。
- 辅助装备中大量 `装备特效号=2` 不应解释成所有辅助装备实际特效都是 2；它更像“辅助装备类别标记”。
- 辅助装备的实际效果应优先查看 `类型` 字段；消耗/道具常见 `装备特效号=3`，也更像类别标记。
- 当前宝物页可见 UI 不再显示“实际效果号/效果说明”，避免新版冗长联动干扰旧版编辑体验；内部仍保留该纠偏逻辑供隐藏列、筛选和后续诊断使用。

#### 2026-06-03 宝物特效目录

- 新增项目侧 `宝物特效` 编辑器，入口位于 `宝物设定` 页面工具栏，紧邻 `读取宝物/物品`、`保存宝物/物品`。
- 目录文件位于 `CCZModStudio_Notes\{项目名}_ItemEffectCatalog.json`，保存前会自动给旧 JSON 生成同目录时间戳 `.bak.json` 备份。
- 目录字段：
  - `EffectId`：特效号，可重复。
  - `Name`：特效名，变长中文。
  - `Description`：特效说明，变长中文。
- 宝物页、字段说明面板和跨表引用解释统一采用同一口径：
  - `0` 仍解释为 `无`。
  - `255` 仍解释为 `普通装备/无扩展特效`。
  - 其余编号优先查项目侧目录；未命中时回退基础装备特效名称表。
- 该方案用于解决三类旧表限制：
  - 固定宽度 GBK 字段导致的中文截断/编码异常。
  - 同一特效号无法稳定维护多条中文名。
  - 跨设备后不同系统区域设置下的编码不稳定。
