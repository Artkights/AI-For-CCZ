﻿# 人工智能绘图 / 图像工作室 MCP

## 结论速览

- CCZModStudio MCP 已接入无头人工智能绘图素材流程：先生成提示词和目标映射，再调用图像工作室兼容上游，最后后处理为 6.5 图片资源可用尺寸。
- 所有产物只写入 `CCZModStudio_Exports\AiImageAssets`，并做替换预览；不直接写入 `E5`、`Pmapobj.e5`、`Unit_*.e5` 或 动态链接库资源。
- S 形象提示词已从“单帧小人”升级为“4x6、24 格动作表”。后处理按格切帧，导出 `Unit_mov.e5`、`Unit_atk.e5`、`Unit_spc.e5` 三套条带。
- 本地 6.5 样本尺寸和编号映射为“已验证”；OpenAI 文档、精灵图表/像素动画资料为“外部佐证”。
- RetroDiffusion / Image Studio 现在只作为 R/S 的可选 AI 草稿路线；当用户要求不通过外部生图模型时，必须改走本机像素编辑 MCP 主线。
- 本机像素编辑 MCP 主线不调用外部 provider，不使用 Python/Pillow 或几何脚本制造成品，只通过工具整合包 C# Core/MCP 对本地 BMP 条带做帧级像素编辑、接触表预览和结构化报告。

## 适用版本

- 默认适用：曹操传加强版 6.5 未加密当前基底。
- 不自动适配 6.6/通用 6.x；跨版本需要重新验证资源文件、编号范围、条带尺寸和写回护栏。

## 绘图工具选择

- 外部 AI 草稿路线中，R/S 形象可使用 RetroDiffusion，原因是它面向像素画和游戏素材，公开资料显示支持像素角色、精灵图表、原生小尺寸和参考图/编辑类工作流，比通用 Image2 更适合低分辨率点阵约束。
- 当用户要求“不通过 RetroDiffusion / Image Studio / 外部生图模型”时，`r_actor` / `s_unit` 不走 `CCZ_PIXEL_IMAGE_PROVIDER`，改用 `create_rs_pixel_edit_workspace` 等本机像素编辑 MCP 工具。
- `r_background` / `face` / `dll_icon` 仍使用 图像工作室 兼容上游；这些素材不是当前低分辨率 S/R 点阵痛点。
- 若 RetroDiffusion 暂不可用，可临时设置 `CCZ_PIXEL_IMAGE_PROVIDER=image_studio` 回退原 图像工作室 请求形态，但 R/S 质量预期应降级为草稿辅助。

RetroDiffusion 环境变量：

```text
RETRO_DIFFUSION_BASE_URL=https://api.retrodiffusion.ai
RETRO_DIFFUSION_API_KEY=<本机私有密钥，不写入仓库>
RETRO_DIFFUSION_MODEL=RD_FLUX
CCZ_PIXEL_IMAGE_PROVIDER=retrodiffusion
```

图像工作室 环境变量仍用于非 R/S 素材和回退：

```text
IMAGE_STUDIO_BASE_URL=<兼容 OpenAI Images/Responses 的上游>
IMAGE_STUDIO_API_KEY=<本机私有密钥，不写入仓库>
IMAGE_STUDIO_TEXT_MODEL=<文本/Responses 模型>
IMAGE_STUDIO_IMAGE_MODEL=<图像模型>
IMAGE_STUDIO_API_MODE=responses|images
```

## 本机像素编辑 MCP 路线

该路线适用于用户明确要求“不通过 RetroDiffusion / Image Studio / 外部生图模型”的 R/S 制作任务。

执行顺序：

1. 从本地 6.5 基底导出真实格式参考，整理为含 `front.bmp`、`back.bmp`、`mov.bmp`、`atk.bmp`、`spc.bmp` 的文件夹。
2. 调用 `create_rs_pixel_edit_workspace` 创建工作区。`design_image_path` 只作为设计观察图；`format_reference_root` 必须提供五件套格式参考。
3. 调用 `build_rs_pixel_edit_plan` 生成本机编辑 recipe。
4. 用 `apply_rs_pixel_frame_edits` 分轮执行清脸、擦旧武器、重绘甲胄、重绘披风、画唯一长枪、画枪尖特效和洋红键清理。
5. 每轮调用 `export_rs_pixel_contact_sheets`，人工检查原始尺寸和 4x/8x 放大图。
6. 调用 `validate_rs_pixel_edit_workspace` 做格式、帧、脸部、单枪风险检查。
7. 视觉验收通过后，才允许追加 `validate_rs_pixel_material_package` 和只读预览；仍不得直接调用 `replace_*`。

允许的像素编辑 op：

```text
recolor_palette
clean_face_box
erase_weapon_residue
erase_effect_residue
erase_rect_to_magenta
draw_spear_axis
draw_spear_tip
draw_spear_effect
repaint_armor_blocks
repaint_cape_blocks
magenta_key_cleanup
copy_region_from_reference
copy_region_from_frame
```

`erase_weapon_residue` 用于选择性清理武器亮色残留，不得默认整块清空 bbox。旧白灰/浅粉斩击弧优先用 `erase_effect_residue`。只有确认目标区域应为空背景时，才允许用 `erase_rect_to_magenta` 做小块整擦。
`copy_region_from_frame` 只能用于同条带内修复已确认的空帧、稀疏帧或原生未使用槽位；必须写入 `edit_log.jsonl`，不得作为伪造整套重复帧的手段。

边界：

- 不使用外部生图 provider。
- 不使用系统 imagegen。
- 不使用 Python/Pillow 或仓库外脚本制造 R/S 成品。
- `copy_region_from_reference` 只能用于局部参考，不得伪造整套重复帧。
- `copy_region_from_frame` 只能用于局部修复或填补明确坏帧，不得替代正常像素编辑。
- 工具输出接触表和报告不等于视觉合格，仍需人工看图。

## R/S 参考图参数

AI 草稿路线仍必须真实传入参考图，而不是只把参考图改写成文字。新版 MCP schema 支持：

```text
reference_image_paths: string[]
reference_roles: string[]
```

推荐角色：

- `design`：角色形象图。只负责身份、服装、主色、轮廓、气质和明确允许的设计特征。
- `format_action`：曹操传 6.5 R/S 格式图。只负责动作语法、格子顺序、朝向、短身比例、占格、洋红键背景和低像素风格。
- `style_optional`：可选补充风格图。只能补充像素边缘、色块和特效尺度。

R/S AI 草稿整包优先使用 `build_rs_pixel_character_design`，该入口统一记录 `design` 和 `format_action` 参考图、生成 S 4x6 动作表计划、生成 R `2列x20行` 正背动作表计划，并把结果整理到 `CCZModStudio_Exports\RS_PixelDesign\<PackageId>`。如果当前客户端工具表仍未暴露 `build_rs_pixel_character_design` 或参考图参数，说明运行中的 MCP 进程未刷新，应先重启/刷新 MCP，必要时通过 `工具整合包\MCP配置\start-ccz-mcp.ps1` 直连新版 JSON-RPC。

计划和 manifest 必须记录每张参考图的角色、路径和 SHA256。没有参考图记录的 R/S 生成，不得声称“使用了形象图”。

## S 形象提示词工程

### 蛋卷 人工智能 形象包本地经验

- `D:\Downloads\蛋卷人工智能形象包0608\01提示词`：记录了 S/R 形象的两种提示词思路。`S提示词一` 是自由动作型，适合没有足够格式参考图时让模型按 4x6 描述自行组织动作；`S提示词二` 是替换型，要求用第一张角色图替换第二张格式图中的角色，并保留第二张图的布局、动作和朝向。当前默认优先采用替换型。
- `D:\Downloads\蛋卷人工智能形象包0608\02参考图`：包含 R 参考图、S 4x4/4x6/5x5/8x8 参考图和 SR 参考图。其中 `s-4-6-*.png` 为 `1024x1536`，本质是把低分辨率曹操传 S 小人按 4 倍硬边放大，便于通用绘图模型理解比例、动作和像素边缘。
- `D:\Downloads\蛋卷人工智能形象包0608\03成品（修完）` / `04未修待处理的跑图`：显示人工智能输出通常只是草稿，需要缩回目标尺寸后再检查、修图和整理成 `mov.bmp`、`atk.bmp`、`spc.bmp`。不要把通用模型输出直接当作可入库成品。
- `01提示词\03说明.txt` 明确提醒“现阶段只有辅助作用”，并提示不要随意使用未公开 R/S 形象。未公开素材只能作为私有参考或学习观察；除非后续确认授权，不能默认进入训练集、公开模型或公开资料包。

### 输入图角色分工

- 第一张参考图：角色身份、服装、发型、配色、武器、关键装饰；可以是全身像、立绘、补全后的全身图或三视图。
- 第二张参考图：曹操传 模组 S 格式图，负责动作、朝向、人物比例、占格比例、像素画法、布局和风格。优先使用 4 倍硬边放大的 `1024x1536` 4x6 格式图。
- 后续真实 `mov.bmp / atk.bmp / spc.bmp` 或 Unit 条带参考图：用于补充原生像素画法、洋红色键背景、轮廓粗细、动作幅度和特效尺度。
- 不要让模型复制参考图背景、界面、文字、水印；只抽取角色设计和 精灵 风格。
- 武器不能写死为剑；必须从参考图或描述中抽取“当前武器”，例如枪、戟、刀、剑、斧、锤、弓、弩、扇、杖、法器或徒手格斗武器。
- 有第二张格式参考图时，提示词优先要求“把第一张角色替换到第二张格式图的所有动作格”，而不是让模型重新设计整套动作。

### 曹操传 S 风格约束

- 已验证帧规格：`Unit_mov.e5` 单帧 `48x48`，`Unit_atk.e5` 单帧 `64x64`，`Unit_spc.e5` 单帧 `48x48`。
- 已验证真实样例：`D:\Downloads\mov.bmp` 为 `48x528`、11 帧；`atk.bmp` 为 `64x768`、12 帧；`spc.bmp` 为 `48x240`、5 帧；背景为接近 `RGB(247,0,255)` 的纯洋红色键。
- 真实移动帧几乎撑满 `48x48`，样例平均可见包围盒约 `46.8x47.8`；攻击帧利用 `64x64` 大画布，样例平均包围盒约 `60.8x57.5`；特技帧可按动作需要略小，但仍有明确状态/施法/倒地姿态。
- 角色在格内应是低分辨率战棋小人，而不是高清像素插画：粗轮廓、硬边、大块色、少量明暗层次、有限调色盘。
- 比例应接近曹操传 S 小人：头盔/头部偏大但不是巨头，躯干紧凑，腿短，肩背宽，整体约 `2 到 3 头身`。
- 细节要概括：服装、甲片、披风、发色、阵营色只保留缩小到 `48x48/64x64` 后仍可读的关键色块。
- 每格角色、当前武器、裙摆和特效应接近撑满单帧，允许贴近边界但不能被裁断；不要画成小图标悬在大空白里。
- 负面方向：不要精细立绘、写实长身比例、柔边抗锯齿、复杂光影、现代高清像素画、白底/透明底、大面积留白，也不要把非剑武器改成剑。

### 失败样例复盘

- 仅写“像素风、居中、不出界、留边距”会诱导模型生成现代像素贴纸：角色小、白底、细节多、攻击只是站立角色加小特效。
- 真实曹操传 S 的关键不是“干净居中”，而是“目标帧内原生绘制、洋红色键背景、几乎撑满格、低色块、粗轮廓、强动作剪影”。
- 4x6 中间表可继续用于工程化切帧，但提示词必须说明最终条带语义：move 11 帧、attack 12 帧、special 5 帧，攻击帧要使用 `64x64` 的全画布。
- 替换型提示词比自由动作型更稳定：自由动作型容易改布局、漏格、动作重复、比例漂移；替换型能把布局和动作压力交给第二张格式图。
- 即使用替换型提示词，通用绘图模型仍可能产生白底、柔边、脏色、格内比例漂移或局部出界，必须后处理和手修检查。

### 替换型模板（默认推荐）

替换要求：把第一张角色图中的人物替换到第二张曹操传 S 格式图中的所有动作格里；保留第二张图的格子布局、行列顺序、每格朝向、动作姿态、动作节奏、人物短身比例、像素风格和特效尺度。不要改变第二张格式图布局，不要重排格子，不要把 24 格画成一张连续场景。不要复制参考图里的背景、界面、文字、水印。

曹操传 S 风格约束：必须像真实 6.5 Unit_mov/Unit_atk/Unit_spc 条带，而不是现代高清像素贴纸。真实移动/特技帧是 48x48，攻击帧是 64x64；角色、当前武器、裙摆和特效应接近撑满单帧，允许贴近边界但不能被裁断。不要把人物画成小图标悬在大空白里。每格必须像在目标低分辨率格内原生绘制，再最近邻放大展示：硬边像素、深色粗轮廓、有限调色盘、少量明暗层次、底部曹操传式深色落地阴影。人物是短身战棋小人：头盔/头部偏大但不能巨头，躯干紧凑，腿短，肩背宽，整体约 2 到 3 头身。

背景和输出：纯洋红色键背景，使用接近 #F700FF / RGB(247,0,255) 的单一平整背景色；不要白底、透明底、透明棋盘格、地面或场景。只保留曹操传式深色落地阴影。每格必须是不同姿态，不要复制粘贴同一帧。人工智能 输出只作为可编辑草稿，后处理会负责切格、缩放、洋红色键透明化和替换预览，必要时仍需要手工修图校正比例、动作和边缘像素。

```text
为曹操传加强版 6.5 模组 绘制 S 战场角色小人动作表。角色描述：{描述}

输出一张完整像素图 精灵图表：4 列 x 6 行，共 24 格；从左上到右下为动画/GIF 播放顺序；每格等宽等高，无边框、无网格线、无间距、无裁切。

参考图使用规则：第一张参考图只用于角色身份、服装、发型、配色、阵营气质、当前武器和关键装饰；后续真实 Unit/S 条带参考图只用于曹操传 S 形象的原生像素画法、人物比例、占格比例、洋红色键背景、轮廓粗细、动作幅度和特效尺度。不要复制参考图里的背景、界面、文字、水印。

武器规则：不要默认画剑。必须识别并沿用角色参考图或描述中的当前武器，例如枪、戟、刀、剑、斧、锤、弓、弩、扇、杖、法器或徒手格斗武器；如果参考图没有明确武器，就选择与角色职业最相称的一种古代武器，并在 24 格中保持同一武器。所有“格挡、庆祝、蓄力、攻击、收招、支撑休息”动作都使用这个当前武器；弓弩角色用拉弓、放箭和收弓表现攻击，法师/扇杖角色用挥扇/举杖施法表现攻击，长兵器角色用突刺或横扫，重武器角色用劈砍或砸击。

曹操传 S 风格约束：必须像真实 6.5 `Unit_mov/Unit_atk/Unit_spc` 条带，而不是现代高清像素贴纸。真实移动/特技帧是 48x48，攻击帧是 64x64；角色、当前武器、裙摆和特效应接近撑满单帧，允许贴近边界但不能被裁断。不要把人物画成小图标悬在大空白里。移动帧平均应占满约 44-48 像素宽、46-48 像素高；攻击帧应利用约 58-64 像素宽和 50-64 像素高，武器轨迹或法术特效可占半个格子。每格必须像在目标低分辨率格内原生绘制，再最近邻放大展示：硬边像素、深色粗轮廓、有限调色盘、少量明暗层次、底部曹操传式深色落地阴影。人物是短身战棋小人：头盔/头部偏大但不能巨头，躯干紧凑，腿短，肩背宽，整体约 2 到 3 头身；服装细节要概括成缩小后仍可读的色块，不要画成高清插画、精细立绘、写实长身比例或现代高清像素画。

背景和输出：纯洋红色键背景，使用接近 #F700FF / RGB(247,0,255) 的单一平整背景色；不要白底、透明底、透明棋盘格、地面或场景。只保留曹操传式深色落地阴影。每格必须是不同姿态，不要复制粘贴同一帧。

第1-4格：正面视角迈左脚行走；正面视角迈右脚行走；正面视角使用当前武器或盾牌格挡；正面视角受到攻击向后倾倒但仍握住当前武器。
第5-8格：侧面视角迈左脚行走；侧面视角迈右脚行走；侧面视角使用当前武器或盾牌格挡；正面视角举手或举起当前武器庆祝。
第9-12格：背面视角迈左脚行走；背面视角迈右脚行走；背面视角使用当前武器或盾牌格挡；正面视角用当前武器支撑或手扶武器半跪休息。
第13-16格：正面视角静止站立；正面视角举起当前武器蓄力准备攻击；正面视角使用当前武器攻击，加入低像素武器轨迹和少量光效；正面视角攻击后收回当前武器。
第17-20格：背面视角静止站立；背面视角举起当前武器蓄力准备攻击；背面视角使用当前武器攻击，加入低像素武器轨迹和少量光效；背面视角攻击后收回当前武器。
第21-24格：侧面视角静止站立；侧面视角举起当前武器蓄力准备攻击；侧面视角使用当前武器攻击，加入低像素武器轨迹和少量光效；侧面视角攻击后收回当前武器。

攻击动作必须夸张但仍像曹操传小人：身体明显位移和扭转，当前武器有大摆幅或明确施法/射击动作，轨迹/法术效果要大、粗、低像素，预备-打击-收招节奏清楚。攻击帧不要只是站立角色旁边加小特效；必须利用 64x64 攻击格的大画布。每一格里角色、当前武器和特效可以接近边缘但不能被裁断；动作之间要能连贯衔接，角色身份、服装配色和武器形状在 24 格中保持一致。
```

## 接口 请求形态

RetroDiffusion R/S 默认请求形态：

```json
{
  "prompt": "<替换型或自由动作型 S/R 提示词>",
  "negative_prompt": "<负面约束>",
  "model": "${RETRO_DIFFUSION_MODEL}",
  "width": 1024,
  "height": 1536,
  "num_images": 1,
  "image_format": "png",
  "pixel_art": true,
  "style": "low-resolution pixel art 精灵图表",
  "palette": "limited",
  "transparent_background": false,
  "background_color": "#F700FF",
  "remove_background": false,
  "reference_images": ["<base64 image 1>", "<base64 image 2>"]
}
```

说明：新版 MCP 支持 `reference_image_paths` / `reference_roles`。AI 草稿路线必须记录 `design`、`format_action` 等参考图角色，并在 RetroDiffusion 分支把参考图作为 `reference_images` 传给上游。没有参考图路径和 hash 记录时，不得声称“使用了形象图”。如果 provider 是 Image Studio 且不能处理参考图，应报错或降级为非 R/S 草稿，不得静默忽略参考图。

Responses 接口 可用于文字生图，也可在 content 中加入 `input_image` 作为参考图。Images 接口 可用于生成和编辑，编辑端点适合“角色参考图 + 像素风格参考图”的工作流。常用参数包括 `size`、`quality`、`output_format`，响应可返回 base64 图片。

Responses 接口 文字生图示例：

```json
{
  "model": "${IMAGE_STUDIO_TEXT_MODEL}",
  "input": [{"role": "user", "content": [{"type": "input_text", "text": "<prompt>"}]}],
  "tools": [{
    "type": "image_generation",
    "model": "${IMAGE_STUDIO_IMAGE_MODEL}",
    "size": "1024x1536",
    "quality": "high",
    "output_format": "png"
  }],
  "tool_choice": {"type": "image_generation"},
  "store": false,
  "stream": true
}
```

Images 接口 编辑/参考图示例：

```text
POST /v1/images/edits
multipart/form-data:
- model = gpt-image-1.5 或兼容图像编辑模型
- prompt = <4x6 S形象动作表提示词>
- image = <角色参考图>
- image = <Unit/S风格参考图>
- size = 1024x1536
- quality = high
- output_format = png
```

## 后处理规则

- 背景：按目标比例居中裁切，缩放到 `640x400`，保存 JPG。
- 头像：居中裁切为正方形，缩放到目标尺寸，保存 PNG；纯洋红或纯色边角背景会转透明。
- 动态链接库图标：居中缩放到方形画布，输出 PNG，再由 动态链接库 预览服务缩放为目标 RT_BITMAP 尺寸。
- R 形象：旧静态条带只能作为失败草稿经验；新版 AI 草稿路线要求 R 上游输出 `2列x20行` 正背动作表，本机像素编辑路线则直接编辑 `front.bmp/back.bmp` 五件套中的 R 条带。
- S 形象：优先要求上游生成 `4x6/24格` 动作表；后处理按格切帧并生成 move/attack/special 三条带，不再复制单帧。

## 风险与边界

- S 形象 v2 已避免重复单帧，但 `Unit_mov/atk/spc` 的精确方向、动作语义和实机表现仍需逐条验证。
- 生成后只做替换预览，不直接写 E5 或 动态链接库；正式替换必须显式调用现有 `replace_*` 工具，并走备份、复读和报告。
- 不依赖模型直接输出透明背景；提示词要求纯洋红或纯色背景，后处理负责透明化。
- RetroDiffusion / Image Studio 是可选 AI 草稿路线，不是 R/S 成品唯一入口；用户要求本机制作时，优先使用本机像素编辑 MCP 主线。
- 若 RetroDiffusion/API key 不可用，AI 草稿路线停止在 pending 状态；不得回退 Pillow、几何脚本或 fake upstream 生成成品。若要继续，只能切到本机像素编辑 MCP 工具链。
- 通用绘图模型输出只视为草稿，不能直接作为入库成品；必须缩回目标尺寸、检查条带、必要时手修。
- 未公开 R/S 形象只能作为私有参考或学习观察；除非确认授权，不能默认进入训练集、公开模型或公开资料包。
- 垂域 S 形象模型计划已延期归档到 `10-制作案例/人工智能垂域S形象模型-延期计划.md`，当前不训练模型、不接入生产流程。
- 接口 Key 不得写入仓库、知识库或 generated config；只允许本机环境变量或本机私有配置。
