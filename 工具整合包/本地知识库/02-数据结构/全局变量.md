##### 单位数组（两种编号体系）⭐

| 地址 | 名称 | 条目大小 | 条目数 | 索引方式 | 用途 |
|------|------|----------|--------|----------|------|
| `0x004A7B20` | TACTICAL_UNIT_ARRAY | 0x30 (48B) | 250 | **战场编号** (0-249) | 运行时状态：HP/MP/位置/行动状态 |
| `0x004A3E77` | CHARACTER_DATA_ARRAY | 0x61 (97B) | 1024 | **Data号** (0-1023) | 固有属性：五维/兵种/装备槽/个人特技 |

- `0x4061F9`：`TacticalUnit* = 0x4A7B20 + 战场编号 × 0x30`
- `0x484002`：`CharacterData* = 0x4A3E77 + Data号 × 0x61`

> 关于两种编号体系的完整说明，参见 [单位结构体.md — 两种编号体系](./单位结构体.md#两种编号体系-)。

#### 状态数组详解

##### 0x508B00 - 二次行动状态数组

用于跟踪每个单位的二次行动使用状态，每个单位占用1字节：

| 值 | 含义 |
|----|------|
| `0x00` | 本回合尚未触发，等待激活 |
| `0x01` | 单位不拥有该特技，不可用 |
| `0xFF` | 本回合已触发过，已使用（受保护） |

**状态转换流程**：

```
回合开始 → 重置循环将所有 0x508B00[i] 设为 0x00
    ↓
行动处理 → 调用点2判定：有特技→写0x00，无特技→写0x01
    ↓
行动结束 → 调用点1检查=0x00 → 判定通过 → 恢复行动 → 写0xFF
    ↓
再次行动 → 调用点1检查=0xFF → ≠0 → 跳过（不再触发）
```

##### 0x500E9D - 目标单位数组

引导攻击系统使用此数组存储候选目标列表：

```
[0x500E9D]     = 目标1单位编号
[0x500E9D + 1] = 目标2单位编号
[0x500E9D + 2] = 目标3单位编号
...
[0x500E9D + n] = 0xFF  (列表终止符)
```
