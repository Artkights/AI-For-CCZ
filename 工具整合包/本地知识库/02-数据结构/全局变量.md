| 地址 | 名称 | 条目大小 | 条目数 | 索引方式 | 用途 |
|------|------|----------|--------|----------|------|

**转换函数**：
- `4061F9`：`TacticalUnit* = 4A7B20 + 战场编号 × 30`

- `40658F`：从战术单位 `[unit+00]` 取得 Data 号并转人物运行时记录

#### 状态数组详解

##### 508B00 - 二次行动状态数组

用于跟踪每个单位的二次行动使用状态，每个单位占用1字节：

| 值 | 含义 |
|----|------|
| `00` | 本回合尚未触发，等待激活 |
| `01` | 单位不拥有该特技，不可用 |
| `FF` | 本回合已触发过，已使用（受保护） |

**状态转换流程**：

```
回合开始 → 重置循环将所有 508B00[i] 设为 00
    ↓
行动处理 → 调用点2判定：有特技→写00，无特技→写01
    ↓
行动结束 → 调用点1检查=00 → 判定通过 → 恢复行动 → 写FF
    ↓
再次行动 → 调用点1检查=FF → ≠0 → 跳过（不再触发）
```

##### 500E9D - 目标单位数组

引导攻击系统使用此数组存储候选目标列表：

```
[500E9D]     = 目标1单位编号
[500E9D + 1] = 目标2单位编号
[500E9D + 2] = 目标3单位编号
...
[500E9D + n] = FF  (列表终止符)
```
