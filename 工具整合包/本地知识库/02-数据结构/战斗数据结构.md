## 详细记录

#### 概述

`0x4927F0` 是物理攻击过程中所有战斗信息的集中存储区，在游戏代码中经常通过 `MOV ECX, 0x4927F0` 指令访问，然后通过各种偏移来读取或写入战斗数据。

**⭐ 此结构是两种编号体系的汇聚枢纽**：头部同时存储了**战场编号**（+0x00/+0x01）和 **Data号**（+0x04），并提供了两种指针（+0x08=角色数据指针, +0x0C=战术单位指针），是补丁代码中实现"从一种编号找到另一种编号"的关键桥梁。

> 关于两种编号体系的完整说明，参见 [单位结构体.md — 两种编号体系](./单位结构体.md#两种编号体系-)。

#### 数据结构详解

##### 头部区域 (0x4927F0 - 0x4927FF)

> 头部是 **两种编号体系交汇的核心**，同时存放战场编号和 Data号，以及对应的两种指针。

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+0x00` | 1字节 | 字节 | ATTACKER_BATTLE_ID | 攻击方的**战场编号**<br>范围: 00-0E(我军), 0F-22(友军), 23-72(敌军) |
| `+0x01` | 1字节 | 字节 | CURRENT_TARGET_BATTLE_ID | 当前被攻击方的**战场编号** |
| `+0x04` | 4字节 | 双字 | ATTACKER_DATA_ID | 攻击方的**Data号**（永久角色编号）<br>512人数据: 000-1FF<br>1024人数据: 000-3FF |
| `+0x08` | 4字节 | 双字 | ATTACKER_DATA_ADDR | 攻击方角色数据指针<br>等同于 `0x484002(ATTACKER_DATA_ID)` |
| `+0x0C` | 4字节 | 双字 | ATTACKER_TACTICAL_ADDR | 攻击方战术单位指针<br>等同于 `0x4061F9(ATTACKER_BATTLE_ID)` |

##### 受攻击武将列表区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+0x10` | 0x74字节 | 字节[0x73] | TARGET_BATTLE_ID_ARRAY | 所有被攻击武将的战场编号数组<br>含穿透攻击中其他受影响武将 |

##### 伤害值区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+0x84` | 0x1CC字节 | 双字[0x73] | HP_DAMAGE_ARRAY | 全体被攻击武将按顺序存放HP伤害值 |
| `+0x254` | 0x1CC字节 | 双字[0x73] | MP_DAMAGE_ARRAY | 全体被攻击武将按顺序存放MP伤害值 |

##### 穿透攻击与经验区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+0x424` | 1字节 | 字节 | PIERCE_RANGE | 穿透范围 |
| `+0x425` | 1字节 | 字节 | TARGET_X | 当前被攻击武将X坐标 |
| `+0x426` | 1字节 | 字节 | TARGET_Y | 当前被攻击武将Y坐标 |
| `+0x428` | 4字节 | 双字 | PERSONAL_EXP | 攻击者获得的个人经验值 |
| `+0x42C` | 4字节 | 双字 | WEAPON_EXP | 攻击者获得的武器经验值 |
| `+0x430` | 0x1CC字节 | 双字[0x73] | ARMOR_EXP_ARRAY | 全体被攻击武将获得的防具经验值 |

##### 战斗状态与特效区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+0x600` | - | - | AI_ATTACK_FLAG | 是否人工智能攻击 |
| `+0x604` | 4字节 | 双字 | CRITICAL_HIT_FLAG | 是否发生暴击 (0=无, 非0=是) |
| `+0x608` | 1字节 | 字节 | DOUBLE_ATTACK_FLAG | 是否连击 (0=无, 非0=是) |
| `+0x60C` | 4字节 | 双字 | SHAKE_X | 挨打动作播放时位移抖动的横坐标 |
| `+0x610` | 4字节 | 双字 | SHAKE_Y | 挨打动作播放时位移抖动的纵坐标 |
| `+0x614` | 4字节 | 双字 | COUNTER_ATTACK_FLAG | 反击以及反击后反击的标志<br>通常也通过0x492E04访问 |

#### 使用示例

##### 获取攻击者数据指针

```asm
mov ecx, 0x4927F0         ; ECX = 战斗数据基址
mov ecx, [ecx + 0x08]    ; ECX = CharacterData*
```

##### 获取攻击者战术单位指针

```asm
mov ecx, 0x4927F0
mov ecx, [ecx + 0x0C]    ; ECX = TacticalUnit*
```

##### 调用点代码中访问

在 [物理攻击.md](../03-机制详解/物理攻击.md) 中提到的0x43DADA函数：
```asm
0x43DB53: mov ecx, 0x4927F0    ; 设置战斗数据基址
0x43DB58: call 0x4064F1        ; 写入战斗数据
```

#### 注意事项

1. **数组大小**：所有与被攻击武将相关的数组大小都是 0x73 (115)，支持最多115个单位
2. **坐标对齐**：+0x425和+0x426是连续的两个字节，可作为字读取
3. **攻击者信息**：+0x08和+0x0C存储的是指针，可直接用于特技判定系统
4. **常用别名**：0x492E04是0x4927F0 + 0x614的别名，用于快速访问反击标志

#### 2026-06-11 动态补充：物理伤害方向与反击时头部翻转

测验1续的友方/敌方自动阶段确认：`0x004927F0` 头部描述的是“当前正在结算的伤害方向”，不是固定的“玩家主动发起者”。

- `+0x00` 当前攻击者战场编号。
- `+0x01` 当前目标战场编号。
- `+0x04` 当前攻击者 Data 号。
- `+0x0C` 当前攻击者战术单位指针。
- 主动攻击、友军自动攻击、敌军自动攻击、反击都会复用同一套字段。
- 反击发生时，`+0x00/+0x01` 会按当前反击方向翻转；Hook 代码应以当前字段为准，不要缓存上一段伤害的攻击者/目标后直接复用。

与 HP 应用点的配合：

- 在 `0x00405AD5`，`[EBP-0x20]` 是当前目标 `unit*`，`[EBP-0x14]` 是本段最终伤害，`EAX` 是旧 HP。
- 可用 `([EBP-0x20] - 0x004A7B20) / 0x30` 反推出目标战场编号，并与 `0x004927F0+0x01` 交叉校验。
- 经验相关字段 `+0x428/+0x42C/+0x430` 已确认由当前 6.5 物理攻击流程写入战斗上下文；主动攻击样本中的写入点分别为 `0x00406043`、`0x00406054`、`0x00406074`。击杀、反击、连击和最终落盘/升级显示规则仍需分样本验证。

#### 2026-06-11 动态补充：曹操普通攻击 EXP 写入闭环

第二回合曹操 `unit[0]` 普通攻击敌军 `unit[61]` 的样本确认了 `+0x428/+0x42C/+0x430` 的写入时机。该样本未触发连击，`0x004E3000` 返回 `EAX=0`，`0x00406520` 未命中。

攻击初始化清零：

- `0x0040632D`：写 `ctx+0x428 = 0`，清个人经验缓存。
- `0x00406333`：写 `ctx+0x42C = 0`，清武器 EXP 缓存。
- `0x00406342 -> 0x0047FD13`：清 `ctx+0x430` 防具 EXP 数组。

奖励写入：

| 字段 | 写入点 | 计算函数/来源 | 本次值 | 说明 |
|------|--------|---------------|--------|------|
| `+0x428` | `0x00406043` | `0x004037A3` | `10` | 与旧值取较大值后写回 |
| `+0x42C` | `0x00406054` | `0x00403870` | `2` | `add [ctx+0x42C], eax` |
| `+0x430+slot*4` | `0x00406074` | `0x00403A8A` | `0` | 本次 `slot=0`，无防具 EXP |

调用链：

```text
0x0040654A
    -> 0x00452807
        -> 0x00406043 / 0x00406054 / 0x00406074
```

边界：

- 旧版文档线索 `0x00406113/0x00406153/0x00406171` 在本次 6.5 曹操样本中未命中。
- 这些字段是战斗上下文中的本次结算缓存，不等同于角色永久经验、装备永久 EXP 或五维功勋存储字段。
- 仍需拆分验证：击杀、反击、连击第二段、策略伤害、策略击杀和升级显示/落盘。

#### 2026-06-11 动态补充：暴击击杀时的战斗数据字段

第二回合曹操普通攻击触发暴击并击杀 `unit[69]` 的样本确认：

- `+0x00 = 0`，当前攻击者为曹操 `unit[0]`。
- `+0x01 = 0x45`，当前目标为 `unit[69]`。
- `+0x0C = 0x004A7B20`，当前攻击者单位指针。
- `+0x84` 第一个 HP 伤害值为 `0x44`，与 `0x00405AD5` 的 `[EBP-0x14]=0x44` 一致。
- `+0x428 = 38`，本次读到的个人经验字段值。
- `+0x42C = 3`，本次读到的武器 EXP 字段值。
- `+0x604 = 1`，暴击标志置位。
- `+0x608 = 0`，本段不是连击。
- `+0x614 = 0`，本段不是反击。

经验字段写入时机边界：

- 在 HP setter 返回后才设置 `+0x428/+0x42C/+0x430` 写断点，没有命中；但字段已经有值。
- 因此这些字段很可能在战斗数据生成/伤害计算阶段早于 `0x00405AD5` 写入，下一轮应在攻击命令确认前提前布设写断点。

#### 2026-06-11 动态补充：第二次自动阶段字段表现

第二次进入友方/敌方自动阶段后，`0x00405AD5` 改为暂停采样后又切换为 log-only。当前可人工确认的暂停样本如下：

| 当前伤害方向 | 旧 HP | 伤害 | 新 HP | `+0x604` | `+0x608` | `+0x614` | `+0x428/+0x42C` |
|--------------|-------|------|-------|----------|----------|----------|-----------------|
| `unit[24] -> unit[65]` | 27 | 27 | 0 | 0 | 0 | 0 | 0 / 0 |
| `unit[25] -> unit[63]` | 100 | 69 | 31 | 1 | 0 | 0 | 0 / 0 |
| `unit[20] -> unit[68]` | 84 | 84 | 0 | 1 | 0 | 0 | 0 / 0 |
| `unit[22] -> unit[74]` | 118 | 67 | 51 | 0 | 0 | 0 | 0 / 0 |

结论：

- `+0x604` 在自动阶段暴击样本中同样置 1，不限于玩家手动攻击。
- 当前暂停采集到的自动阶段样本中 `+0x608/+0x614` 均为 0，说明这些段不是连击段、不是反击段；反击样本仍以测验1续前半段的反击链为准。
- `+0x428/+0x42C` 在这些自动阶段样本中为 0，说明它们没有写入我军个人经验/武器 EXP；但不能据此否定我军主动攻击、反击、击杀时会写这些字段。
- `0x00410F1C` log-only hit_count 在本轮自动阶段增长到 `12`，说明它与 `0x00405AD5` 后伤害应用之间有稳定前后关系；内部公式、命中判定和暴击倍率仍待单步解析。

#### 2026-06-11 动态补充：撤退台词后的反击 EXP 上下文

张梁撤退台词触发后，战斗仍继续运行。本轮在后续自动阶段/反击链中再次命中曹操 EXP 缓存写入点，说明 `0x004927F0` 会被剧情台词、自动阶段和反击结算连续复用；Hook 不应把“出现撤退台词”解释为战斗数据结构失效或关卡结束。

本次 `0x00406043` 暂停时可确认：

- `ctx+0x00 = 0`：当前攻击者为曹操 `unit[0]`。
- `ctx+0x01 = 0x3C`：当前目标为 `unit[60]`。
- `ctx+0x0C = 0x004A7B20`：当前攻击者单位指针。
- `unit[60] = 0x004A7B20 + 0x3C * 0x30 = 0x004A8660`，与栈中的目标单位指针一致。
- `ctx+0x84 = 0x24`：本段 HP 伤害缓存为 36。
- `ctx+0x428` 在 `0x00406043` 写入后为 10；随后 `0x00406054` 的 `EAX=2` 将累加武器 EXP。
- `ctx+0x614 = 1`：本次处于反击/嵌套行动标志下。

边界：

- 该样本证明反击/自动阶段会进入曹操 EXP 缓存链，但不证明五维功勋字段位置。
- `0x00406074` 防具 EXP 点在同一链路到达，但本次补读寄存器时 x32dbg MCP 超时；数值语义继续沿用主动攻击样本，不能从本样本新增结论。

#### 原版文档字段对照

来源：`../90-来源归档/原始文档/解读KOEI曹操传代码.docx`，证据等级为原版/旧代码参考。

- 原版文档同样把 `0x004927F0+0x84` 作为 HP 伤害数组、`+0x254` 作为 MP 伤害数组。
- 原版文档把 `+0x428`、`+0x42C`、`+0x430` 分别描述为物理攻击个人经验、武器 EXP 和防具 EXP 记录区，并给出旧版写入点 `0x00406113`、`0x00406153`、`0x00406171`。
- 原版文档把 `+0x604` 描述为致命一击/暴击标志，当前 6.5 已由曹操暴击击杀和自动阶段暴击样本实测支持。
- 原版文档把 `+0x608` 用作攻击循环计数/连击相关字段；当前 6.5 已确认该字段在非连击样本为 0，连击成功后的准确置位时机仍待验证。
- 原版文档把 `+0x614` / `0x00492E04` 作为反击相关标志；当前 6.5 已观察反击执行链，但谓词和计数语义仍待拆分。
