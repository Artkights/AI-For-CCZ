## 详细记录

#### 概述

`4927F0` 是物理攻击过程中所有战斗信息的集中存储区，在游戏代码中经常通过 `MOV ECX, 4927F0` 指令访问，然后通过各种偏移来读取或写入战斗数据。

**⭐ 此结构是两种编号体系的汇聚枢纽**：头部同时存储了**战场编号**（+00/+01）和 **Data号**（+04），并提供了两种指针（+08=人物运行时记录指针, +0C=战术单位指针），是补丁代码中实现"从一种编号找到另一种编号"的关键桥梁。

> 关于两种编号体系的完整说明，参见 [单位结构体.md — 两种编号体系](./单位结构体.md#两种编号体系-)。

#### 数据结构详解

##### 头部区域 (4927F0 - 4927FF)

> 头部是 **两种编号体系交汇的核心**，同时存放战场编号和 Data号，以及对应的两种指针。

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+00` | 1字节 | 字节 | ATTACKER_BATTLE_ID | 攻击方的**战场编号**<br>范围: 00-0E(我军), 0F-22(友军), 23-72(敌军) |
| `+01` | 1字节 | 字节 | CURRENT_TARGET_BATTLE_ID | 当前被攻击方的**战场编号** |
| `+04` | 4字节 | 双字 | ATTACKER_DATA_ID | 攻击方的**Data号**（永久角色编号）<br>512人数据: 000-1FF<br>1024人数据: 000-3FF |
| `+08` | 4字节 | 双字 | ATTACKER_DATA_ADDR | 攻击方人物运行时记录指针；当前 6.5 人物记录由 `[004CEA00] + Data号 * 48H` 定位，可通过 `004061E4/0040658F` 取得 |
| `+0C` | 4字节 | 双字 | ATTACKER_TACTICAL_ADDR | 攻击方战术单位指针<br>等同于 `4061F9(ATTACKER_BATTLE_ID)` |

##### 受攻击武将列表区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+10` | 74字节 | 字节[73] | TARGET_BATTLE_ID_ARRAY | 所有被攻击武将的战场编号数组<br>含穿透攻击中其他受影响武将 |

##### 伤害值区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+84` | 1CC字节 | 双字[73] | HP_DAMAGE_ARRAY | 全体被攻击武将按顺序存放HP伤害值 |
| `+254` | 1CC字节 | 双字[73] | MP_DAMAGE_ARRAY | 全体被攻击武将按顺序存放MP伤害值 |

##### 穿透攻击与经验区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+424` | 1字节 | 字节 | PIERCE_RANGE | 穿透范围 |
| `+425` | 1字节 | 字节 | TARGET_X | 当前被攻击武将X坐标 |
| `+426` | 1字节 | 字节 | TARGET_Y | 当前被攻击武将Y坐标 |
| `+428` | 4字节 | 双字 | PERSONAL_EXP | 攻击者获得的个人经验值 |
| `+42C` | 4字节 | 双字 | WEAPON_EXP | 攻击者获得的武器经验值 |
| `+430` | 1CC字节 | 双字[73] | ARMOR_EXP_ARRAY | 全体被攻击武将获得的防具经验值 |

##### 战斗状态与特效区域

| 偏移 | 大小 | 类型 | 名称 | 说明 |
|------|------|------|------|------|
| `+600` | - | - | AI_ATTACK_FLAG | 是否人工智能攻击 |
| `+604` | 4字节 | 双字 | CRITICAL_HIT_FLAG | 是否发生暴击 (0=无, 非0=是) |
| `+608` | 1字节 | 字节 | DOUBLE_ATTACK_FLAG | 是否连击 (0=无, 非0=是) |
| `+60C` | 4字节 | 双字 | SHAKE_X | 挨打动作播放时位移抖动的横坐标 |
| `+610` | 4字节 | 双字 | SHAKE_Y | 挨打动作播放时位移抖动的纵坐标 |
| `+614` | 4字节 | 双字 | COUNTER_ATTACK_FLAG | 反击以及反击后反击的标志<br>通常也通过492E04访问 |

#### 使用示例

##### 获取攻击者数据指针

```asm
mov ecx, 4927F0         ; ECX = 战斗数据基址
mov ecx, [ecx + 08]    ; ECX = RuntimeCharacter*（48H 人物运行时记录）
```

##### 获取攻击者战术单位指针

```asm
mov ecx, 4927F0
mov ecx, [ecx + 0C]    ; ECX = TacticalUnit*
```

##### 调用点代码中访问

在 [物理攻击.md](../03-机制详解/物理攻击.md) 中提到的43DADA函数：
```asm
43DB53: mov ecx, 4927F0    ; 设置战斗数据基址
43DB58: call 4064F1        ; 写入战斗数据
```

#### 注意事项

1. **数组大小**：所有与被攻击武将相关的数组大小都是 73 (115)，支持最多115个单位
2. **坐标对齐**：+425和+426是连续的两个字节，可作为字读取
3. **攻击者信息**：+08和+0C存储的是指针，可直接用于特技判定系统
4. **常用别名**：492E04是4927F0 + 614的别名，用于快速访问反击标志

#### 策略战斗数据结构 `497AF8`

6.4 文档、论坛旧资料和当前 6.5 静态调用点共同支持：`00497AF8` 是策略施放/结算上下文，不是通用暴击倍率变量。当前 6.5 `0044A5EB` 从 `[00497AF8]` 读取策略号后调用 `00484002`，而 `00484002` 按 `strategy_id * 61H + 004A3E77` 返回策略记录指针。

| 偏移 | 大小 | 推荐语义 | 当前证据 |
|------|------|----------|----------|
| `+00` | 1 字节 | 当前策略号 | 当前 EXE 静态 + 外部资料 |
| `+01` | 1 字节 | 施放者战场编号 | 外部佐证 |
| `+04` | 4 字节 | 施放者人物运行时记录指针 | 外部佐证 |
| `+08` | 4 字节 | 施放者战术单位指针 | 外部佐证 |
| `+0C` | 1 字节 | 当前目标战场编号 | 外部佐证 |
| `+0D` | 目标数组 | 全体目标战场编号，`FF` 结束 | 外部佐证 + 6.5 注入文本 |
| `+84` | DWORD 数组 | 各目标策略伤害 | 外部佐证 |
| `+254` | 1 字节 | 总体策略效果标识 | 外部佐证 |
| `+255` | 字节数组 | 各目标实际效果标识 | 外部佐证 |
| `+2C9/+2CA` | 各 1 字节 | 攻击范围/穿透范围 | 外部佐证 |
| `+2D0/+2D4/+2D8` | DWORD/数组 | 个人、武器、防具 EXP 缓存 | 外部佐证 |
| `+4AC` | 4 字节 | 是否成功使用：`0` 返回策略选择，`1` 成功并结束行动 | 外部佐证，6.5 待动态 |

完整来源、道具上下文和版本护栏见 [战场运行时内存：6.4 资料与 6.5 复核](./战场运行时内存-6.4资料与6.5复核.md)。

#### 2026-06-11 动态补充：物理伤害方向与反击时头部翻转

测验1续的友方/敌方自动阶段确认：`004927F0` 头部描述的是“当前正在结算的伤害方向”，不是固定的“玩家主动发起者”。

- `+00` 当前攻击者战场编号。
- `+01` 当前目标战场编号。
- `+04` 当前攻击者 Data 号。
- `+0C` 当前攻击者战术单位指针。
- 主动攻击、友军自动攻击、敌军自动攻击、反击都会复用同一套字段。
- 反击发生时，`+00/+01` 会按当前反击方向翻转；Hook 代码应以当前字段为准，不要缓存上一段伤害的攻击者/目标后直接复用。

与 HP 应用点的配合：

- 在 `00405AD5`，`[EBP-20]` 是当前目标 `unit*`，`[EBP-14]` 是本段最终伤害，`EAX` 是旧 HP。
- 可用 `([EBP-20] - 004A7B20) / 30` 反推出目标战场编号，并与 `004927F0+01` 交叉校验。
- 经验相关字段 `+428/+42C/+430` 已确认由当前 6.5 物理攻击流程写入战斗上下文；主动攻击样本中的写入点分别为 `00406043`、`00406054`、`00406074`。击杀、反击、连击和最终落盘/升级显示规则仍需分样本验证。

#### 2026-06-11 动态补充：曹操普通攻击 EXP 写入闭环

第二回合曹操 `unit[0]` 普通攻击敌军 `unit[61]` 的样本确认了 `+428/+42C/+430` 的写入时机。该样本未触发连击，`004E3000` 返回 `EAX=0`，`00406520` 未命中。

攻击初始化清零：

- `0040632D`：写 `ctx+428 = 0`，清个人经验缓存。
- `00406333`：写 `ctx+42C = 0`，清武器 EXP 缓存。
- `00406342 -> 0047FD13`：清 `ctx+430` 防具 EXP 数组。

奖励写入：

| 字段 | 写入点 | 计算函数/来源 | 本次值 | 说明 |
|------|--------|---------------|--------|------|
| `+428` | `00406043` | `004037A3` | `10` | 与旧值取较大值后写回 |
| `+42C` | `00406054` | `00403870` | `2` | `add [ctx+42C], eax` |
| `+430+slot*4` | `00406074` | `00403A8A` | `0` | 本次 `slot=0`，无防具 EXP |

调用链：

```text
0040654A
    -> 00452807
        -> 00406043 / 00406054 / 00406074
```

边界：

- 旧版文档线索 `00406113/00406153/00406171` 在本次 6.5 曹操样本中未命中。
- 这些字段是战斗上下文中的本次结算缓存，不等同于角色永久经验、装备永久 EXP 或五维功勋存储字段。
- 仍需拆分验证：击杀、反击、连击第二段、策略伤害、策略击杀和升级显示/落盘。

#### 2026-06-11 动态补充：暴击击杀时的战斗数据字段

第二回合曹操普通攻击触发暴击并击杀 `unit[69]` 的样本确认：

- `+00 = 0`，当前攻击者为曹操 `unit[0]`。
- `+01 = 45`，当前目标为 `unit[69]`。
- `+0C = 004A7B20`，当前攻击者单位指针。
- `+84` 第一个 HP 伤害值为 `44`，与 `00405AD5` 的 `[EBP-14]=44` 一致。
- `+428 = 38`，本次读到的个人经验字段值。
- `+42C = 3`，本次读到的武器 EXP 字段值。
- `+604 = 1`，暴击标志置位。
- `+608 = 0`，本段不是连击。
- `+614 = 0`，本段不是反击。

经验字段写入时机边界：

- 因此这些字段很可能在战斗数据生成/伤害计算阶段早于 `00405AD5` 写入，下一轮应在攻击命令确认前提前布设写断点。

#### 2026-06-11 动态补充：第二次自动阶段字段表现

第二次进入友方/敌方自动阶段后，`00405AD5` 改为暂停采样后又切换为 log-only。当前可人工确认的暂停样本如下：

| 当前伤害方向 | 旧 HP | 伤害 | 新 HP | `+604` | `+608` | `+614` | `+428/+42C` |
|--------------|-------|------|-------|----------|----------|----------|-----------------|
| `unit[24] -> unit[65]` | 27 | 27 | 0 | 0 | 0 | 0 | 0 / 0 |
| `unit[25] -> unit[63]` | 100 | 69 | 31 | 1 | 0 | 0 | 0 / 0 |
| `unit[20] -> unit[68]` | 84 | 84 | 0 | 1 | 0 | 0 | 0 / 0 |
| `unit[22] -> unit[74]` | 118 | 67 | 51 | 0 | 0 | 0 | 0 / 0 |

结论：

- `+604` 在自动阶段暴击样本中同样置 1，不限于玩家手动攻击。
- 当前暂停采集到的自动阶段样本中 `+608/+614` 均为 0，说明这些段不是连击段、不是反击段；反击样本仍以测验1续前半段的反击链为准。
- `+428/+42C` 在这些自动阶段样本中为 0，说明它们没有写入我军个人经验/武器 EXP；但不能据此否定我军主动攻击、反击、击杀时会写这些字段。
- `00410F1C` log-only hit_count 在本轮自动阶段增长到 `12`，说明它与 `00405AD5` 后伤害应用之间有稳定前后关系；内部公式、命中判定和暴击倍率仍待单步解析。

#### 2026-06-11 动态补充：撤退台词后的反击 EXP 上下文

张梁撤退台词触发后，战斗仍继续运行。本轮在后续自动阶段/反击链中再次命中曹操 EXP 缓存写入点，说明 `004927F0` 会被剧情台词、自动阶段和反击结算连续复用；Hook 不应把“出现撤退台词”解释为战斗数据结构失效或关卡结束。

本次 `00406043` 暂停时可确认：

- `ctx+00 = 0`：当前攻击者为曹操 `unit[0]`。
- `ctx+01 = 3C`：当前目标为 `unit[60]`。
- `ctx+0C = 004A7B20`：当前攻击者单位指针。
- `unit[60] = 004A7B20 + 3C * 30 = 004A8660`，与栈中的目标单位指针一致。
- `ctx+84 = 24`：本段 HP 伤害缓存为 36。
- `ctx+428` 在 `00406043` 写入后为 10；随后 `00406054` 的 `EAX=2` 将累加武器 EXP。
- `ctx+614 = 1`：本次处于反击/嵌套行动标志下。

边界：

- 该样本证明反击/自动阶段会进入曹操 EXP 缓存链，但不证明五维功勋字段位置。
- `00406074` 防具 EXP 点在同一链路到达，但本次补读寄存器时 x32dbg MCP 超时；数值语义继续沿用主动攻击样本，不能从本样本新增结论。

#### 原版文档字段对照

来源：`../90-来源归档/原始文档/解读KOEI曹操传代码.docx`，证据等级为原版/旧代码参考。

- 原版文档同样把 `004927F0+84` 作为 HP 伤害数组、`+254` 作为 MP 伤害数组。
- 原版文档把 `+428`、`+42C`、`+430` 分别描述为物理攻击个人经验、武器 EXP 和防具 EXP 记录区，并给出旧版写入点 `00406113`、`00406153`、`00406171`。
- 原版文档把 `+604` 描述为致命一击/暴击标志，当前 6.5 已由曹操暴击击杀和自动阶段暴击样本实测支持。
- 原版文档把 `+608` 用作攻击循环计数/连击相关字段；当前 6.5 已确认该字段在非连击样本为 0，连击成功后的准确置位时机仍待验证。
- 原版文档把 `+614` / `00492E04` 作为反击相关标志；当前 6.5 已观察反击执行链，但谓词和计数语义仍待拆分。
