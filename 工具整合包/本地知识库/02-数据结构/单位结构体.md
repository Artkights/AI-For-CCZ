# 单位结构体

## 结论速览

- 当前 6.5 同时存在战术单位记录、人物运行时记录和策略记录，三者不能混用。
- 战术单位数组为 `004A7B20`，每槽 `30H` 字节，战场编号只用于计算槽位地址。
- `unit+00` 的低 WORD 是 Data 号，`FFFF` 表示空槽；`unit+04` 是出场/显示编号载体，不是 Data 号。
- 人物运行时记录基址来自指针槽 `[004CEA00]`，每人 `48H` 字节；`004061E4` 按 Data 号取人物记录。
- `004A3E77 + id * 61H` 是策略记录表，`00484002` 是策略号转策略记录指针。旧知识库曾把它误写为人物 Data 数组，本页已纠正。
- 当前 HP 是 `unit+10` 的 DWORD，已由 6.5 x32dbg 写断点闭环；当前 MP 是 `unit+14` 的 DWORD，已有多源支持但 setter 仍待动态闭环。

## 适用版本

- 默认适用：当前规范 6.5 未加密 `Ekd5.exe`，SHA256 `F9A357585F4D77C273A8BADD3E4EF4062983CC81D8040C0CB74768FAB7C04F5F`。
- 6.4 文档和旧扳手源码只用于字段对照；升级为 6.5 写入规则的项目必须有当前 EXE 静态证据或动态证据。
- 论坛旧版的 `004B2C50 + id * 24H` 只说明结构演化，不能替代当前 `004A7B20 + id * 30H`。

## 已确认事实

| 对象 | 当前 6.5 定位方式 | 主要用途 | 证据等级 |
|------|-------------------|----------|----------|
| 战术单位 | `004A7B20 + 战场编号 * 30H` | 当前 HP/MP、坐标、阵营、行动、状态 | 静态 + 动态 |
| 人物运行时记录 | `[004CEA00] + Data号 * 48H` | 基础能力、兵种、装备、等级、经验 | 当前 EXE 静态 |
| 策略记录 | `004A3E77 + 策略号 * 61H` | 策略名称、范围、消耗、类别等 | 当前 EXE 静态 + 6.4 资料 |

当前 6.5 的关键函数：

| 地址 | 推荐名称 | 作用 |
|------|----------|------|
| `004061E4` | `data_id_to_runtime_character` | 16 位 Data 号转人物运行时记录指针 |
| `004061F9` | `battlefield_id_to_unit` | 8 位战场编号转战术单位指针 |
| `0040658F` | `unit_to_runtime_character` | 读取 `[unit+00]` 后调用 `004061E4` |
| `0041DF70` | `unit_get_job` | 战术单位转人物记录，再读取兵种 |
| `00484002` | `strategy_id_to_record` | 策略号转 `61H` 策略记录指针 |

## 实现/使用方法

### 两种单位编号

#### Data 号

- 角色在项目数据中的持久编号。
- 当前函数以 16 位无符号值接收，不能只读一个字节。
- 战术单位记录的 `unit+00..+01` 保存该值；`FFFF` 表示该战术槽无单位。
- 需要人物固有属性时，通过 `004061E4` 或 `0040658F` 取得人物运行时记录。

#### 战场编号

- 当前战斗内的临时槽位编号，范围 `0..249`。
- 不要求再从战术单位内部读取，已知编号即可调用 `004061F9`。
- 反击、多目标策略和 AI 阶段会改变“当前攻击者/目标”，应以当下上下文字段为准。

关系如下：

```text
战场编号
  -> 004061F9
  -> TacticalUnit* = 004A7B20 + id * 30H
  -> WORD [unit+00] = Data号
  -> 004061E4
  -> RuntimeCharacter* = [004CEA00] + data_id * 48H
```

策略号走另一条链：

```text
策略号
  -> 00484002
  -> StrategyRecord* = 004A3E77 + strategy_id * 61H
```

### 战术单位数组

```asm
004061F9  push  ebp
004061FC  movzx ecx, cl
004061FF  imul  ecx, ecx, 30
00406202  add   ecx, 004A7B20
00406209  ret
```

计算式：

```text
unit_ptr = 004A7B20 + battlefield_id * 30H
```

### 战术单位字段

| 偏移 | 大小 | 推荐语义 | 证据等级/边界 |
|------|------|----------|---------------|
| `+00` | 2 字节 | Data 号；`FFFF` 空槽 | 当前 EXE + 旧工具源码 |
| `+02` | 2 字节 | Data 号载体高位或扩展字段 | 多处按 DWORD 搬运，高位语义待证 |
| `+04` | 4 字节载体 | 出场/显示编号 | 当前 UI 调用读取 DWORD；不是 Data 号 |
| `+05` | 1 字节 | 阵营：`00` 我、`01` 友、`>=02` 敌 | 多源一致 |
| `+06/+07` | 各 1 字节 | 当前 X/Y | 读取已用；写入函数待“只移动”动态闭环 |
| `+08` | 1 字节 | AI 攻击/跟随目标战场编号 | 外部佐证 |
| `+09/+0A` | 各 1 字节 | AI 移动目标 X/Y | 外部佐证 |
| `+0B` | 1 字节 | AI 行动顺序 | 外部佐证 |
| `+0C` | 1 字节 | 显示/隐藏/退场状态 | 外部佐证 |
| `+0D` | 1 字节 | 行动状态位，`08` 与 S 形象翻转相关 | 当前逻辑 + 旧工具源码；bit 语义未全闭环 |
| `+0E` | 1 字节 | AI/控制策略；`07` 为可控 | 外部佐证 + 旧工具源码 |
| `+0F` | 1 字节 | 朝向 `0/1/2/3 = 上/右/下/左` | 多源一致 |
| `+10` | 4 字节 | 当前 HP | 当前 6.5 动态闭环 |
| `+14` | 4 字节 | 当前 MP | 多源一致；setter 待动态 |
| `+18..+1D` | 6 字节 | 攻、防、精、爆、士、移动临时档位 | 旧工具 + 补丁样本 |
| `+1E` | 1 字节 | 异常状态 | 旧工具 + 补丁样本 |
| `+1F` | 1 字节 | 撤退台词状态 | 6.4 外部佐证；待动态 |
| `+20/+21` | 2 字节 | 限制区域或第二目标坐标 | 来源冲突，不可写 |
| `+22` | 1 字节 | 未知扩展字段 | 待证 |
| `+23` | 1 字节 | 回合扩展/戮力同心计数候选 | 当前回合路径会清零，业务语义待证 |
| `+24..+2F` | 12 字节 | 扩展区 | 未建立字段级规则 |

`+0D` 的当前保守用法：

```asm
; 00406690，按掩码清行动位
movzx eax, byte ptr [ebp+08]
not   eax
and   byte ptr [ecx+0D], al
ret   04
```

- 参数是位掩码，不是位号。
- 旧工具以 `unit+0D |= 06` 标“已行动”，以 `unit+0D &= F8` 清行动相关位。
- `01/02/04/06` 在移动、攻击、待命和二次行动中的准确语义仍需分动作验证。

`+1E` 的 6.4 状态组合以四组 2-bit 状态表达麻痹、禁咒、混乱、中毒。6.5 注入文本直接写过 `C0/30/0C/03`；在没有 setter 和状态显示样本前，新增实现应优先调用游戏状态函数，而不是裸写该字节。

### 人物运行时记录

```asm
004061E4  movzx ecx, word ptr [ebp+08]
004061EB  imul  ecx, ecx, 48
004061EE  add   ecx, dword ptr [004CEA00]
004061F5  ret   04
```

人物记录常用偏移：

| 偏移 | 语义 | 当前状态 |
|------|------|----------|
| `+11..+1A` | 攻、防、精、爆、士 | 6.4 扩展资料；6.5 需逐能力函数复核 |
| `+1B..+1E` | HP 上限 | 外部佐证 |
| `+1F..+20` | MP 上限 | `0040728F` 静态读取支持 |
| `+21..+25` | 武、统、智、敏、运 | 外部佐证 |
| `+2A` | 是否我军 | 旧工具源码 |
| `+2B` | 兵种 | `00406610` 静态确认 |
| `+2C/+2D` | 等级/经验 | 旧工具源码 |
| `+2E..+34` | 武器、防具、辅助装备 | 旧工具源码 |
| `+35` | 击杀数 | 旧工具源码 |

完整字段和跨版本边界见 [战场运行时内存：6.4 资料与 6.5 复核](./战场运行时内存-6.4资料与6.5复核.md)。

### 注入时的取值规则

1. 已有战场编号：调用 `004061F9` 取战术单位指针。
2. 已有战术单位指针：从 `unit+00` 取 16 位 Data 号；不要从 `unit+04` 取 Data 号。
3. 需要人物记录：调用 `0040658F`，或把 Data 号传给 `004061E4`。
4. 需要策略记录：从 `00497AF8+00` 等上下文取得策略号，再调用 `00484002`。
5. 需要当前 HP/MP：读战术单位 `+10/+14`；需要 HP/MP 上限：读人物运行时记录对应字段或调用已验证 getter。

## 风险与边界

- 现有源代码中仍有 `UnitDataIdOffset=0x04`、`get_char_data_ptr=00484002` 等旧命名。本页只修知识库，不修改代码；自动解释器在修正前可能继续输出错误类型。
- `unit+00` 多处以 DWORD 搬运，但当前 Data 号转换函数只读取低 WORD。不要自行解释高 2 字节。
- `unit+04` 虽与战场显示/更新有关，是否总等于槽位编号仍需正负样本；这里只确认它不是 Data 号。
- 战术单位字段是运行时状态。直接改 HP、MP、坐标、退场或状态可能绕过刷新、上限、事件和剧情逻辑。
- 跨版本必须重新确认数组基址、步长、动态人物基址指针槽和 getter 指令签名。

## 待验证项

- 用至少两个 Data 号大于 `255` 的单位动态确认 `unit+00` 低 WORD、`unit+04` 和人物身份，防止再次发生低字节误判。
- 对 `unit+06/+07` 做“只移动”写断点，闭环坐标 setter。
- 对 `unit+14` 做耗 MP、回 MP、上限截断三类写断点，确认 DWORD 宽度和 setter。
- 分别采集移动、攻击、策略、道具、待命、二次行动和新回合，拆清 `unit+0D/+23`。
- 分别构造限制区域、AI 第二目标和戮力同心样本，解决 `unit+20..+23` 的来源冲突。

## 2026-06-11 动态确认：当前 HP 写入闭环

证据等级：当前 6.5 x32dbg live 命中 + 硬件写断点 + 调用栈闭环。场景为曹操攻击黄巾兵并触发连击。

| 项目 | 结论 |
|------|------|
| 战术单位数组基址 | `004A7B20` |
| 单位步长 | `30H` 字节 |
| 当前 HP 字段 | `unit+10`，4 字节 |
| HP setter 入口 | `0043F6DB`，`ECX=TacticalUnit*`、`EDX=new_hp` |
| 最终写入 | `0043F70C: mov dword ptr [ecx+10],edx` |

本次样本：

| 画面角色 | 战场编号 | 单位地址 | 当前 HP 地址 | 观察 |
|----------|----------|----------|--------------|------|
| 曹操 | `0` | `004A7B20` | `004A7B30` | 攻击者，HP 曾读到 `167` |
| 黄巾兵 | `77` | `004A8990` | `004A89A0` | `118 -> 59 -> 15`，连击两次写入 |

边界：本样本只闭环了槽位、步长和当前 HP 写入；人物身份、坐标 setter、当前 MP setter及其它扩展字段不能由此自动外推。
