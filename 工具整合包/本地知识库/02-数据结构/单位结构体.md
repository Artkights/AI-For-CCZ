## 详细记录

#### 概述

曹操传中存在**两套独立**的单位数据结构：

| 结构 | 用途 | 大小 | 基址 | 获取函数 |
|------|------|------|------|----------|
| **战术单位** (Tactical) | 战场运行时数据（HP/位置/状态） | 0x30 | 0x4A7B20 | 0x4061F9 |
| **角色单位** (Character) | 角色固有属性（能力/装备/兵种） | 0x61 | 0x4A3E77 | 0x484002 |

本文档描述**战术单位结构体**。

#### 两种编号体系 ⭐

曹操传引擎中存在**两套独立且互补**的单位编号体系，在战斗中频繁互转：

##### Data号（角色数据号）— 角色"身份证"

| 特性 | 说明 |
|------|------|
| 范围 | **0 ~ 1023**（共 1024 个角色） |
| 分配 | 编译期固定，按角色数据在设计工具中的顺序排列 |
| 生命周期 | **永久不变** —— 曹操永远是同一个 Data号 |
| 作用 | 定位角色的**固有属性**：五维、兵种、装备槽、个人特技、成长率等 |
| 存储 | 角色数据数组 `0x4A3E77`，每个 0x61 字节 |
| 转换函数 | `0x484002`：`CharacterData* = 0x4A3E77 + Data号 × 0x61` |

##### 战场编号（战场编号）— 出战"座位号"

| 特性 | 说明 |
|------|------|
| 范围 | **0 ~ 249**（最多 250 个单位同时在场） |
| 分配 | **每场战斗动态分配**，取决于角色在战场上的部署位置 |
| 生命周期 | **仅单场战斗有效** —— 同一角色在不同战斗中战场编号不同 |
| 作用 | 定位单位的**运行时状态**：HP、MP、位置坐标、行动状态、增益档位等 |
| 存储 | 战术单位数组 `0x4A7B20`，每个 0x30 字节 |
| 转换函数 | `0x4061F9`：`TacticalUnit* = 0x4A7B20 + 战场编号 × 0x30` |

##### 两者关系

```
战场编号 ──→ 战术单位 (TacticalUnit) ──→ unit[...] 读取 Data号
                                              │
                                              ↓
Data号    ──→ 角色数据 (CharacterData) ──→ 装备槽 / 个人特技 / 五维
```

- 战术单位通过内部字段（`unit[0x04]` 附近）存储对应的 **Data号**，从而能追溯到角色的固有属性
- 战斗数据结构 `0x4927F0` 同时存有两套编号（`+0x00`=战场编号, `+0x04`=Data号），是连接二者的枢纽
- 特技判定时：核心引擎收到**战术单位指针**（战场侧），内部读出 **Data号**，转到角色数据检查装备/个人特技

##### 实战：一个完整的特技判定调用链

```
游戏逻辑层 push 4参数 + mov ecx, TacticalUnit*
    ↓
call 0x4101D9 (核心引擎)
    ↓ ECX = 战术单位指针 (战场侧)
    ├── 从战术单位读出 Data号
    ├── call 0x484002: Data号 → CharacterData*
    ├── 装备渠道: 遍历装备槽，比对装备特效号
    └── 个人渠道: 比对个人特技号
    ↓ 返回 EAX = 效果值 (或 0=无)
游戏逻辑层使用返回值
```

#### 内存布局 — 战术单位 (战术单位)

##### 数组基址

```c
// 战术单位数组基址
#define TACTICAL_UNIT_BASE 0x004A7B20

// 战术单位结构体大小
#define TACTICAL_UNIT_SIZE 0x30  // 48字节
```

##### 单位指针计算

```c
// 根据单位索引获取战术单位指针
Unit* get_unit_ptr(byte unit_index) {
    return (Unit*)(TACTICAL_UNIT_BASE + unit_index * TACTICAL_UNIT_SIZE);
}

// 公式
unit_ptr = 0x4A7B20 + unit_index × 0x30
```

##### 角色单位指针计算（参考）

```c
// 角色单位数组基址与大小（用于属性读取）
#define CHAR_UNIT_BASE 0x004A3E77
#define CHAR_UNIT_SIZE 0x61

// 公式
char_ptr = CHAR_UNIT_BASE + character_id × CHAR_UNIT_SIZE
// 实现于 0x484002: imul ecx,ecx,0x61; add ecx,0x4A3E77
```

##### 汇编实现

```asm
; 函数 0x4061F9 - 单位编号转指针
004061F9: push ebp
004061FA: mov ebp, esp
004061FC: movzx ecx, cl              ; 单位编号（1字节）
004061FF: imul ecx, ecx, 0x30        ; × 48（单位结构体大小）
00406205: add ecx, 0x004A7B20        ; + 单位数组基址
0040620B: pop ebp
0040620C: ret
```

#### 结构体字段

##### 已确认字段

| 偏移 | 大小 | 类型 | 说明 |
|------|------|------|------|
| `0x00` | 2字节 | 字 | 单位有效性标记（0xFFFF = 无效） |
| `0x01` | 1字节 | 字节 | **战场编号 (战场编号)** — 战术单位数组索引，值和自身在 `0x4A7B20` 中的槽位一致 |
| `0x04` | 1字节 | 字节 | **Data号 (角色数据号)** — 永久角色编号（0-1023），可追溯到角色数据 `0x4A3E77` |
| `0x05` | 1字节 | 字节 | 阵营标识（0x00=我军, 0x01=友军, 0x02=敌军） |
| `0x06` | 1字节 | 字节 | 大阵营标识（如 魏/蜀/吴 等） |
| `0x07` | 1字节 | 字节 | 子阵营级别（同大阵营内的细分） |
| `0x0D` | 1字节 | 字节 | 行动状态位掩码 |
| `0x10` | 4字节 | 双字 | 当前HP |
| `0x14` | 4字节 | 双字 | 最大HP |
| `0x1E` | 1字节 | 字节 | 状态标志（异常状态） |
| `0x1F` | 2字节 | 字 | MP相关属性 |
| `0x21` | 6字节 | 字节[6] | 五维/移动力属性数组（攻击/防御/精神/爆发/士气/移动） |
| `0x23` | 1字节 | 字节 | 回合相关状态标志 |
| `0x84` | 4字节 | 双字 | 物理伤害输出值（攻击后写入） |
| `0x108` | 4字节 | 双字 | 策略伤害值 |

> **注意**：`0x84` 和 `0x108` 偏移超出战术单位 0x30 字节范围，实际属于**战斗数据结构**（[战斗数据结构.md](./战斗数据结构.md) 中 `0x4927F0` 的 `+0x84` 为 HP_DAMAGE_ARRAY）。在补丁代码中，这些偏移通过函数参数栈帧（如 `[ebp+0x08]`）传入，指向的是战斗数据块而非战术单位。

##### 字段详解

###### 0x0D - 行动状态位掩码

每位代表一种行动类型的可用状态：

| 位 | 含义 |
|----|------|
| 位 0 | 移动能力（0=可用，1=已使用） |
| 位 1 | 攻击能力（0=可用，1=已使用） |
| 位 6 | 二次行动（0=可用，1=已使用） |

**清除操作**：
```asm
; 恢复行动能力
; 函数：0x406690
; 调用约定：__thiscall（ECX = 单位指针）
; 参数：位掩码（要清除的位）

00406690: push ebp
00406691: mov ebp, esp
00406693: movzx eax, byte [ebp+0x08]  ; eax = 参数（位掩码）
00406697: not eax                      ; eax = ~eax（按位取反）
00406699: and byte [ecx+0x0D], al    ; [ecx+0x0D] &= al
0040669C: mov esp, ebp
0040669E: pop ebp
0040669F: ret 0x04

; 公式：unit[0x0D] &= ~参数
; 注意：参数是位掩码，不是位号！
```

**常用参数**：

| 参数 | 二进制 | 清除的位 | 效果 |
|------|--------|----------|------|
| `0x01` | `00000001` | 位 0 | 恢复移动能力 |
| `0x02` | `00000010` | 位 1 | 恢复攻击能力 |
| `0x03` | `00000011` | 位 0 + 位 1 | 恢复移动+攻击 |
| `0x06` | `00000110` | 位 1 + 位 2 | 恢复特殊行动能力 |

###### 0x10 - 当前HP

```asm
; 函数 0x41B500 - 读取当前HP
0041B500: push ebp
0041B501: mov ebp, esp
0041B503: mov eax, [ecx+0x10]        ; 读取单位结构体偏移0x10
0041B506: pop ebp
0041B507: ret
```

###### 0x1E - 状态标志

存储单位的异常状态，使用位编码：

| 位范围 | 含义 |
|--------|------|
| 位 0-1 | 状态类型 |
| 位 2-3 | 状态等级/强度 |
| 位 6-7 | 中毒状态 |

**中毒检测示例**：
```asm
; 参数0x10的匹配过程：
; 迭代0: 0x10>>1=0x08, bit0=0 → 跳过
; 迭代1: 0x08>>1=0x04, bit0=0 → 跳过
; 迭代2: 0x04>>1=0x02, bit0=0 → 跳过
; 迭代3: 0x02>>1=0x01, bit0=1 → 检查 (unit[0x1E]>>6)&0x03
```

###### 0x01 - 战场编号 (战场编号)

战术单位在 `0x4A7B20` 数组中的槽位索引（同时也是自身的战场编号）。通过 `0x4061F9` 可将此编号还原为战术单位指针：

```asm
movzx ecx, byte [ecx+0x01]    ; 读取战场编号 (0-249)
call 0x4061F9                 ; 战场编号 → 战术单位指针
                              ; 公式: (0x4A7B20 + id * 0x30)
```

**典型用法**：补丁中常需从某个结构体（如战斗数据）中取出一个字节的战场编号，然后通过此函数获取完整的战术单位指针，进而读取 HP、阵营、位置等信息。

> **注意**：此字段的值与自身在数组中的位置一致（自引用），主要用于"已知指针但不知编号"的场景，或从其他结构体的 `+0x00`/`+0x01`（仅存战场编号）还原指针。

###### 0x04 - Data号 (角色数据号)

战术单位对应的**永久角色编号**（0-1023）。通过 `0x484002` 可将 Data号 转为角色数据指针：

```asm
movzx ecx, byte [ecx+0x04]    ; 读取 Data号
call 0x484002                 ; Data号 → 角色数据指针
                              ; 公式: (0x4A3E77 + id * 0x61)
```

角色数据中包含五维基础值、兵种、装备槽（武器/防具/辅助）、个人特技号等固有属性。核心引擎 `0x4101D9` 最终通过此路径读取装备特技和个人特技配置。

> **注意**：若 Data号 范围超过 255，此 1 字节字段可能需要与相邻字节组合读取。模组 版本（1024 角色）中实际存储方式需进一步确认。

###### 0x05 - 阵营标识

| 值 | 含义 |
|----|------|
| `0x00` | 我军（玩家控制） |
| `0x01` | 友军（人工智能友方） |
| `0x02` | 敌军 |

在策略偷钱等补丁中用于区分增减逻辑。

###### 0x06 - 大阵营标识

标识单位所属大阵营（如魏/蜀/吴等势力），用于敌潮逆噬等补丁的阵营差判定（`|阵营A - 阵营B| ≤ 1` 视为同系）。

###### 0x07 - 子阵营级别

同大阵营内部的细分标识，与 `0x06` 配合用于严格阵营比对。

###### 0x21 - 五维/移动力属性数组

连续 6 字节的属性数组（偏移 0x21-0x26），用于存储单位的能力增益档位值（0-7档）：

| 子偏移 | 属性 |
|--------|------|
| `+0x00` | 攻击力增益 |
| `+0x01` | 防御力增益 |
| `+0x02` | 精神/智力增益 |
| `+0x03` | 爆发力/敏捷增益 |
| `+0x04` | 士气/运气增益 |
| `+0x05` | 移动力增益 |

```asm
mov al, byte [eax+ecx+0x21]   ; 读取特定属性的档位值
```

###### 0x84 - 物理伤害输出值（战斗数据结构，非战术单位）

攻击结算后写入的最终伤害数值，回MP攻击补丁从此处读取伤害以计算MP恢复量。

###### 0x108 - 策略伤害值（战斗数据结构，非战术单位）

策略攻击结算时写入的伤害数值，策略偷钱补丁从此处读取以计算金钱变动。

###### 0x1F - MP相关属性

存储与MP计算相关的属性值，配合 `0x40728F` 获取最大MP。

#### 单元数组容量

系统支持最多 250 个单位（索引 0-249），新回合重置时遍历所有单位。

```asm
; 重置循环
0044AF7B: call 0x4061F9               ; 获取单位对象
0044AF80: mov byte [ecx+0x23], 0x00   ; 清除单位偏移0x23
0044AF8B: mov byte [eax+0x508B00], 0x00  ; 重置状态
0044AF97: jmp 0x44AF70                ; 继续循环
; ...
0044AF76: cmp cl, 0xFA                ; i < 250 (0xFA)?
```

#### 2026-06-11 测验1动态确认：战场单位 HP 存储与写入闭环

证据等级：x32dbg live 命中 + 硬件写断点 + 调用栈闭环。目标为当前 6.5 未加密 `Ekd5.exe` 的一次实机战斗样本：我军曹操攻击敌军黄巾兵，并由用户确认本次出现连击。

已确认规则：

| 项目 | 结论 |
|------|------|
| 战场单位数组基址 | `0x004A7B20` |
| 单位步长 | `0x30` 字节 |
| 战场单位地址 | `unit_addr = 0x004A7B20 + battle_index * 0x30` |
| 当前 HP 字段 | `unit + 0x10`，4 字节 |
| 当前 HP 地址 | `0x004A7B20 + battle_index * 0x30 + 0x10` |

本次样本：

| 画面角色 | 战场编号 | 单位地址 | 当前 HP 地址 | 本轮观察 |
|----------|----------|----------|--------------|----------|
| 曹操 | `unit[0]` | `0x004A7B20` | `0x004A7B30` | 攻击者，HP 曾读到 `167` |
| 黄巾兵 | `unit[77]` | `0x004A8990` | `0x004A89A0` | 被攻击者，连击导致 HP 两次下降 |

HP 写入函数链：

```asm
0x0043F6DB  unit_set_hp_clamped_entry
0x0043F70C  mov dword ptr [ecx+0x10], edx
```

调用语义：

- `ECX = TacticalUnit*`，即战场单位结构指针。
- `EDX = new_hp`，即写回后的当前 HP。
- `0x0043F70C` 是已确认的当前 HP 最终写入点；它不是伤害公式，也不是连击判定函数。

本次黄巾兵 `unit[77]` 的两次写入：

| 次序 | 地址 | 旧 HP | 新 HP | 差值 |
|------|------|-------|-------|------|
| 第 1 段 | `0x004A89A0` | `118` | `59` | `59` |
| 第 2 段 | `0x004A89A0` | `59` | `15` | `44` |

注意：

- 这里的 `battle_index` 是战场单位槽位编号，不等同于角色 Data 号，也不等同于画面人物名。
- `unit+0x04` 附近的 Data 号字段不能单独用来给当前画面人物命名；必须结合战场上下文、画面观察和本轮调试证据。
- 本次没有动态闭环移动写入函数。坐标字段仍需用移动动作 + `unit` 坐标候选写断点单独验证，不能把 HP 写入链外推为移动链。
