## 详细记录

#### 概述

曹操传的物理攻击体系是一个复杂的多阶段处理流程，涉及初始化、攻击者能力检查、目标检查、伤害计算、伤害应用、战斗数据记录和动画播放等多个环节。整个体系以 `0x43DADA` 为核心调度函数，协同多个子系统完成一次完整的物理攻击流程。

#### 核心函数：0x43DADA

##### 函数签名与入口处理

```
0x43DADA: push ebp
0x43DADB: mov ebp, esp
0x43DADD: sub esp, 0x14              ; 分配20字节栈空间
0x43DAE0: mov [ebp-0x0C], ecx        ; 保存this指针(攻击者单位结构)
0x43DAE3: xor eax, eax                ; eax = 0
0x43DAE5: mov [ebp-0x10], eax        ; 初始化临时变量为0
0x43DAE8: mov al, 0xFF                ; al = 0xFF
0x43DAEA: mov [ebp-0x08], eax        ; [ebp-0x08] = 0xFFFFFFFF
0x43DAED: call 0x4109A1              ; 调用初始化函数
0x43DAF2: mov [ebp-0x04], al         ; 保存初始化结果
```

栈空间布局：

| 偏移 | 用途 |
|------|------|
| `[ebp-0x04]` | 初始化结果/临时状态 |
| `[ebp-0x08]` | 状态标志(0xFFFFFFFF初始值) |
| `[ebp-0x0C]` | ECX保存(攻击者单位指针) |
| `[ebp-0x10]` | 临时变量 |
| `[ebp-0x14]` | 栈分配边界 |


#### 数据传递机制

##### ECX寄存器（攻击者单位指针）

在物理攻击流程中，ECX寄存器始终指向攻击者的单位数据结构。函数调用约定使用 `thiscall`，第一个参数通过ECX传递。

##### 栈参数传递

关键函数使用栈传递额外参数：

| 函数 | 参数1 | 参数2 | 说明 |
|------|-------|-------|------|
| `0x4109A1` | - | - | 无参数，使用ECX |
| `0x43F8C0` | - | - | 无参数，使用ECX |
| `0x45533D` | 目标单位 | - | ECX=this, [ebp+0x08]=目标 |
| `0x410F1C` | 攻击者 | 防御者 | [ebp+0x08], [ebp+0x0C] |
| `0x4363DC` | 单位 | 攻击类型 | [ebp-0x04], [ebp+0x08] |
| `0x4064F1` | 攻击者 | 伤害值 | [ebp+0x08], [ebp+0x0C] |

##### 返回值约定

- `al = 0xFF`：特殊状态（如必杀、无法攻击等）
- `al = 0x00`：正常失败
- `al = 0x01`：成功
- `eax`：复合返回值（如伤害值、指针等）

#### 战斗数据区：0x4927F0

##### 数据结构

战斗数据结构位于 `0x4927F0`，由 `0x4064F1` 函数写入。

##### 写入时机

物理攻击完成后记录战斗数据。

##### 数据内容

记录的信息包括：
- 攻击者ID
- 防御者ID
- 伤害值
- 攻击类型
- 是否必杀
- 攻击结果

#### 状态标志系统

##### 单位状态字节 [ecx+0x0D]

| 位 | 含义 |
|----|------|
| bit 0 | 移动能力（0=可移动，1=已移动） |
| bit 1 | 攻击能力（0=可攻击，1=已攻击） |
| bit 2 | 特殊行动能力 |
| bit 3-7 | 其他标志 |

##### 状态修改时机

在物理攻击流程中，**仅在** `0x442718` 函数中修改行动状态。攻击结束后，bit 1 被设置为1，表示该单位本回合已完成攻击。

##### 恢复行动能力

如果需要恢复行动能力，需要调用 `0x406690` 函数：

```asm
; 公式：unit[0x0D] &= ~参数
; 参数0x01：恢复移动能力
; 参数0x02：恢复攻击能力
; 参数0x03：恢复移动+攻击能力
```

#### 与特技系统的关系

物理攻击体系与特技系统是**独立**的：

1. **物理攻击主流程**（0x43DADA）处理标准的攻击逻辑
2. **特技系统**在特定触发点进行检查，决定是否执行额外效果
3. **常规物理攻击**不会自动恢复行动力

##### 常见物理攻击相关机制

| 机制名称 | 类型 | 功能 | 详细说明 |
|----------|------|------|----------|
| 暴击 | 伤害加成 | 特定条件下伤害提升 | 见 critical-hit.md |
| 连击 | 多段攻击 | 同一目标连续攻击多次 | 见 double-attack.md |
| 引导攻击 | 特技 | 击杀后追击相邻敌人 | 见 chase-attack.md |
| 二次行动 | 特技 | 行动结束后额外行动 | 见 second-action.md |
| 属性下降 | 后伤害效果 | 命中后随机降低目标属性增益 | 见 attribute-reduction-on-hit.md |

##### 特技触发点

在物理攻击流程中，可在以下位置插入特技判定：

1. **0x4363DC 返回后**：伤害应用完成，可检查击杀事件
2. **0x442718 返回后**：动画播放完成，可检查显示相关效果
