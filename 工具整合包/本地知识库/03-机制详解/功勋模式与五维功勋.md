# 功勋模式与五维功勋

## 结论速览

- 功勋模式可在开局 R 场景与许子将对话后选择；进入战场后，点击人物头像可查看该人物功勋。
- 五维功勋包括武力、统率、智力、敏捷、运气。需要提升的能力应提前勾选；获得对应功勋且满足点数条件时，对应能力自动提升 1 点。
- 功勋模式下能力上限为 110。
- 2026-06-09 实机观察：曹操在颍川之战物理攻击张梁后，张梁反击被曹操格挡，战后功勋显示武力、敏捷、统率各增加 1 点。
- 当前已完成 UI 规则和前后快照差分；功勋实际存储地址、写入函数和持久化字段仍需 x32dbg 写断点闭环。

## 适用版本

- 默认适用：曹操传加强版 6.5（未加密）当前项目基底。
- 本轮实机目标：`基底\加强版6.5未加密版\Ekd5.exe`，SHA256 `84E3A1DC085AE6F9900D1E8C388A9CD6766379832DDF51BC7BDF780C6615B4A3`。
- 其它 6.x、刘备传改造版或论坛资料只能作为外部线索，不得直接套用为 6.5 默认规则。

## 已确认事实

### 功勋获取规则

以下规则来自 2026-06-09 用户提供的功勋模式说明图与同场实机观察；在字段写入点未确认前，证据等级为“实机 UI 已确认，内存写入待闭环”。

| 功勋 | 获取条件 |
|------|----------|
| 武力功勋 | 每进行一次物理攻击获得 1 点；无论是否命中均有效，反击也有效。 |
| 统率功勋 | 被物理攻击或被策略攻击每次获得 1 点；无论是否命中均有效。 |
| 智力功勋 | 施放策略或中策略每次获得 1 点；有益法术也有效。 |
| 敏捷功勋 | 双击或格挡每次获得 1 点。 |
| 运气功勋 | 暴击或策略格挡每次获得 1 点。 |
| 击退奖励 | 每击退一个敌军，五种功勋各增加 1 点。 |
| 过关奖励 | 过关后我方单位每项能力获得功勋奖励；剩余回合越多奖励越多，未参战或暂加入人员无法获取。 |
| 吃果 | 只提供对应 BUFF，不增加五维功勋。 |

## 实现/使用方法

- 采证时优先采用“人类操作员 + 调试采证员”流程：调试侧先抓前快照，用户执行一个明确动作，动作结束后立即抓后快照并比较。
- 每次只验证一种事件来源：物理攻击、被物理攻击、格挡、双击、策略使用、被策略命中、暴击、策略格挡、击退。
- 不要把一次复合战斗直接拆成多个已验证结论。本轮“曹操攻击张梁且格挡反击”只能确认 UI 显示的三项增加同时发生；仍需单独构造“不格挡的物理攻击”“只被攻击”“只格挡”等样本拆分写入原因。
- 下一轮定位功勋字段时，对本轮 `+1` 候选地址及其邻近结构设置硬件写入断点或内存写断点，记录 EIP、调用栈、写入前后值和当前战斗事件。

## 风险与边界

- 功勋 UI 规则已经由实机图示和用户观察确认，但具体内存字段尚未闭环，不能作为补丁写入或 EXE 注入依据。
- `004A3E77` 策略记录区本轮无差异与功勋持久化无直接关系；下一轮应采 `[004CEA00]` 指向的人物运行时记录、战后结算、存档结构和 UI 缓冲区。
- 本轮动作同时包含“曹操主动攻击”“张梁反击”“曹操格挡”，不能直接推导每个功勋的唯一写入时机。
- 若后续使用测试副本、内存强制或脚本夹具构造功勋场景，必须与正式基底实机观察分开标注。

### 2026-06-11 Hook 注入口结论

测验1续已经确认物理伤害、反击和击杀派生的可靠观测点，但没有确认五维功勋的实际写入函数。

可用于功勋事件归因的已确认物理侧观测点：

- `00405AD5`：单段物理伤害应用点，可判断当前伤害方向、旧 HP、伤害值和新 HP。
- `00405966`：攻击结果/后伤害结果循环，可观察连击等多段结果。
- `004927F0+00/+01/+04/+0C`：当前攻击者/目标上下文；反击时字段按当前伤害方向翻转。
- `0043F70C`：最终 HP 写入点，只适合做兜底写入监控，不适合作为功勋规则入口。

仍未闭环的功勋/经验点：

- 五维功勋旧候选地址均已证伪或改判：`00492F9C` 是战场输入/选择状态字段，`00492C20` 是防具 EXP 缓存起点，`004B1B1A` 是对话链中 dword `[004B1B18]` 写入的覆盖字节，`00496E79` 位于资源路径字符串缓冲区。五维功勋存储结构必须重新定位。
- 个人经验、武器 EXP、防具 EXP 的战斗上下文缓存已确认写点：`00406043 -> +428`、`00406054 -> +42C`、`00406074 -> +430+slot*4`；但最终角色/装备永久字段、升级显示和五维功勋写入函数仍未确认。
- 击杀奖励、连击奖励、反击奖励、格挡奖励、策略功勋和过关奖励应拆成单因子样本；不能用一次复合战斗直接归因。

### 2026-06-11 证伪记录：00492F9C

暴击击杀样本中，HP setter 返回后重新设置 `00492F9C` 硬件写断点。该断点命中：

- EIP：`0040FB7E`
- 指令：`mov dword ptr [ecx+08], 0`
- `ECX=00492F90`，因此写入地址为 `00492F98` 到 `00492F9B`，覆盖 `00492F9C` 邻近状态结构。
