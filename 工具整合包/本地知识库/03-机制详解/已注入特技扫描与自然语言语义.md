# 已注入特技扫描与自然语言语义

CCZModStudio 把当前 EXE 中的特技分为三层：`Effects` 是供 UI/MCP 使用的统一逻辑特技主集合；`InjectedEffects`、`NativeEffects` 是兼容视图；`Diagnostics` 是普通跳转、wrapper、参数未恢复和 CFG 不完整等线索，不作为已安装特技。主集合只在签名/语义规则一致，或四参数渠道、模式和触发阶段一致时合并，不能只按中文名称合并。

知识库的 17 个注入文本只提供结构签名和语义规则。只有当前 EXE 实际命中后，才允许用于解释已安装状态。

## 四参数 ABI

```text
push 效果值模式
push 叠加方式
push 装备特效号
push 个人特技号
ECX = 当前单位指针
call 004101D9
```

- 效果值模式 `0`：EAX 返回配置的特效值，未拥有时返回 0。
- 效果值模式 `1`：EAX 只返回是否拥有，结果为 0 或 1。
- 叠加方式 `0`：个人与装备渠道允许累加。
- 叠加方式 `1`：不可叠加，只取一个渠道。
- 叠加方式 `2`：优先装备渠道，未命中再回退个人渠道。

## 两个编号空间

- 个人特技号和兵种特效号共用 `6.X-7 兵种特效`名称空间；`6.X-7-2` 提供兵种/武将分配，`6.X-7-3` 提供人物专属和套装绑定。
- 装备/宝物特效号属于独立编号空间，名称优先来自项目侧宝物特效目录，再回退引擎名称表。
- 装备特效号 `00` 表示不使用宝物渠道，不能参与特技命名；个人特技号 `FF` 可以是有效扩展编号，不能全局当作空值。
- 当前注入代码语义与目录名称不一致时，以已安装代码语义解释实际功能，同时保留目录冲突，提示复核人物、兵种或宝物绑定。

## 专属参数语义

四参数只能说明如何判定，不能单独说明业务效果。自然语言解释还必须命中功能体签名或数据读写规则：噬心毒咒的特效值是概率百分比；大杀四方是每回合次数上限；回 MP 攻击是伤害倍率；策略冲锋、聚势伐谋和敌潮逆噬是百分比系数；策略保底/限伤的 `16..30` 按百分比解释，其他值按固定伤害解释；护卫还有范围和台词开关。信息传送 29、整型变量 4003 属于引擎扩展，不是普通四参数特技。

无法同时证明调用 ABI、功能体签名和数据读写时，必须输出“尚未完整解析”，不得根据特技号猜测。

## MCP

```text
scan_installed_effects
read_effect_instance(instance_id)
preview_effect_parameter_update(request)
apply_effect_patch(preview.Package)
```

一个逻辑参数可以对应多个物理参数位。当前值一致时，预览必须同步生成全部写入段，并为每一处建立当前字节锁；当前值不一致时显示“配置不一致”并阻止一键修改。参数修改还必须带 EXE SHA 锁。
