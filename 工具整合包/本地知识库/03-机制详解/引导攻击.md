## 详细记录

#### 概述

**引导攻击**是一种物理攻击特技，其效果为：攻击击杀目标后，继续攻击范围内的其他敌人，最多重复3次。

#### 特技参数

| 参数 | 值 |
|------|-----|
| 装备特效号 | 54 (0x36) |
| 个人特效号 | 24 (0x18) |
| 效果值标志 | 1 (不需要效果值) |
| 叠加标志 | 1 (不可叠加) |

#### 调用链总览

```
游戏行动系统
├── [行动分发] 0x43DEE2 (switch, al==1=物理攻击)
│   └── 调用者4 (0x43DADA) — 目标选择与攻击处理
│
├── [主战斗处理] 0x43B2E0
│   └── 调用者3 (0x43AAF7) — 主编排器
│       ├── [路径A] 调用者1 (0x417C67) — 目标列表构建
│       │   └── 包装函数 (0x4212F1) — 引导循环
│       └── [路径B] 包装函数 (0x4212F1) — 直接调用
│
├── [攻击处理A] 0x41D6ED
│   └── 调用者1 (0x417C67)

```

#### 核心函数详解

##### 桩函数 - 0x4210D0

特技判定入口，向调度函数传递参数。

##### 包装函数 - 0x4212F1

引导攻击的核心循环执行器。

**伪代码**：

```c
int wrapper(Unit* attacker) {
    if (!has_ability(attacker))    // call 0x4210D0
        return 0;
    
    init();                        // call 0x425B12
    target_list = get_list();      // call 0x436499
    
    chain_count = 0;
    while (true) {
        target = *target_list;
        if (target == 0xFF) break;         // 列表结束
        
        execute_attack(target);            // call 0x435829
        chain_count++;
        
        if (chain_count >= 3) break;       // 最多3次
        
        if (!target_is_dead(target)) break; // 目标未死则停止
        
        target_list++;                      // 下一目标
    }
    return chain_count;
}
```

##### 调用者1 - 0x417C67 (目标列表构建器)

遍历全局数组 [0x500E9D] 中的候选目标，验证有效性，标记列表终止符(0xFF)，然后调用包装函数执行引导攻击。

**参数**：

| 参数位置 | 内容 |
|----------|------|
| `[ebp+0x08]` | 参数1（单位编号） |
| `[ebp+0x0C]` | 攻击者单位指针 |
| `[ebp+0x10]` | 目标数量上限 |

##### 调用者2 - 0x423BD7 (轻量查询)

简洁的特技判定函数，仅判断单位是否拥有引导攻击能力。接收一个参数（单位指针），返回判定结果。

##### 调用者3 - 0x43AAF7 (主编排器)

引导攻击系统的顶层调度器，包含双分支决策逻辑。

**流程**：

1. 检查全局标志 [0x505FDC]
2. 检查单位 0x80 标志位
3. 检查游戏状态 != 4
4. 获取角色编号和坐标
5. 坐标相邻判定（找攻击范围内的敌人）
6. 获取游戏子状态 (call 0x43F8C0)

**路径分支**：

| 路径 | 条件 | 处理 |
|------|------|------|
| 路径A | 子状态 ≠ 2/3/4 | 调用 调用者1，构建目标列表并执行引导 |
| 路径B | 子状态 = 2/3/4 | 直接调用包装函数，目标由技能系统预先指定 |

**后处理**（两条路径汇合）：
- 0x438E78 — 效果应用
- 0x442718 — 后处理
- 0x443AC3 — 收尾

##### 调用者4 - 0x43DADA (行动分发入口)

由行动类型分发器在 al==1 时调用，处理物理攻击行动。

**流程**：
1. 通过 0x44533D 获取目标单位
2. 坐标操作 (0x435829)
3. 前置条件检查 (0x4363DC, 0x4061F9, 0x41B500)
4. 调用包装函数 (0x4212F1)
5. 后处理 (0x438E78, 0x442718, 0x443AC3)

#### 死亡判定

##### 判定流程

在包装函数 0x4212F1 的引导循环中，每次攻击后执行：

```asm
00421343: mov ecx, [ebp-0x10]      ; ecx = 目标单位编号
00421346: call 0x004061F9         ; 单位编号 → 单位指针
0042134B: call 0x0041B500         ; 读取 HP
00421350: test eax, eax            ; HP == 0 ?
00421352: jnz 0x00421359          ; HP != 0 → 存活，停止引导
00421354: inc dword [ebp-0x14]    ; HP == 0 → 死亡，继续引导
```

##### 判定逻辑

| HP状态 | 结果 | 行为 |
|--------|------|------|
| HP == 0 | 目标死亡 | 继续引导，攻击下一个目标 |
| HP != 0 | 目标存活 | 引导终止，停止连锁 |

> **注意**：没有额外的"死亡标记"——HP 归零本身就是死亡的唯一判据。
