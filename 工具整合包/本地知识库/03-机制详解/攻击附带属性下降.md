## 详细记录

#### 概述

曹操传中存在一种"攻击命中后使目标随机属性下降"的能力，其个人特效号为 `0C` (12)，装备特效号为 `5E` (94)。该能力的检测点位于 `472D63` 附近。

**关键特征**：该能力虽然仍遵循桩函数→调度函数→核心引擎的判定架构，但其桩函数不在常规的 `421080-421180` 区域，而是**直接内联**在战斗效果处理函数 `472D48` 中。

---

#### 完整调用链

```
后伤害效果处理 472D48 (attacker, target)
        │
        ├── 436392(p1, p2, 0E, attacker, 00): 前置条件检查
        │        │
        │        ├── 返回 ≠ 0 → 跳过特效检查, 直接进入属性下降循环
        │        │
        │        └── 返回 = 0 → 进入特技检测
        │
        ├── ★ 内联桩函数 (472D5F-472D67)
        │   push 01        ; 效果值标志
        │   push 01        ; 叠加标志
        │   push 5E        ; 装备特效号 = 94
        │   push 0C        ; 个人特效号 = 12
        │   call 600127    ; → 调度函数
        │        │
        │        └── 600127(EDX=34E0): 标准调度函数
        │             │
        │             ├── 9BBFE4: 查表验证
        │             └── 4101D9: 核心引擎执行
        │
        ├── 特技未激活 → 直接返回
        │
        └── 特技激活 → 进入属性下降循环 (最多2次)
             │
             ├── 循环延迟器: 47FB13(06)  ★ 随机属性选择
             │
             └── 409821(target, attr_index, 00)  ★ 应用属性下降
                  │
                  └── 414361(target, attr_index, 00, 01)
                       │
                       └── 45AA37(...)  ★ 写入属性 + UI更新
```

#### 调用上下文

该效果由 **[后伤害效果调度器 405966](../04-函数速查/函数速查表.md)** 统一调用。405966 在伤害应用后执行，按顺序调用多个效果处理函数：

```
405966 (后伤害效果调度器)
    ├── 40399B: 伤害效果执行
    ├── 472D48: ★ 属性下降效果（本文档所述）
    ├── 4039F4: 伤害动画/音效
    ├── 436392: 前置条件检查
    └── 472C41: 另一伤害效果检查
```

这就是为什么"回MP攻击"的注入补丁（见 [注入补丁机制](../01-核心引擎/代码空洞与安全补丁.md)）选择在 405966 内部插 Hook——它是所有后伤害效果的统一入口。

#### 入口函数 472D48 详细反汇编

```asm
; 472D48: 后伤害属性下降效果处理
; 参数: [ebp+08] = attacker_ptr (攻击者)
;       [ebp+0C] = target_ptr  (被攻击者)
472D48: push ebp
472D49: mov ebp, esp
472D4B: push 00               ; param5 = 0
472D4D: push ecx                ; param4 = attacker (ecx原值)
472D4E: push 0E               ; param3 = 0E
472D50: push [ebp+08]         ; param2 = attacker
472D53: call 436392           ; ★ 前置条件检查
472D58: test eax, eax
472D5A: jnz 472D75            ; 前置满足 → 跳过特技检查

; ===== 内联桩函数: 检测 5E/0C 特技 =====
472D5C: mov ecx, [ebp-08]     ; ecx = attacker
472D5F: push 01               ; 效果值标志 = 1 (无效果值)
472D61: push 01               ; 叠加标志 = 1 (不可叠加)
472D63: push 5E               ; ★ 装备特效号 = 5E (94)
472D65: push 0C               ; ★ 个人特效号 = 0C (12)
472D67: call 600127           ; → 调度函数 (EDX=34E0)
472D6C: test eax, eax
472D6E: jz 472D92             ; 无特技 → 退出
472D70: jmp 472D75            ; 有特技 → 进入循环

; ===== 属性下降循环 (最多2次) =====
472D72: inc [ebp-04]          ; 循环计数器++
472D75: cmp byte [ebp-04], 02  ; i < 2?
472D79: jnb 472D92            ; i >= 2 → 退出
472D7B: push 06               ; 参数 = 06 (6种属性)
472D7D: call 47FB13           ; ★ 随机选择属性索引 (0-5)
472D82: nop ; nop ; nop         ; 3字节NOP填充
472D85: push 00               ; arg3 = 0 (下降模式)
472D87: push eax                ; arg2 = 随机属性索引
472D88: mov ecx, [ebp+0C]     ; ecx = target_ptr
472D8B: call 409821           ; ★ 应用属性下降
472D90: jmp 472D72            ; 继续循环

472D92: mov esp, ebp
472D94: pop ebp
472D95: ret 08
```

#### 关键子函数详解

##### 1. 前置条件检查 — 436392

```asm
436392: push ebp
436393: mov ebp, esp
436395: mov al, [ebp+08]      ; 参数1
436398: cmp al, 00            ; 参数1 == 0?
43639A: jz 4363BA             ; 是 → 返回0
43639C: cmp al, [ebp+0C]      ; 参数1 == 参数2?
43639F: jnz 4363BA            ; 否 → 返回0
4363A1: mov eax, [ecx]          ; 读取ecx指向的值
4363A3: call 40658F
4363AD: mov edx, [500C37]     ; ★ 读取全局状态
4363B3: call 403599           ; 比对验证
4363B8: jmp 4363BC
4363BA: xor eax, eax            ; 返回0
4363BC: pop ebp
4363BD: ret 08
```

该函数检查 `[500C37]` 全局状态变量。如果全局状态满足条件（如特定回合、特定战场规则等），则自动触发属性下降效果，无需检查特技。

##### 2. 调度函数 — 600127

```asm
600127: push ebp
600128: mov ebp, esp
60012A: push ecx
60012B: mov ecx, [ecx]          ; 获取攻击者单位指针
60012D: mov edx, 34E0         ; ★ 特技数据表偏移
600132: call 9BBFE4           ; 查表验证
600137: test eax, eax
600139: jnz 60014D            ; 未查到 → 返回
60013B: pop ecx
60013C-600145: push [ebp+14/10/C/8]
600148: call 4101D9           ; ★ 核心引擎
60014D: leave
60014E: ret 10
```

**这是标准的调度函数模板**，唯一的差异是 EDX 偏移为 `34E0`（而非常规区域的 35C0、3740 等）。

##### 3. 随机属性选择 — 47FB13

```asm
47FB13: push ebp
47FB14: mov ebp, esp
47FB16: xor eax, eax
47FB18: cmp [ebp+08], 02    ; 参数 >= 2?
47FB1C: jb 47FB2A             ; < 2 → 返回0
47FB1E: call 47FAE7           ; ★ 生成随机数 (0-32767)
47FB23: sub edx, edx            ; edx = 0
47FB25: div [ebp+08]          ; eax / 6, 余数→edx
47FB28: mov eax, edx            ; eax = 余数 (0-5)
47FB2A: pop ebp
47FB2B: ret 04
```

**返回值**：0-5 之间的随机整数，代表属性索引。

##### 4. 随机数生成器 — 47FAE7 (LCG算法)

```asm
47FAE7: mov eax, [48ECD4]     ; ★ 读取种子
47FAEC: imul eax, 41C64E6D    ; 种子 × 41C64E6D
47FAF2: add eax, 3039         ; + 3039
47FAF7: mov [48ECD4], eax     ; ★ 写回种子
47FAFC: and eax, 7FFF0000     ; 取高15位
47FB01: shr eax, 10           ; 右移16位 → 值域 [0, 32767]
47FB04: ret
```

经典线性同余生成器 (LCG)：`seed = seed × 41C64E6D + 3039`

##### 5. 属性下降包装 — 409821

```asm
409821: push ebp
409822: mov ebp, esp
409824: push 01               ; arg4 = 01
409826: push [ebp+0C]         ; arg3 = mode (00 = 下降模式)
409829: push [ebp+08]         ; arg2 = attr_index (0-5)
40982C: push ecx                ; arg1 = target_ptr
40982D: call 414361           ; ★ 核心修改函数
409832: pop ebp
409833: ret 08
```

##### 6. 属性核心修改 — 414361

```asm
; 参数:
;   ecx = target unit ptr
;   [ebp+08] = target unit ptr (via push)
;   [ebp+0C] = attr_index (0-5)
;   [ebp+10] = mode (00 = 下降)
;   [ebp+14] = context flag (01)
414361: push ebp
414362: mov ebp, esp
414364: add ecx, [ebp+0C]     ; ecx = target + attr_index
414367: movzx eax, byte [ecx+18]  ; ★ 读取 [target + attr_index + 18]

; === 下降模式 (mode=0) ===
41436B: cmp [ebp+10], 00
41436F: jz 41437D             ; mode=0 → 下降路径

; === 上升模式 (mode≠0, 此调用不使用) ===
414371: cmp al, 03
414373: jb 414379
414375: mov al, 06            ; 当前值>=3 → 设为6(满)
414377: jmp 414387
414379: mov al, 03            ; 当前值<3 → 设为3

; === 下降模式 ===
41437D: cmp al, 03
41437F: jnbe 414385           ; 当前值 > 3?
414381: xor eax, eax            ; 值 ≤ 3 → 归零 (清除该属性)
414383: jmp 414387
414385: mov al, 03            ; 值 > 3 → 降至3 (降1档)

; === 写入属性 + UI更新 ===
414387: mov ecx, [ebp+08]     ; ecx = target ptr
41438A: push [ecx+04]         ; unit ID
41438D: push [ebp+14]         ; context flag = 01
414390: push eax                ; new_value
414391: push [ebp+0C]         ; attr_index
414394: push [ecx]              ; unit valid flag
414396: call 45AA37           ; ★ 写入属性并更新UI
41439B: pop ebp
41439C: ret 10
```

**属性下降规则**：

| 当前属性值 | 下降后 | 说明 |
|-----------|--------|------|
| `value ≤ 3` | `0` | 该属性增益完全清零 |
| `value > 3` | `3` | 降至下一档（中档） |

---

#### 受影响的属性

`attr_index` 取值 0-5，对应战术单位 `[target + attr_index + 18]` 处的 6 个字节：

| 索引 | 实际偏移 | 待证假设属性 |
|------|----------|----------|
| 0 | `target+18` | 攻击力增益 |
| 1 | `target+19` | 防御力增益 |
| 2 | `target+1A` | 精神/智力增益 |
| 3 | `target+1B` | 爆发力/敏捷增益 |
| 4 | `target+1C` | 士气/运气增益 |
| 5 | `target+1D` | 移动力增益 |

这些属性值范围通常是 0-6（无增益→满增益），下降将减少/清除受影响的增益。

#### 与常规桩函数路径的对比

##### 常规路径 (421080区域)

```
游戏主循环/事件分发
        │
        └── call 4210XX (桩函数在421080区域)
                 │
                 └── call 6000XX (调度函数)
                          │
                          └── call 4101D9 (核心引擎)
```

##### 本能力的路径 (内联桩)

```
后伤害效果处理 472D48
        │
        ├── 436392 前置条件检查
        │        └── 通过 → 直接进入效果
        │
        └── 内联桩代码 (push/push/push/call)
                 │
                 └── call 600127 (调度函数, EDX=34E0)
                          │
                          └── call 4101D9 (核心引擎)
```

##### 差异总结

| 特性 | 421080区域桩函数 | 472D48内联桩 |
|------|-------------------|---------------|
| 位置 | 独立的桩函数 | 嵌入在效果处理函数内 |
| 入口方式 | `call 4210XX` | 函数内部的 `push ... call` |
| 前置条件 | 无（桩函数本身即入口） | 有 (436392 检查) |
| 调度EDX | 各不同 (35C0-3A20) | 34E0 |
| 效果执行 | 由调用者处理 | 在自身函数内完成循环 |

**实质相同点**：调度函数模板完全一致（`mov ecx,[ecx]; mov edx,xxx; call 9BBFE4; call 4101D9`），只是嵌入位置不同。

#### 设计注意事项

如果要在模组中自定义此机制：

1. **修改下降程度**：修改 `41437D-414385` 的条件逻辑（当前是 value>3→降至3，value≤3→归0）
2. **修改随机属性数**：修改 `47FB13` 的参数（当前06）和循环上限（当前02）
3. **添加新能力**：在 `472D48` 模式中添加新的 inline stub + effect loop
4. **修改属性映射**：修改 `414364` 的 `+18` 偏移
