## 详细记录

#### 概述

曹操传中的暴击（也称"必杀"）是一种在物理攻击时触发的高伤害特殊效果。当暴击触发时，攻击伤害会大幅提升。

暴击判定分为两层：
1. **基础暴击判定**：由 `426B33` 函数进行，基于攻击者状态和能力
2. **必定暴击特技**：当基础判定失败时，检查攻击者是否拥有"必定暴击"特技

#### 第一层：基础暴击判定函数 426B33

##### 函数反汇编

```asm
426B33: push ebp
426B34: mov ebp, esp
426B36: call 41B500           ; mov eax, [ecx+10] — 检查HP > 0
426B3B: test eax, eax
426B3D: jz 426B4E            ; HP=0则跳转, 返回al=02
426B3F: push 08               ; 参数 = 08（暴击标志位）
426B41: call 423FE8           ; 位匹配检查 unit[1E] & 08
426B46: test al, al
426B48: jnz 426B50           ; 匹配成功则跳转
426B4A: xor eax, eax            ; 匹配失败, eax=0
426B4C: jmp 426B50
426B4E: mov al, 02            ; HP=0, 返回02
426B50: pop ebp
426B51: ret
```

**423FE8** — 状态位匹配函数：
```asm
423FE8: push ebp
423FE9: mov ebp, esp
423FEB: push 00

423FF1: push edx
...
; 循环移位匹配：检查参数08是否在unit[1E]的位模式中
423FF5: cmp al, 04              ; 最多4次迭代
423FF9: shr [ebp+08], 1         ; 参数右移1位
423FFF: test al, 01             ; 检查最低位
424001: jz 42400D
424003: mov eax, [ebp-08]
424006: and eax, 03             ; 取状态的低2位
424009: test eax, eax
42400B: jnz 424018              ; 匹配成功
42400D: shr [ebp-08], 2         ; 状态右移2位
424011: inc [ebp-04]            ; 计数器++
424014: jmp 423FF2              ; 继续循环
424016: xor eax, eax              ; 匹配失败
```

##### 函数返回值

| 返回值 | 含义 |
|--------|------|
| `al = 00` | 暴击判定失败（HP正常但位不匹配） |
| `al = 02` | 暴击判定失败（HP=0，攻击者已阵亡） |
| `al = 非零` | 暴击判定成功（unit[1E]的位匹配08） |

#### 第二层：必定暴击特技 410F0A

当基础暴击判定（426B33）失败时，游戏会进行第二层判定——检查攻击者是否拥有"必定暴击"特技。

#### 暴击位的设置/清除机制

##### unit[1E] 是一个衍生字段

`unit[1E]` 位于**战术单位结构体**中（30字节结构，基址 4A7B20），而非角色单位结构体（61字节结构，基址 4A3E77）。该字段的值**不是直接写入**的，而是在战斗准备阶段从多个数据源聚合计算得出。

##### 两套单位结构体对比

| 属性 | 战术单位 (Tactical) | 角色单位 (Character) |
|------|-------------------|---------------------|
| 结构体大小 | `30` (48字节) | `61` (97字节) |
| 基址 | `4A7B20` | `4A3E77` |
| 指针公式 | `index × 30 + 4A7B20` | `id × 61 + 4A3E77` |
| 获取函数 | `4061F9` | `484002` |
| 内容 | 战场运行时数据 | 角色固有属性 |

##### 攻击者能力检查流程 43F8C0 详析

在物理攻击执行之前，43F8C0 会检查攻击者的各项状态：

```asm
; 43F8C0 — 攻击者能力与状态检查
43F8C4: call 40658F            ; 获取UI上下文
43F8D7: mov eax, 500E9A
43F8DC: cmp [eax], 02          ; 检查攻击类型=物理攻击
43F8DF: jnz 43F8F5
43F8E1: cmp [eax+01], 00     ; 检查 500E9B != 0
43F8E5: jz 43F8F5
43F8E7: mov cl, [eax+01]       ; 攻击子类型
43F8EA: call 41E918            ; 查表: cl*10 + 4A7510
43F8EF: mov al, [ecx+0C]       ; 读取攻击类型的标志字节
43F8F2: mov [ebp-08], al       ; 暂存到局部变量

; Mod引擎状态检查序列:
43F8FB: call 4D3242(05,00,32)  ; 检查状态A
         → 如果返回非全0 → [ebp-07] = al
43F90C: call 4D25B6(00, attacker)   ; 检查状态B
         → 如果返回非0 → [ebp-06] = 00 (清除)
43F91E: call 4D25CB(00, attacker)   ; 检查状态C
         → 如果返回非0 → [ebp-05] = 00 (清除)
43F931: call 4E4000(al, attacker)     ; 遍历全单位检查
         → 修改 [ebp-08] (攻击目标)

; 最终检查: 攻击者是否被阻止攻击
43F939: push 04               ; ★ 检查 bit 04
43F93E: call 423FE8            ; unit[1E] 的 bit 04?
43F945: jz 43F94E             ; 如果未设置 → 攻击正常
43F947: or eax, -1              ; 如果已设置 → 攻击被阻止
43F94A: mov al, 0B            ; 返回 0B = 攻击不可执行
```

##### 暴击位 (08) 的设置来源（推断）

| 来源 | 触发条件 | 清除条件 |
|------|----------|----------|
| 兵种特性 | 特定兵种天然暴击 | 无（永久） |
| 武器/宝物 | 装备暴击加成物品 | 卸下装备 |
| 攻击类型 | 特定攻击指令自带暴击 | 攻击结束 |
| Buff/法术 | "奋起"/"会心"类法术 | 回合结束/被解除 |

##### 静态分析瓶颈

当前纯静态分析无法确定 `unit[1E]` 的写入代码，原因：
1. 模组引擎函数（4DXXXX/4EXXXX）深度封装，调用链复杂
2. 状态位可能由 4E4000 内部的 4EE0C8 → 4EE09C/4EE060 子函数写入
3. 也可能在战斗初始化阶段（44AF8F 回合重置循环附近）预设

**建议**：在 x64dbg 中对战术单位的 `[unit_ptr + 1E]` 内存地址设置**硬件写入断点**，然后执行一次物理攻击，即可直接捕获写入点。

#### 与连击机制的关系

暴击和连击是独立的判定：
- 基础暴击：通过 unit[1E] 位匹配 + HP检查
- 必定暴击：通过特技桩→调度→核心引擎
- 连击：通过"爆发力"比对（见 连击.md）
- 两种效果可以同时触发，独立计算

#### 设计注意事项

如果要实现自定义暴击系统：

1. **基础暴击修改**：修改 `426B33` 中的判定逻辑

#### 2026-06-11 动态确认：曹操暴击击杀样本

证据等级：x32dbg live 命中 + 寄存器/栈/战斗上下文/HP 写入闭环。场景为第二回合我方可控阶段，曹操普通攻击敌军并触发暴击，用户确认画面发生暴击。

补充结论：

- `004927F0+604` 不只在玩家曹操手动攻击中有效；自动阶段物理攻击发生暴击时也会置 1。
- 上述两段样本中 `+608=0`、`+614=0`，说明暴击标志可独立于连击和反击标志出现。
