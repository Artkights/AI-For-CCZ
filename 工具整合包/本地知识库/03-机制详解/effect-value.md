## 详细记录

#### 概述

**特效值**是特技的"强度数值"，由游戏数据配置决定。例如：MP恢复特技的特效值=10，则每回合恢复10点MP。

#### 效果值标志

桩函数的第一个 push 参数是"效果值标志"，用于控制返回值类型：

| 标志值 | 含义 | 典型用途 | 返回值 |
|--------|------|----------|--------|
| `1` | 不需要效果值 | 纯开关型特技 | 1（有）或0（无） |
| `0` | 需要效果值 | 有强度等级的特技 | 具体数值或0 |

#### 效果值标志对比

| 特性 | 效果值标志=1 | 效果值标志=0 |
|------|--------------|--------------|
| 典型特技 | 二次行动、引导攻击 | MP恢复、HP恢复 |
| 桩函数返回值 | 1（有）或0（无） | 具体数值或0 |
| 调用者用法 | `test eax, eax; jz skip` | `add accumulator, eax` |
| 强度概念 | 无（纯开关） | 有（数值强度） |

#### 特效值传递路径

##### 标准路径（引擎原生）

```
游戏数据（特效值配置）
    ↓
核心引擎 0x4101D9（双渠道判定 + 读取效果值）
    ↓ 返回 EAX = 效果值
调度函数 0x60002A（透传）
    ↓ 返回 EAX = 效果值
桩函数 0x421106（透传）
    ↓ 返回 EAX = 效果值
调用者 0x43CE8F（使用效果值）
    ↓ add [ebp-0x04], eax → 累加到MP恢复量
```

##### 内联直接调用路径（Mod补丁常用 ⭐）

```
游戏数据（特效值配置）
    ↓
代码洞中的内联桩代码:
    push <效果值标志:0>
    push <叠加标志>
    push <装备特效号>
    push <个人特效号>
    call 0x4101D9       ← 直接调用核心引擎
    ↓ 返回 EAX = 效果值
调用者（在代码洞中直接使用效果值）
```

> **注意**：在内联直接调用模式下，效果值通过 `call 0x4101D9` 的返回值（EAX）直接携带，无需经过调度函数透传。效果值标志=0 时返回配置的效果值数值；效果值标志=1 时返回 0/1 布尔值。

#### 叠加标志

桩函数的第二个 push 参数是"叠加标志"，用于控制效果值能否叠加：

| 标志值 | 含义 | 说明 |
|--------|------|------|
| `1` | 不可叠加 | 只取其中一个渠道的效果值 |
| `0` | 可叠加 | 两个渠道的效果值累加 |

**示例**：MP恢复特技的叠加标志=0（可叠加），如果角色自身拥有该特技（个人特效号=1），且装备也提供该特技（装备特效号=0x1B），两个渠道的效果值会叠加。

#### 代码示例：MP恢复特技

##### 桩函数 - 0x421106

```asm
; MP恢复特技判定入口
; 效果值标志=0（需要效果值）
; 叠加标志=0（可叠加）

00421106: push ebp
00421107: mov ebp, esp
00421109: push 0x00        ; 效果值标志 = 0（需要效果值！）
0042110B: push 0x00        ; 叠加标志 = 0（可叠加）
0042110D: push 0x1B        ; 装备特效号 = 27
0042110F: push 0x01        ; 个人特效号 = 1
00421111: call 0x0060002A  ; 调度函数
00421116: pop ebp
00421117: ret
```

##### 调度函数 - 0x60002A

```asm
; 设置特技类型标识 EDX=0x3430，调用包装层和核心引擎

0060002A: push ebp
0060002B: mov ebp, esp
0060002D: push ecx
0060002E: mov ecx, [ecx]           ; 解引用单位指针
00600030: mov edx, 0x3430          ; 特技类型标识
00600035: call 0x009BBFE4          ; 包装层调用（DLL导入）
0060003A: test eax, eax
0060003C: jnz 0x00600050           ; 包装层返回非0 → 跳过核心引擎
0060003E: pop ecx
0060003F: push [ebp+0x14]          ; 效果值标志
00600042: push [ebp+0x10]          ; 叠加标志
00600045: push [ebp+0x0C]          ; 装备特效号
00600048: push [ebp+0x08]          ; 个人特效号
0060004B: call 0x004101D9          ; 核心引擎
00600050: leave
00600051: ret 0x10
```

##### 核心引擎返回值 - 0x4101D9

```asm
; 执行双渠道判定后，通过以下路径返回结果：

00410230: call 0x0041301E          ; 双渠道判定
00410235: mov [ebp-0x08], edx      ; 保存判定结果标志
0041023E: call 0x00413009          ; 获取效果值
00410243: mov [ebp-0x10], eax      ; 保存效果值

; ...（中间可能有循环处理多个效果值）...

004102B7: mov eax, [ebp-0x10]      ; EAX = 效果值
004102BA: mov edx, [ebp-0x08]      ; EDX = 判定结果标志
004102BD: mov esp, ebp
004102BF: pop ebp
004102C0: ret 0x10
```

#### 回合恢复函数 - 0x43CE8F

##### 完整恢复流程

```c
void process_turn_recovery(Unit* unit) {
    // HP恢复
    hp_recovery = percent_calc(base_hp, rate);  // 0x425AD9
    hp_recovery = check_poison(hp_recovery);   // 0x423FA1
    if (hp_recovery > 0) {
        if (unit->hp + hp_recovery > unit->max_hp)
            hp_recovery = unit->max_hp - unit->hp;
    }

    // MP恢复（通过桩函数获取效果值）
    mp_recovery = 0;
    mp_recovery += stub_mp_recovery_1();    // call 0x421106 → 返回效果值
    mp_recovery += stub_mp_recovery_2();    // call 0x4210AC → 返回效果值
    mp_recovery = check_poison(mp_recovery); // 中毒则减半
    if (mp_recovery > 0) {
        max_mp = get_max_mp(unit);           // 0x40728F
        if (unit->mp + mp_recovery > max_mp)
            mp_recovery = max_mp - unit->mp;
    }

    apply_recovery(unit, hp_recovery, mp_recovery);
}
```

##### MP恢复计算（汇编）

```asm
; === MP恢复计算（调用桩函数获取效果值）===
0043CE8C: mov ecx, [ebp-0x1C]      ; 单位指针
0043CE8F: call 0x00421106          ; ★ 调用MP恢复桩函数
0043CE94: add [ebp-0x04], eax      ; ★ EAX=效果值，累加到MP恢复量

; === 另一个MP恢复特技 ===
0043CE9A: call 0x004210AC          ; 个人特效号=0x32, 装备特效号=0x44
0043CE9F: add [ebp-0x04], eax      ; 累加效果值
```

#### 辅助函数

##### 百分比计算 - 0x425AD9

计算 `param1 * param2 / 100`

```asm
00425AD9: push ebp
00425ADA: mov ebp, esp
00425ADC: mov eax, [ebp+0x0C]      ; param2
00425ADF: imul eax, [ebp+0x08]     ; param1 * param2
00425AE3: xor edx, edx
00425AE5: mov ecx, 0x64            ; 100
00425AEA: div ecx                  ; (param1 * param2) / 100
00425AEC: pop ebp
00425AED: ret 0x08
```

##### 中毒检测 - 0x423FA1

检测单位是否处于中毒状态，中毒则恢复量减半。

```asm
00423FA1: push ebp
00423FA2: mov ebp, esp
00423FA4: push 0x10                ; 参数=0x10，对应中毒状态位
00423FA6: call 0x00423FE8          ; 检查单位状态标志
00423FAB: test eax, eax
00423FAD: jz 0x423FB2              ; 未中毒 → 不调整
00423FAF: shr [ebp+0x08], 1        ; 中毒 → 恢复量减半
00423FB2: mov eax, [ebp+0x08]
00423FB5: pop ebp
00423FB6: ret 0x04
```

##### 状态标志检测 - 0x423FE8

通用状态标志检测函数，检查单位偏移0x1E的特定位是否置位。

```asm
00423FE8: push ebp
00423FE9: mov ebp, esp
00423FEB: push 0x00                    ; 循环计数器=0
00423FED: movzx edx, byte [ecx+0x1E] ; 读取单位状态标志
; ... 更多逻辑见 unit-structure.md
```

**参数0x10的匹配过程**：
- 迭代0: 0x10>>1=0x08, bit0=0 → 跳过
- 迭代1: 0x08>>1=0x04, bit0=0 → 跳过
- 迭代2: 0x04>>1=0x02, bit0=0 → 跳过
- 迭代3: 0x02>>1=0x01, bit0=1 → 检查 (unit[0x1E]>>6)&0x03

##### 获取最大MP - 0x40728F

```asm
0040728F: push ebp
00407290: mov ebp, esp
00407293: push 0x06                ; 属性索引 = 6（MP）
00407295: call 0x00407C35          ; 获取基础属性
0040729E: movzx eax, word [ecx+0x1F] ; 读取单位偏移0x1F
004072A3: push 0x1E                ; 参数
004072A5: call 0x0040D0CF          ; 计算最终MP值
```
