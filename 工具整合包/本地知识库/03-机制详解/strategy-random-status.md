## 详细记录

#### 概述

本补丁实现了一个新特技：持有该特技的角色在**策略攻击**命中并造成伤害后，有一定概率使目标陷入**随机异常状态**（中毒/麻痹/禁咒/混乱之一），触发概率由特技的**特效值**控制（如特效值=50则50%触发）。

| 属性 | 值 |
|------|-----|
| 个人特效号 | `0xC0` (192) |
| 装备特效号 | `0x00` (不可通过装备获取) |
| 注入点 | `0x4259AF` |
| Code Cave | `0x41A627` |
| 兼容性 | 与策略偷钱补丁兼容（注入点在其之前） |

#### 注入策略

```
策略攻击结算流程
    │
0x4259A7: call 0x41E5C2              ; 策略相关处理
0x4259AC: mov ecx, [ebp-0x04]        ; ecx = 策略数据结构
0x4259AF: ★ jmp 0x41A627             ; → 本补丁注入（替换原 call 0x422F2E）
    │
    └── 0x41A627: Code Cave 逻辑
        ├── 校验策略伤害值 > 0
        ├── 调用 0x4101D9 核心引擎（效果值标志=0，获取特效值）
        ├── 特效值%概率判定 via 0x47FB13
        ├── 随机选择异常状态 (0-3)
        ├── 写入目标 unit[0x1E] 状态字节
        ├── call 0x422F2E（执行原被替换的调用）
        └── jmp 0x4259B4（回到原流程，若策略偷钱已安装则链入）
```

#### 关键数据结构

`[ebp-0x04]` 指向策略战斗数据结构：

| 偏移 | 内容 |
|------|------|
| `+0x01` | 攻击方战场编号 |
| `+0x0D` | 目标方战场编号 |
| `+0x108` | 策略伤害值 |

#### 异常状态位编码

状态写入 `unit[0x1E]`，使用位掩码：

| 状态 | 索引 | 掩码 | 位 |
|------|------|------|-----|
| 中毒 | 0 | `0xC0` | bit 6-7 |
| 麻痹 | 1 | `0x30` | bit 4-5 |
| 禁咒 | 2 | `0x0C` | bit 2-3 |
| 混乱 | 3 | `0x03` | bit 0-1 |

#### Code Cave 反汇编

```asm
0x41A627: mov ecx, [ebp-0x04]          ; ECX = 策略数据结构
0x41A62A: cmp dword [ecx+0x108], 1     ; 策略伤害 >= 1?
0x41A631: jl .trampoline                ; 无伤害 → 跳过

0x41A637: movzx ecx, byte [ecx+0x01]   ; 攻击方战场编号
0x41A63B: call 0x4061F9                ; → 战术单位指针
0x41A640: push 0x00                    ; 效果值标志 = 0 (需要特效值!)
0x41A642: push 0x01                    ; 叠加标志 = 1 (不可叠加)
0x41A644: push 0x00                    ; 装备特效号 = 0x00
0x41A646: push 0xC0                    ; 个人特效号 = 0xC0
0x41A64B: call 0x4101D9                ; → 核心引擎判定
0x41A650: test eax, eax
0x41A652: jz .trampoline               ; EAX=0 → 无特技

0x41A658: mov ebx, eax                 ; EBX = 特效值 (概率%)
0x41A65A: push 0x64                    ; 参数 = 100
0x41A65C: call 0x47FB13                ; → 随机数 0-99
0x41A661: cmp eax, ebx
0x41A663: ja .trampoline               ; 随机值 > 概率 → 不触发

0x41A669: mov ecx, [ebp-0x04]          ; 重新获取结构
0x41A66C: movzx ecx, byte [ecx+0x0D]   ; 目标方战场编号
0x41A670: call 0x4061F9                ; → 目标战术单位指针
0x41A675: push 0x04                    ; 参数 = 4
0x41A677: call 0x47FB13                ; → 随机状态索引 0-3

0x41A67C: mov dl, [ecx+0x1E]           ; 读取当前状态
0x41A67F: cmp eax, 0
0x41A682: jz .poison
0x41A684: cmp eax, 1
0x41A687: jz .paralysis
0x41A689: cmp eax, 2
0x41A68C: jz .silence
0x41A68E: or dl, 0x03                  ; 混乱
0x41A691: jmp .write
.poison:
0x41A693: or dl, 0xC0                  ; 中毒
0x41A696: jmp .write
.paralysis:
0x41A698: or dl, 0x30                  ; 麻痹
0x41A69B: jmp .write
.silence:
0x41A69D: or dl, 0x0C                  ; 禁咒
.write:
0x41A6A0: mov [ecx+0x1E], dl           ; 写入状态

.trampoline:
0x41A6A3: mov ecx, [ebp-0x04]          ; 恢复 ECX
0x41A6A6: call 0x422F2E                ; 执行原被替换的调用
0x41A6AB: jmp 0x4259B4                 ; 回到原流程
```
