# 6.5 特效注入索引

## 结论速览

- 本页集中整理 `D:\Downloads` 中 17 个 6.5 注入补丁文本，作为 6.5 特效专栏的入口。
- 注入文本格式统一为 `.1.0.6.5`、`o`、`-地址`、十六进制字节；`#...号` 后面的单字节地址通常就是可改参数。
- 多数新增特效使用内联桩：先设置 `ECX=单位指针`，再按 `效果值标志/叠加标志/装备号/个人号` 顺序 `push`，最后调用 `0x4101D9`。
- 已发现原样冲突：两个“回MP攻击”文件互为变体，不能同时应用；“护卫-最终版”和“殭屍大法”主代码区重叠，不能原样同时应用。

## 适用版本

- 默认适用：曹操传加强版 6.5 未加密基底。
- 这些补丁以 `.1.0.6.5` 标识版本；未针对 6.6/6.6x 做迁移验证。

## 已确认事实

- 已按 UTF-8 读取并解析全部本地注入文本，地址块、字节长度和可改参数地址来自本地文件复读。
- 功能说明主要来自补丁文件注释；部分变量和结构含义结合现有知识库与机器码上下文归纳。
- 本页确认的是“补丁文本结构”和“参数位置”，不是已经对当前 `Ekd5.exe` 完成实机安装验证。

## 实现/使用方法

- 应用前先备份目标 `Ekd5.exe`，确认目标是 6.5 未加密基底。
- 修改特效号时优先改补丁末尾的单字节参数块，例如 `-4529BF` 下的 `BF`；不要在主代码块里手工找同一字节乱改。
- 同一地址若先在主代码块出现、后面又出现单字节参数块，以后面的参数块作为最终配置值。
- `68 xx 00 00 00` 是 32 位立即数 `push`，通常可把 `xx` 改到 `00-FF`；`6A xx` 是带符号 8 位立即数 `push`，一般只安全用于 `00-7F`。
- 策略保底、策略限伤这类使用 `FF` 的特效号，依赖补丁使用 `68 FF 00 00 00` 编码；不要改成 `6A FF`。
- 叠加多个补丁时先查本页“冲突与链式关系”，再用反汇编或二进制复读确认目标地址未被其它补丁覆盖。

## 风险与边界

- 大杀四方、护卫、策略保底/限伤属于多入口或大段逻辑补丁，风险高于单点内联桩。
- 部分补丁直接改策略伤害计算链 `0x43C2xx` 或策略结算链 `0x4259xx`，多个策略补丁叠加时必须确认回跳地址和 Hook 顺序。
- `0x41AA98-0x41AB9E` 与 `0x41AB00-0x41AB2F` 重叠，护卫与殭屍大法原样不能共存，需要重新分配代码洞或合并逻辑。
- 本页不把这些补丁视为当前基底已安装状态；这里只记录可复用样本和参数格式。

## 详细记录

#### 通用注入格式

```text
.1.0.6.5
o

-<虚拟地址>
<空格分隔的十六进制字节>

#参数说明
-<参数地址>
<单字节或多字节值>
```

- `-43C2B0` 这类地址按 6.5 未加密基底的虚拟地址口径记录。
- 多字节立即数和地址采用 x86 小端序，例如 `CF 94 42 00` 表示写入指针 `0x004294CF`。
- 单字节参数块通常覆盖主代码块里的默认字节，方便不用重算机器码即可换特效号。

#### 内联桩参数格式

常见机器码模式：

```asm
mov ecx, <单位指针>
push <效果值标志>      ; 00=返回特效值，01=只返回是否拥有
push <叠加标志>        ; 00=可叠加，01=不可叠加
push <装备特效号>
push <个人特效号>
call 0x4101D9
```

实际补丁中常见字节：

| 字节 | 含义 |
|------|------|
| `6A 00` | `push 0`，常用于需要特效值或装备号 0 |
| `6A 01` | `push 1`，常用于不可叠加或纯开关 |
| `68 AD 00 00 00` | `push 0xAD`，用于个人号大于 `0x7F` 的安全编码 |
| `E8 xx xx xx xx` | `call rel32`，调用目标需按当前地址重新计算 |
| `E9 xx xx xx xx` | `jmp rel32`，Hook 和回跳常用 |

#### 冲突与链式关系

| 关系 | 结论 |
|------|------|
| 回MP攻击 171B 版 vs 136B 版 | 两者 Hook 地址和代码区相同，必须二选一。171B 版覆盖 `0x418335` 7 字节，136B 版覆盖 5 字节。 |
| 护卫 vs 殭屍大法 | `护卫` 主代码区 `0x41AA98-0x41AB9E` 覆盖 `殭屍大法` 的 `0x41AB00-0x41AB2F`，原样冲突。 |
| 噬心毒咒 vs 策略偷钱 | 设计上可链式共存：噬心从 `0x4259AF` 跳入代码洞后回到 `0x4259B4`，策略偷钱正好 Hook `0x4259B4`。 |
| 策略冲锋/聚势伐谋/策略保底限伤/无视策略减伤 | 都在策略伤害计算链附近，地址不重叠，但属于同一计算阶段，安装后必须验证伤害顺序。 |
| 大杀四方 vs 二次行动机制 | 使用 `0x412D79/0x412D7B` 附近和 `0x508B00` 状态数组，需和已有二次行动补丁共同复读。 |

#### 单项补丁要点

##### 盛气凌人

- Hook `0x472D92` 改为 `jmp 0x4529A7`，代码洞末尾执行原 epilogue。
- 判定 `0x4927F0+0x604 != 0` 才继续。
- 内联桩默认 `装备=0x00`、`个人=0xBF`、纯开关。
- 循环属性索引 `0-5`，对目标调用 `0x409821(attr_index, 0)`。

##### 噬心毒咒 v3

- Hook `0x4259AF`，代码洞 `0x41A627`。
- 默认 `装备=0x00`、`个人=0xC0`，需要特效值，特效值按百分比解释。
- v3 版本遍历所有策略受击目标，跳过 `0xFF` 结束符和无伤害目标。
- 随机状态写入 `unit+0x1E`：中毒 `0xC0`、麻痹 `0x30`、禁咒 `0x0C`、混乱 `0x03`。

##### 大杀四方

- 属于高复杂度补丁：物理击杀、策略击杀、回合重置、行动恢复链均有改动。
- 默认 `装备=0x00`、`个人=0xC1`，特效值 N 控制每回合最多触发 N 次。
- 样本使用 `0x4D3500` 起的大段扩展区和 `0x508B00` 状态数组；安装前必须查是否已有其它扩展占用 `0x4D35xx`。

##### 4003 保底伤害

- Hook `0x43BD58`，新增逻辑读取 `0x505E8C`。
- 注释口径为“整型变量 4003=X，则物理攻击最小伤害=攻击力×X%”。
- 机器码使用 `movzx ecx, byte ptr [0x505E8C]`，实际只读取低 1 字节。
- 若 `X=0` 则跳过保底；若原伤害已经高于保底，保留原伤害。

##### 多多益善

- Hook `0x43F936`，代码洞 `0x441D1C`。
- 默认 `装备=0x00`、`个人=0xB3`，纯开关。
- 代码内含攻击范围编号映射表，按周围友军数量逐级推进，到同类范围上限不再增加。

##### 回MP攻击

- 两个文件都是“物理攻击后获得当前伤害×特效值的 MP”，默认 `装备=0x00`、`个人=0xAD`。
- 171B 版覆盖 `0x418335` 后补 `90 90`，代码洞到 `0x4529A6`。
- 136B 版只覆盖 `0x418335` 5 字节，代码洞到 `0x452983`。
- 选择哪一版必须以当前基底 `0x418335` 附近原指令长度和回跳地址为准。

##### 殭屍大法

- Hook `0x406006`，代码洞 `0x41AB00`。
- 默认 `装备=0x00`、`个人=0xB2`，纯开关。
- 逻辑在 MP 防御结算中把“拟损失 MP”限制到不超过当前 HP。
- 与护卫最终版原样冲突。

##### 策略冲锋

- Hook `0x43C2B0`，代码洞 `0x445F6A`。
- 默认 `装备=0x00`、`个人=0xB0`，需要特效值。
- 读取 `0x500C48` 作为策略攻击前移动步数，增伤为 `步数 × K%`。

##### 聚势伐谋

- Hook `0x43C2B5`，代码洞 `0x44616C`。
- 默认 `装备=0x00`、`个人=0xB1`，需要特效值。
- 统计被策略攻击者周围同阵营人数，并包含被攻击者自身，增伤为 `(人数+1) × K%`。

##### 敌潮逆噬

- Hook `0x43CB4C`，代码洞 `0x43D307`。
- 默认 `装备=0x00`、`个人=0xB4`，需要特效值。
- 通过坐标差和阵营判断周围敌军数量，再按人数与 K 调整数值。

##### 策略偷钱

- Hook `0x4259B4`，代码洞 `0x41A5A7`。
- 默认 `装备=0x00`、`个人=0xAA`，纯开关。
- 读取策略伤害并写全局金钱 `0x4B077C`；上限 `999999`，下限 `0`。

##### 强化攻击穿透

- 直接改写 `0x407831` 一段逻辑，并在 `0x41AA4F` 放置辅助函数和映射表。
- 默认 `装备=0x00`、`个人=0xDD`，纯开关。
- 辅助函数读取 `0x505FDC` 和特技判定结果，返回穿透相关字节。

##### 策略保底与策略限伤

- Hook `0x43C2D5`，桥接代码 `0x43C528` 调用 `0x43D3AE`。
- 保底默认 `装备=0x00`、`特技=0xFF`；限伤默认 `装备=0x00`、`特技=0xFF`。
- 特效值 `16-30` 按百分比解释，其它范围按固定伤害解释。
- 同时存在时限伤优先；保底已满足时增伤 20%，限伤未达到时减伤 20%；策略暴击会继续放大结果。

##### 无视策略减伤

- 改写 `0x43C242`、`0x43C266`，代码洞 `0x43D0D6`。
- 默认 `装备=0x00`、`个人=0xEF`，纯开关。
- 用于绕过“减少策略伤害 X%”类效果。

##### 信息传送 29 根据兵种指定 S

- `0x47D953` 写入 `CF 94 42 00`，把处理入口指向 `0x4294CF`。
- 这不是个人/装备特效补丁，属于信息传送命令扩展。
- 代码中把计算出的 S 写入 `0x501000` 相关表项；具体命令参数仍需回到剧本命令页验证。

##### 护卫最终版

- 多处 Hook：`0x4105C8`、`0x43ACA2`、`0x43DB3E`、`0x410F23`。
- 默认 `装备=0x00`、`个人=0xD1`。
- `-41AAE4` 控制生效范围，按攻击范围序号解释：`00=四向`，`01=八向`。
- `-41AB79` 控制是否说话：`00=不说话`，`01=说话`。
- `-41AFE0` 为触发提示语，使用游戏文本编码并以 `00` 结尾；长度可调但必须保留结束字节。
- 文件注释说明：目标被围攻时无效，混乱时无效，特效值不等于 1 时护卫者必须在敌人攻击范围内才触发。
