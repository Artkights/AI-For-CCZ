﻿﻿# 6.5 特效注入索引

## 结论速览

- 本页集中整理当前工作区 `特效整理/6.5` 中 17 个 6.5 注入补丁文本，作为 6.5 特效专栏的入口。
- 注入文本格式统一为 `.1.0.6.5`、`o`、`-地址`、十六进制字节；`#...号` 后面的单字节地址通常就是可改参数。
- 多数新增特效使用内联桩：先设置 `ECX=单位指针`，再按 `效果值标志/叠加标志/装备号/个人号` 顺序 `push`，最后调用 `4101D9`。
- 已发现原样冲突：两个“回MP攻击”文件互为变体，不能同时应用；“护卫-最终版”和“殭屍大法”主代码区重叠，不能原样同时应用。

## 适用版本

- 默认适用：曹操传加强版 6.5 未加密基底。
- 这些补丁以 `.1.0.6.5` 标识版本；未针对 6.6/6.6x 做迁移验证。

## 实现/使用方法

- 应用前先备份目标 `Ekd5.exe`，确认目标是 6.5 未加密基底。
- 修改特效号时优先改补丁末尾的单字节参数块，例如 `-4529BF` 下的 `BF`；不要在主代码块里手工找同一字节乱改。
- 同一地址若先在主代码块出现、后面又出现单字节参数块，以后面的参数块作为最终配置值。
- `68 xx 00 00 00` 是 32 位立即数 `push`，通常可把 `xx` 改到 `00-FF`；`6A xx` 是带符号 8 位立即数 `push`，一般只安全用于 `00-7F`。
- 策略保底、策略限伤这类使用 `FF` 的特效号，依赖补丁使用 `68 FF 00 00 00` 编码；不要改成 `6A FF`。
- 叠加多个补丁时先查本页“冲突与链式关系”，再用反汇编或二进制复读确认目标地址未被其它补丁覆盖。
- 取人物运行时记录时使用 `004061E4/0040658F`；`00484002` 是策略号转 `61H` 策略记录，不得再作为人物 Data 转换函数。
- 物理、策略、道具上下文分别从 `004927F0`、`00497AF8`、`00497750` 进入，完整字段与证据分级见 [战场运行时内存复核](../02-数据结构/战场运行时内存-6.4资料与6.5复核.md)。

## 风险与边界

- 大杀四方、护卫、策略保底/限伤属于多入口或大段逻辑补丁，风险高于单点内联桩。
- 部分补丁直接改策略伤害计算链 `43C2XX` 或策略结算链 `4259XX`，多个策略补丁叠加时必须确认回跳地址和 Hook 顺序。
- `41AA98-41AB9E` 与 `41AB00-41AB2F` 重叠，护卫与殭屍大法原样不能共存，需要重新分配代码洞或合并逻辑。
- 本页不把这些补丁视为当前基底已安装状态；这里只记录可复用样本和参数格式。
- 当前工具源码仍把 `00484002` 和 `unit+04` 赋予了错误人物语义。本轮按要求只改知识库、不改代码；在工具实现修正并重跑烟测前，新行为草案不能依赖这两项自动解释。
- 若“不改动代码”指不改 CCZModStudio 源码，已登记数据、直接参数和已有适配器仍可通过现有工具链预览/写入；若指不改 `Ekd5.exe` 代码字节，则不能新增行为分支，只能配置原生表项、绑定和参数。

## 详细记录

#### 通用注入格式

```text
.1.0.6.5
o

-<虚拟地址>
<空格分隔的十六进制字节>

#参数说明
-<参数地址>
<单字节或多字节值>
```

- `-43C2B0` 这类地址按 6.5 未加密基底的虚拟地址口径记录。
- 多字节立即数和地址采用 x86 小端序，例如 `CF 94 42 00` 表示写入指针 `004294CF`。
- 单字节参数块通常覆盖主代码块里的默认字节，方便不用重算机器码即可换特效号。

#### 内联桩参数格式

常见机器码模式：

```asm
mov ecx, <单位指针>
push <效果值标志>      ; 00=返回特效值，01=只返回是否拥有
push <叠加标志>        ; 00=可叠加，01=不可叠加
push <装备特效号>
push <个人特效号>
call 4101D9
```

实际补丁中常见字节：

| 字节 | 含义 |
|------|------|
| `6A 00` | `push 0`，常用于需要特效值或装备号 0 |
| `6A 01` | `push 1`，常用于不可叠加或纯开关 |
| `68 AD 00 00 00` | `push AD`，用于个人号大于 `7F` 的安全编码 |
| `E8 xx xx xx xx` | `call rel32`，调用目标需按当前地址重新计算 |
| `E9 xx xx xx xx` | `jmp rel32`，Hook 和回跳常用 |

#### 冲突与链式关系

| 关系 | 结论 |
|------|------|
| 回MP攻击 171B 版 vs 136B 版 | 两者 Hook 地址和代码区相同，必须二选一。171B 版覆盖 `418335` 7 字节，136B 版覆盖 5 字节。 |
| 护卫 vs 殭屍大法 | `护卫` 主代码区 `41AA98-41AB9E` 覆盖 `殭屍大法` 的 `41AB00-41AB2F`，原样冲突。 |
| 噬心毒咒 vs 策略偷钱 | 设计上可链式共存：噬心从 `4259AF` 跳入代码洞后回到 `4259B4`，策略偷钱正好 Hook `4259B4`。 |
| 策略冲锋/聚势伐谋/策略保底限伤/无视策略减伤 | 都在策略伤害计算链附近，地址不重叠，但属于同一计算阶段，安装后必须验证伤害顺序。 |
| 大杀四方 vs 二次行动机制 | 使用 `412D79/412D7B` 附近和 `508B00` 状态数组，需和已有二次行动补丁共同复读。 |

#### 单项补丁要点

##### 盛气凌人

- Hook `472D92` 改为 `jmp 4529A7`，代码洞末尾执行原 epilogue。
- 判定 `4927F0+604 != 0` 才继续。
- 内联桩默认 `装备=00`、`个人=BF`、纯开关。
- 循环属性索引 `0-5`，对目标调用 `409821(attr_index, 0)`。

##### 噬心毒咒 v3

- Hook `4259AF`，代码洞 `41A627`。
- 默认 `装备=00`、`个人=C0`，需要特效值，特效值按百分比解释。
- v3 版本遍历所有策略受击目标，跳过 `FF` 结束符和无伤害目标。
- 随机状态写入 `unit+1E`：中毒 `C0`、麻痹 `30`、禁咒 `0C`、混乱 `03`。

##### 大杀四方

- 属于高复杂度补丁：物理击杀、策略击杀、回合重置、行动恢复链均有改动。
- 默认 `装备=00`、`个人=C1`，特效值 N 控制每回合最多触发 N 次。
- 样本使用 `4D3500` 起的大段扩展区和 `508B00` 状态数组；安装前必须查是否已有其它扩展占用 `4D35XX`。

##### 4003 保底伤害

- Hook `43BD58`，新增逻辑读取 `505E8C`。
- 注释口径为“整型变量 4003=X，则物理攻击最小伤害=攻击力×X%”。
- 机器码使用 `movzx ecx, byte ptr [505E8C]`，实际只读取低 1 字节。
- 若 `X=0` 则跳过保底；若原伤害已经高于保底，保留原伤害。

##### 多多益善

- Hook `43F936`，代码洞 `441D1C`。
- 默认 `装备=00`、`个人=B3`，纯开关。
