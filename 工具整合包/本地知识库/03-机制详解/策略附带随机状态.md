## 详细记录

#### 概述

本补丁实现了一个新特技：持有该特技的角色在**策略攻击**命中并造成伤害后，遍历所有受击目标，有一定概率使每个有效目标陷入**随机异常状态**（中毒/麻痹/禁咒/混乱之一）。触发概率由特技的**特效值**控制（如特效值=50 则约 50% 触发）。

| 属性 | 值 |
|------|-----|
| 个人特效号 | `C0` (192) |
| 装备特效号 | `00` (不可通过装备获取) |
| 注入点 | `4259AF` |
| 代码空洞 | `41A627-41A6DB`（181 字节，v3） |
| 兼容性 | 与策略偷钱补丁兼容（注入点在其之前） |

#### 注入策略

```
策略攻击结算流程
    │
4259A7: call 41E5C2              ; 策略相关处理
4259AC: mov ecx, [ebp-04]        ; ecx = 策略数据结构
4259AF: ★ jmp 41A627             ; → 本补丁注入（替换原 call 422F2E）
    │
    └── 41A627: 代码空洞逻辑
        ├── 校验攻击者特技与特效值
        ├── 从第 0 个目标开始遍历目标数组
        ├── 跳过 FF 结束符和无伤害目标
        ├── 调用 4101D9 核心引擎（效果值标志=0，获取特效值）
        ├── 特效值%概率判定 via 47FB13
        ├── 随机选择异常状态 (0-3)
        ├── 写入当前目标 unit[1E] 状态字节
        ├── 循环处理下一个目标
        ├── call 422F2E（执行原被替换的调用）
        └── jmp 4259B4（回到原流程，若策略偷钱已安装则链入）
```

#### 关键数据结构

`[ebp-04]` 指向策略战斗数据结构：

| 偏移 | 内容 |
|------|------|
| `+01` | 攻击方战场编号 |
| `+0D+i` | 第 i 个目标方战场编号；`FF` 为结束符 |
| `+108+4*i` | 第 i 个目标策略伤害值，`>=1` 才处理 |

#### 异常状态位编码

状态写入 `unit[1E]`，使用位掩码：

| 状态 | 索引 | 掩码 | 位 |
|------|------|------|-----|
| 中毒 | 0 | `C0` | 位 6-7 |
| 麻痹 | 1 | `30` | 位 4-5 |
| 禁咒 | 2 | `0C` | 位 2-3 |
| 混乱 | 3 | `03` | 位 0-1 |

#### v3 代码空洞要点

```asm
4259AF: jmp 41A627

41A627: mov ecx, [ebp-04]          ; ECX = 策略数据结构
...
41A63F: push 00                    ; 效果值标志 = 0，需要特效值
41A641: push 01                    ; 叠加标志 = 1，不可叠加
41A643: push 00                    ; 装备特效号 = 00，可改地址 41A645
41A645: push C0                    ; 个人特效号 = C0，可改地址 41A647
41A64A: call 4101D9
...
41A655: push edi                     ; 保存 edi
41A656: mov edi, eax                 ; edi = 特效值
41A658: mov [ebp-08], 0            ; 目标索引 i = 0
.loop:
  ; 读取 [策略数据+0D+i]，FF 结束
  ; 检查 [策略数据+108+4*i] >= 1
  push 64
  call 47FB13                        ; 随机 0-99
  cmp eax, edi
  jae .next                            ; 随机值 >= 特效值则不触发，避免 off-by-one
  ; 取目标单位并随机写入 unit+1E 状态位
.next:
  inc [ebp-08]
  cmp [ebp-08], 73
  jl .loop
  pop edi
  mov ecx, [ebp-04]
  call 422F2E
  jmp 4259B4
```

v3 修复点：

- 不再用 `ebx` 保存特效值，改用 `edi` 并在进入/离开时保存恢复。
- 增加目标遍历循环，支持多目标策略。
- 概率比较使用 `jae`，随机值等于特效值时不触发，概率范围更符合 `0..99 < 特效值`。
