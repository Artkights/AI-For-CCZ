## 详细记录

#### 概述

曹操传中存在一种"攻击命中后使目标随机属性下降"的能力，其个人特效号为 `0x0C` (12)，装备特效号为 `0x5E` (94)。该能力的检测点位于 `0x472D63` 附近。

**关键特征**：该能力虽然仍遵循桩函数→调度函数→核心引擎的判定架构，但其桩函数不在常规的 `0x421080-0x421180` 区域，而是**直接内联**在战斗效果处理函数 `0x472D48` 中。

---

#### 完整调用链

```
后伤害效果处理 0x472D48 (attacker, target)
        │
        ├── 0x436392(p1, p2, 0x0E, attacker, 0x00): 前置条件检查
        │        │
        │        ├── 返回 ≠ 0 → 跳过特效检查, 直接进入属性下降循环
        │        │
        │        └── 返回 = 0 → 进入特技检测
        │
        ├── ★ 内联桩函数 (0x472D5F-0x472D67)
        │   push 0x01        ; 效果值标志
        │   push 0x01        ; 叠加标志  
        │   push 0x5E        ; 装备特效号 = 94
        │   push 0x0C        ; 个人特效号 = 12
        │   call 0x600127    ; → 调度函数
        │        │
        │        └── 0x600127(EDX=0x34E0): 标准调度函数
        │             │
        │             ├── 0x9BBFE4: 查表验证
        │             └── 0x4101D9: 核心引擎执行
        │
        ├── 特技未激活 → 直接返回
        │
        └── 特技激活 → 进入属性下降循环 (最多2次)
             │
             ├── 循环延迟器: 0x47FB13(0x06)  ★ 随机属性选择
             │
             └── 0x409821(target, attr_index, 0x00)  ★ 应用属性下降
                  │
                  └── 0x414361(target, attr_index, 0x00, 0x01)
                       │
                       └── 0x45AA37(...)  ★ 写入属性 + UI更新
```

---

#### 调用上下文

该效果由 **[后伤害效果调度器 0x405966](../04-函数速查/function-index.md)** 统一调用。0x405966 在伤害应用后执行，按顺序调用多个效果处理函数：

```
0x405966 (后伤害效果调度器)
    ├── 0x40399B: 伤害效果执行
    ├── 0x472D48: ★ 属性下降效果（本文档所述）
    ├── 0x4039F4: 伤害动画/音效
    ├── 0x436392: 前置条件检查
    └── 0x472C41: 另一伤害效果检查
```

这就是为什么"回MP攻击"的注入补丁（见 [注入补丁机制](../01-核心引擎/code_caves_and_safe_patching.md)）选择在 0x405966 内部插 Hook——它是所有后伤害效果的统一入口。

#### 入口函数 0x472D48 详细反汇编

```asm
; 0x472D48: 后伤害属性下降效果处理
; 参数: [ebp+0x08] = attacker_ptr (攻击者)
;       [ebp+0x0C] = target_ptr  (被攻击者)
0x472D48: push ebp
0x472D49: mov ebp, esp
0x472D4B: push 0x00               ; param5 = 0
0x472D4D: push ecx                ; param4 = attacker (ecx原值)
0x472D4E: push 0x0E               ; param3 = 0x0E
0x472D50: push [ebp+0x08]         ; param2 = attacker
0x472D53: call 0x436392           ; ★ 前置条件检查
0x472D58: test eax, eax
0x472D5A: jnz 0x472D75            ; 前置满足 → 跳过特技检查

; ===== 内联桩函数: 检测 0x5E/0x0C 特技 =====
0x472D5C: mov ecx, [ebp-0x08]     ; ecx = attacker
0x472D5F: push 0x01               ; 效果值标志 = 1 (无效果值)
0x472D61: push 0x01               ; 叠加标志 = 1 (不可叠加)
0x472D63: push 0x5E               ; ★ 装备特效号 = 0x5E (94)
0x472D65: push 0x0C               ; ★ 个人特效号 = 0x0C (12)
0x472D67: call 0x600127           ; → 调度函数 (EDX=0x34E0)
0x472D6C: test eax, eax
0x472D6E: jz 0x472D92             ; 无特技 → 退出
0x472D70: jmp 0x472D75            ; 有特技 → 进入循环

; ===== 属性下降循环 (最多2次) =====
0x472D72: inc [ebp-0x04]          ; 循环计数器++
0x472D75: cmp byte [ebp-0x04], 0x02  ; i < 2?
0x472D79: jnb 0x472D92            ; i >= 2 → 退出
0x472D7B: push 0x06               ; 参数 = 0x06 (6种属性)
0x472D7D: call 0x47FB13           ; ★ 随机选择属性索引 (0-5)
0x472D82: nop ; nop ; nop         ; 3字节NOP填充
0x472D85: push 0x00               ; arg3 = 0 (下降模式)
0x472D87: push eax                ; arg2 = 随机属性索引
0x472D88: mov ecx, [ebp+0x0C]     ; ecx = target_ptr
0x472D8B: call 0x409821           ; ★ 应用属性下降
0x472D90: jmp 0x472D72            ; 继续循环

0x472D92: mov esp, ebp
0x472D94: pop ebp
0x472D95: ret 0x08
```

#### 关键子函数详解

##### 1. 前置条件检查 — 0x436392

```asm
0x436392: push ebp
0x436393: mov ebp, esp
0x436395: mov al, [ebp+0x08]      ; 参数1
0x436398: cmp al, 0x00            ; 参数1 == 0?
0x43639A: jz 0x4363BA             ; 是 → 返回0
0x43639C: cmp al, [ebp+0x0C]      ; 参数1 == 参数2?
0x43639F: jnz 0x4363BA            ; 否 → 返回0
0x4363A1: mov eax, [ecx]          ; 读取ecx指向的值
0x4363A3: call 0x40658F
0x4363AD: mov edx, [0x500C37]     ; ★ 读取全局状态
0x4363B3: call 0x403599           ; 比对验证
0x4363B8: jmp 0x4363BC
0x4363BA: xor eax, eax            ; 返回0
0x4363BC: pop ebp
0x4363BD: ret 0x08
```

该函数检查 `[0x500C37]` 全局状态变量。如果全局状态满足条件（如特定回合、特定战场规则等），则自动触发属性下降效果，无需检查特技。

##### 2. 调度函数 — 0x600127

```asm
0x600127: push ebp
0x600128: mov ebp, esp
0x60012A: push ecx
0x60012B: mov ecx, [ecx]          ; 获取攻击者单位指针
0x60012D: mov edx, 0x34E0         ; ★ 特技数据表偏移
0x600132: call 0x9BBFE4           ; 查表验证
0x600137: test eax, eax
0x600139: jnz 0x60014D            ; 未查到 → 返回
0x60013B: pop ecx
0x60013C-0x600145: push [ebp+14/10/C/8]
0x600148: call 0x4101D9           ; ★ 核心引擎
0x60014D: leave
0x60014E: ret 0x10
```

**这是标准的调度函数模板**，唯一的差异是 EDX 偏移为 `0x34E0`（而非常规区域的 0x35C0、0x3740 等）。

##### 3. 随机属性选择 — 0x47FB13

```asm
0x47FB13: push ebp
0x47FB14: mov ebp, esp
0x47FB16: xor eax, eax
0x47FB18: cmp [ebp+0x08], 0x02    ; 参数 >= 2?
0x47FB1C: jb 0x47FB2A             ; < 2 → 返回0
0x47FB1E: call 0x47FAE7           ; ★ 生成随机数 (0-32767)
0x47FB23: sub edx, edx            ; edx = 0
0x47FB25: div [ebp+0x08]          ; eax / 6, 余数→edx
0x47FB28: mov eax, edx            ; eax = 余数 (0-5)
0x47FB2A: pop ebp
0x47FB2B: ret 0x04
```

**返回值**：0-5 之间的随机整数，代表属性索引。

##### 4. 随机数生成器 — 0x47FAE7 (LCG算法)

```asm
0x47FAE7: mov eax, [0x48ECD4]     ; ★ 读取种子
0x47FAEC: imul eax, 0x41C64E6D    ; 种子 × 0x41C64E6D
0x47FAF2: add eax, 0x3039         ; + 0x3039
0x47FAF7: mov [0x48ECD4], eax     ; ★ 写回种子
0x47FAFC: and eax, 0x7FFF0000     ; 取高15位
0x47FB01: shr eax, 0x10           ; 右移16位 → 值域 [0, 32767]
0x47FB04: ret
```

经典线性同余生成器 (LCG)：`seed = seed × 0x41C64E6D + 0x3039`

##### 5. 属性下降包装 — 0x409821

```asm
0x409821: push ebp
0x409822: mov ebp, esp
0x409824: push 0x01               ; arg4 = 0x01
0x409826: push [ebp+0x0C]         ; arg3 = mode (0x00 = 下降模式)
0x409829: push [ebp+0x08]         ; arg2 = attr_index (0-5)
0x40982C: push ecx                ; arg1 = target_ptr
0x40982D: call 0x414361           ; ★ 核心修改函数
0x409832: pop ebp
0x409833: ret 0x08
```

##### 6. 属性核心修改 — 0x414361

```asm
; 参数:
;   ecx = target unit ptr
;   [ebp+0x08] = target unit ptr (via push)
;   [ebp+0x0C] = attr_index (0-5)
;   [ebp+0x10] = mode (0x00 = 下降)
;   [ebp+0x14] = context flag (0x01)
0x414361: push ebp
0x414362: mov ebp, esp
0x414364: add ecx, [ebp+0x0C]     ; ecx = target + attr_index
0x414367: movzx eax, byte [ecx+0x18]  ; ★ 读取 [target + attr_index + 0x18]

; === 下降模式 (mode=0) ===
0x41436B: cmp [ebp+0x10], 0x00
0x41436F: jz 0x41437D             ; mode=0 → 下降路径

; === 上升模式 (mode≠0, 此调用不使用) ===
0x414371: cmp al, 0x03
0x414373: jb 0x414379
0x414375: mov al, 0x06            ; 当前值>=3 → 设为6(满)
0x414377: jmp 0x414387
0x414379: mov al, 0x03            ; 当前值<3 → 设为3

; === 下降模式 ===
0x41437D: cmp al, 0x03
0x41437F: jnbe 0x414385           ; 当前值 > 3?
0x414381: xor eax, eax            ; 值 ≤ 3 → 归零 (清除该属性)
0x414383: jmp 0x414387
0x414385: mov al, 0x03            ; 值 > 3 → 降至3 (降1档)

; === 写入属性 + UI更新 ===
0x414387: mov ecx, [ebp+0x08]     ; ecx = target ptr
0x41438A: push [ecx+0x04]         ; unit ID
0x41438D: push [ebp+0x14]         ; context flag = 0x01
0x414390: push eax                ; new_value
0x414391: push [ebp+0x0C]         ; attr_index
0x414394: push [ecx]              ; unit valid flag
0x414396: call 0x45AA37           ; ★ 写入属性并更新UI
0x41439B: pop ebp
0x41439C: ret 0x10
```

**属性下降规则**：

| 当前属性值 | 下降后 | 说明 |
|-----------|--------|------|
| `value ≤ 3` | `0` | 该属性增益完全清零 |
| `value > 3` | `3` | 降至下一档（中档） |

---

#### 受影响的属性

`attr_index` 取值 0-5，对应战术单位 `[target + attr_index + 0x18]` 处的 6 个字节：

| 索引 | 实际偏移 | 待证假设属性 |
|------|----------|----------|
| 0 | `target+0x18` | 攻击力增益 |
| 1 | `target+0x19` | 防御力增益 |
| 2 | `target+0x1A` | 精神/智力增益 |
| 3 | `target+0x1B` | 爆发力/敏捷增益 |
| 4 | `target+0x1C` | 士气/运气增益 |
| 5 | `target+0x1D` | 移动力增益 |

这些属性值范围通常是 0-6（无增益→满增益），下降将减少/清除受影响的增益。

#### 与常规桩函数路径的对比

##### 常规路径 (0x421080区域)

```
游戏主循环/事件分发
        │
        └── call 0x4210xx (桩函数在0x421080区域)
                 │
                 └── call 0x6000xx (调度函数)
                          │
                          └── call 0x4101D9 (核心引擎)
```

##### 本能力的路径 (内联桩)

```
后伤害效果处理 0x472D48
        │
        ├── 0x436392 前置条件检查
        │        └── 通过 → 直接进入效果
        │
        └── 内联桩代码 (push/push/push/call)
                 │
                 └── call 0x600127 (调度函数, EDX=0x34E0)
                          │
                          └── call 0x4101D9 (核心引擎)
```

##### 差异总结

| 特性 | 0x421080区域桩函数 | 0x472D48内联桩 |
|------|-------------------|---------------|
| 位置 | 独立的桩函数 | 嵌入在效果处理函数内 |
| 入口方式 | `call 0x4210xx` | 函数内部的 `push ... call` |
| 前置条件 | 无（桩函数本身即入口） | 有 (0x436392 检查) |
| 调度EDX | 各不同 (0x35C0-0x3A20) | 0x34E0 |
| 效果执行 | 由调用者处理 | 在自身函数内完成循环 |

**实质相同点**：调度函数模板完全一致（`mov ecx,[ecx]; mov edx,xxx; call 0x9BBFE4; call 0x4101D9`），只是嵌入位置不同。

#### 设计注意事项

如果要在Mod中自定义此机制：

1. **修改下降程度**：修改 `0x41437D-0x414385` 的条件逻辑（当前是 value>3→降至3，value≤3→归0）
2. **修改随机属性数**：修改 `0x47FB13` 的参数（当前0x06）和循环上限（当前0x02）
3. **添加新能力**：在 `0x472D48` 模式中添加新的 inline stub + effect loop
4. **修改属性映射**：修改 `0x414364` 的 `+0x18` 偏移
