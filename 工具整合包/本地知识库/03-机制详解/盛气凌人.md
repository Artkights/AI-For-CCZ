## 详细记录

#### 概述

"盛气凌人"是一个**自定义注入式特技**：攻击者在物理攻击发生暴击时，使目标全部 6 项属性（攻击、防御、精神、爆发、士气、移动）各下降 1 档。

- **个人特效号**：`0xBF` (191)
- **装备特效号**：`0x00`（不可通过装备获取）
- **Hook 模式**：函数尾部 epilogue 劫持（不破坏任何原指令）

#### 完整调用链

```
后伤害效果调度 0x405966 (attacker, target)
  │
  ├── call 0x40399B              ← 伤害效果执行
  ├── call 0x472D48              ← ★ 原生属性下降 + 盛气凌人
  │     │
  │     ├── 前置条件检查 (0x436392)
  │     ├── 特技检测 (0x5E/0x0C → 0x600127 → 0x4101D9)
  │     ├── 随机降 2 项属性 (循环调用 0x409821)
  │     ├── 0x472D92: jmp 0x4529A7 ──────────┐  ★ Hook 劫持
  │     │                                      │
  │     └── ★ 盛气凌人代码洞 0x4529A7 ────────┘
  │         ① mov eax, 0x4927F0
  │         ② cmp [eax+0x604], 0     ; 检测暴击标志
  │         ③ jz .epilogue           ; 非暴击 → 跳过
  │         ④ mov ecx, [ebp-0x08]    ; ecx = attacker
  │         ⑤ push 1/1/0x00/0xBF → call 0x4101D9
  │         ⑥ jz .epilogue           ; 无特技 → 跳过
  │         ⑦ mov [ebp-0x04], 0      ; 计数器归零
  │         ⑧ .loop: push 0x00; push [ebp-0x04]
  │              mov ecx, [ebp+0x0C]; call 0x409821
  │              inc [ebp-0x04]; cmp [ebp-0x04], 6; jl .loop
  │         ⑨ .epilogue: mov esp, ebp; pop ebp; ret 0x08
  ├── call 0x4039F4              ← 伤害动画/音效
  └── ...
```

#### 设计理念

##### 为什么选择 0x472D48 尾部 Hook

1. **与原生属性下降（`0x0C/0x5E`）完全一致的触发时序**：伤害已结算，属性下降在同一后伤害调度循环中执行
2. **复用现有栈帧**：`[ebp+0x0C]` = target 指针，`[ebp-0x08]` = attacker 指针，无需额外查询
3. **无需恢复被覆盖指令**：Hook 替换的是 epilogue（`mov esp, ebp; pop ebp; ret 0x08`），在代码洞中重新执行这 3 条指令即可
4. **遍历全部 6 项属性**：与原版随机降 2 项的"攻击附带属性下降"形成差异化设计

##### 为什么不使用 ebx/esi 寄存器

该代码洞位于 `0x472D48` 的栈帧上下文中，原函数不使用 `ebx`/`esi`。
代码洞全程使用栈局部变量 `[ebp-0x04]` 做循环计数，这直接沿用了原版 `0x472D48` 的写法
（见 `0x472D72: inc dword [ebp-0x04]`），保持了与原版完全一致的编码风格。

此外，`[ebp-N]` 栈变量的使用方式在原版引擎中广泛存在：
- `0x472D48` 用 `[ebp-0x04]` 做属性下降循环计数
- `0x412D52` 用 `[ebp-0x04]` 保存单位指针、`[ebp-0x08]` 保存状态标志地址
- `0x43DADA` 用 `[ebp-0x0C]` 保存 this 指针、`[ebp-0x04]` 保存临时状态

使用栈变量而非 `ebx`/`esi` 等 被调用方保存 寄存器，可以避免不必要的 push/pop 操作，
降低代码出错的概率，同时让注入代码的风格与所在原函数的上下文保持一致。

#### 完整反汇编（代码洞 0x4529A7，72 字节）

```asm
; ─── 暴击标志检测 ───
0x4529A7: mov eax, 0x4927F0            ; eax = 战斗数据基址
0x4529AC: cmp dword [eax+0x604], 0     ; [0x4927F0+0x604] = 暴击标志
0x4529B3: jz 0x4529E9                  ; 非暴击 → 跳 epilogue

; ─── 特技判定（复用原版 [ebp-0x08] = attacker） ───
0x4529B5: mov ecx, [ebp-0x08]          ; ecx = attacker
0x4529B8: push 0x01                     ; 效果值标志 = 1
0x4529BA: push 0x01                     ; 叠加标志 = 1
0x4529BC: push 0x00                     ; 装备特效号 = 0x00
0x4529BE: push 0xBF                     ; 个人特效号 = 0xBF
0x4529C3: call 0x4101D9                ; ★ 核心引擎
0x4529C8: test eax, eax
0x4529CA: jz 0x4529E9                  ; 无特技 → 跳 epilogue

; ─── 全属性下降循环 ───
0x4529CC: mov dword [ebp-0x04], 0      ; 计数器 = 0
0x4529D3: push 0x00                     ; arg3 = 0 (下降模式)
0x4529D5: push dword [ebp-0x04]         ; arg2 = 属性索引 (0-5)
0x4529D8: mov ecx, [ebp+0x0C]           ; ecx = target 指针
0x4529DB: call 0x409821                ; ★ 属性下降包装
0x4529E0: inc dword [ebp-0x04]         ; 计数器++
0x4529E3: cmp dword [ebp-0x04], 6      ; i < 6 ?
0x4529E7: jl 0x4529D3                  ; 继续循环

; ─── 恢复原 epilogue ───
0x4529E9: mov esp, ebp
0x4529EB: pop ebp
0x4529EC: ret 0x08
```

#### 受影响的属性

`attr_index` 取值 0-5，对应战术单位 `[target + attr_index + 0x18]` 处的 6 个字节：

| 索引 | 实际偏移 | 属性 |
|------|----------|------|
| 0 | `target+0x18` | 攻击力增益 |
| 1 | `target+0x19` | 防御力增益 |
| 2 | `target+0x1A` | 精神/智力增益 |
| 3 | `target+0x1B` | 爆发力/敏捷增益 |
| 4 | `target+0x1C` | 士气/运气增益 |
| 5 | `target+0x1D` | 移动力增益 |

属性下降规则同 `0x414361`（与原生属性下降一致）：
- `当前值 > 3` → 降至 3
- `当前值 ≤ 3` → 归零

#### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 个人特效号 | `0xBF` (191) | 修改补丁文件 `-4529BF` |
| 装备特效号 | `0x00` (0) | 修改补丁文件 `-4529BD` |
| Hook 地址 | `0x472D92` | 替换 `0x472D48` 函数 epilogue |
| 代码洞 | `0x4529A7` | 空洞 D-1，72 字节 |
| 暴击标志偏移 | `+0x604` | `0x4927F0` 战斗数据结构 |

#### 与原版属性下降（0x0C/0x5E）的对比

| 特性 | 原版（0x0C/0x5E） | 盛气凌人（0xBF） |
|------|------------------|-----------------|
| 触发条件 | 攻击命中后（前置检查通过/特技激活） | 攻击命中 + 暴击 + 特技激活 |
| 下降项数 | 随机 2 项 | 固定全部 6 项 |
| 下降档位 | 随机 | 各降 1 档 |
| 代码路径 | `0x472D48` 内部完整逻辑 | `0x472D48` 尾部注入 |
| 栈帧复用 | 自身 | 复用 `0x472D48` |
| 计数器 | `[ebp-0x04]` | `[ebp-0x04]` |

#### 开发历程

此次特技补丁经历了多轮迭代：

| 版本 | Hook 点 | 问题 |
|------|---------|------|
| v1-v2 | `0x43DB53`（0x43DADA 内） | `push ebx/push esi` 干扰 `0x4064F1` 栈参数 |
| v3 | `0x43DB53`（修正 push 顺序后） | 与 模组 版引擎不兼容 |
| v4 (模组) | `0x405F24`（模组 暴击汇合点） | 伤害结算前触发，时序不对 |
| v5 (模组) | `0x406023`（经验结算） | 太靠后，不符合"伤害→属性下降"时序 |
| **v6 最终版** | `0x472D92`（原生属性下降函数尾部） | ✅ 与原版完全一致的触发时序 |

最终版放弃使用 `ebx`/`esi`，改用 `[ebp-0x04]` 栈局部变量计数，与原版编码风格统一。
