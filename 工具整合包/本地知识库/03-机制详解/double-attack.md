## 详细记录

#### 概述

曹操传中的连击并非独立的"特技"系统，而是**游戏物理攻击流程中的基础战斗判定机制**。该机制通过比对攻击方与被攻击方的"爆发力"属性，计算出三项核心战斗参数：**连击率**、**命中率**、**回避率**，并将这些参数应用于物理攻击的全过程。

#### 核心概念：爆发力

##### 什么是爆发力

"爆发力"是曹操传中用于衡量单位战斗爆发能力的综合属性。该属性直接影响战斗中的三项关键判定：

| 战斗参数 | 计算来源 | 作用 |
|----------|----------|------|
| **连击率** | 攻击方爆发力 vs 防御方爆发力 | 决定是否发动追加攻击 |
| **命中率** | 攻击方爆发力 vs 防御方爆发力 | 决定攻击是否命中目标 |
| **回避率** | 防御方爆发力 vs 攻击方爆发力 | 决定目标能否闪避攻击 |

##### 爆发力与敏捷的关系

根据游戏机制分析，爆发力的数值来源与单位的**敏捷属性**高度相关。敏捷越高的单位，其爆发力数值越大，在战斗判定中越容易：
- 发动连击
- 命中敌人
- 回避敌人攻击

#### 爆发力比对的计算公式

##### 核心公式（待证假设）

基于游戏机制分析，爆发力比对可能采用以下形式的计算：

```
连击率 = (攻击方爆发力 / (攻击方爆发力 + 防御方爆发力)) × 基础系数
命中率 = (攻击方爆发力 / (攻击方爆发力 + 防御方爆发力)) × 技能加成
回避率 = (防御方爆发力 / (攻击方爆发力 + 防御方爆发力)) × 地形加成
```

##### 随机判定

所有百分比判定（连击率、命中率、回避率）最终都通过随机数进行比较：

```
if (随机数(0-100) < 判定阈值) {
    // 判定成功
}
```

#### 与特技系统的区别

##### 关键区分

| 特性 | 战斗基础判定 | 特技系统 |
|------|-------------|----------|
| **性质** | 物理攻击流程的固有部分 | 需通过特技桩函数触发 |
| **触发方式** | 自动计算，无需显式触发 | 特定条件满足后触发 |
| **判定依据** | 爆发力属性比对 | 特技定义数据 |
| **修改方式** | 修改属性计算逻辑 | 添加特技定义 |

##### 错误理解纠正

**❌ 错误理解**：连击是一种特技，通过特定装备或能力获得

**✅ 正确理解**：连击是战斗的基础判定机制，所有物理攻击都会进行爆发力比对，进而决定是否触发连击
