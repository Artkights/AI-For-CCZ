## 详细记录

#### 概述

曹操传的物理攻击体系是一个复杂的多阶段处理流程，涉及初始化、攻击者能力检查、目标检查、伤害计算、伤害应用、战斗数据记录和动画播放等多个环节。整个体系以 `43DADA` 为核心调度函数，协同多个子系统完成一次完整的物理攻击流程。

#### 2026-06-09 战场实机操作入口

- 进入正式战场后，点击我军单位会显示可移动范围和攻击范围。
- 紫色格子表示当前可移动目标；再次点击无人占据的紫色格子会移动单位。
- 红色框格子表示当前攻击范围；执行物理攻击前，目标敌军必须位于红色框内。
- 人工操作物理攻击时，先选中我军单位，再点击右侧最上方“攻击”指令，最后点击红色框范围内的敌军。
- 颍川之战功勋模式样本：曹操原地攻击上方张梁，张梁反击时曹操格挡，UI 显示武力、敏捷、统率功勋各增加 1 点。详见 [功勋模式与五维功勋](./功勋模式与五维功勋.md)。
- 除二次行动、再次移动等特技外，单位一回合通常只能执行一次行动；行动可为物理攻击、释放策略、使用道具或待命。行动后单位形象变灰，点击会提示“已经行动完毕”。

#### 核心函数：43DADA

##### 函数签名与入口处理

```
43DADA: push ebp
43DADB: mov ebp, esp
43DADD: sub esp, 14              ; 分配20字节栈空间
43DAE0: mov [ebp-0C], ecx        ; 保存this指针(攻击者单位结构)
43DAE3: xor eax, eax                ; eax = 0
43DAE5: mov [ebp-10], eax        ; 初始化临时变量为0
43DAE8: mov al, FF                ; al = FF
43DAEA: mov [ebp-08], eax        ; [ebp-08] = FFFFFFFF
43DAED: call 4109A1              ; 调用初始化函数
43DAF2: mov [ebp-04], al         ; 保存初始化结果
```

栈空间布局：

| 偏移 | 用途 |
|------|------|
| `[ebp-04]` | 初始化结果/临时状态 |
| `[ebp-08]` | 状态标志(FFFFFFFF初始值) |
| `[ebp-0C]` | ECX保存(攻击者单位指针) |
| `[ebp-10]` | 临时变量 |
| `[ebp-14]` | 栈分配边界 |

#### 数据传递机制

##### ECX寄存器（攻击者单位指针）

在物理攻击流程中，ECX寄存器始终指向攻击者的单位数据结构。函数调用约定使用 `thiscall`，第一个参数通过ECX传递。

##### 栈参数传递

关键函数使用栈传递额外参数：

| 函数 | 参数1 | 参数2 | 说明 |
|------|-------|-------|------|
| `4109A1` | - | - | 无参数，使用ECX |
| `43F8C0` | - | - | 无参数，使用ECX |
| `45533D` | 目标单位 | - | ECX=this, [ebp+08]=目标 |
| `410F1C` | 攻击者 | 防御者 | [ebp+08], [ebp+0C] |
| `4363DC` | 单位 | 攻击类型 | [ebp-04], [ebp+08] |
| `4064F1` | 攻击者 | 伤害值 | [ebp+08], [ebp+0C] |

##### 返回值约定

- `al = FF`：特殊状态（如必杀、无法攻击等）
- `al = 00`：正常失败
- `al = 01`：成功
- `eax`：复合返回值（如伤害值、指针等）

#### 战斗数据区：4927F0

##### 数据结构

战斗数据结构位于 `4927F0`，由 `4064F1` 函数写入。

##### 写入时机

物理攻击完成后记录战斗数据。

##### 数据内容

记录的信息包括：
- 攻击者ID
- 防御者ID
- 伤害值
- 攻击类型
- 是否必杀
- 攻击结果

#### 状态标志系统

##### 单位状态字节 [ecx+0D]

| 位 | 含义 |
|----|------|
| 位 0 | 移动能力（0=可移动，1=已移动） |
| 位 1 | 攻击能力（0=可攻击，1=已攻击） |
| 位 2 | 特殊行动能力 |
| 位 3-7 | 其他标志 |

##### 状态修改时机

在物理攻击流程中，**仅在** `442718` 函数中修改行动状态。攻击结束后，位 1 被设置为1，表示该单位本回合已完成攻击。

##### 恢复行动能力

如果需要恢复行动能力，需要调用 `406690` 函数：

```asm
; 公式：unit[0D] &= ~参数
; 参数01：恢复移动能力
; 参数02：恢复攻击能力
; 参数03：恢复移动+攻击能力
```

#### 与特技系统的关系

物理攻击体系与特技系统是**独立**的：

1. **物理攻击主流程**（43DADA）处理标准的攻击逻辑
2. **特技系统**在特定触发点进行检查，决定是否执行额外效果
3. **常规物理攻击**不会自动恢复行动力

#### 2026-06-11 测验1动态确认：曹操攻击黄巾兵的结果链

证据等级：x32dbg live 命中 + 调用栈闭环 + HP 硬件写断点。场景为我军曹操攻击可攻击范围内敌军黄巾兵，用户确认本次两次 HP 写入来自连击。

已确认攻击链：

```text
0043DF0A  攻击命令类型分支，命中时 AL=1
    -> 0043DADA  攻击处理入口，本次 ECX=004A7B20（曹操 unit*）
        -> 00418330  post-damage caller
            -> 00405966  攻击结果/伤害结果循环
                -> 00405ADC  call 0043F6DB
                    -> 0043F6DB  HP setter 入口
                        -> 0043F70C  mov [ecx+10], edx
```

本轮动态事实：

- 攻击者：曹操 `unit[0] = 004A7B20`。
- 被攻击者：黄巾兵 `unit[77] = 004A8990`。
- 最终被修改的核心数值：目标 `unit+10` 当前 HP。
- 本次连击使同一目标经历两段扣血：`118 -> 59`，随后 `59 -> 15`。
- `00405966` 会循环处理已生成的伤害/结果记录；本轮确认它对同一目标两次走到 `00405ADC -> 0043F6DB`。

边界：

- `00405966` 是攻击结果/后伤害结果循环，不等同于最初伤害公式函数。
- `0043F70C` 是最终 HP 写入点，不等同于伤害计算点。
- `00405AB9 -> 00450986` 是事件/效果派发路径，本轮不是 HP 写入点。
- `00405B6B -> 00406690`、`00404B69 -> 00442718` 属于攻击后状态/行动位处理路径，本轮未闭环每个位的语义。
- 本次未动态验证移动动作；移动入口、坐标写入函数和坐标字段必须单独用移动样本验证。

#### 2026-06-11 测验1续：友军/敌军自动阶段物理攻击与反击链

证据等级：x32dbg live 命中 + 暂停断点寄存器/栈观察 + HP 写入闭环。场景从我方曹操行动结束后进入友方阶段、敌方阶段，再回到第二回合我方可控阶段。用户确认友方与敌方阶段玩家不可控制。

本轮升级的关键观测点：

```text
物理主伤害链：
0045280F
  -> 0040654A
    -> 0043ACBD
      -> 0043B2E5
        -> 0044EA34
          -> 00418335
            -> 00405AD5  sub eax, [ebp-14]
            -> 00405ADC  call 0043F6DB
              -> 0043F6DB
                -> 0043F70C  mov [ecx+10], edx

反击伤害链：
0041797E
  -> 004064DA
    -> 00406581
      -> 0043ACBD
        -> ... -> 00405AD5 -> 00405ADC -> 0043F6DB -> 0043F70C
```

`00405AD5` 是目前最适合做物理后伤害观测的地址：

- 指令语义：`sub eax, [ebp-14]`。
- `EAX`：扣血前 HP。
- `[EBP-14]`：本段最终伤害。
- `[EBP-20]`：目标战术单位指针。
- 新 HP：`EAX - [EBP-14]`，随后由 `00405ADC -> 0043F6DB -> 0043F70C` 写入 `unit+10`。

已确认单位结构与 HP 规则：

- 战术单位数组基址：`004A7B20`。
- 单位槽大小：`30`。
- `unit_addr = 004A7B20 + battle_index * 30`。
- 当前 HP 字段：`unit+10`，4 字节。
- 最终 HP 写入：`0043F70C: mov [ecx+10], edx`，其中 `ECX=unit*`，`EDX=new_hp`。

已确认战斗上下文：

- `004927F0+00`：当前攻击者战场编号。
- `004927F0+01`：当前目标战场编号。
- `004927F0+04`：攻击者 Data 号。
- `004927F0+0C`：攻击者战术单位指针。
- 发生反击时，头部攻击者/目标会随当前伤害方向翻转；因此 Hook 中不能把“本回合主动发起者”和“当前伤害攻击者”混为一个概念。

曹操主动攻击 EXP 与连击检查样本：

场景：第二回合曹操 `unit[0]` 普通攻击敌军 `unit[61]`，本次未触发连击。

已确认：

- `00410F1C` hit_count 从 `12` 增至 `13`，仍先于 `00405AD5`，继续支持其作为物理伤害计算候选。
- `00405AD5` hit_count 从 `25` 增至 `26`，本段伤害应用仍走 `old_hp - damage -> HP setter` 链。
- `004927F0+84` 第一个 HP 伤害值为 `3E`，即 62。
- `004927F0+428` 个人经验由 `004037A3` 计算，`00406043` 写入，本次为 `10`。
- `004927F0+42C` 武器 EXP 由 `00403870` 计算，`00406054` 累加，本次为 `2`。
- `004927F0+430+slot*4` 防具 EXP 由 `00403A8A` 计算，`00406074` 写入，本次 `slot=0`、值为 `0`。
- `00406555 -> 004E3000 -> 0040655C` 是当前 6.5 未连击样本中的连击/继续攻击检查链；`004E3000` 返回 `EAX=0`，因此未跳到 `0040651D/00406520` 追加攻击分支。

边界：

- `0040632D/00406333/00406342` 是攻击开始时清经验缓存，不是奖励结算。
- 旧版 docx 的 `00406113/00406153/00406171` 在本次当前 6.5 样本中未命中。
- `004E3000` 尚不能直接命名为完整“连击概率公式函数”；当前只确认它在未连击样本中返回 0 并控制攻击循环是否进入追加分支。

移动写入的动态结论仍保持较低证据等级：

- 目前有候选坐标写入链：`0040659B -> 004065A4: mov word ptr [ecx], dx`。
- 观察口径：`ECX=unit+06`，`DX` 为新坐标字，低字节 X、高字节 Y。
- 需要用“只移动不攻击”的单动作样本复核；在完成前不得把该地址作为跨版本移动 Hook 的最终入口。

跨版本/跨游戏复用口径：

- 优先抽象为“数据结构 + 调用关系 + 指令语义”，不要只记死地址。
- 可复用特征包括：单位数组 stride、HP setter 的 `unit+10` 写入、伤害应用点的“旧 HP - damage -> setter”模式、战斗上下文中当前攻击者/目标的翻转规则。
- 跨版本定位时应先找最终 HP setter，再向上追踪伤害应用点和结果循环；不要直接把 `00405AD5` 或 `0043F70C` 套到其它 EXE。

#### 2026-06-11 剧情撤退台词与战斗流程边界

张梁撤退台词触发后，战斗仍在继续；本轮随后进入自动阶段/反击链，并命中曹操 EXP 缓存写入点。当前应把“撤退台词”理解为剧本/剧情事件插入，而不是物理攻击系统、自动阶段调度或战斗结算的结束信号。

已确认的后续链：

```text
auto_phase_action_driver_after_unit_action  0044EA34
  -> auto_action_after_damage_or_attack_flow  0043ACBD
    -> battle_context_dispatch_retaliation_or_nested_action  004064DA
      -> counterattack_or_retaliation_damage_flow  0041797E
        -> 0040654A
          -> 00452807
            -> 00406043 / 00406054 / 00406074
```

对自动化和 Hook 的影响：

- 自动阶段、敌方阶段和剧情台词插入期间，玩家通常不可控，但物理伤害、反击和 EXP 缓存仍会继续结算。
- 若自动化控制器只看画面台词或弹窗，很容易误判为流程结束；必须结合单位数组、行动状态、`004927F0` 当前攻击者/目标和调度断点判断阶段。
- `on_post_damage` 和 EXP 缓存 Hook 要允许同一回合内多次进入，不能按“玩家一次点击攻击只产生一次结算”设计。
- “撤退”类剧情可能会让单位消失、触发台词或设置剧情变量；这与 HP 扣减、击杀派生和战斗胜负判定仍需分开验证。

#### 原版 KOEI 代码解析文档的参考价值

来源：`../90-来源归档/原始文档/解读KOEI曹操传代码.docx`。该文档解析的是原版/旧版曹操传代码，不是当前 6.5 未加密 EXE；因此只作为断点选址和结构理解线索，实作以本地动态证据为准。

可参考的流程形态：

- 原版 `004064F1` 被描述为核心攻击循环，先把 `004927F0+608` 清 0，再最多循环两段攻击。
- 原版循环中依次调用：`00405E03` 攻击伤害、`00405744` 攻击动作显示、`00405889` 攻击后附加效果、`00406181` 连击判定、`00405C6C` 经验/升级显示、`0040639A` 反击处理。
- 原版 `00405E03` 先清 `004927F0+84` HP 伤害数组和 `+254` MP 伤害数组，再调用 `0043BC4B` 计算物理伤害，暴击后把 `+604` 置 1，并写入 `+428/+42C/+430` 经验字段。
- 原版 `0040639A` 会判断反击后反击、无反击攻击、反击者 HP/状态、射程，成立后再调用攻击循环。

与当前 6.5 实测的关系：

- `004927F0` 的字段偏移在当前样本中高度吻合，尤其是 `+84`、`+254`、`+428`、`+42C`、`+430`、`+604`、`+608`、`+614`。
