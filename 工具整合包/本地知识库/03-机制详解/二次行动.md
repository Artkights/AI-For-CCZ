## 详细记录

#### 概述

**二次行动**是一种特技，效果为：角色行动结束后可额外行动一次，每回合仅生效一次。

> ⚠️ **专属性说明**：本文档涉及的所有函数（`412D52`、`421175`、`406690` 在二次行动流程中的调用、`508B00` 状态数组）**仅服务于二次行动这一个特技**。引擎不会为没有二次行动特技的单位自动恢复行动力——没有该特技的单位在攻击、施法、移动等行动结束后，行动标志位不会被清除，不会获得额外行动机会。

#### 特技参数

| 参数 | 值 |
|------|-----|
| 装备特效号 | 90 (5A) |
| 个人特效号 | 56 (38) |
| 效果值标志 | 1 (不需要效果值) |
| 叠加标志 | 1 (不可叠加) |

#### 核心机制

二次行动的核心机制围绕**状态数组 508B00**展开，确保每回合只触发一次。

##### 状态数组 508B00

每个单位占用1字节，存储二次行动的使用状态：

| 值 | 含义 |
|----|------|
| `00` | 本回合尚未触发，等待激活 |
| `01` | 单位不拥有该特技，不可用 |
| `FF` | 本回合已触发过，已使用（受保护） |

##### 状态转换流程

```
回合开始 → 重置循环将所有 508B00[i] 设为 00
    ↓
行动处理 → 调用点2判定：有特技→写00，无特技→写01
    ↓
行动结束 → 调用点1检查=00 → 判定通过 → 恢复行动 → 写FF
    ↓
再次行动 → 调用点1检查=FF → ≠0 → 跳过（不再触发）
```

#### 完整调用时序

##### 1. 回合开始 - 重置循环

**函数**：44AF8F

在新回合开始时，遍历所有单位，重置二次行动状态。

```asm
void reset_turn() {
    for (int i = 0; i < 250; i++) {
        Unit* unit = get_unit_ptr(i);       // 4061F9
        unit->field_23 = 0;                 // 清除回合状态
        restore_action(unit, 06);          // 406690: 恢复行动
        state_flags[i] = 0;                  // 508B00[i] = 0（重置为可用）
    }
    global_turn_flag = FF;                // 501BF9 = FF
}
```

##### 2. 行动处理 - 调用点2

**地址**：43D522

在行动处理流程中，初始化/更新二次行动的激活状态。

```asm
0043D522: call 421175             ; 调用二次行动桩函数
0043D527: test al, al
0043D529: jz 43D52D               ; 判定失败 → al保持0
0043D52B: mov al, 01              ; 判定成功 → al=1
0043D52D: xor al, 01              ; 翻转：成功→0, 失败→1
; ...
0043D53C: cmp byte [edx], FF      ; 已是FF（已使用）？
0043D540: jz 43D544               ; 是 → 跳过，保护已使用状态
0043D542: mov byte [edx], al        ; 写入新状态
```

**逻辑说明**：
- 判定通过 → al=0 → 写入 508B00[unit_id] = 0（标记"已激活"）
- 判定失败 → al=1 → 写入 508B00[unit_id] = 1（标记"不可用"）
- 如果已经是 FF，调用点2不会覆盖

##### 3. 行动结束 - 调用点1

**函数**：412D52

在角色行动结束后，检查并恢复二次行动能力。

```c
bool try_second_action(byte unit_index) {
    unit = get_unit_ptr(unit_index);           // 4061F9
    if (unit == INVALID) return false;
    if (unit->hp == 0) return false;          // 死亡单位不触发
    if (state_flags[unit_index] != 0)         // 508B00[index]
        return false;                         // 本回合已用过

    if (!stub_second_action())                 // call 421175
        return false;                         // 特技判定失败

    restore_action(unit, 06);               // call 406690
    update_action_ui(unit, 88, 06);      // call 43E6AA
    state_flags[unit_index] = FF;           // 标记已使用
    return true;
}
```

##### 4. 再次行动 - 调用点1重复检查

调用点1会检查 state_flags[index] 是否为00，只有状态为"等待触发"时才会恢复行动能力。

#### 关键函数详解

##### 桩函数 - 421175

二次行动特技判定入口。

```asm
00421175: push ebp
00421176: mov ebp, esp
00421178: push 01        ; 效果值标志 = 1
0042117A: push 01        ; 叠加标志 = 1
0042117C: push 5A        ; 装备特效号 = 90
0042117E: push 38        ; 个人特效号 = 56
00421180: call 00600819  ; 调度函数
00421185: pop ebp
00421186: ret
```

##### 行动恢复函数 - 406690

通过位清除操作恢复单位的行动能力。

```asm
00406690: push ebp
00406691: mov ebp, esp
00406693: movzx eax, byte [ebp+08]  ; 取参数（行动类型位）
00406697: not eax                      ; 取反（如 06 → FFFFFFF9）
00406699: and byte [ecx+0D], al     ; 清除 unit[0D] 的对应位
0040669C: mov esp, ebp
0040669E: pop ebp
0040669F: ret 04
```

**公式**：`unit[0D] &= ~参数`

**重要说明**：
- 参数是**位掩码**，不是"位号"
- 函数会**清除**参数指定的位（即恢复行动能力）
- 参数 = 00 时，`~00 = FF`，`unit[0D] &= FF`，**所有位不变**

**常用参数映射**：

| 参数 | 二进制 | 清除的位 | 效果 |
|------|--------|----------|------|
| `01` | `00000001` | 位 0 | 恢复移动能力 |
| `02` | `00000010` | 位 1 | 恢复攻击能力 |
| `03` | `00000011` | 位 0 + 位 1 | 恢复移动+攻击 |
| `06` | `00000110` | 位 1 + 位 2 | 恢复特殊行动能力 |

**示例**：
```asm
; 恢复攻击能力
push 02              ; 参数 = 02
mov ecx, ebx           ; ecx = 单位指针
call 406690          ; 执行后：unit[0D] 的 bit 1 被清除
```

##### 界面更新函数 - 43E6AA

更新行动界面状态，让玩家看到行动能力已恢复。

#### 关键数据结构

| 地址/偏移 | 说明 |
|-----------|------|
| `508B00[index]` | 二次行动状态数组（最多250个槽位） |
| `501BF9` | 全局行动类型/回合标志 |
| `unit+04` | 单位ID（1字节，用作508B00的索引） |
| `unit+0D` | 行动状态位掩码（1字节，每位对应一种行动类型） |
| `unit+23` | 回合相关状态标志（1字节，新回合清零） |
