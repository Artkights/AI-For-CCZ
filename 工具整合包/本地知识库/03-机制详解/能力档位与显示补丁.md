# 能力档位与显示补丁

## 结论速览

- 当前 6.5 未加密样本的能力档位核心函数位于 `0041A534`，负责把单位五维原始值映射到档位编号。
- 本地 6.5 机器码保留 7 个分支：`C/B/A/S/X/V/Z`，其中阈值分别是 `50/70/90/100/111/127`；项目实际可按 6 档使用，即不启用 `Z=7`。
- 档位修改至少涉及三类补丁点：计算函数、封顶函数、显示指针表；不能只改显示文字或只改一个阈值。
- 4-7 档可优先采用“原函数分支合并 + 封顶 + 显示表”低风险方案；8-10 档需要迁移档位计算函数到代码洞并重定向 5 个调用点。
- 外部论坛地址只能作为候选模板，正式写入必须以当前 `Ekd5.exe` 的 SHA256、PE 映射和机器码签名为准。

## 适用版本

- 默认目标：曹操传加强版 6.5 未加密当前本地基底。
- 6.5p、6.5pl、6.6/6.6x、大型打包 EXE 或已改过档位函数的 MOD 只能通过签名扫描生成候选报告，不能复用固定地址写入。

## 档位适配方法

### 4-7 档低风险方案

4-7 档优先不迁移函数，而是合并高档返回值：

| 目标档位数 | 有效显示 | 返回值合并规则 | 封顶字节 |
| ---: | --- | --- | --- |
| 4 | `C/B/A/S` | `Z,V,X,S -> 4`，`A -> 3`，`B -> 2`，默认 `C -> 1` | `3C 05 72 02 B0 04` |
| 5 | `C/B/A/S/X` | `Z,V,X -> 5`，`S -> 4`，`A -> 3`，`B -> 2` | `3C 06 72 02 B0 05` |
| 6 | `C/B/A/S/X/V` | `Z,V -> 6`，`X -> 5`，`S -> 4`，`A -> 3`，`B -> 2` | `3C 07 72 02 B0 06` |
| 7 | `C/B/A/S/X/V/Z` | 保持原样 | `3C 07 72 02 B0 07` |

返回值立即数位置：

| 分支 | 当前返回 | 返回值 VA | 文件偏移 |
| --- | ---: | ---: | ---: |
| `Z` | 7 | `0041A547` | `019947` |
| `V` | 6 | `0041A54F` | `01994F` |
| `X` | 5 | `0041A557` | `019957` |
| `S` | 4 | `0041A55F` | `01995F` |
| `A` | 3 | `0041A567` | `019967` |
| `B` | 2 | `0041A56F` | `01996F` |
| `C` | 1 | `0041A573` | `019973` |

本地 6 档有效 profile 的最小写法：

```text
0041A547: 07 -> 06
00406CB4: 3C 07 72 02 B0 07 -> 3C 07 72 02 B0 06
显示标签：C/B/A/S/X/V
```

### 8-10 档迁移方案

8-10 档不应硬塞进 `0041A534` 原函数。推荐流程：

1. 扫描并确认代码洞和数据洞，例如 `004748F0`、`00474961` 只能作为候选，必须复读为全 NOP。
2. 在代码洞写入新的档位计算函数。
3. 将 5 个 call site 改为调用新函数。
4. 保留或复制原函数尾部特技/效果值叠加语义。
5. 将封顶点改为目标档位。
6. 新建显示指针表和字符串表。
7. 将 `00478205` 的 `mov edx, imm32` 改为新显示表地址。

封顶字节规律：

```text
3C (档位数+1) 72 02 B0 (档位数)
```

| 目标档位数 | 封顶字节 |
| ---: | --- |
| 8 | `3C 09 72 02 B0 08` |
| 9 | `3C 0A 72 02 B0 09` |
| 10 | `3C 0B 72 02 B0 0A` |

## 显示标签规则

当前显示层本质是“档位编号 -> 字符串指针”。标签既可以是数字，也可以是字母，也可以是作者自定义字符。

推荐支持分级：

| 标签类型 | 支持策略 |
| --- | --- |
| 单字母 ASCII | 可原地或新表写入 |
| 数字 `1..9` | 可原地或新表写入 |
| 数字 `10` | 必须新建字符串区，因为 `10\0` 需要 3 字节 |
| 自定义 ASCII | 推荐新建字符串区和指针表 |
| GBK 中文/符号 | 可作为候选支持，但必须实机验证 UI 宽度和编码 |
| 无法 GBK 编码、含 NUL、过长标签 | 禁止自动写入 |

## 工具实现边界

工具中应区分：

| 字段 | 含义 |
| --- | --- |
| `EngineTierCapacity` | 当前机器码最大可返回档位，例如 7 |
| `EffectiveTierCount` | 当前 MOD 作者实际启用档位，例如 6 |
| `ThresholdRules` | 原始值到档位的阈值表 |
| `DisplayLabels` | 实际显示标签 |
| `PatchMode` | `MergeOriginalBranches` 或 `RelocateCalculationFunction` |

正式写入前必须完成：

1. PE 映射成功。
2. 档位函数签名匹配。
3. 5 个 call site 可解释。
4. 封顶点签名匹配。
5. 显示表指针可解释。
6. 代码洞/数据洞确认不会覆盖非 NOP 或未知补丁。
7. 写测试副本、复读 profile、记录 SHA256 before/after。
8. 正式项目写前备份，写后复读和生成 JSON 报告。
