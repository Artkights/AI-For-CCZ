# 19 全剧本指令详解 2023 归纳

来源：本地资料 `工具整合包\本地知识库\全剧本指令详解（2023）.docx`。本页只记录当前 CCZModStudio 与刘备传前三关制作直接需要的规则；原始 docx 不作为 GitHub 上传对象。

## 结构规则

- `.eex` 是 R/S 剧本主格式；`.E5S` 不作为 R/S 剧本主线格式。
- 剧本按 `Scene -> Section -> Command -> ChildBlock` 理解；完整树保存必须重建 Scene 偏移表、Section 长度、子块长度和 `0x76` 跳转位移。
- `00 事件结束` 是 Section 或子事件结束标识，不应单独添加、复制、剪切、删除。
- `01 子事件设定` 是子事件开始标识；复制子事件时必须连同 `01` 一起复制。
- `0C 结束 Section` 多用于自由 R 或同 Scene 多 Section 流程，普通 R/S 制作不应随意插入。
- `0D 结束 Scene` 是常规 Scene 结束关键命令，战斗胜利、失败、出战测试等流程常用。
- `76 无条件跳转` 按目标命令偏移跳转，修改后必须由完整树 Writer 重新计算并复读校验。

## R 剧本常用规则

- R 剧本开头通常使用 `0A 初始化局部变量`，清空局部变量和上一关临时状态。
- `06 我军出场限制`、`4A 我军出场强制设定` 常用于战前选择限制；进入 S 前通常还需要配套一次 false 处理。
- `07 出战测试` 是 R 切入 S 的入口，通常与 `2D 武将点击测试` 放在同一 Scene。
- `17 场所名`、`18 事件名称设定` 只在 R 场景使用，名称长度要受旧工具限制。
- `14 对话1` 是常规对白命令：人名前带 `&`，人名独占一行，正文另起；长句应拆成多条命令。
- `15 对话2` 可处理不在场说话等特殊情况，前三关优先使用 `14`。
- `16 信息` 可作旁白或历史情报，也可用不带 `&人名` 的 `14` 替代。

## S 战场常用规则

- `19 胜利条件` 存储菜单中查看的胜败目标，通常位于 S 剧本第一个 Scene。
- `1A 显示胜利条件` 是战前展示给玩家看的目标提示；`19` 不会自动展示。
- `42 战斗胜利测试`、`43 战斗失败测试` 用于测试战斗结果。
- `0E 战斗失败`、`49 战斗结束`、`0D 结束 Scene` 常连用处理失败或胜利流程。
- `44 战斗初始化`、`45 战场全局变量`、`46 友军出场设定`、`47 敌军出场设定`、`4B 我军出场设定` 是战场部署核心命令；当前只确认候选，不批量强写。
- `25 武将进入指定地点测试`、`26 武将进入指定区域测试` 用于坐标或区域触发，坐标必须结合地图格复核。
- `2E 武将相邻测试` 常用于相邻或可攻击范围内触发的战场对白事件。
- `36 武将状态测试` 常用于检测某武将当前 HP 是否为 0。
- `3F 回合测试`、`40 行动方测试`、`41 战场人数测试` 用于回合、行动方和人数触发。
- `53 战场撤退` 常与撤退显示/确认类命令配合。
- `59 战利品` 用于战场奖励。
- `5A 战场操作开始` 是进入玩家可操作阶段的重要节点。

## 变量规则

- `05 变量测试` 与 `0B 变量赋值` 常配合使用。
- 局部变量 `0..255` 适合关内临时事件；全局变量 `256..1535` 用于跨关剧情记录。
- 局部变量在战后或 R 开头应清空，避免上一关残留。
- 一次性事件常用“`05` 测 false -> 事件 -> `0B` 赋 true”的结构；测试变量与赋值变量必须一致。
- `77 变量运算`、`78 整型变量赋值`、`79 变量测试` 属于高级变量体系，`4076` 用于辅助指定 byte/word/dword 运算宽度。

## 当前工具写回策略

- 短文本原地写回只适合小幅修改，且必须满足原 GBK 容量。
- 重写长对白、胜败条件、事件名时，应走 `LegacyScenarioReader` + 修改文本参数 + `LegacyScenarioWriter.Save` 的完整树保存链路。
- 完整树文本写回可扩容文本参数，但仍必须保持命令结构不乱：不改 `00/01/0C/0D`，不直接编辑跳转命令和可变数组。
- 刘备传前三关当前只重写 `14/16/17/18/19/1A` 的文本参数；出场、坐标、AI、触发条件保持原结构，等待逐命令核对后再改。
