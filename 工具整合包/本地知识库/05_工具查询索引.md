# 05 工具查询索引

本页是后续开发遇到格式疑问时的“先查哪里”索引。原则：先看本知识库和 `plan.md`，再查对应旧工具目录；没有确认的格式不做强写。

## 用户指定的优先查询来源

| 问题/模块 | 必查工具文件夹 | 当前用途 | 实施约束 |
| --- | --- | --- | --- |
| 人物形象 | `B形象指定器` | 校对人物 R/S 编号指定表、`System.ini` 中的 `FileHead/RFileHead/UserPath2`、形象资源编号习惯 | 人物形象链路固定为 `Ekd5.exe R/S 指定表 -> Pmapobj.e5/Unit_*.e5/Face.e5 (资源封包)`；不得把 `.E5S` 当人物形象 |
| 剧本编辑 | `a新剧本编辑器v0.23` | 对齐 `CczSceneEditor2_v0.23.exe` 的传统制作习惯，核对 R/S `.eex` 剧本读取、命令树、文本和字典配置 | `.eex` 才是 R/S 剧本/资源相关格式；读取列表只扫描 `RS\R_*.eex/S_*.eex`，未知命令结构不强写 |
| 兵种设定 | `B形象指定器`、`CczRSX 6.5` | 校对兵种/形象联动、`HexTable.xml` 表定义、数据偏移和字段含义 | 只写已确认表字段；保存前备份，保存后复读校验 |
| 宝物设定 | `B形象指定器`、`CczRSX 6.5` | 校对宝物表、装备/特效字段、旧工具字段命名 | 未确认特效位和扩展字段前只做只读解释或原字段写回 |
| 地形编辑 | `地形编辑器` | 校对 `Hexzmap.e5`、地图地形块、画笔和保存习惯 | 地形块写回必须保留未知字节；拖拽绘制要做缓存和节流，不能影响地图制作页稳定性 |

## 当前最高优先级查询规则

1. 剧本/战场相关如果再次出现“读取卡死”或格式错判，先查 `a新剧本编辑器v0.23`，再核对 `ScenarioFileReader` 是否仍只读 `.eex` 头部索引。
2. 人物形象如果出现 R/S 资源错位，先查 `B形象指定器\形象指定器6.5\System.ini`，再核对 `ImageAssignmentService` 的 R=`0xE1000`、S=`0xD2800` 护栏。
3. 兵种、宝物、角色字段如果出现偏移疑问，先查 `CczRSX 6.5\ConfigTable\HexTable.xml`；旧工具显示习惯只作为字段命名和 UI 参考。
4. 所有写入功能必须经过：目标文件备份 -> 保留未知字节写回 -> 复读校验 -> 失败提示最近备份。

## 仍需用户或旧工具进一步提供的信息

- `a新剧本编辑器v0.23` 对 `.eex` 的完整命令长度、Section 边界和插入/删除规则。
- `B形象指定器` 对 R/S `.eex` 帧级图像解码、调色板和动作序列的真实规则。
- `CczRSX 6.5` 中宝物特效、兵种扩展位、人物扩展属性的字段解释。
- `地形编辑器` 对 `Hexzmap.e5` 变量尺寸地形段、段头 2 字节、地图编号映射和附加结构的解释。
