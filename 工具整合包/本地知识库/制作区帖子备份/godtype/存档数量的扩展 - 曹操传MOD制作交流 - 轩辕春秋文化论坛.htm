追加修正自动存档功能会令战场中的禁止存档无效的问题
无意中发现的，战场上设置禁止存档的效果，但是自动存档会无视这个效果，除了自动保存一个新档，同时会重新恢复可以存档功能。
修正如下：
0044E1CD   .^E9 F1FEFFFF    JMP 0044E0C3
0044E1D2     90             NOP

0044E0C3     0F84 0A010000  JE 0044E1D3
0044E0C9     833D FC0F5000 >CMP DWORD PTR DS:[500FFC],0
0044E0D0    -FF25 0CD14C00  JMP DWORD PTR DS:[<&AutoDLL.ASave>]             ;  AutoDLL.ASave

003E12B0 > 75 07            JNZ SHORT AutoDLL.003E12B9
003E12B2   B8 EEE64400      MOV EAX,44E6EE
003E12B7   FFE0             JMP EAX
最后的3E*段是AutoDLL，用UE改，位置是12B3，改成EE E6。

===========
追加自动生成存档文件
一般的引擎必须要有一个存档D文件才能成功保存，如果没有是不能成功保存的。
这个问题由STAR提供解决方法：
0041B3D7  |.  E8 80DBFFFF   CALL 00418F5C      ; \返回eax 如果是0 表示文件不存在
在这后面增加自动创建文件的代码即可

LEA EAX,DWORD PTR SS:[EBP-11C]     文件名在这里
PUSH EAX                         ; /Arg1
CALL 00418ED4               ; 取得完整的路径
ADD ESP,4
PUSH EAX                         ; /Arg1
LEA ECX,DWORD PTR SS:[EBP-10]    ; |
CALL 004190BB               ; 创建文件
一般人就跳到其他空白地方改写吧。我是将这条函数重写了一次，以前B档部分代码要跳出去写，这次就顺便写在一起，压缩了一部分代码。
