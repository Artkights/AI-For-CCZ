# 剧本编辑器 v0.24 静态对照

## 结论速览

- `老版游戏制作工具\剧本编辑器v0.24` 只提供 `cczEditor_v0.24.exe`、`CczString.ini`、`CczSceneEditor2.ini`、`CczCustom.ini` 和 `说明.txt`，没有源码。
- `cczEditor_v0.24.exe` 是原生 MFC/Win32 程序，不是 .NET 程序；无法通过托管反射读取实现，只能通过配置、PE 资源、导入表、字符串和运行说明做静态取证。
- v0.24 在剧本树/旧式对话框主线仍延续 v0.23 资源体系；可作为操作习惯和新增功能线索，但完整 `.eex` 读写规则仍以 v0.23 源码、当前 `LegacyScenarioReader/Writer`、样本复读和烟测为准。
- v0.24 明确新增 R 场景预览操作习惯：`Ctrl` 为吸附模式，鼠标滚轮改变朝向，`Shift+鼠标滚轮` 改变动作。

## 证据来源

- 目录：`F:\从0开始的AI制作曹操传MOD\曹操传加强版6.5（未加密）\老版游戏制作工具\剧本编辑器v0.24`
- 程序：`cczEditor_v0.24.exe`
  - 文件大小：`4,649,472` 字节。
  - SHA256：`8E198C7975AEA8A27C22B7391BEAF388542FB13B31AD832B916F1702E3CB76F4`。
  - `[Reflection.AssemblyName]::GetAssemblyName()` 报告“原本应包含程序集清单”，确认不是托管程序集。
- 配置：
  - `CczCustom.ini` 与 v0.23 哈希一致。
  - `CczSceneEditor2.ini` 主要差异是本机 `ExePath`。
  - `CczString.ini` 比 v0.23 多出一段 `wchar_t xxcs[34][30]` 源码式文本，内容与 `信息传送 0..33` 子功能列表同源。
- PE 静态资源：
  - v0.23：`MENU=9`、`DIALOG=78`、`STRING=18`、`ACCELERATOR=1`。
  - v0.24：`MENU=10`、`DIALOG=102`、`STRING=47`、`ACCELERATOR=2`，另有大量 MFC 皮肤 `PNG/STYLE_XML` 资源。
  - v0.24 主菜单比 v0.23 多出 `导出为JSON文件...`、`启动HTTP服务器`、`停止HTTP服务器`。
- PE 字符串/导入表：
  - 可见字符串含 `EEX File(*.eex)|*.eex;*.eex_new||`、`Ekd5.exe`、`E5\Mmap.e5`、`Pmapobj.e5`。
  - 导入 `GDI+`、`GdipCreateBitmapFromStream`、`GdipDrawImageI`、`GdipBitmapGetPixel`、`MSIMG32.TransparentBlt` 等图像绘制 接口。

## R 场景预览行为

`说明.txt` 明确写明：

- `按住ctrl是吸附模式`
- `鼠标滚轮是改变朝向`
- `按住shift+鼠标滚轮是改变动作`

结合 `Mmap.e5`、`Pmapobj.e5` 字符串和图像 接口 导入，可确认 v0.24 的 R 场景预览不是纯文本说明，而是实际加载 R 背景和人物帧做画布式预览。由于没有源码，本页不把鼠标坐标换算、吸附阈值、帧序列和写回规则升级为已验证实现规则。

## 与当前整合包对照

当前普罗工具整合包已经具备的能力：

- `LegacyScenarioReader/Writer` 已按旧版完整树读取/重建 `R_*.eex/S_*.eex`。
- `LegacyMfcDialogCatalog` 已覆盖 v0.23 源码分派的旧式对话框主线，含 `0x30 武将出现`、`0x32 武将移动`、`0x33 武将转向`、`0x34 武将动作` 等 R 场景视觉命令。
- `RSceneDraftService` 可从 `0x27/0x2F/0x30/0x31/0x32/0x33/0x34` 构建当前 段落 的 R 场景状态快照。
- `MainForm.RScene` 已能读取 `Mmap.e5` 背景、渲染 `Pmapobj.e5` 人物帧、拖放插入 `0x30 武将出现`、拖拽同步已有 `0x30` 的 X/Y、控制面板同步方向/动作，并通过 `完整保存R剧本` 落盘。

主要差异：

- v0.24 的画布交互把普通滚轮用于朝向，`Shift+滚轮` 用于动作，`Ctrl` 用于吸附；当前整合包 `HandleRSceneCanvasMouseWheel()` 把滚轮用于缩放，`Ctrl` 只是放大缩放步长。
- v0.24 的拖拽吸附是显式操作提示；当前整合包拖拽时总是通过菱形坐标换算落到格点，没有提供 `Ctrl` 开关式吸附/自由移动预览。
- v0.24 暴露 JSON 导出和 HTTP 服务器菜单；当前整合包没有同名 界面，但已有 MCP server、结构化 JSON 报告和项目侧 JSON 草稿，能力方向不同。
- v0.24 的 `CczString.ini` 带有 `信息传送` 子功能源码式数组；当前整合包已从旧资源抽取 `Dialog_114` 模板，但后续仍应把 v0.24 这份配置作为 `72 信息传送` 子功能核对来源之一。

## 优化点

1. R 场景画布交互兼容 v0.24 习惯：
   - 选中/编辑角色时，普通滚轮循环方向；
   - `Shift+滚轮` 调整动作帧；
   - 缩放改为 `Ctrl+滚轮` 或工具栏按钮，避免和 v0.24 操作习惯冲突；
   - 修改后继续只写内存模型，提示必须点击 `完整保存R剧本`。
2. 增加 `Ctrl` 吸附模式语义：
   - 当前格点换算可保留为默认安全写回；
   - 预览层可区分“自由像素拖动”和“按 控制键 吸附到菱形格”，但写回 `.eex` 前仍只能落到已确认的格点坐标槽。
3. 补充 R 场景姿态写回烟测：
   - 当前已有 `0x30` X/Y 写回烟测；
   - 应增加 `0x30` 方向/动作槽、`0x33` 转向和 `0x34` 动作的内存修改、完整保存、复读校验；
   - 未完成前，方向/动作写回只能按当前已确认的界面行为和旧对话框 规则保守处理。
4. 将 v0.24 `CczString.ini` 纳入旧工具资源候选：
   - `ProjectDetector` 当前优先固定 `a新剧本编辑器v0.23\CczString.ini`；
   - 可增加 v0.24 配置作为只读补充来源，尤其服务于 `信息传送` 子功能名称核对；
   - 写入/命令结构仍不得因为 v0.24 配置文本自动放开。
5. 对 JSON/HTTP 能力做产品层取舍：
   - 整合包已有 MCP 和结构化报告，不必复刻 v0.24 内置 HTTP server；
   - 可补一个“导出当前 R/S 旧版树为 JSON”菜单或 MCP 工具，作为只读审查/版本对比能力。

## 边界

- v0.24 没有源码，PE 资源和字符串不能替代 v0.23 源码级证据。
- v0.24 的 R 预览交互可作为 UX 对齐依据，但坐标换算、吸附阈值、图层顺序、帧序列和写回语义仍需样本、源码、烟测或实机验证。
- v0.24 新增 JSON/HTTP 菜单只确认“存在入口”，没有确认协议、输出结构和安全边界，不能直接进入整合包写回流程。
