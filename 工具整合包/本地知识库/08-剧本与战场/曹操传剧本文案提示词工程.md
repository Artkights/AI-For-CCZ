# 曹操传剧本文案提示词工程

## 结论速览

- 本专题沉淀通用曹操传系 MOD 剧本文案提示词，不绑定刘备军、曹操军、姜维、蜀汉传或任何固定主角。
- 主线由输入字段决定：MOD 名称、主线阵营、主角/视角人物、时代节点、世界观、史实/架空程度和当前战役视角。
- “姜维传水准”只表示人物塑造质量标准：有血肉、有迟疑、有情感逻辑，不表示固定采用姜维或蜀汉视角。
- 文案创作与可导入命令转换分两步：先写剧本，再按 `剧本文本导入AI说明模板.md` 转成 CCZModStudio 可解析格式。

## 适用版本

- 默认适用：曹操传加强版 6.5（未加密）当前项目基底。
- 适用于曹操传系 MOD 的文案设计，包括魏传、蜀传、吴传、晋传、姜维传风格作品、刘备传、群雄传、架空外传和独立单关。
- 若进入 `.eex` 写回、地图部署、资源替换或表数据修改，仍必须按 6.5 本地样本、旧工具源码、备份、复读和烟测规则执行。

## 已确认事实

- R 场景和 S 战场在制作上职责不同：R 更偏过场叙事，S 更偏战斗部署、触发事件、胜败分支和战后收束。
- 现有 `剧本文本导入AI说明模板.md` 是格式转换器，负责把定稿文案转换为 `@dialog`、`@narration`、`@appear`、`@move`、`@turn`、`@action`、`@battle-turn`、`@battle-action` 等可导入命令。
- 文案提示词不能让 AI 在缺少人物 ID、坐标、方向、动作编号时编造可写入命令；这些缺失必须在转换阶段报告。
- `我军` 在战场设计中表示本关玩家可控或剧情主视角阵营，不等同于任何固定历史势力。

## 实现/使用方法

- 场景过场使用 `制作提示词模板.md` 的“通用场景过场剧本提示词”。必须填写或可推断：主线阵营、主角/视角人物、时代节点、世界观、史实约束、人物存亡约束和剧情梗概。
- 战场设计使用 `制作提示词模板.md` 的“通用战场剧本设计提示词”。必须明确“本关我军定义”，再动态判定友军和敌军。
- 主线判定优先级：用户明确设定 > 当前战役可控方 > 叙事焦点 > 情感焦点。仍无法判断时输出“缺少主线设定”，不得自动套用魏、蜀、吴或姜维传。
- 二段式流程：先生成可读剧本文案，再转可导入文本。转换阶段不得继续扩写剧情，只做格式映射和缺失信息报告。
- 独立单关工具链仍走 `StandaloneScenarioDesign -> force_open ModPackage -> preview -> test_copy apply -> auto_validate_mod -> export_mod_report`，但主线字段必须来自用户输入或可靠推断。

## 风险与边界

- 本专题是提示词工程，不是 `.eex` 指令格式、二进制偏移或写回规则。
- 不得把“姜维传水准”误解为蜀汉视角默认值；模板必须支持任意阵营、原创势力和多主角群像。
- 缺少坐标、人物 ID、方向、动作编号、触发条件时，只能输出缺失信息，不能为了完整性臆造可写入命令。
- 多主角 MOD 中，本关视角只用于当前场景或当前战场，不自动改写全局主线。

## 详细记录

### 主线字段

| 字段 | 用途 |
|------|------|
| MOD名称 | 确定作品语境和命名风格 |
| 主线阵营 | 确定长期叙事立场，不固定为某一势力 |
| 本关我军定义 | 确定 S 战场中的玩家可控或主视角阵营 |
| 主角/视角人物 | 确定当前剧情情感焦点 |
| 时代节点 | 防止人物死亡后登场、事件顺序错位 |
| 世界观 | 区分史实向、演义向、架空改写和原创外传 |
| 史实约束 | 保留必须遵守的历史事件和关系 |
| 人物存亡约束 | 明确不可登场、只能回忆或传闻提及的人物 |

### 文案与导入边界

- 文案阶段负责人物、冲突、台词、分幕、战场节点和特殊机制。
- 导入阶段负责把定稿转换为结构化命令，不能继续扩写剧情。
- 可导入命令所需的 ID、坐标、方向、动作编号必须来自用户、工程数据或明确映射表。
