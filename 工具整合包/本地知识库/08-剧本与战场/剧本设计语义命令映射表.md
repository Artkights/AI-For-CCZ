# 剧本设计语义命令映射表

## 结论速览

- 本页把“单挑、奖励、坐标触发、胜败条件、阶段推进”等设计语义映射到 6.5 R/S 剧本命令码，供生成 `ScenarioDesign`、`StandaloneScenarioDesign` 和后续 `ModPackage` 前查表使用。
- 本页不是提示词替代品，而是制作约束：生成单关 R/S 时必须先查本表，再写事件图和可导入草稿。
- 命令号来自本地知识库、旧剧本编辑器源码对照、`ScenarioCommandParameterTemplateService`、`ScenarioTextImportService` 和 `ModPackageService`。凡未确认参数槽、动作编号、跳转位移或结构写入边界的内容，必须标注风险并进入待核验项。

## 使用规则

1. R/S 设计稿里的每个事件节点都要写出：`设计语义`、`触发命令`、`动作命令`、`变量/防重复`、`写回风险`。
2. 玩家可见的战前情报卡必须能逐条映射到 S 事件图；若没有对应命令落点，不得写成已实现机制。
3. `0x76 无条件跳转` 不手填偏移，只能由旧版完整树写入器重算并复读。
4. `0x46/0x47` 是循环数组部署命令，不按普通短命令理解；坐标、AI、装备必须绑定具体 Record。
5. `append_command/insert_command` 在 `ModPackage V1` 当前偏预览；正式写结构命令优先走旧版完整树工具、测试副本、复读报告。
6. 6.5 原版 `R_00.eex` 是开局启动容器；独立精简基底不必保留全部启动骨架，但必须从本地样本确认首关入口和战前准备流程，不能把文学分幕写成多个真实剧情 Section。
7. `0x11 剧本跳转` 的目标参数必须按本地工具显示和复读确认：`0 -> S_00.eex`、`1 -> R_01.eex`、`2 -> S_01.eex`、`3 -> R_02.eex`，不得按“0=R_00、1=S_00”的直觉填写；但采用玩家切换 Scene 的战前准备路线时，R 侧不硬写 `0x11`。

## 样本式 S 事件结构

刘备 `S_01` 与三英 `S_00` 共同确认，战中事件应把触发条件放在 Section 顶层、正文根之前：

```text
02 内部信息
-> 25/2E/36/3F/41 等触发测试
-> 05 变量门或防重复
-> 00 正文根
-> 14/16/23/3D/4E/4F/50/53/55/59/49/0E 等正文动作
-> 0B 变量赋值
-> 00 事件结束
```

不要生成 `02 -> 00 -> 触发测试`。这会把触发条件放进正文块，战场循环无法测试事件。

## 轻量单挑标准组合

用于“刘生 VS 祁戍”这类不强求完整单挑场景的事件：

| 步骤 | 命令 | 作用 |
| --- | --- | --- |
| 触发 | `0x2E 武将相邻测试` | 刘生与祁戍相邻后进入事件 |
| 防重复 | `0x05 变量测试` + `0x0B 变量赋值` | 检查并设置 `scale_broken_liu` |
| 台词 | `0x14 对话` 或 `0x16 信息` | 播放单挑对白 |
| 演出 | `0x4F 战场转向` + `0x50 战场动作` | 刘生、祁戍转向与攻击/受击 |
| 结果 | `0x53 战场撤退` | 祁戍撤退 |
| 奖励 | `0x38 武将能力设定` 或 `0x3D 获得物品` | 刘生攻击提升或发物品 |

## 完整单挑命令族边界

刘备 `S_01` 样本可见完整单挑命令族：

| 命令 | 用途 |
| --- | --- |

| `0x60 出场` | 单挑双方入场 |
| `0x63 对话` | 单挑内对白 |
| `0x64 动作` | 单挑动作 |
| `0x65/0x66 攻击` | 攻击与受击 |
| `0x61 胜负` | 胜负处理 |

该命令族样本可见但自动化默认不启用。首版可玩关卡优先使用轻量单挑；完整单挑必须另建样本、复读参数、测试副本和实机验收。

## 待验证项

- 部分命令的完整参数长度、动作编号、特效编号、胜败类型枚举仍需结合旧工具、样本复读和实机验证。

- `0x76` 跳转和跨 Section 重排不得手工拼接，必须等完整树写入器重算并复读。
