#### 结构硬规则

- `.eex` 是 R/S 剧本主格式；`.E5S` 不作为 R/S 剧本主线格式。
- 剧本按 `Scene -> Section -> Command -> ChildBlock` 理解；完整树保存必须重建 Scene 偏移表、Section 长度、子块长度和 `0x76` 跳转位移。
- `00 事件结束` 是 Section 或子事件结束标识，不应单独添加、复制、剪切、删除。
- `01 子事件设定` 是子事件开始标识；复制子事件时必须连同 `01` 一起复制，不能只复制里面的子事件内容。
- 会自动生成子事件套子的测试类指令包括 `02`、`03`、`04`、`05`、`12`、`13`、`25`、`26`、`2E`、`36`、`37`、`3F`、`40`、`41`、`6E`、`73`、`79` 等；修改时必须连同子块结构一起核对。
- `0C 结束 Section` 多用于自由 R 或同 Scene 多 Section 流程，普通 R/S 制作不应随意插入。
- `0D 结束 Scene` 是常规 Scene 结束关键命令，战斗胜利、失败、出战测试等流程常用。
- `76 无条件跳转` 按目标命令偏移跳转，修改后必须由完整树 Writer 重新计算并复读校验；当前不开放直接手填跳转偏移。

#### R/S 教程迁移规则

- 外部教程统一作为“阅读和排错口径”，不作为二进制写回规则。能迁移的是流程结构、常见命令组合和风险提醒；不能迁移的是具体偏移、自动改写、跳转号手填或跨版本变量行为。
- R/S 剧本的基础术语可按教程理解：`R_XX.eex` 支配过场/R 剧情，`S_XX.eex` 支配战斗/S 关卡；剧本有 `Scene -> Section -> 指令/子事件` 层次。当前工具实现以旧源码确认的 `Scene/Section/CommandBlock` 模型为准。
- 子事件必须保持 `0x01 子事件设定 + 可承接子块的命令 + 子块内容 + 0x00 结束` 这条结构链。教程中“子事件后面要接可展开命令”的经验，与旧源码 `code_test[id] != 0` 承接子块规则一致；不能在 `0x01` 和承接命令之间插入普通命令。
- 教程和经验贴都强调 `0x76 无条件跳转` 高风险。当前知识库统一收口为：不手填跳转位移，不复制带跳转的子事件后直接使用；如确需复制/移动，必须由完整树 Writer 重算并用旧版 Reader 复读。
- `即时判定`、跨 Scene 跳转、R/S 跳转和战斗胜败后的剧本切换都属于流程高风险。外部教程可用于提示“应该检查哪里”，不能直接生成自动修复。
- 修改后复读仍是硬规则：教程经验中的“另存、重新打开、确认不报错”映射到当前工具链就是备份、临时文件、`LegacyScenarioReader` 复读校验和结构化报告。

#### R 剧本常用规则

- R 剧本开头通常使用 `0A 初始化局部变量`，清空局部变量和上一关临时状态。
- `06 我军出场限制`、`4A 我军出场强制设定` 常用于战前选择限制；进入 S 前通常还需要配套一次 false 处理。
- `07 出战测试` 是 R 切入 S 的入口，通常与 `2D 武将点击测试` 放在同一 Scene。该 Scene/Section 若缺测试指令，旧资料认为可能导致剧本损坏。
- 外部教程口径：同类 R 剧本之间可用 `11 剧本跳转` 做即时跳转；R 进入 S 通常要先经过出战测试，再加入跳转到目标 S 的流程。该经验只作为流程检查项，具体命令位置和跳转目标必须回到本地 R/S 样本核对。
- `17 场所名`、`18 事件名称设定` 只在 R 场景使用，名称长度要受旧工具限制。
- `14 对话1` 是常规对白命令：人名前带 `&`，人名独占一行，正文另起；长句应拆成多条命令。
- `15 对话2` 可处理不在场说话等特殊情况，前三关优先使用 `14`。
- `16 信息` 可作旁白或历史情报，也可替代受限的 `69 旁白`。

#### R 场景演出命令补充

- `27 背景显示` 使用 R 场景背景 `Mmap.e5`；R 背景与 S 战场 `Map\Mxxx.jpg` 不同。资料口径中外场景约为 1-40，内场景从 41 起。
- `27 背景显示` 与 `1C 绘图` 常配套使用，制作 R 场景时应先核对背景图号和后续绘图命令是否成对。
- `30 武将出现`、`31 武将消失`、`32 武将移动`、`33 武将转向`、`34 武将动作`、`1E 武将重绘` 是 R 场景人物摆放、移动、朝向、动作预览的核心命令。
- R 场景方向按“上北、下南、左西、右东”记录；工具 UI 预览也按这个口径切换方向。
- `34 武将动作` 对应 `Pmapobj.e5` 的 R 形象帧；人物 R 编号仍来自 Ekd5.exe 的 `6.5-0-4 R形象` 指定表。
- `35 武将形象改变` 属于旧方式。Star175 新引擎口径中，R 形象变化更推荐用 `77 变量运算` 配合 `78 整型变量赋值` 写到对应人物 R 形象变量；NPC 可通过改名和切换 R 形象在不同 R 场景复用。
- 当前 CCZModStudio 的 `R场景制作` 页仅把上述命令解析为视觉候选，并保存项目侧草稿；未确认参数槽前，不直接写回 `30/32/33/34/35`。

#### S 战场常用规则

- 外部 S 剧本教程把典型战场分为战前设置、战中处理、战后处置三段，可作为阅读 S 剧本时的导航口径。
- 传统 S 剧本常被教程概括为 3 个 Scene：战前设置、战中处理、战后处置。这个三段式适合做阅读导航，但不能作为强制结构约束；实际文件仍以 `LegacyScenarioReader` 读出的 Scene/Section 为准。
- 传统战前设置常见 `44` 战斗初始化、`45` 战场全局变量、`46/47` 友敌出场控制、我军设定子事件、背景显示、延时、绘图、胜利条件、菜单处理等序列；这只说明常见流程，不证明每个项目都按同一顺序或同一参数槽。
- 外部教程提到 `5A 战场操作开始` 前后常配合变量赋值和菜单处理。当前知识库只把它作为“进入玩家可操作阶段”的关键检查点；是否缺失、提前或延后必须结合该关剧情、隐藏出场和实机流程验证。
- 战后处置常见菜单处理、调色板和剧本跳转等命令；跨 Scene/Section 的结构编辑仍必须由完整树 Writer 负责，不能手工拼接跳转。
- 外部教程口径：S 进入 R 通常在胜利测试或失败测试后通过 `11 剧本跳转` 回到目标 R。当前只作为流程阅读提示，不自动改写 `42/43/0E/49/0D/11` 的组合。
- `19 胜利条件` 存储菜单中查看的胜败目标；`1A 显示胜利条件` 是战前展示给玩家看的目标提示。
- `42 战斗胜利测试`、`43 战斗失败测试` 用于测试战斗结果。
- `0E 战斗失败`、`49 战斗结束`、`0D 结束 Scene` 常连用处理失败或胜利流程。
- `44 战斗初始化`、`45 战场全局变量`、`46 友军出场设定`、`47 敌军出场设定`、`4B 我军出场设定` 是战场部署核心命令；当前只确认候选，不批量强写。
- `25 武将进入指定地点测试`、`26 武将进入指定区域测试` 用于坐标或区域触发，坐标必须结合地图格复核。
- `2E 武将相邻测试` 常用于相邻或可攻击范围内触发的战场对白事件。
- `36 武将状态测试` 常用于检测某武将当前 HP 是否为 0。
- `3F 回合测试`、`40 行动方测试`、`41 战场人数测试` 用于回合、行动方和人数触发。
- `53 战场撤退` 常与撤退显示/确认类命令配合。
- `59 战利品` 用于战场奖励。
- `5A 战场操作开始` 是进入玩家可操作阶段的重要节点。

#### 变量与扩展变量规则

- `05 变量测试` 与 `0B 变量赋值` 常配合使用。
- 局部变量 `0..255` 适合关内临时事件；全局变量 `256..1535` 用于跨关剧情记录。
- 局部变量在战后或 R 开头应清空，避免上一关残留。
- 一次性事件常用 `05` 测 false -> 事件 -> `0B` 赋 true；测试变量与赋值变量必须一致。
- `77 变量运算`、`78 整型变量赋值`、`79 变量测试` 属于高级变量体系，必须结合 `4076` 的 byte/word/dword 宽度解释。

#### `58` 战场障碍与区域操作

- docx 记录 `58` 可设置火、火船、船等障碍；`21` 只能添加火，很多场景可用 `58` 替代。
- 坐标 `X` 或 `Y` 大于等于 40 可被用作无效坐标技巧；资料中用 `X-40` 表示地雷位置。
- 区域障碍需先用 `4076=2` 指定 word 运算，再用 `4050` 记录左上角坐标，`58` 参数中的坐标表示右下角。
- `4049` 可控制未知白框是否显示。
- 原资料提示障碍数量存在上限，超过后可能无效果；当前不开放 MCP 写入。
